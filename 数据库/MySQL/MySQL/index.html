    一、MySQL 概述¶

    MySQL 是一个关系型数据库管理系统，使用 SQL 语言（结构化查询语言）进行数据管理。MySQL 以其高效、稳定的性能以及丰富的功能而广泛应用于 Web 应用、内容管理系统、在线交易等场景。

    MySQL 的主要特点：

    • 高性能：使用了多种存储引擎，特别是 InnoDB 提供事务支持。
    • 高度扩展性：支持大规模并发连接和数据存储。
    • 丰富的存储引擎：如 InnoDB（支持事务和外键）、MyISAM（轻量但不支持事务）。
    • 跨平台支持：兼容多种操作系统和开发语言。

    二、MySQL 架构与原理分析¶

    MySQL 的架构包括客户端、连接层、SQL 层、存储引擎层等。

    1. MySQL 的架构¶

    • 连接层：负责客户端与服务器之间的连接和权限验证。连接可以基于 TCP/IP 或 UNIX Socket。
    • SQL 层：MySQL 的核心部分，负责 SQL 解析、优化、查询缓存等操作，生成执行计划并调度存储引擎。
    • 存储引擎层：实际存储和处理数据的模块，支持多种存储引擎。最常用的是 InnoDB，它支持事务、行级锁和崩溃恢复。

    2. MySQL 存储引擎¶

    • InnoDB：事务型存储引擎，支持 ACID 特性和外键约束，使用 B+Tree 作为索引结构。
    • MyISAM：不支持事务和外键，适用于读多写少的应用场景，采用表级锁。
    • Memory：将数据存储在内存中，速度非常快，但不适合持久化存储。

    3. MySQL 的主要组件¶

    • SQL 查询解析和优化：包括 SQL 的解析、预处理和查询优化器。优化器会选择最佳执行路径。
    • 存储引擎：每个表可以选择不同的存储引擎。不同引擎有不同的锁机制和数据存储方式。
    • 缓冲池：InnoDB 使用缓冲池来缓存索引和数据页，提升读取性能。

    4. MySQL 索引结构¶

    MySQL 中最常用的是 B+Tree 索引，存储数据和索引条目，通过优化的节点查找效率提高查询速度。

    • B+Tree 索引：常用在主键和普通索引中，适合范围查询和排序。
    • 全文索引：用于字符串的全文搜索，主要用于 MyISAM 和 InnoDB 存储引擎。
    • 哈希索引：适合精确匹配查询，范围查询效率低。

    三、MySQL 的基本使用¶

    1. 数据库管理¶

    • 创建数据库：
    CREATE DATABASE mydatabase;
    • 选择数据库：
    USE mydatabase;

    2. 表操作¶

    • 创建表：
    CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100),
    age INT,
    email VARCHAR(100)
);
    • 查看表结构：
    DESCRIBE users;

    3. 数据操作¶

    • 插入数据：
    INSERT INTO users (name, age, email) VALUES ('Alice', 25, 'alice@example.com');
    • 查询数据：
    SELECT * FROM users WHERE age > 20;
    • 更新数据：
    UPDATE users SET age = 30 WHERE name = 'Alice';
    • 删除数据：
    DELETE FROM users WHERE name = 'Alice';

    4. 索引操作¶

    • 创建索引：
    CREATE INDEX idx_name ON users(name);
    • 删除索引：
    DROP INDEX idx_name ON users;

    7.跳过权限¶

    # 停止进程
sudo systemctl stop mysql
# 跳过验证
sudo mysqld_safe --skip-grant-tables --skip-networking --skip-mysqlx &
# 查看错误
sudo tail -f /var/log/mysql/error.log
# 可能出现的错误解决方法
sudo mkdir /var/run/mysqld
sudo chown -R mysql:mysql /var/run/mysqld
# 删掉配置中的syslog
# 无密码登录
mysql -u root
# 清空密码
use mysql;
update user set authentication_string='' where user='root';

# 杀掉刚刚的跳过验证的启动进程
# 然后开启mysql服务
sudo systemctl start mysql
sudo systemctl status mysql

# 进入mysql，更新密码
ALTER user 'root'@'localhost' IDENTIFIED BY 'p@ssw0rd1';
FLUSH PRIVILEGES;

# 备用
/usr/sbin/mysqld --basedir=/usr --datadir=/var/lib/mysql --plugin-dir=/usr/lib/mysql/plugin --user=mysql --skip-grant-tables --skip-networking --skip-mysqlx --log-error=/var/log/mysql/error.log --pid-file=/var/run/mysqld/mysqld.pid --socket=/var/run/mysqld/mysqld.sock --port=3306

    四、MySQL 优化¶

    1. 查询优化¶

    • 使用索引：对频繁查询的列加索引，如主键、外键、过滤条件列等。
    • 避免全表扫描：通过索引或条件过滤减少扫描数据量。
    • 合理使用 JOIN：多表查询时选择合适的 JOIN 方式，并确保关联字段有索引。

    2. 配置优化¶

    • 内存优化：配置 innodb_buffer_pool_size 提升 InnoDB 的缓存命中率。
    • 查询缓存：通过 query_cache_size 启用查询缓存，但注意缓存机制的开销。
    • 连接数配置：根据应用负载调整 max_connections 设置。

    3. 表结构设计¶

    • 范式设计：避免数据冗余，设计合理的表结构。
    • 分区：对于大表使用分区提高性能，分区方式包括范围、哈希等。
    • 分表：对于特别大的数据集，考虑进行水平分表，分散数据压力。

    五、MySQL 使用场景¶

    1. 事务性应用：MySQL 的 InnoDB 存储引擎支持事务，适合电商系统、金融系统等需要强一致性的应用。
    2. Web 应用：MySQL 的性能和扩展性使其在 Web 开发中广泛应用，适合内容管理系统、用户数据存储等。
    3. 日志系统：可以使用 MySQL 来存储和分析日志数据，配合表分区和索引优化。

    六、MySQL 优缺点¶

    1.优点¶

    • 开源免费：MySQL 社区版本开源，广泛应用且生态丰富。
    • 扩展性强：可以通过分片、复制等方式实现大规模应用。
    • 易用性高：易于管理和优化，有完善的文档支持。

    2.缺点¶

    • 水平扩展较难：虽然支持分片和复制，但对于海量数据的水平扩展还是较为复杂。
    • 复杂查询性能差：不适合非常复杂的多表关联查询。

    八、MySQL 高频面试题¶

    1. MySQL 索引的种类有哪些？¶

    • 主键索引：聚簇索引，数据行的物理存储顺序与索引顺序一致。
    • 唯一索引：确保列中的所有值唯一。
    • 普通索引：最常见的索引类型，没有唯一性限制。
    • 全文索引：用于全文搜索（支持 MyISAM 引擎）。
    • 联合索引：多个列组合的索引。

    2. InnoDB 和 MyISAM 的区别？¶

    • 事务支持：InnoDB 支持事务，MyISAM 不支持。
    • 锁机制：InnoDB 支持行级锁，MyISAM 只支持表级锁。
    • 外键支持：InnoDB 支持外键，MyISAM 不支持。
    • 全文索引：MyISAM 支持全文索引，InnoDB 在 MySQL 5.6 后才开始支持。

    3. MySQL的事务四大特性（ACID）？¶

    • 原子性（Atomicity）：事务中的操作要么全部执行，要么全部不执行。
    • 一致性（Consistency）：事务执行前后，数据库保持一致状态。
    • 隔离性（Isolation）：一个事务的执行不被其他事务干扰。
    • 持久性（Durability）：事务执行完成后，结果持久保存在数据库中。

    4. MySQL 的四种隔离级别？¶

    • READ UNCOMMITTED（未提交读）：可以读取未提交的数据。
    • READ COMMITTED（已提交读）：只能读取已提交的数据。
    • REPEATABLE READ（可重复读）：事务开始后，看到的数据是一致的，防止幻读（InnoDB的默认级别）。
    • SERIALIZABLE（串行化）：最高级别，完全串行化执行事务，避免一切并发问题。

    5. MySQL 的锁机制？¶

    • 表锁：一次锁住整张表，适合 MyISAM。
    • 行锁：锁住表中的一行数据，适合 InnoDB。
    • 意向锁：用于标记一个事务将要对某些行进行加锁，InnoDB 中使用。

    6. 什么是覆盖索引？¶

    • 覆盖索引指的是索引包含了查询所需的所有字段，不需要回表查询数据行。可以通过联合索引实现。

    7. Explain 查询分析工具中各字段的含义？¶

    • id：查询的序列号，表示查询的执行顺序。
    • select_type：表示查询的类型（简单查询、联合查询、子查询等）。
    • table：表示查询涉及的表。
    • type：表示查询的类型，可能值如 ALL、index、range 等，反映查询的效率。
    • key：表示查询使用的索引。
    • rows：估算需要扫描的行数。

    8. MySQL 性能优化的方法有哪些？¶

    • 使用适当的索引。
    • 避免 SELECT *。
    • 使用连接池。
    • 查询缓存。
    • 读写分离，主从复制。
