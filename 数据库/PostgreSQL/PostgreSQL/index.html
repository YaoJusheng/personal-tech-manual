    一、PostgreSQL 概述¶

    PostgreSQL 是一款开源、功能强大的对象关系型数据库管理系统，以其高扩展性和标准兼容性广泛应用于企业系统、地理信息系统、数据仓库等场景。它支持复杂查询、事务处理、视图、触发器、函数等丰富的特性。

    PostgreSQL 的特点¶

    1. 完全的 ACID 支持：事务处理确保数据一致性和可靠性。
    2. 扩展性：用户可以定义新的数据类型、函数、操作符等。
    3. 多版本并发控制（MVCC）：有效支持高并发操作。
    4. 先进的数据类型：如 JSON、数组、XML、GIS 数据类型等。
    5. 支持标准 SQL：包括窗口函数、递归查询等高级 SQL 特性。

    二、PostgreSQL 架构和原理分析¶

    PostgreSQL 由多个模块组成，包括客户端、连接管理器、SQL 解析器、查询优化器、执行器和存储引擎。

    1. PostgreSQL 的体系架构¶

    • 客户端/连接层：负责接收客户端连接并进行权限认证。支持多种连接方式，包括 TCP、UNIX Socket 等。
    • SQL 解析器：将 SQL 查询转换成抽象语法树（AST），供优化器使用。
    • 查询优化器：根据统计信息选择最佳查询计划，并生成执行计划。
    • 执行器：负责执行查询计划，调用存储引擎读取数据并返回给客户端。
    • 存储引擎：管理数据的物理存储，包括表、索引、事务日志等。PostgreSQL 使用单一存储引擎，提供统一的接口和操作逻辑。
    • WAL（Write-Ahead Logging）日志：确保数据的持久性和一致性，保证在系统崩溃后数据不会丢失。

    2. 主要组件¶

    • MVCC（多版本并发控制）：PostgreSQL 通过维护多个数据版本，实现高效的并发控制，避免读写冲突。
    • WAL 日志：使用预写日志技术，确保数据写操作的原子性和系统的崩溃恢复能力。
    • 存储和索引：PostgreSQL 支持 B-Tree、GIN、GiST 等多种索引结构，用于加速查询。
    • Buffer Pool（缓冲池）：将常用的数据和索引缓存到内存中，提高查询性能。

    3. PostgreSQL 的存储结构¶

    • 表：数据按行存储，每行记录有多个列。PostgreSQL 的表文件采用行存储。
    • 索引：常用的索引结构包括 B-Tree、Hash、GIN、GiST 等。不同的索引类型适用于不同的查询场景。
    • 事务管理：通过 MVCC、WAL、锁机制实现高效事务管理，避免事务冲突。

    三、常用 PostgreSQL CLI 命令¶

    PostgreSQL 的常用 CLI 命令和 SQL 查询语句涵盖了数据库管理、表操作、数据查询等方面。以下是一些常见的 PostgreSQL CLI 命令和 SQL 查询语句的分类和说明。

    1. 连接到 PostgreSQL
    psql -U username -h hostname -d dbname -p port
    • username: 连接数据库的用户名。
    • hostname: 数据库服务器的地址（通常是 localhost）。
    • dbname: 要连接的数据库名称。
    • port: 数据库端口（默认是 5432）。

    示例：

    psql -U postgres -h localhost -d mydb -p 5432
    1. 列出所有数据库
    \l
    1. 创建数据库
    CREATE DATABASE [数据库名];
    1. 删除数据库
    DROP DATABASE [数据库名];
    1. 切换到某个数据库
    \c dbname
    1. 列出当前数据库的表
    \dt
    1. 显示表的结构
    \d tablename
    1. 退出 psql
    \q
    1. 显示帮助信息
    \h

    这将显示 SQL 命令的帮助。

    1. 列出所有用户
    \du

    1. 查看 PostgreSQL 的版本

      SELECT version();

    2. 导入和导出数据库

      • 导出：
      pg_dump -U username dbname > backup.sql
      • 导入：
      psql -U username dbname < backup.sql

    四、字段类型¶

    1. 9 大核心类型（必须会）¶

    类型族 具体写法 尺寸 范围/精度 何时用它
    整数 smallint / int2 2 B –32 768 ~ +32 767 状态码、月份、库存量等绝对超不过 3 万的小数字
    integer / int4 4 B –21 亿 ~ +21 亿 主键、计数器、外键；默认先用它
    bigint / int8 8 B –9E18 ~ +9E18 微博级计数、雪花 ID、分布式序列
    精确小数 numeric(p,s) / decimal 每 4 位≈2 B 最大 131 072 位 金额、税率、报表——要求分毫不差
    浮点 real / float4 4 B 6 位十进制精度 科学计算、传感器值，可容忍误差
    double precision / float8 8 B 15 位十进制精度 机器学习特征、GIS 距离，默认浮点选它
    字符串 varchar(n) 1 B + 数据 最大 1 GB 长度分布差异大，又想加硬上限（手机号、邮箱）
    text 1 B + 数据 最大 1 GB 默认字符串——性能与 varchar 相同，先选它
    char(n) 定长 n 不足补空格 仅用于定长编码：ISO 货币代码、身份证号
    布尔 boolean / bool 1 B（实际 1 bit） true/false/null 是否启用、是否删除；不要再用 char(1) 存 Y/N
    日期时间 date 4 B 4713 BC ~ 5874897 AD 生日、账单日——不带时分秒
    timestamp[(p)] 8 B 同上+毫秒 订单创建、日志时间——默认选它
    timestamptz / timestamp with time zone 8 B 同上，内部 UTC 跨时区业务（航班、全球活动）——存它，展示层转时区
    time[(p)] / timetz 8 B 00:00:00 ~ 24:00:00 每日重复时段（打铃、抢购）很少用
    区间 interval 16 B –178 000 000 年 ~ +178 000 000 年 时长、倒计时、排班跨度

    2. 4 个通用扩展（知道即可）¶

    场景 类型 说明
    主键、分布式 ID bigserial / identity 语法糖，背后仍是 bigint + 序列；PG 10+ 推荐 generated by default as identity
    货币 money 固定小数 2 位，受 locale 影响，新项目建议 numeric(19,2)
    枚举 create type mood as enum ('happy','sad') 值列表固定且极少变；比 check 省空间、可读好
    JSON json 存原文，jsonb 存二进制 需要路径查询、索引 → 选 jsonb；只接收/转发 → json 更快

    3. 选型口诀¶

    1. 整数默认 integer，主键自增 bigint + identity
    2. 金额必须 numeric(19,2)，浮点只给科学/统计
    3. 字符串先 text，真想限制长度再加 check 约束
    4. 时间戳默认 timestamptz，省得跨时区翻车
    5. 布尔就 boolean，别再 char(1) 存 Y/N

    五、常用 PostgreSQL 查询 SQL¶

    1. 查询所有数据库
    SELECT datname FROM pg_database;
    1. 查询所有表
    SELECT table_name FROM information_schema.tables WHERE table_schema='public';
    1. 查询表的结构
    SELECT column_name, data_type FROM information_schema.columns WHERE table_name='your_table_name';
    1. 创建表
    CREATE TABLE employees (
    id SERIAL PRIMARY KEY,
    name VARCHAR(100),
    position VARCHAR(50),
    salary DECIMAL
);
    1. 插入数据
    INSERT INTO employees (name, position, salary) VALUES ('Alice', 'Manager', 75000);
    1. 查询数据
    SELECT * FROM employees;
    1. 更新数据
    UPDATE employees SET salary = 80000 WHERE name = 'Alice';
    1. 删除数据
    DELETE FROM employees WHERE name = 'Alice';
    1. 条件查询
    SELECT * FROM employees WHERE salary > 50000;

    1. 分页查询

      SELECT * FROM employees ORDER BY id LIMIT 10 OFFSET 20;

    2. 统计查询

      • 统计记录数：
      SELECT COUNT(*) FROM employees;
      • 统计平均工资：
      SELECT AVG(salary) FROM employees;

    3. 分组查询

      SELECT position, AVG(salary) FROM employees GROUP BY position;

    4. 创建索引

      CREATE INDEX idx_position ON employees(position);

    5. 查看表的索引

      SELECT indexname, indexdef FROM pg_indexes WHERE tablename = 'employees';

    6. 删除表

      DROP TABLE employees;

    六、其他实用的 PostgreSQL SQL 语句¶

    1. 查看数据库连接数
    SELECT datname, numbackends FROM pg_stat_database;
    1. 查看当前活动的查询
    SELECT pid, datname, usename, state, query FROM pg_stat_activity;
    1. 强制终止某个查询进程
    SELECT pg_terminate_backend(pid);
    1. 创建用户
    CREATE USER username WITH PASSWORD 'password';
    1. 授予用户权限
    GRANT ALL PRIVILEGES ON DATABASE dbname TO username;
    1. 修改用户密码
    ALTER USER username WITH PASSWORD 'newpassword';
    1. 删除用户
    DROP USER username;

    七、性能监控查询¶

    1. 查看最慢的 SQL 查询
    SELECT query, calls, total_time, mean_time FROM pg_stat_statements ORDER BY mean_time DESC LIMIT 10;
    1. 查看数据库的 IO 状况
    SELECT * FROM pg_stat_bgwriter;
    1. 查看表的锁信息
    SELECT * FROM pg_locks WHERE relation IN (SELECT oid FROM pg_class WHERE relname = 'employees');

    八、索引与优化查询¶

    1. 分析表以优化查询计划
    ANALYZE employees;
    1. 查看表的索引状态
    SELECT * FROM pg_stat_user_indexes WHERE relname = 'employees';
    1. 查询缺失索引的信息
    SELECT * FROM pg_stat_user_tables WHERE schemaname = 'public' AND idx_scan = 0;

    这些 PostgreSQL 常用的 CLI 命令和 SQL 查询可以帮助你进行日常的数据库管理和优化操作。根据需要选择合适的命令和查询语句。

    九、常用命令¶

    • 连接查询: 使用 psql 命令连接数据库时，如果不指定要连接的数据库名，它默认会尝试连接一个与用户名相同的数据库。
    # 设置默认的数据库名
export PGDATABASE=postgres

# 注意
postgres： 一个默认数据库。它通常用于管理目的，比如创建和删除数据库、用户管理等。它的存在是为了提供一个默认的管理数据库，方便数据库管理员进行日常管理操作。
template0：一个模板数据库。它是一个只读数据库，不能被修改。template0 的存在是为了提供一个干净的、最初始状态的模板。

template1：一个模板数据库，但与 template0 不同，它是可写的。template1 通常包含一些用户自定义的设置或扩展模块。

    十、PostgreSQL 高频面试题及解答¶

    1. PostgreSQL 与 MySQL 的区别是什么？¶

    • 事务支持：PostgreSQL 对事务的支持更加完备，支持复杂的事务隔离级别。
    • 数据类型：PostgreSQL 支持更多的数据类型，如数组、JSON、XML 等，适用于数据处理需求复杂的场景。
    • 扩展性：PostgreSQL 可以扩展新的数据类型和函数，更具灵活性。
    • 性能：MySQL 在读操作较多的情况下性能更好，而 PostgreSQL 在复杂查询和写密集型任务上表现优异。

    2. 什么是 MVCC？如何实现？¶

    • MVCC（Multi-Version Concurrency Control） 是一种多版本并发控制机制，避免了长事务导致的阻塞问题。
    • PostgreSQL 通过版本控制为每条记录保存多个版本，读操作读取旧版本数据，写操作创建新版本，同时利用 xmin 和 xmax 实现事务隔离。

    3. PostgreSQL 的锁机制有哪些？¶

    • 行级锁：包括共享锁和排他锁，锁粒度较小。
    • 表级锁：如访问独占锁、行共享锁等，控制并发访问。
    • 乐观锁与悲观锁：乐观锁通过版本控制检测冲突，悲观锁通过阻塞方式避免冲突。

    4. PostgreSQL 如何实现崩溃恢复？¶

    PostgreSQL 通过 WAL（Write-Ahead Logging）实现崩溃恢复。事务执行前将操作写入 WAL 日志，即便系统崩溃也可通过日志恢复未完成的事务，确保数据一致性。

    5. PostgreSQL 中的常用索引类型及适用场景？¶

    • B-Tree 索引：适用于等值、范围查询，默认索引类型。
    • Hash 索引：适用于等值查询，支持较快的哈希定位。
    • GIN（Generalized Inverted Index）索引：用于全文搜索、数组、JSON 字段查询。
    • GiST（Generalized Search Tree）索引：支持空间数据查询。

    6. 如何优化 PostgreSQL 的查询性能？¶

    • 合理使用索引：创建合适的索引，提高查询效率。
    • SQL 优化：尽量减少复杂查询，拆分子查询。
    • 缓存机制：调整 shared_buffers 大小，减少磁盘 IO。
    • 分区表：对大表分区，减少扫描的数据量。
    • VACUUM 操作：定期进行 VACUUM 操作，回收死元组，减少存储空间浪费。

    7. PostgreSQL 的事务隔离级别有哪些？¶

    PostgreSQL 支持 4 种事务隔离级别：

    • READ UNCOMMITTED：允许读取未提交的数据，容易产生脏读。
    • READ COMMITTED：只读取已提交的数据，防止脏读。
    • REPEATABLE READ：在事务中保证多次读取的数据一致性，防止不可重复读。
    • SERIALIZABLE：最高级隔离，保证事务串行执行，避免幻读。

    8. 如何实现 PostgreSQL 的主从复制？¶

    • 流复制（Streaming Replication）：主库将 WAL 日志实时同步到从库，实现数据复制。
    • 逻辑复制（Logical Replication）：只复制特定表或数据，适合多租户场景。
    • 同步复制：主从库同步写入，确保数据一致性。

    9. PostgreSQL 的常用命令有哪些？¶

    • 连接数据库：psql -U username -d dbname
    • 显示数据库：\l
    • 显示表：\dt
    • 描述表结构：\d tablename
    • 导出数据：pg_dump -U username dbname > dump.sql
    • 恢复数据：psql -U username -d dbname < dump.sql

    10. PostgreSQL 中的分区表如何使用？¶

    PostgreSQL 支持表分区，适用于大表。分区方法包括范围分区、哈希分区等。通过分区可以显著减少查询数据量，提高查询效率。
