    1.CPU查询¶

    总核数 = 物理CPU个数 X 每颗物理CPU的核数
    总逻辑CPU数 = 物理CPU个数 X 每颗物理CPU的核数 X 超线程数

    lscpu
cat /proc/cpuinfo
cat /proc/cpuinfo | grep name | cut -f2 -d: | uniq -c   # cpu型号
cat /proc/cpuinfo| grep "physical id"| sort| uniq| wc -l  # 查看物理CPU个数
cat /proc/cpuinfo| grep "cpu cores"| uniq  # 查看每个物理CPU中core的个数(即核数)
cat /proc/cpuinfo| grep "processor"| wc -l  # 查看逻辑CPU的个数（超线程）
    2.CPU架构¶

    uname -m 常见输出值及其含义:

    x86:

    • i386: 32位Intel 80386（也称为x86）架构。
    • i486: 32位Intel 80486架构。
    • i586: 32位Intel Pentium架构。
    • i686: 32位Intel Pentium Pro和后续版本的架构，通常用于32位Linux系统。
    • x86_64: 64位Intel和AMD架构，也称为AMD64或Intel 64。表示系统支持64位指令集。

    ARM:

    • arm: ARM架构的通用标识，通常是32位。
    • armv7l: ARMv7架构，32位。
    • armv8l: ARMv8架构的32位版本。
    • aarch64: 64位ARM架构（ARMv8-A），也称为ARM64。

    PowerPC:

    • ppc: 32位PowerPC架构。
    • ppc64: 64位PowerPC架构。
    • ppc64le: 64位PowerPC架构，采用小端序（Little Endian）。

    SPARC:

    • sparc: 32位SPARC架构。
    • sparc64: 64位SPARC架构。

    MIPS:

    • mips: 32位MIPS架构，通常是大端序（Big Endian）。
    • mipsel: 32位MIPS架构，采用小端序（Little Endian）。
    • mips64: 64位MIPS架构。
    • mips64el: 64位MIPS架构，采用小端序（Little Endian）。

    Itanium:

    • ia64: Intel Itanium 64位架构，不同于x86_64。

    Others:

    • s390: IBM的System/390架构。
    • s390x: IBM的System/390架构的64位扩展版本。
    x86 家族通常代表Intel和AMD制造的基于CISC指令集的处理器，它们广泛用于桌面电脑、笔记本和服务器。

    3.平均负载¶

    Tasks: 376, 3360 thr; 1 running
Load average: 1.79 1.52 1.18
Uptime: 18 days, 04:43:09

    解释：

    Load average（负载平均值）：

    1. 概念
    2. 负载平均值表示系统在过去一段时间内的平均负载情况。它是一个衡量系统繁忙程度的指标，主要考虑的是 CPU 和等待 CPU 的可运行进程（包括线程）的数量。
    3. 三个时间范围的值（1.79、1.52、1.18）
    4. 这三个值分别代表过去 1 分钟、5 分钟和 15 分钟的系统负载平均值。例如，1.79 的 1 分钟负载平均值表示在过去 1 分钟内，系统平均有 1.79 个进程（线程）在等待 CPU 或者正在使用 CPU。
    5. 如果系统的负载平均值长期大于系统的 CPU 核心数，说明系统 CPU 负载较重，可能存在性能瓶颈。例如，对于一个有 4 个 CPU 核心的系统，如果负载平均值长时间大于 4，那么可能需要优化系统的 CPU 使用情况或者增加 CPU 资源。

    3.2.1.查找¶
    方法1：使用 lsof 命令¶
    # 统计每个进程打开的文件数
lsof | awk '{print $2}' | sort | uniq -c | sort -rn | head -20

    方法2：使用 /proc 文件系统¶
    # 统计每个进程的文件描述符数量
ls -1 /proc/*/fd 2>/dev/null | awk -F/ '{print $3}' | sort | uniq -c | sort -rn | head -20

# 更详细的统计（显示进程名）
for pid in /proc/*/cmdline; do
    pid_num=$(echo $pid | cut -d/ -f3)
    if [[ $pid_num =~ ^[0-9]+$ ]]; then
        count=$(ls /proc/$pid_num/fd 2>/dev/null | wc -l)
        name=$(cat /proc/$pid_num/cmdline 2>/dev/null | tr '\0' ' ' | head -c 50)
        echo "$count $pid_num $name"
    fi
done 2>/dev/null | sort -rn | head -20

    6.3.自动保活¶

    方法一：修改用户级别的ssh_config文件¶

    修改~/.ssh/config文件， 添加：

    Host *
    ServerAliveInterval 60
    ServerAliveCountMax 3
    方法二：使用命令行参数临时设置¶

    如果不想修改配置文件，也可以在连接时通过命令行参数临时设置保持活动状态：

    ssh -o ServerAliveInterval=60 -o ServerAliveCountMax=3 user@remote_host
    • ServerAliveInterval=60：每60秒向服务器发送一次保持活动状态的数据包。
    • ServerAliveCountMax=3：如果服务器在3个间隔内没有响应，SSH客户端将自动断开连接。
    方法三：修改全局的ssh_config文件¶

    如果希望所有用户都使用这个设置，可以修改全局的ssh_config文件：

    1. 打开/etc/ssh/ssh_config文件：
    sudo nano /etc/ssh/ssh_config
    1. 添加或修改配置： 在文件中找到Host *部分（如果没有则添加），并添加以下内容：
    Host *

    6.7.rsync¶

    rsync 是一个用于文件和目录同步的强大工具，常用于备份和镜像。以下是 rsync 的详细解析和常用选项：

    1.基本语法¶
    rsync [options] source destination

    常用选项:

    • -a: 归档模式，等同于 -rlptgoD，保留链接、权限、时间戳、组和设备文件等。
    • -v: 详细输出，显示传输过程。
    • -z: 压缩文件数据，适合传输大文件。
    • -r: 递归传输目录及其内容。
    • -e: 指定远程 shell 程序，如 ssh。
    • --delete: 删除目标目录中源目录没有的文件。
    • --progress: 显示传输进度。
    • --exclude: 排除指定的文件或目录。
    2.示例¶
    1. 本地目录同步：
    rsync -av /source/directory/ /destination/directory/
    1. 通过 SSH 同步到远程服务器：
    rsync -avz -e "ssh -p <port>" /local/directory/ user@remote:/remote/directory/
    1. 从远程服务器同步到本地：
    rsync -avz -e ssh user@remote:/remote/directory/ /local/directory/
    1. 同步并删除目标中多余的文件：
    rsync -av --delete /source/directory/ /destination/directory/
    1. 排除特定文件或目录：
    rsync -av --exclude 'node_modules' /source/directory/ /destination/directory/

    注意事项:

    • 斜杠的用法：在源目录路径后加斜杠（/）表示同步目录内容，不加则表示同步整个目录。
    • 权限和所有权：确保具有足够的权限进行文件同步，特别是在远程服务器上。
    • 安全传输：通过 SSH 使用 rsync 时，确保 SSH 密钥配置正确，以便安全和无密码访问。
