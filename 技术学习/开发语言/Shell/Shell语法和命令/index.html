    1.6.3.工具对比¶

    工具 压缩率 压缩速度 解压速度 格式 特点和适用场景
    zstd 较高 极快 极快 .zst Zstandard，由 Facebook 开发。非常适合大文件的快速压缩解压，且压缩率和速度有很好的平衡。适合备份和大型数据文件的压缩。
    gzip 中等 快 快 .gz 广泛使用且兼容性好，常用于日志文件的压缩。压缩率一般，但解压速度不错，适合系统日志或通用用途。
    bzip2 较高 慢 较慢 .bz2 压缩率较高，但速度较慢，适合压缩率要求较高但速度要求不高的文件。
    xz 非常高 很慢 较慢 .xz 适合高压缩率需求的场景，比如软件发行包等。压缩速度较慢，但解压速度尚可。
    lz4 较低 极快 极快 .lz4 速度极快但压缩率较低。适合对压缩时间敏感、但对压缩率要求不高的场景，如数据库备份等。
    7zip 非常高 较慢 较慢 .7z 支持多种压缩算法，压缩率较高，适合多文件或目录的归档压缩。
    tar 不压缩 N/A N/A .tar 只进行打包，结合其他压缩算法使用（如 tar.gz、tar.xz），适合将多个文件打包为一个归档。
    zip 中等 中等 中等 .zip 跨平台兼容性好，压缩率和速度均中等，适合在不同操作系统间共享的文件。
    1. zstd (Zstandard)¶
    • 开发者: Facebook
    • 特点: 速度极快，同时在压缩率上表现优异。可以在压缩效率和压缩比之间灵活调整，提供不同的压缩级别（1-19）。
    • 适用场景: 大文件、备份文件、日志压缩等，尤其适合对速度有较高要求的场景。
    2. gzip¶
    • 特点: 使用 Deflate 算法，压缩率中等，但速度相对较快。几乎所有 Linux 发行版都默认支持。
    • 适用场景: 日志文件压缩、备份、软件分发等。较为通用且兼容性好。
    3. bzip2¶
    • 特点: 使用 Burrows-Wheeler 压缩算法，压缩率比 gzip 高，但速度较慢。
    • 适用场景: 数据压缩率要求高，但速度要求不高的场景，如源代码压缩。
    4. xz¶
    • 特点: 使用 LZMA2 算法，压缩率极高，但速度较慢，尤其是在压缩时。
    • 适用场景: 需要极高压缩率但不关注速度的场景，如发行版包压缩（如 Linux 内核和软件包）。
    5. lz4¶
    • 特点: 速度极快，但压缩率较低。支持流式压缩，因此适合实时数据压缩。
    • 适用场景: 速度优先的场景，如数据库备份或实时数据传输。
    6. 7zip (p7zip)¶
    • 特点: 支持多种压缩算法，默认使用 LZMA，压缩率较高但速度较慢。支持加密和压缩多个文件。
    • 适用场景: 多文件打包和压缩，适合对压缩率和安全性有较高要求的场景。
    7. tar¶
    • 特点: 只打包不压缩。常与 gzip、bzip2 或 xz 等压缩工具结合使用，例如 tar.gz、tar.xz。
    • 适用场景: 归档大量文件，尤其是在备份和分发多个文件时使用。
    8. zip¶
    • 特点: 支持多平台解压和基本的压缩率。可以对多个文件进行归档并压缩。
    • 适用场景: 适用于在不同平台之间（如 Windows 和 Linux）传输文件时的兼容性要求。
    9.使用场景的推荐工具¶
    1. 实时压缩: 使用 lz4，适合对速度有极高要求的场景。
    2. 大文件压缩且兼顾速度: 使用 zstd，在压缩速度和压缩率之间有良好平衡。
    3. 压缩率优先，速度可以较慢: 使用 xz，压缩率最高，但速度慢。
    4. 普通日志压缩和系统备份: 使用 gzip，适合通用用途。
    5. 归档多个文件或目录: 使用 tar，可结合其他压缩工具。
    6. 多平台兼容的压缩: 使用 zip，适合在 Linux、Windows、macOS 间传输。

# -x：要求整行匹配。
# -f file1：从 file1 中读取要排除的模式。
# sed 's/[[:space:]]*$//' file1：去掉 file1 中每一行的行末空白字符（包括空格和制表符）。
# sed 's/[[:space:]]*$//' file2：去掉 file2 中每一行的行末空白字符。
# <(...)：在 shell 中使用过程替换，将处理后的 file1 和 file2 直接传给 grep。

# 方法 2：使用 comm 命令
# 首先，需要确保文件是排序的（因为 comm 要求有序输入）：
sort file1 -o file1
sort file2 -o file2
# 然后使用 comm 命令：
comm -23 file2 file1
comm -23 <(sort <(sed 's/[[:space:]]*$//' file2)) <(sort <(sed 's/[[:space:]]*$//' file1))
# 解释：
# -2：隐藏 file1 独有的行。
# -3：隐藏 file1 和 file2 共有的行。
# sed 's/[[:space:]]*$//'：去掉行末空格或其他空白字符。
# sort：对去掉空白字符的内容进行排序，以便 comm 正常工作。
# <(...)：使用过程替换来处理和排序 file1 和 file2，然后传给 comm。

# 方法 3：使用 awk 命令
awk 'NR==FNR {a[$0]; next} !($0 in a)' file1 file2
# 解释：
# NR==FNR：处理 file1 时，逐行读取并将行内容存入数组 a。
# !($0 in a)：在 file2 中检查每行，如果该行内容不在数组 a 中，则输出。

    1.11.FTP上传/下载¶

    参考-ftp上传下载

    FTP文件传输类型有： ascii、binary、ebcdic、image、local M 和 tenex。

    • – ascii：将文件传输类型设置为网络 ASCII。此类型为缺省值，即默认使用ascii方式进行传输。
    • – binary：将文件传输类型设置为二进制映像。需要使用binary方式传输的文件类型有ISO文件、可执行文件、压缩文件、图片等。此类型可能比 ASCII 传送更有效。

    • – image：将文件传输类型设置为二进制映像。此类型可能比 ASCII 传送更有效。
    • – local M：将文件传输类型设置为本地。M 参数定义每计算机字位的十进制数。此参数没有缺省值。
    • – tenex：将文件传输类型设为 TENEX 机器需要的类型。

    2.3.替换单引号和布尔值¶

    假设查询结果的 JSON 形如 {'Manip KNWL': True, 'AirExo': True, 'Brain': True}，我们可以按照以下步骤进行处理：

    2.3.1.方法一：使用 sed 替换单引号和布尔值¶

    可以通过 sed 同时替换单引号为双引号，并将 True 和 False 转换为符合 JSON 标准的小写布尔值 true 和 false。

    message="{'Manip KNWL': True, 'AirExo': True, 'Brain': True}"

# 使用 sed 将单引号替换为双引号，并转换布尔值为小写的 true 和 false
formatted_message=$(echo "${message}" | sed "s/'/\"/g; s/True/true/g; s/False/false/g")

# 输出格式化的 JSON
echo "${formatted_message}" | jq --indent 4 -r

    解释

    • s/'/\"/g: 将单引号 ' 替换为双引号 "。
    • s/True/true/g: 将 True 替换为小写的 true，符合 JSON 标准。
    • s/False/false/g: 将 False 替换为小写的 false，符合 JSON 标准。

    2.3.2.方法二：使用 tr 和 sed 结合¶

    你也可以使用 tr 和 sed 的组合来进行更精细的控制：

    message="{'Manip KNWL': True, 'AirExo': True, 'Brain': True}"

# 使用 tr 将所有单引号替换为双引号
formatted_message=$(echo "${message}" | tr "'" "\"")

# 使用 sed 将布尔值替换为小写
formatted_message=$(echo "${formatted_message}" | sed "s/True/true/g; s/False/false/g")

# 输出格式化后的 JSON
echo "${formatted_message}" | jq --indent 4 -r

    输出

    无论使用哪种方法，最终会得到如下格式化输出：

    {
    "Manip KNWL": true,
    "AirExo": true,
    "Brain": true
}

    3.参数传递¶

    3.1.位置参数¶

    Shell 脚本通过位置参数接收命令行输入，这些参数按顺序编号，格式为 $n（n 为正整数）：

    • $0：脚本自身的文件名（包含路径时为完整路径）
    • $1：第一个参数，$2：第二个参数，以此类推（$9 之后需用 ${10} 形式访问）
    • 示例：
    • 执行 ./script.sh arg1 arg2 arg3 时，脚本内 $1=arg1，$2=arg2，$3=arg3。

    3.2.特殊变量-关键变量¶

    除了位置参数，Shell 提供了一组特殊变量用于处理参数集合和状态：

    变量 含义
    $# 参数的总数量（不包含 $0）
    $* 所有参数的集合，以空格分隔，视为单个字符串（"$*" 等价于 "$1 $2 ... $n"）
    $@ 所有参数的集合，每个参数独立（"$@" 等价于 "$1" "$2" ... "$n"）
    $? 上一条命令的退出状态（0 表示成功，非 0 表示失败）
    $$ 当前脚本的进程 ID（PID）
    $! 上一个后台进程的 PID

    在bash中，"${*}"和"$@"都是用来处理传递给脚本或函数的所有参数，但它们的行为有所不同：

    1. "$*"：将所有参数视为一个单一的字符串。如果使用双引号括起来，所有参数会被合并成一个字符串，参数之间的分隔符（通常是空格）会被保留，但参数之间的边界会丢失。这意味着如果某个参数本身包含空格，使用"$*"可能会导致参数被错误地分割。
    2. "$@"：保留每个参数的独立性。当使用双引号括起来时，"$@"会展开成每个参数作为一个独立的单词，保持原有的参数边界。这是处理参数列表的推荐方式，特别是当参数可能包含空格或其他特殊字符时。

    示例脚本（test.sh）：

    #!/bin/bash
echo "使用 \$* 遍历："
for arg in "$*"; do
  echo "参数：$arg"
done

echo -e "\n使用 \$@ 遍历："
for arg in "$@"; do
  echo "参数：$arg"
done

    执行 ./test.sh "hello world" foo，输出：

    使用 $* 遍历：
参数：hello world foo

使用 $@ 遍历：
参数：hello world
参数：foo

    3.3.参数解析方法¶

    根据参数类型（普通参数、带选项的参数），解析方式不同：

    1. 基础解析：直接读取位置参数¶

    适用于参数数量固定、无选项的简单场景。

    示例：计算两个数的和（sum.sh）

    #!/bin/bash
# 检查参数数量是否为 2
if [ $# -ne 2 ]; then
  echo "用法：$0 <数1> <数2>"
  exit 1  # 非 0 退出表示错误
fi

num1=$1
num2=$2
sum=$((num1 + num2))
echo "$num1 + $num2 = $sum"

    执行 ./sum.sh 3 5，输出 3 + 5 = 8。

    2. 选项解析：处理带 - 或 -- 的参数¶

    当脚本需要支持选项（如 -h 显示帮助、-v 开启 verbose 模式）时，需用解析工具：

    （1）getopts：处理短选项（推荐）¶

    getopts 是 Shell 内置命令，用于解析短选项（如 -a、-b val），格式：getopts "选项字符串" 变量名

    • 选项字符串中，: 表示该选项需要参数（如 "a:b" 表示 -a 需要参数，-b 不需要）
    • ? 表示未知选项，: 表示缺少选项参数

    示例（option_demo.sh）：

    #!/bin/bash
verbose=0
output_file=""

# 解析选项：-v（无参数）、-o（需参数）
while getopts "vo:" opt; do
  case $opt in
    v)  verbose=1 ;;  # 开启 verbose 模式
    o)  output_file="$OPTARG" ;;  # 保存 -o 后的参数到 output_file
    \?) echo "无效选项: -$OPTARG" >&2; exit 1 ;;
    :)  echo "选项 -$OPTARG 需要参数" >&2; exit 1 ;;
  esac
done

# 解析完选项后，剩余参数从 $OPTIND 开始（$OPTIND 是 getopts 自动维护的索引）
shift $((OPTIND - 1))  # 移除已解析的选项，保留剩余参数

echo "verbose: $verbose"
echo "output_file: $output_file"
echo "剩余参数: $*"

    执行 ./option_demo.sh -v -o result.txt file1 file2，输出：

    verbose: 1
output_file: result.txt
剩余参数: file1 file2
    （2）getopt：支持长选项（如 --help）¶

    getopt 是外部工具（需安装），支持长选项（--help）和短选项，语法较复杂，适合复杂场景。

    示例（简化版）：

    #!/bin/bash
# 解析短选项 -v、长选项 --output
args=$(getopt -o v --long output: -n "$0" -- "$@")
eval set -- "$args"  # 重设位置参数

while true; do
  case "$1" in
    -v) verbose=1; shift ;;
    --output) output="$2"; shift 2 ;;
    --) shift; break ;;  # 结束选项解析
  esac
done

echo "verbose: $verbose, output: $output, 剩余参数: $*"

    执行 ./long_opt.sh -v --output log.txt data，输出：

    verbose: 1, output: log.txt, 剩余参数: data
    （3） 参数默认值¶

    使用 ${n:-默认值} 为参数设置默认值（当参数未提供时生效）：

    #!/bin/bash
# 若 $1 未提供，默认使用 "default"
name=${1:-default}
echo "Hello, $name"

    执行 ./default.sh 输出 Hello, default；执行 ./default.sh Alice 输出 Hello, Alice。

    （4） 检查参数是否存在¶

    结合 $# 和条件判断，确保必要参数已提供：

    #!/bin/bash
if [ $# -eq 0 ]; then
  echo "错误：请提供至少一个参数" >&2  # >&2 表示输出到错误流
  exit 1
fi
    （5）处理含空格 / 特殊字符的参数¶

    传递含空格或特殊字符（如 *、?）的参数时，需用双引号包裹，并在脚本中用 "$@" 接收：

    # 执行脚本时：
./script.sh "hello world" "a*b"

# 脚本内用 "$@" 遍历，会正确保留参数原样
for arg in "$@"; do
  echo "参数：$arg"
done

    4.数组操作¶

    4.1.数组合并¶

    示例代码：

    #!/bin/bash

# 获取终端输入的参数,组成数组
opts=("${@:2}")
# 定义两个数组
array1=(1 2 3)
array2=(4 5 6)

# 合并 array1 和 array2
array1+=("${array2[@]}")

# 输出合并后的 array1
echo "Merged array: ${array1[@]}"

    说明：

    • array1+=("${array2[@]}")：使用 += 操作符将 array2 的所有元素 ("${array2[@]}") 追加到 array1 中。
    • ("${array2[@]}") 是为了将 array2 中的所有元素展开并追加到 array1 中。

    输出结果：

    Merged array: 1 2 3 4 5 6

    4.2.数组去重¶

    如果需要在合并两个数组后去掉重复的元素，可以使用以下方法：

    #!/bin/bash

# 定义两个数组
array1=(1 2 3 4)
array2=(3 4 5 6)

# 合并两个数组
array1+=("${array2[@]}")

# 去重并合并为新数组
unique_array=($(echo "${array1[@]}" | tr ' ' '\n' | sort -u | tr '\n' ' '))

# 输出去重后的数组
echo "Merged unique array: ${unique_array[@]}"

    说明：

    • tr ' ' '\n'：将数组元素转换为换行符分隔的格式。
    • sort -u：对元素进行排序并去重。
    • tr '\n' ' '：将换行符转换回空格，以便重新将元素组成数组。

    4.3.数组判断¶

    shell判断字符串是否在数组中

    4.3.1.方法 1：使用 for 循环¶

    示例：

    #!/bin/bash

# 定义数组
arr=("apple" "banana" "orange" "grape")

# 要查找的字符串
target="banana"

# 默认未找到
found=0

# 遍历数组
for item in "${arr[@]}"; do
    if [[ "$item" == "$target" ]]; then
        found=1
        break
    fi
done

# 判断结果
if [[ $found -eq 1 ]]; then
    echo "$target 在数组中"
else
    echo "$target 不在数组中"
fi

    4.3.2.方法 2：使用 printf 和 grep 检查字符串¶

    通过 printf 将数组转换为字符串，并用 grep 检查是否存在目标字符串。

    示例：

    #!/bin/bash

# 定义数组
arr=("apple" "banana" "orange" "grape")

# 要查找的字符串
target="banana"

# 使用 printf 和 grep 检查是否存在
if printf "%s\n" "${arr[@]}" | grep -q -w "$target"; then
    echo "$target 在数组中"
else
    echo "$target 不在数组中"
fi

    4.3.3.方法 3：使用 Shell 关联数组（Bash 4.0+）¶

    在 Bash 4.0+ 中，可以使用关联数组（hash map）提高查找效率。

    示例：

    #!/bin/bash

# 定义数组
arr=("apple" "banana" "orange" "grape")

# 使用关联数组来创建查找表
declare -A lookup
for item in "${arr[@]}"; do
    lookup["$item"]=1
done

# 要查找的字符串
target="banana"

# 检查是否在数组中
if [[ ${lookup[$target]} ]]; then
    echo "$target 在数组中"
else
    echo "$target 不在数组中"
fi

    4.3.4.方法 4：使用内置的 [[ 运算符和 *¶

    在数组中直接检查是否包含某个字符串。

    示例：

    #!/bin/bash

# 定义数组
arr=("apple" "banana" "orange" "grape")

# 要查找的字符串
target="banana"

# 直接用字符串模式匹配
if [[ " ${arr[@]} " =~ " ${target} " ]]; then
    echo "$target 在数组中"
else
    echo "$target 不在数组中"
fi

    4.3.5.总结：¶

    1. for 循环：最常用的方法，简单易懂，适用于小数组。
    2. printf 和 grep：通过 grep 来查找目标字符串，简洁高效。
    3. 关联数组：适用于 Bash 4.0+，使用哈希表提高查找效率，适合处理大数组。
    4. 内置匹配：简洁但性能不如关联数组高，适合简单的匹配操作。

    4.4.从数组中获取对象¶

    4.4.1.访问数组所有元素的值¶

    echo ${arr[@]}
echo ${arr[*]}

    4.4.2.获取数组所有索引¶

    echo ${!arr[@]}
echo ${!arr[*]}

    4.4.3.获取数组中非空元素的个数¶

    echo ${#arr[@]}
echo ${#arr[*]}

    4.4.4.遍历数组¶

    使用 for、while 或 until 来实现循环。例如：

    # 遍历方式1：使用for...in ${arr[@]}，可以获取每个元素的值
arr=(1 2 3 4 5)
for i in ${arr[@]}; do
    echo "Number: $i"
done

# 遍历方式2：使用for...in ${!arr[@]}，可以获取每个元素的索引，间接的可以获取每个元素的值
for i in ${!arr[@]};do
    echo index:$i
    echo value: ${arr[$i]}
done

# 遍历方式3：如果是数值索引类型，且每个元素都不为空，则还可以使用数组长度的方式遍历
for((i=0;i<${#arr[@]};i++)){
  echo $i
  echo ${arr[$i]}
}

    4.4.5.随机获取¶

    示例代码：

    # 定义一个数组
my_array=("apple" "banana" "cherry" "date" "elderberry")

# 获取数组的长度
array_length=${#my_array[@]}

# 使用 $RANDOM 对数组长度取模来生成随机索引
random_index=$((RANDOM % array_length))

# 获取随机索引对应的数组元素
random_element=${my_array[$random_index]}

# 输出随机选择的元素
echo "Random element: $random_element"

    解释：

    1. my_array=("apple" "banana" "cherry" "date" "elderberry"): 定义了一个包含多个字符串的数组。
    2. array_length=${#my_array[@]}: 获取数组的长度。
    3. random_index=$((RANDOM % array_length)): 生成一个 0 到 array_length-1 范围内的随机索引，$RANDOM 是 bash 内置的随机数生成器，生成的数字在 0 到 32767 之间。通过对数组长度取模，生成合法的数组索引。
    4. random_element=${my_array[$random_index]}: 使用随机索引从数组中获取随机字符串。
    5. echo "Random element: $random_element": 输出随机字符串。

    4.5.关联数组¶

    关联数组需要先使用declare -A来定义，然后才表示是关联数组。

    declare -A aarr
aarr[one]=1      # key部分可不加引号
aarr[six]=6
aarr[five]=5
aarr[nine]=9

    也可以使用括号直接赋值：

    aarr=([one]=1 [two]=2)

    访问关联数组中元素：

    $ echo ${aarr[one]}
1
$ echo ${aarr[six]}

    4.6.向shell函数传递数组引用¶

    多数人对Bash数组有误解，认为无法将Bash数组传递给函数参数，但实际上是可以的。

    Bash的declare命令有一个选项-n，可以定义变量的引用。例如：

    function ff(){
  declare -n _inner_arr__=$1 # 现在_inner_arr__和$1的值是同一个变量
  echo ${_inner_arr__[@]}
  #...do something...
  declare +n _inner_arr__  # 取消变量引用
}

# 传递数组
$ arr=(11 22 33)
$ ff arr
11 22 33

    declare命令是深入shell脚本必学的命令之一，请记住它很重要，以后遇到一些变量相关的功能需求，记得去找declare。

    4.7.扩展(插入)数组和剔除(删除)数组元素¶

    • [x] 将一个或多个元素添加到数组尾部(或者新建数组)，即其它编程语言中push和extend到数组的操作：：
    # 尾部添加一个元素、多个元素，如果arr当前不存在，则自动新建数组
$ arr+=(1)
$ arr+=(2 3 4)
$ declare -p arr
declare -a arr=([0]="1" [1]="2" [2]="3" [3]="4")
    • [x] 删除数组中某个索引位置的元素，即其它编程语言中对数组进行的delete或者remove操作：
    # 注意，对非尾部元素进行unset，数组的索引位不会改变，
# 即：unset所删除元素后面的元素不会自动左移
$ arr=(1 2 3 4)
$ unset arr[-1]
$ declare -p arr
declare -a arr=([0]="1" [1]="2" [2]="3")

$ unset arr[0]
$ declare -p arr  # 注意输出结果中索引值是从1开始的，并不是0
declare -a arr=([1]="2" [2]="3")
    • [x] 删除数组尾部元素，即其它编程语言中对数组进行pop的操作，只需unset最后一个元素即可：
    $ arr=(1 2 3 4)
$ unset arr[-1]
$ declare -p arr
declare -a arr=([0]="1" [1]="2" [2]="3")
    • [x] 删除数组头部元素(第一个元素)，即其它编程语言中对数组进行shift的操作：
    # 从索引位1开始取到最后一个元素，并将它们重新构建成数组
$ arr=(1 2 3 4)
$ arr=("${arr[@]:1}")
    • [x] 在数组头部插入元素，即其它编程语言中对数组进行unshift的操作：
    # 向arr数组头部插入1和2两个元素
$ arr=(3 4)  # 原数组
$ arr1=(1 2)
$ arr1+=("${arr[@]}")

    5.循环控制¶

    5.1.for循环¶

    1.基本语法¶

    for variable in list
do
    commands
done

    2.工作原理¶

    • 变量赋值：每次循环从 word_list 中取一个值赋给 variable
    • 循环执行：对列表中的每个值执行一次循环体
    • 循环结束：当列表中的所有值都被处理完后循环结束

    3.示例¶

    for i in 1 2 3 4 5
do
    echo "Number: $i"
done

# 使用序列
for i in {1..5}
do
    echo "Number: $i"
done

# 字母序列
for letter in {a..e}
do
    echo "Letter: $letter"
done

# 带步长的序列
for i in {1..10..2}
do
    echo "Number: $i"
done

# C语言风格的for循环
for ((i=0; i<5; i++))
do
    echo "Count: $i"
done

# 遍历文件
for file in *.txt
do
    echo "Processing: $file"
done

# 使用脚本参数
for param in "$@"
do
    echo "Parameter: $param"
done

# 简写方式
for param
do
    echo "Parameter: $param"
done

    5.2.while循环¶

    1.基础用法¶

    1.1.基本语法结构¶
    while condition
do
    commands
done
    1.2.工作原理¶
    • 条件检查：在每次循环开始前检查条件
    • 循环执行：条件为真时执行循环体
    • 循环结束：条件为假时结束循环
    1.3.示例¶
    # 基本格式


# 示例：计数器
count=1

    2.特殊用法¶

    2.1.while [[ $# -gt 0 ]]; do¶
    1.整体作用¶

    while [[ $# -gt 0 ]]; do 用于持续处理命令行参数，直到所有参数被解析完毕。

    语法元素 含义
    while condition; do ... done Bash循环结构：当condition为true时，重复执行do和done之间的代码块
    [[ ... ]] Bash的条件表达式语法（比传统[ ... ]功能更强，支持更多运算符）
    $# 特殊变量：表示当前脚本/函数接收到的位置参数总数（即命令行参数个数）
    -gt 比较运算符：表示“大于”（greater than）
    0 数值0
    2.逻辑解读¶

    [[ $# -gt 0 ]] 的字面意思是：“判断当前参数个数是否大于0”。 结合while循环，整体逻辑为：“只要还有未处理的命令行参数（参数个数>0），就持续执行循环体中的代码”。

    3.典型配套用法¶

    循环体内通常会配合 case 语句解析参数，并使用 shift 命令“移除”已处理的参数（使$#减1），直到$#变为0，循环结束。

    while [[ $# -gt 0 ]]; do
  case "$1" in          # 解析第一个参数（$1）
    --version)
      VERSION="$2"      # 处理带值的选项（如--version v1.0.0）
      shift 2           # 移除已处理的2个参数（--version和v1.0.0）
      ;;
    -h|--help)
      print_help        # 处理无值的选项（如--help）
      shift             # 移除已处理的1个参数（-h或--help）
      ;;
    *)
      # 处理未知参数
      echo "未知参数: $1"
      exit 1
      ;;
  esac
done
    4.使用场景推荐¶

    1）命令行参数解析（最常用）

    这是该语法的核心应用场景，用于处理脚本的选项（如--version、--output）、标志（如-v、-f）或位置参数。

    示例： 解析命令行参数（如--version、--registry等）：

    # ============================================
# Main Script
# ============================================
# Parse options
while [[ $# -gt 0 ]]; do  # 循环处理所有参数，直到参数个数为0
    case $1 in
        --version)
            VERSION="$2"
            shift 2  # 移除已处理的--version和值
            ;;
        --registry)
            DOCKER_REGISTRY="$2"
            IMAGE_NAME="${DOCKER_REGISTRY}/${DOCKER_NAMESPACE}/${APP_NAME}"
            shift 2
            ;;
        # ... 其他参数解析 ...
        *)
            log_error "Unknown option: $1"
            print_usage
            exit 1
            ;;
    esac
done

    2）批量处理输入参数

    当脚本需要接收多个“待处理对象”（如文件路径、ID列表等）时，可通过此循环逐个处理。

    示例：批量删除指定文件

    # 脚本用法：./delete_files.sh file1.txt file2.log file3.csv
while [[ $# -gt 0 ]]; do
  if [[ -f "$1" ]]; then
    rm "$1"
    echo "已删除: $1"
  else
    echo "跳过不存在的文件: $1"
  fi
  shift  # 处理下一个文件
done

    3）交互式参数确认

    结合 read 命令，可实现“持续接收用户输入直到满足条件”的交互逻辑。

    示例：接收用户输入的密码，直到非空

    while [[ $# -gt 0 ]]; do  # 此处$#可替换为自定义条件（如密码为空）
  read -p "请输入密码: " PASSWORD
  if [[ -z "$PASSWORD" ]]; then
    echo "密码不能为空，请重试"
  else
    break  # 密码非空，退出循环
  fi
done

    5.3.until 循环¶

    1.基本语法¶

    # 基本格式（与while相反，条件为假时执行）

do
    commands
done

    2.工作原理¶

    • 条件检查：在每次循环开始前检查条件
    • 循环执行：条件为假时执行循环体
    • 循环结束：条件为真时结束循环

    3.与while循环的区别¶

    循环类型 执行时机 使用场景
    while 条件为真时执行 满足条件时持续执行
    until 条件为假时执行 等待条件变为真

    4.示例¶

    count=1

do
    echo "Count: $count"
    ((count++))
done

until systemctl is-active --quiet nginx
do
    echo "Nginx is not running, waiting..."
    sleep 2
done
echo "Nginx is now running!"

# 等待文件创建
until [ -f /var/run/service.pid ]
do
    echo "PID file not found, waiting..."
    sleep 1
done
echo "Service started with PID: $(cat /var/run/service.pid)"

# 等待网络连接
until ping -c1 google.com &> /dev/null
do
    echo "No network connection, retrying..."
    sleep 5
done
echo "Network is up!"

    7.8.2.rsync¶

    rsync 是一个用于文件和目录同步的强大工具，常用于备份和镜像。以下是 rsync 的详细解析和常用选项：

    1.基本语法¶
    rsync [options] source destination

    常用选项:

    • -a: 归档模式，等同于 -rlptgoD，保留链接、权限、时间戳、组和设备文件等。
    • -v: 详细输出，显示传输过程。
    • -z: 压缩文件数据，适合传输大文件。
    • -r: 递归传输目录及其内容。
    • -e: 指定远程 shell 程序，如 ssh。
    • --delete: 删除目标目录中源目录没有的文件。
    • --progress: 显示传输进度。
    • --exclude: 排除指定的文件或目录。
    2.示例¶
    1. 本地目录同步：
    rsync -av /source/directory/ /destination/directory/
    1. 通过 SSH 同步到远程服务器：
    rsync -avz -e "ssh -p <port>" /local/directory/ user@remote:/remote/directory/
    1. 从远程服务器同步到本地：
    rsync -avz -e ssh user@remote:/remote/directory/ /local/directory/
    1. 同步并删除目标中多余的文件：
    rsync -av --delete /source/directory/ /destination/directory/
    1. 排除特定文件或目录：
    rsync -av --exclude 'node_modules' /source/directory/ /destination/directory/

    注意事项:

    • 斜杠的用法：在源目录路径后加斜杠（/）表示同步目录内容，不加则表示同步整个目录。
    • 权限和所有权：确保具有足够的权限进行文件同步，特别是在远程服务器上。
    • 安全传输：通过 SSH 使用 rsync 时，确保 SSH 密钥配置正确，以便安全和无密码访问。

function _prune_resource() {
    res_file="docs/resource.json"
    if [ -z "${example_RESOURCE_PRUNE:-""}" ]; then
        pages=(
            "ProjectToolboxList"
            "ProjectDatasetProcess"
            "DeploymentDownloadCenter"
            "BackstageManagementLicenseManagement"
            "BackstageManagementSoftware"
            "BackstageManagementDocker"
        )
    else
        IFS=$'\n' read -r -d '' -a pages < <(echo "${example_RESOURCE_PRUNE}" | tr ',;' '\n')
    fi

    for key in "${pages[@]}"; do
        echo -e "\t\033[31mDeleting resource:\033[0m \033[36m$key ...\033[0m"
        jq --indent 4 'def walk(f):
        . as $in |
        if type == "object" then
          reduce keys[] as $key
            ( {}; . + { ($key): ( $in[$key] | walk(f) ) } )
        elif type == "array" then
          map( walk(f) | select(type != "object" or .name != "'"$key"'") )
        else
          f
        end;
        walk(if type == "object" and .name == "'"$key"'" then empty else . end)' $res_file | sponge $res_file
    done
    log_info "Resource of service has pruned completed!"
}
