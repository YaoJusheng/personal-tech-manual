                    • ✔️ 粒度 1：基础导出 pub → 【上级可见，最常用，90% 场景】

                    • ✔️ 粒度 4：路径导出 pub(in 路径) → 【精准可见，高级封装】
                  • 2.导入语句
                    • ✔️ 语法 1：基础单项导入 → use 完整路径; 【最常用，70% 场景】
                    • ✔️ 语法 2：批量导入 → use 路径::{项1,项2,项3}; 【高频，20% 场景】
                    • ✔️ 语法 3：重命名导入 → use 路径::项 as 别名; 【重中之重，必学，解决命名冲突】
                    • ✔️ 语法 4：嵌套导入 → use 路径::{子路径1, 子路径2::{项}}; 【进阶高频，优雅简洁】
                    • ✔️ 语法 5：通配符导入 → use 路径::*; 【按需使用，谨慎】
                    • ✔️ 语法 6：父模块导入 → use 路径::{self, 项}; 【进阶，导入模块本身 + 模块内项】
                    • ✔️ 语法 7：根 / 父模块导入 → use crate::* / use super::* 【模块化必备】
                    • ✔️ 语法 8：从当前 crate 中的模块导入 （但不是从根开始）

    整型溢出¶

    假设有一个 u8 ，它可以存放从 0 到 255 的值。那么当你将其修改为范围之外的值，比如 256，则会发生整型溢出。关于这一行为 Rust 有一些有趣的规则：当在 debug 模式编译时，Rust 会检查整型溢出，若存在这些问题，则使程序在编译时 panic(崩溃,Rust 使用这个术语来表明程序因错误而退出)。

    在当使用 --release 参数进行 release 模式构建时，Rust 不检测溢出。相反，当检测到整型溢出时，Rust 会按照补码循环溢出（two’s complement wrapping）的规则处理。简而言之，大于该类型最大值的数值会被补码转换成该类型能够支持的对应数字的最小值。比如在 u8 的情况下，256 变成 0，257 变成 1，依此类推。程序不会 panic，但是该变量的值可能不是你期望的值。依赖这种默认行为的代码都应该被认为是错误的代码。

    要显式处理可能的溢出，可以使用标准库针对原始数字类型提供的这些方法：

    • 使用 wrapping_* 方法在所有模式下都按照补码循环溢出规则处理，例如 wrapping_add
    • 如果使用 checked_* 方法时发生溢出，则返回 None 值
    • 使用 overflowing_* 方法返回该值和一个指示是否存在溢出的布尔值
    • 使用 saturating_* 方法，可以限定计算后的结果不超过目标类型的最大值或低于最小值，例如:
    assert_eq!(100u8.saturating_add(1), 101);
assert_eq!(u8::MAX.saturating_add(127), u8::MAX);

    下面是一个演示wrapping_*方法的示例：

    fn main() {
    let a : u8 = 255;
    let b = a.wrapping_add(20);
    println!("{}", b);  // 19
}

    2.浮点类型¶

    浮点类型数字 是带有小数点的数字，在 Rust 中浮点类型数字也有两种基本类型： f32 和 f64，分别为 32 位和 64 位大小。默认浮点类型是 f64，在现代的 CPU 中它的速度与 f32 几乎相同，但精度更高。

    下面是一个演示浮点数的示例：

    fn main() {
    let x = 2.0; // f64

    let y: f32 = 3.0; // f32
}

    浮点数根据 IEEE-754 标准实现。f32 类型是单精度浮点型，f64 为双精度。

    运算符 说明

    | 位或 相同位置只要有1时则为1，否则为0
    ^ 异或 相同位置不相同则为1，相同则为0

    5.序列(Range)¶

    Rust 提供了一个非常简洁的方式，用来生成连续的数值，例如 1..5，生成从 1 到 4 的连续数字，不包含 5 ；1..=5，生成从 1 到 5 的连续数字，包含 5，它的用途很简单，常常用于循环中：

    for i in 1..=5 {
    println!("{}",i);
}

    最终程序输出:

    1
2
3
4
5

    序列只允许用于数字或字符类型，原因是：它们可以连续，同时编译器在编译期可以检查该序列是否为空，字符和数字值是 Rust 中仅有的可以用于判断是否为空的类型。如下是一个使用字符类型序列的例子：

    for i in 'a'..='z' {
    println!("{}",i);
}

    1.2.字符类型¶

    Rust 的字符不仅仅是 ASCII，所有的 Unicode 值都可以作为 Rust 字符，包括单个的中文、日文、韩文、emoji 表情符号等等，都是合法的字符类型。Unicode 值的范围从 U+0000 ~ U+D7FF 和 U+E000 ~ U+10FFFF。

    Rust 的字符只能用 '' 来表示， "" 是留给字符串的。

    1.3.布尔类型¶

    Rust 中的布尔类型有两个可能的值：true 和 false，布尔值占用内存的大小为 1 个字节

    1.4.单元类型(unit)¶

    在 Rust 中，unit 类型是一种特殊的基础类型，用 () 表示（空元组）。它是 Rust 类型系统中最简单的类型：没有任何数据，且仅包含一个值—— 同样用 () 表示（称为 “unit 值”）。

    unit 类型的核心作用是在需要 “占位” 但不需要实际数据时使用，常见场景包括：

    1.1. 函数的 “无返回值” 场景¶

    Rust 中没有 “无返回值” 的函数 —— 所有函数都必须返回一个值。当函数不需要返回实际数据时，会隐式返回 unit 值 ()，此时函数的返回类型可省略（或显式声明为 ()）。

    // 显式声明返回类型为 unit
fn print_hello() -> () {
    println!("Hello");
    // 隐式返回 `()`，无需写 return 语句
}

// 省略返回类型（默认返回 unit）
fn do_nothing() {
    // 函数体执行完后自动返回 `()`
}

fn main() {
    let result1 = print_hello(); // result1 的类型是 `()`，值为 `()`
    let result2 = do_nothing();  // result2 的类型是 `()`，值为 `()`
    println!("{:?}, {:?}", result1, result2); // 输出：(), ()
}
    1.2. 作为 “无数据” 的占位符¶

    当需要一个类型但不需要实际数据时，unit 类型可作为占位符，常见于泛型、元组或集合中。

    （1）泛型中的 “空数据”¶

    例如，Result<T, E> 泛型中，若操作成功但无需返回数据，可用 Result<(), E> 表示：

    use std::fs::File;

// 尝试创建文件，成功无返回值（Ok(())），失败返回错误
fn create_file(path: &str) -> Result<(), std::io::Error> {
    File::create(path)?; // 若失败，返回 Err(错误)
    Ok(()) // 成功返回 unit 值
}

fn main() {
    match create_file("test.txt") {
        Ok(()) => println!("文件创建成功"), // 匹配 unit 值
        Err(e) => println!("创建失败：{}", e),
    }
}
    （2）元组中的占位符¶

    元组中可用 () 作为不需要的元素的占位符：

    // 元组包含“字符串 + unit + 数字”
let data: (&str, (), i32) = ("hello", (), 42);
println!("{}", data.0); // 访问第一个元素：hello
// data.1 是 unit 值，无实际意义
    （3）集合中的 “标记”¶

    有时需要一个集合仅用于 “存在性检查” 而非存储数据，可用 HashSet<()> 或 Vec<()>：

    use std::collections::HashSet;

fn main() {
    let mut flags = HashSet::new();
    flags.insert(()); // 插入一个 unit 值，表示“标记已设置”

    if flags.contains(&()) {
        println!("标记已存在");
    }
}
    1.3. 模式匹配中的空分支¶

    在模式匹配（match、if let 等）中，可用 () 匹配 “无数据” 的场景：

    fn main() {
    let value: () = (); // 显式定义一个 unit 类型变量

    match value {
        () => println!("匹配到 unit 值"), // 唯一可能的分支
    }
}
    1.4. 闭包的返回值¶

    闭包若没有显式返回值，默认返回 ()：

    fn main() {
    let log = || println!("Logging..."); // 闭包返回类型为 `()`
    let result = log(); // result 是 `()`
}

    二、复合类型¶

    2.1.字符串与切片¶

    在 Rust 中，字符串（String）和字符串切片（&str）是处理文本的核心类型，而 “切片（slice）” 是更通用的概念（可用于数组、向量等）。理解它们的区别、联系和设计逻辑，是掌握 Rust 内存安全和所有权模型的关键。

    1.先理解 “切片（Slice）”：通用的视图类型¶

    切片（slice）是 Rust 中一种无所有权的引用类型，用于指向集合（如数组、Vec、字符串）中连续的一部分元素，本质是 “对数据的视图”。它的核心作用是：在不获取所有权的情况下，安全地访问集合的一部分。

    1.1.切片的基本形式¶

    切片的语法为 &[T]（T 为元素类型），由两部分组成：

    • 指向数据起始位置的指针；
    • 切片包含的元素数量（长度）。

    例如，数组切片：

    let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3]; // 切片：指向 arr 的索引 1 到 2（左闭右开），即 [2, 3]
println!("{:?}", slice); // 输出 [2, 3]
    1.2. 切片的核心特性¶
    • 无所有权：切片不拥有数据，数据的所有权属于原集合（如数组、Vec）；
    • 不可变默认：默认是不可变切片（&[T]），也可创建可变切片（&mut [T]），但需保证原集合是可变的；
    • 长度固定：切片的长度在创建后不可变（但指向的原集合可修改，切片会反映修改结果）；
    • 安全访问：切片的索引操作会被 Rust 检查（越界会 panic），避免内存不安全。

    2.字符串切片（&str）：字符串的视图¶

    字符串切片（&str）是专门用于文本的切片，本质是 &[u8] 的特殊形式，但有一个关键约束：它指向的字节序列必须是有效的 UTF-8 编码。这是 Rust 字符串区别于其他语言的核心特性（Rust 强制字符串为 UTF-8）。

    2.1. &str 的定义与内存布局¶
    • 类型标识：&str（读作 “字符串切片” 或 “字符串引用”）；
    • 内存结构：与通用切片类似，包含 “指向 UTF-8 字节序列的指针” 和 “字节长度”；
    • 核心约束：指向的字节必须是有效的 UTF-8（否则会导致未定义行为）。
    2.2. 字符串字面量：&'static str¶

    我们直接写的字符串字面量（如 "hello"）的类型是 &'static str：

    let s = "hello"; // s 的类型是 &'static str
    • 'static 生命周期：表示该字符串存储在程序的二进制文件中，在整个程序运行期间都存在；
    • 本质：是对二进制文件中 UTF-8 字节序列的切片，无所有权，不可修改。
    2.3. &str 的常见来源¶
    • 字符串字面量（"rust" → &'static str）；
    • 从 String 中截取（&s[0..5]，s 是 String）；
    • 函数返回的临时字符串视图（如 String::as_str() 方法）。

    3.String：拥有所有权的动态字符串¶

    String 是 Rust 中拥有所有权的、可动态增长的 UTF-8 字符串类型，适合需要修改、创建或拥有文本数据的场景。

    3.1. String 的定义与内存布局¶
    • 类型标识：String；
    • 底层实现：基于 Vec<u8>（动态字节数组），但保证内部存储的字节序列是有效的 UTF-8；
    • 内存结构：在堆上分配，包含三部分（与 Vec 一致）：
    • 指向堆上字节序列的指针；
    • 已分配的容量（capacity）；
    • 当前长度（length）。
    3.2. String 的核心特性¶
    • 拥有所有权：String 拥有其底层的字节数据，离开作用域时会自动释放内存；
    • 可修改：支持追加（push_str）、拼接（+ 或 format!）、替换等操作；
    • 动态增长：当长度超过容量时，会自动重新分配内存（类似 Vec）；
    • UTF-8 保证：所有修改 String 的方法（如 push_str）都会确保结果仍是有效的 UTF-8，否则编译或运行时报错。
    3.3. 创建 String 的常见方式¶
    // String 与 &str 的转换
// 1. 从字符串字面量创建
let s1 = String::from("hello"); // 推荐方式      (&str -> String)
let s2 = "world".to_string();   // 等价于 from   (t同上)

// String 类型转为 &str
// 取引用, e.g. &s1, &s1[..], s1.as_str()

// 2. 空字符串
let mut s3 = String::new(); // 空 String，可后续修改

// 3. 拼接创建
let s4 = format!("{} {}", s1, s2); // 类似 println!，返回 String
    3.4.字符串的索引：为什么不能直接用 [] 访问？¶

    Rust 字符串不支持直接通过索引（如 s[0]）访问 “字符”，核心原因是：UTF-8 编码中，一个 “字符”（Unicode 标量值）可能由 1~4 个字节组成（如英文字母 1 字节，中文 3 字节，emoji 4 字节）。直接索引访问的是字节，而非 “字符”，可能获取到无效的 UTF-8 片段，导致内存不安全。

    1. 直接索引的编译错误¶

    尝试用 [] 索引字符串会直接编译报错：

    let s = "hello 世界";
// let c = s[0]; // 编译错误：`str` cannot be indexed by `{integer}`

    错误原因：str 类型没有实现 Index<usize> trait，Rust 从语法层面禁止直接索引，避免开发者误用。

    2. 替代方案：按 “字节” 或 “字符” 访问¶

    若需访问字符串的组成部分，需明确是访问字节还是字符：

    • as_bytes() → 返回 &[u8] 只是“借”出内部 UTF-8 字节数组，零成本、无分配。
    • bytes() → 返回 Bytes<'_> 迭代器（定义在 std::str::Bytes） 把 UTF-8 字节逐个 u8 地迭代出来，内部仍引用原字符串，不产生新堆内存，但会做一次迭代器包装。

    （1）访问字节（as_bytes()）

    通过 as_bytes() 方法将字符串转换为字节切片（&[u8]），可按索引访问单个字节（但需注意：单个字节可能不是完整字符）：

    let s = "中"; // 中文“中”的 UTF-8 编码是 3 字节：0xE4 0xB8 0xAD
let bytes = s.as_bytes(); // &[u8; 3]

println!("字节 0: 0x{:X}", bytes[0]); // 0xE4
println!("字节 1: 0x{:X}", bytes[1]); // 0xB8
println!("字节 2: 0x{:X}", bytes[2]); // 0xAD

// ...
let s = "Rust";

// 1. 直接拿到字节切片
let b1: &[u8] = s.as_bytes();   // 不拷贝，O(1)
assert_eq!(b1, &[b'R', b'u', b's', b't']);

// 2. 拿到迭代器
let mut iter = s.bytes();       // 也是零拷贝，但多了一层迭代器状态
assert_eq!(iter.next(), Some(b'R'));
assert_eq!(iter.collect::<Vec<_>>(), vec![b'u', b's', b't']);

    （2）访问字符（chars() 迭代器）

    通过 chars() 方法获取字符迭代器，遍历 Unicode 标量值（即 “字符”）：

    let s = "a中😀"; // 1字节 + 3字节 + 4字节
for c in s.chars() {
    println!("{}", c);
}
// 输出：
// a
// 中
// 😀

    若需获取指定位置的字符，可将迭代器转为集合后索引（注意：性能较低，适合短字符串）：

    let s = "a中😀";
let chars: Vec<char> = s.chars().collect(); // 收集为 Vec<char>
println!("第 0 个字符：{}", chars[0]); // a
println!("第 1 个字符：{}", chars[1]); // 中
println!("第 2 个字符：{}", chars[2]); // 😀
    3.5.字符串的切片：安全截取子串¶

    字符串切片（&str）是对原字符串部分内容的引用，但必须满足：切片的范围对应完整的 UTF-8 字符，否则会导致无效的字符串切片。

    1. 切片语法与规则¶

    字符串切片的语法为 &s[start..end]（左闭右开区间），其中 start 和 end 是字节索引（而非字符索引）。核心规则：start 和 end 必须落在字符的边界上（即指向某个字符的起始字节），否则切片会包含无效的 UTF-8 字节。

    let s = "hello 世界";
// "hello" 是 5 字节（每个字母 1 字节），空格是 1 字节，"世界" 是 6 字节（每个字 3 字节）
// 完整字节索引：0-4（h-e-l-l-o）、5（空格）、6-8（世）、9-11（界）

let hello = &s[0..5]; // 截取 "hello"（0-4 字节，完整字符）
let space = &s[5..6]; // 截取空格（1 字节）
let shi = &s[6..9];   // 截取 "世"（3 字节）
let jie = &s[9..12];  // 截取 "界"（3 字节）

println!("{} {} {} {}", hello, space, shi, jie); // hello 世界
    2. 安全切片：get() 方法¶

    为避免切片越界或边界错误，可使用 get(start..end) 方法：

    • 若范围有效，返回 Some(&str)；
    • 若范围无效（越界或非字符边界），返回 None（不会 panic）。
    let s = "hello 世界";

// 有效范围
let valid = s.get(0..5);
println!("{:?}", valid); // Some("hello")

// 无效范围（"世"的中间）
let invalid = s.get(6..8); // 6-8 是 "世" 的前 2 字节（不完整）
println!("{:?}", invalid); // None

// 越界范围
let out_of_bounds = s.get(100..200);
println!("{:?}", out_of_bounds); // None
    3.6.字符串的常用操作¶

    字符串操作需区分 String（可变，拥有所有权）和 &str（不可变，无所有权）：

    • String 支持原地修改（如追加、插入）；
    • &str 操作会返回新的 String（原切片不变）。
    1. 创建字符串¶
    // 从字面量创建
let s1 = String::from("hello");
let s2 = "world".to_string();

// 空字符串
let mut s3 = String::new(); // 后续可修改

// 格式化创建
let name = "Rust";
let s4 = format!("Hello, {}", name); // 类似 println!，返回 String
    2. 追加内容（push_str、push）¶
    • push_str(&str)：追加字符串切片（不获取所有权）；
    • push(char)：追加单个字符（Unicode 标量值）。

    仅 String 支持（&str 不可变）：

    let mut s = String::from("hello");

s.push_str(" world"); // 追加字符串切片
println!("{}", s); // hello world

s.push('!'); // 追加单个字符
println!("{}", s); // hello world!
    3. 拼接字符串（+ 运算符、format!）¶

    （1）+ 运算符

    • 语法：String + &str（右侧必须是字符串切片，避免所有权转移）；
    • 原理：调用 add 方法，实际是 s1.push_str(s2) 并返回 s1。
    let s1 = String::from("hello");
let s2 = String::from(" world");

let s3 = s1 + &s2; // s1 的所有权被转移，后续不可再使用 s1
println!("{}", s3); // hello world
// println!("{}", s1); // 编译错误：s1 已被移动

    （2）format! 宏（推荐）

    适合多字符串拼接，不转移所有权，语法更清晰：

    let s1 = String::from("hello");
let s2 = String::from("world");
let s3 = String::from("!");

let result = format!("{} {} {}", s1, s2, s3); // 不影响 s1、s2、s3 的所有权
println!("{}", result); // hello world !
    4. 替换内容（replace、replacen）¶
    • replace(pattern, replacement)：替换所有匹配的子串，返回新 String；
    • replacen(pattern, replacement, n)：替换前 n 个匹配的子串。
    • replace_range(range, replacement): 直接操作原来的字符串，不会返回新的字符串。该方法需要使用 mut 关键字修饰。

    String 和 &str 都支持（返回新字符串，不修改原字符串）：

    let s = "hello hello world";

// 替换所有 "hello" 为 "hi"
let s1 = s.replace("hello", "hi");
println!("{}", s1); // hi hi world

// 替换前 1 个 "hello" 为 "hi"
let s2 = s.replacen("hello", "hi", 1);
println!("{}", s2); // hi hello world
    5. 截断与清空（truncate、clear）¶
    • truncate(len)：截断字符串到指定字节长度（需确保 len 是有效字符边界，否则会包含无效 UTF-8，导致后续操作 panic）；
    • clear()：清空字符串（等价于 truncate(0)）。

    仅 String 支持：

    let mut s = String::from("hello world");

s.truncate(5); // 截断到 5 字节（"hello"）
println!("{}", s); // hello

s.clear(); // 清空
println!("{}", s.is_empty()); // true
    6. 插入与删除（insert_str、remove、drain）¶
    • insert_str(pos, &str)：在指定字节位置插入字符串（pos 必须是有效字符边界）；
    • pop(): 删除并返回字符串的最后一个字符
    • remove(pos)：删除指定字节位置的字符（返回删除的字节，pos 必须是有效字符边界）；
    • drain(range)：删除指定范围的字符（返回迭代器，range 必须是有效字符边界）。

    仅 String 支持，且操作需确保字节位置是有效边界：

    let mut s = String::from("hello");

// 插入
s.insert_str(5, " world"); // 在 5 字节位置（"hello" 末尾）插入
println!("{}", s); // hello world

// 删除单个字符（索引 0 是 'h' 的字节位置）
let removed = s.remove(0);
println!("删除的字节：{}，结果：{}", removed as char, s); // e, ello world

// 删除范围（2..6 对应 "lo w"）
let drained: String = s.drain(2..6).collect();
println!("删除的内容：{}，结果：{}", drained, s); // lo w, elo rld
    7. 转换操作¶

    （1）&str 与 String 互转

    • &str → String：to_string() 或 String::from(&str)；
    • String → &str：通过解引用（&s）或 as_str() 方法。
    let str_slice = "hello";
let string = str_slice.to_string(); // &str → String

let s = String::from("world");
let slice = s.as_str(); // String → &str
let slice2 = &s; // 等价于 as_str()

    （2）&str与字节&[u8]切片互转

    • &str → &[u8]：as_bytes()；
    • &[u8] → &str：std::str::from_utf8()（需确保字节是有效的 UTF-8）。
    let s = "hello";
let bytes = s.as_bytes(); // &[u8; 5]

let valid_bytes = b"hello";
let valid_str = std::str::from_utf8(valid_bytes).unwrap(); // "hello"

let invalid_bytes = [0xFF, 0xFF]; // 无效 UTF-8
let invalid_str = std::str::from_utf8(&invalid_bytes);
println!("{:?}", invalid_str); // Err(Utf8Error { ... })

    （3） String 与字节&[u8]切片互转

    • String → &[u8]（不可变字节切片，不转移所有权）:as_bytes()
    let s = String::from("hello 世界");
let bytes: &[u8] = s.as_bytes(); // 获取字节切片

// 打印字节值（"hello" 是 ASCII 字节，"世界" 是多字节 UTF-8）
println!("{:?}", bytes);
// 输出：[104, 101, 108, 108, 111, 32, 228, 184, 173, 231, 149, 140]
    • String → Vec<u8>（拥有所有权的字节向量，转移所有权）:
    let s = String::from("hello");
let byte_vec: Vec<u8> = s.into_bytes(); // 转换为 Vec<u8>，s 不再可用

println!("{:?}", byte_vec); // [104, 101, 108, 108, 111]
// println!("{}", s); // 编译错误：s 已被移动
    • &[u8] → String（通过 &str 中转）
    // 有效 UTF-8 字节
let valid_bytes = b"hello 世界"; // &[u8; 12]（b"" 表示字节字面量）
let valid_str = std::str::from_utf8(valid_bytes).unwrap(); // 转换为 &str
let valid_string = valid_str.to_string(); // 转换为 String
println!("{}", valid_string); // hello 世界

// 无效 UTF-8 字节（如 0xFF 是无效 UTF-8）
let invalid_bytes = [0xFF, 0x00];
let invalid_result = std::str::from_utf8(&invalid_bytes);
println!("{:?}", invalid_result); // Err(Utf8Error { valid_up_to: 0, error_len: Some(1) })
    • Vec<u8> → String（直接转换）
    // 有效 UTF-8 字节向量
let valid_byte_vec = b"hello".to_vec(); // Vec<u8>
let valid_string = String::from_utf8(valid_byte_vec).unwrap(); // 转换为 String
println!("{}", valid_string); // hello

// 无效 UTF-8 字节向量
let invalid_byte_vec = vec![0xFF, 0x00];
let invalid_result = String::from_utf8(invalid_byte_vec);
println!("{:?}", invalid_result); // Err(FromUtf8Error { bytes: [255, 0], error: Utf8Error { ... } })
    3.7.字符串格式化宏¶

    在 Rust 中，println! 和 format! 是两个核心的字符串格式化宏，均用于将数据转换为字符串并按指定格式处理。它们共享相同的格式化语法，但用途不同：println! 用于将格式化后的字符串输出到标准输出（控制台），而 format! 用于生成并返回格式化后的 String。

    1.基本概念与核心区别¶
    宏 作用 返回值 核心场景
    println! 格式化字符串并输出到标准输出 无（()） 控制台打印、日志输出
    print! 无换行的标准输出 无（()） 需要在同一行分多次输出内容时（如动态进度条、分步构建一行文本）
    eprint!/eprintln! 将格式化字符串输出到标准错误流（stderr） () 输出错误、警告等需要与正常内容分离的信息
    format! 格式化字符串并返回结果 String 动态构建字符串、拼接文本
    format_args! 生成一个 std::fmt::Arguments 结构体（包含格式化字符串和参数），作为其他格式化宏的底层实现，支持延迟格式化（先准备参数，后执行格式化） Arguments 需要重复使用相同格式化参数时（避免重复解析格式化字符串，提升性能）
    write!/writeln! 将格式化字符串写入实现了 std::fmt::Write trait 的任意目标（如文件、字符串缓冲区、网络流等） Result<(), std::fmt::Error> 向自定义目标（文件、网络流、内存缓冲区）写入格式化内容
    dbg! 调试专用,打印表达式的值、所在文件和行号，并返回表达式的值，用于调试过程中快速查看变量或表达式的中间结果 表达式原值 调试时快速插入打印语句，查看变量或表达式的值

    两者的格式化语法完全一致（均基于 std::fmt 模块），差异仅在于 “输出目标”：println! 输出到控制台，format! 输出到 String。

    2.基础用法：格式化字符串与参数¶
    // println!：输出到控制台
println!("格式化字符串", 参数1, 参数2, ...);

// format!：返回 String
let result = format!("格式化字符串", 参数1, 参数2, ...);
    • 格式化字符串：包含 “占位符”（如 {}），用于指定参数的插入位置和格式；
    • 参数：需被格式化的值（可以是任意类型，只要支持对应的格式化 trait）。
    3.格式化语法：占位符与格式说明符¶

    格式化字符串中的占位符 {} 可通过 “格式说明符”（:后的内容）控制输出格式，语法为：{参数索引/名称:格式说明符}

    3.1. 参数索引与命名（控制参数匹配方式）:

    默认按参数顺序匹配（{} 对应第 0 个参数，{1} 对应第 1 个参数），也可通过命名参数显式指定。

    （1）位置索引:

    let a = 10;
let b = 20;

// 按索引指定参数（0 是第一个参数）
println!("a={0}, b={1}, sum={0}+{1}={2}", a, b, a + b);
// 输出：a=10, b=20, sum=10+20=30

    （2）命名参数:

    let user = "Alice";
let age = 30;

// 命名参数（参数名与变量名一致）
println!("User: {user}, Age: {age}");
// 输出：User: Alice, Age: 30

// 显式指定命名参数（适合参数顺序与变量名无关的场景）
println!("Name: {u}, Age: {a}", u = user, a = age);
// 输出：Name: Alice, Age: 30

    3.2. 格式说明符（控制输出样式）:

    格式说明符用于定制参数的输出格式（如数字进制、字符串对齐、精度等），常见类型如下：

    （1）通用说明符:

    • {:?}：使用 Debug trait 格式化（用于调试，输出类型的详细信息）；
    • {:#?}：带换行和缩进的 Debug 格式化（适合复杂结构）；
    • {:}：等价于 {}，使用 Display trait 格式化（默认，用于用户友好输出）。
    let s = "hello";
let v = vec![1, 2, 3];

// Display（默认，适合用户阅读）
println!("s = {}", s); // s = hello

// Debug（适合调试，显示类型细节）
println!("v = {:?}", v); // v = [1, 2, 3]

// 带格式的 Debug（# 号增强可读性）
println!("v = {:#?}", v);
// 输出：
// v = [
//     1,
//     2,
//     3,
// ]

    （2）数字格式:

    • 整数进制：b（二进制）、o（八进制）、d（十进制，默认）、x（十六进制小写）、X（十六进制大写）；
    • 浮点数：e（科学计数法小写）、E（科学计数法大写）、f（固定小数位）、?（自动选择）；
    • 精度：.n 控制小数位数（如 .2f 保留 2 位小数）；
    • 符号：+ 强制显示正负号（如 +3、-2）。
    let num = 42;
let pi = 3.1415926;

// 整数进制
println!("二进制: {:b}, 八进制: {:o}, 十六进制: {:x}", num, num, num);
// 输出：二进制: 101010, 八进制: 52, 十六进制: 2a

// 浮点数精度与格式
println!("pi = {:.2f}, 科学计数法: {:.2e}", pi, pi);
// 输出：pi = 3.14, 科学计数法: 3.14e+00

// 强制显示符号
println!("收益: {:+}, 亏损: {:+}", 100, -50);
// 输出：收益: +100, 亏损: -50

    （3）对齐与宽度:

    • 宽度：n 控制输出总长度（不足时填充）；
    • 对齐：<（左对齐）、>（右对齐）、^（居中对齐）；
    • 填充：默认用空格填充，可指定其他字符（如 0 填充数字）。
    let text = "rust";

// 宽度为 10，左对齐（默认）
println!("[{:10}]", text); // [rust      ]

// 宽度为 10，右对齐
println!("[{:>10}]", text); // [      rust]

// 宽度为 10，居中对齐
println!("[{:^10}]", text); // [   rust   ]

// 数字填充（0 填充宽度为 5 的整数）
println!("{:05}", 42); // 00042

    （4）布尔值与字符:

    • 布尔值默认输出 true 或 false；
    • 字符直接输出其 Unicode 表示。
    let flag = true;
let c = '中';

println!("flag: {}, char: {}", flag, c); // flag: true, char: 中
    4.格式化的底层支持：Display 与 Debug trait¶

    println! 和 format! 的格式化能力依赖于 std::fmt 模块中的两个核心 trait：

    • Display：用于 “用户友好” 的输出（对应 {} 或 {:}），需手动为类型实现（标准库类型已实现）；

    • Debug：用于 “调试” 输出（对应 {:?}），可通过 #[derive(Debug)] 自动派生。

    • 为自定义类型实现 Display 和 Debug

    // 自动派生 Debug（无需手动实现）
#[derive(Debug)]
struct Point {
    x: i32,
    y: i32,
}

// 手动实现 Display（用户友好输出）
impl std::fmt::Display for Point {
    fn fmt(&self, f: &mut std::fmt::Formatter<'_>) -> std::fmt::Result {
        // 自定义输出格式：(x, y)
        write!(f, "({}, {})", self.x, self.y)
    }
}

fn main() {
    let p = Point { x: 10, y: 20 };

    // 使用 Display（{}）
    println!("Display: {}", p); // Display: (10, 20)

    // 使用 Debug（{:?}）
    println!("Debug: {:?}", p); // Debug: Point { x: 10, y: 20 }
}
    1. 关键区别：Display vs Debug
    特性 Display Debug
    用途 用户友好的输出（如给终端用户看） 调试用输出（如开发者查错）
    实现方式 必须手动实现 fmt 方法 可通过 #[derive(Debug)] 自动生成
    对应占位符 {} 或 {:} {:?} 或 {:#?}
    1. 示例 (动态宽度与精度（* 占位符）)
    let width = 10;
let precision = 2;
let num = 3.14159;

// 动态宽度（width=10）和精度（precision=2）
println!("{:width$.precision$}", num, width = width, precision = precision);
// 等价于 "{:10.2}"，输出：     3.14
    3.8.关键注意事项¶
    1. UTF-8 有效性：所有字符串操作必须保证结果是有效的 UTF-8，否则会编译错误或运行时 panic（如插入无效字节、截断到非字符边界）。
    2. 所有权与可变性：String 是可变的、拥有所有权的类型，适合修改；&str 是不可变的、无所有权的视图，适合传递和引用。
    3. 性能考量：频繁修改 String 可能导致多次内存重分配（可通过 with_capacity(n) 预分配容量优化）；chars() 迭代器比收集为 Vec<char> 更高效。

    Rust 字符串的索引、切片和操作受 UTF-8 编码的强约束，核心要点：

    • 索引：禁止直接用 [] 访问，需通过 chars()（字符）或 as_bytes()（字节）间接访问；
    • 切片：必须使用有效字符边界的字节范围，推荐用 get() 方法避免 panic；
    • 操作：String 支持原地修改（追加、插入、截断等），&str 操作返回新 String，所有操作需保证 UTF-8 有效性。

    2.2.元组¶

    在 Rust 中，元组（Tuple） 是一种固定长度、异构的复合类型，用于将多个不同类型的值组合成一个单一实体。它的核心特点是 “长度固定、类型灵活”，既适合临时封装少量数据，也常用于函数返回多个值等场景，是 Rust 中简洁且高效的复合类型。

    1. 核心定义¶

    • 元组是有序的复合类型：元素的顺序固定，访问时通过索引定位；
    • 异构性：元素可以是不同类型（如 i32、String、bool 等）；
    • 固定长度：一旦声明，元组的长度无法修改（不能添加 / 删除元素）。

    2. 声明方式¶

    元组用 () 包裹元素，元素间用逗号分隔。支持隐式类型推断（推荐）或显式类型标注。

    （1）隐式类型推断（常用）¶
    // 不同类型的元素组合
let t1 = (10, "hello", true, 3.14);
// 类型自动推断为：(i32, &str, bool, f64)

// 空元组（即 unit 类型 `()`）
let t2 = ();

// 单元素元组（必须加逗号，否则会被解析为表达式括号）
let t3 = (5,); // 类型：(i32,)，注意末尾的逗号
// let t3 = (5); // 错误：被解析为 i32 类型，而非元组
    （2）显式类型标注¶
    // 显式指定每个元素的类型
let t: (i32, String, bool) = (42, String::from("rust"), false);

    3.元组的基本操作¶

    3.1. 解构（Destructuring）：快速提取元素¶

    元组支持 “解构”，即通过模式匹配将元组的元素快速绑定到变量，无需逐个通过索引访问，是元组最常用的操作之一。

    （1）完整解构¶
    let t = (42, String::from("hello"), true);

// 解构元组，绑定到变量 a、b、c
let (a, b, c) = t;
println!("a={}, b={}, c={}", a, b, c); // a=42, b=hello, c=true
    （2）部分解构（忽略无关元素）¶

    用 _ 忽略不需要的元素：

    let t = (10, 20, 30, 40);

// 只提取第二个和第四个元素，忽略其他
let (_, b, _, d) = t;
println!("b={}, d={}", b, d); // b=20, d=40
    （3）解构用于函数返回值¶

    Rust 没有 “多返回值”，但可以通过返回元组模拟，再通过解构提取结果（最经典场景）：

    // 函数返回元组（模拟多返回值）
fn divide(a: i32, b: i32) -> (i32, i32) {
    let quotient = a / b;
    let remainder = a % b;
    (quotient, remainder) // 返回元组
}

fn main() {
    // 解构函数返回的元组
    let (q, r) = divide(10, 3);
    println!("商：{}, 余数：{}", q, r); // 商：3, 余数：1
}
    3.2. 嵌套元组（Tuple of Tuples）¶

    元组的元素可以是另一个元组，形成嵌套结构，访问时用多级索引。

    // 嵌套元组：(i32, (String, bool), f64)
let nested = (10, (String::from("rust"), true), 3.14);

// 访问嵌套元组的元素
println!("外层第一个元素：{}", nested.0); // 10
println!("内层第二个元素：{}", nested.1.1); // true（先访问外层索引 1，再访问内层索引 1）
    3.3. 遍历元组（限制较多）, 用 . 来访问元组¶

    由于元组是异构的（元素类型不同），不能直接用 for 循环遍历（编译器无法统一处理不同类型的元素）。若需遍历，需手动解构或使用宏（如 tuple! 相关宏），但通常不推荐（元组设计初衷是 “少量固定元素”）。

    // 不推荐：手动解构遍历（仅适用于短元组）
let t = (1, 2, 3);
let elements = [t.0, t.1, t.2]; // 转为数组后遍历
for e in elements {
    println!("{}", e);
}
    3.4.特殊元组¶
    1. 空元组（()）：单元类型¶

    空元组 () 即之前讲的单元类型，是元组的特殊情况：

    • 长度为 0，无元素；
    • 唯一值为 ()；
    • 用途：占位、函数无返回值（隐式返回 ()）等。
    let unit = (); // 类型：()
fn do_nothing() -> () {
    println!("无返回值");
}
    2. 单元素元组（(T,)）¶

    单元素元组必须在元素后加逗号，否则会被解析为 “带括号的表达式”（而非元组）：

    let t1 = (5,); // 正确：单元素元组，类型 (i32,)
let t2 = (5);  // 错误：被解析为 i32 类型，不是元组

// 验证类型
println!("t1 类型：{}", std::any::type_name_of_val(&t1)); // (i32,)
println!("t2 类型：{}", std::any::type_name_of_val(&t2)); // i32
    3.5.元组的常见用途¶
    1. 函数返回多个值¶

    这是元组最核心的用途，无需定义结构体即可返回多个不同类型的结果。

    // 示例：解析字符串为数字，返回“是否成功”和“结果”
fn parse_int(s: &str) -> (bool, i32) {
    match s.parse() {
        Ok(num) => (true, num),
        Err(_) => (false, 0),
    }
}

fn main() {
    let (success, num) = parse_int("123");
    if success {
        println!("解析成功：{}", num);
    } else {
        println!("解析失败");
    }
}
    2. 临时封装少量异构数据¶

    当需要临时组合少量不同类型的值（无需长期复用该结构）时，元组比结构体更简洁（无需定义类型）。

    // 临时存储用户信息（无需定义 User 结构体）
let user = (1001, "Alice", 30, true); // (id, 姓名, 年龄, 是否VIP)

// 解构提取信息
let (id, name, age, is_vip) = user;
println!("ID: {}, 姓名: {}, 年龄: {}, VIP: {}", id, name, age, is_vip);
    3. 作为函数参数（传递多个异构值）¶

    当函数需要接收多个不同类型的参数时，元组可简化参数列表（尤其适合参数数量少的场景）。

    // 接收元组作为参数
fn print_info(info: (i32, &str, bool)) {
    println!("ID: {}, 名称: {}, 状态: {}", info.0, info.1, info.2);
}

fn main() {
    let info = (101, "产品A", true);
    print_info(info); // 输出：ID: 101, 名称: 产品A, 状态: true
}
    4. 模式匹配中的占位¶

    在 match 表达式中，元组可用于匹配多个条件。

    let point = (3, 4);

match point {
    (0, 0) => println!("原点"),
    (x, 0) => println!("在 x 轴上，x = {}", x),
    (0, y) => println!("在 y 轴上，y = {}", y),
    (x, y) => println!("在平面内，坐标 ({}, {})", x, y),
}
// 输出：在平面内，坐标 (3, 4)
    3.6.元组与其他复合类型的区别¶
    类型 核心区别 适用场景
    元组（Tuple） 固定长度、异构、无字段名 临时封装少量异构数据、函数多返回值
    数组（Array） 固定长度、同构、通过索引访问 存储多个相同类型的固定长度数据
    向量（Vec） 动态长度、同构、可增删元素 存储多个相同类型的动态长度数据
    结构体（Struct） 固定长度、异构 / 同构、有字段名 长期复用的复合类型（需明确字段含义）

    关键选择准则：

    • 若需 “临时、少量、异构” 数据 → 用元组；
    • 若需 “固定长度、同构” 数据 → 用数组；
    • 若需 “动态长度、同构” 数据 → 用 Vec；
    • 若需 “长期复用、字段含义明确” 数据 → 用结构体。

    2.3.结构体¶

    在 Rust 中，结构体（Struct） 是一种自定义的复合类型，用于将多个相关联的数据（可能是不同类型）组合成一个有意义的整体。与元组（Tuple）相比，结构体的核心优势是为每个数据字段命名，从而使代码更具可读性和可维护性。结构体是 Rust 中组织复杂数据、建模真实世界实体的核心工具。

    1.定义结构体¶

    一个结构体由几部分组成：

    • 通过关键字 struct 定义
    • 一个清晰明确的结构体 名称
    • 几个有名字的结构体 字段

    根据字段的形式可分为三类：普通结构体（具名字段）、元组结构体（匿名字段，类似元组）、单元结构体（无字段，类似单元类型 ()）。

    1.1. 普通结构体（Named-Field Struct）¶

    最常用的结构体类型，为每个字段指定名称和类型，字段名清晰表达数据含义。

    定义语法：

    struct 结构体名称 {
    字段名1: 类型1,
    字段名2: 类型2,
    // ... 更多字段
}

    示例：定义用户结构体

    // 定义普通结构体：包含用户名、年龄、是否为VIP
struct User {
    username: String,
    age: u32,
    is_vip: bool,
}
    1.2. 元组结构体（Tuple Struct）¶

    字段无名称，仅指定类型（类似元组），但整体有结构体名称。适用于需要组合数据但字段含义明确（无需命名）的场景（如坐标、颜色值）。

    定义语法：

    struct 结构体名称(类型1, 类型2, ...);

    示例：定义坐标和颜色结构体

    // 元组结构体：表示二维坐标（x, y）
struct Point(i32, i32);

// 元组结构体：表示RGB颜色（红, 绿, 蓝）
struct Color(u8, u8, u8);
    1.3. 单元结构体（Unit Struct）¶

    无任何字段，类似单元类型 ()。适用于需要实现某个 trait 但无需存储数据的场景（标记作用）。

    它零大小（size_of::<Foo>() == 0），零开销，却在类型系统里是一个真实存在的独立类型。

    当要定义一个类型，但是不关心该类型的内容，只关心它的行为时，可以使用它。

    定义语法：

    struct 结构体名称;

    示例：定义标记结构体

    // 单元结构体：用于标记“空操作”或“默认配置”
struct DefaultConfig;
    1.3.1.类高频使用场景¶
    1. 标记类型 / 标记参数（Phantom Domain）

    把单元结构体当作“编译期标签”来区分不同行为，而不生成任何运行时数据。

    • 例 1：给 HashMap 指定自定义哈希算法

      use std::hash::{BuildHasherDefault, Hasher};

struct MyHash;
impl Hasher for MyHash { ... }

type MyMap<K, V> = HashMap<K, V, BuildHasherDefault<MyHash>>;

    • 例 2：tokio 的 TcpStream 就用了 struct TcpStream; 做协议标记，实现 AsyncRead/AsyncWrite。

    • 零成本状态机（State Machine）

    状态本身不需要携带数据，只需要在类型层面区分。

    struct Idle;
struct Running;
struct Stopped;

impl Machine<Idle> {
    fn start(self) -> Machine<Running> { ... }
}
impl Machine<Running> {
    fn stop(self) -> Machine<Stopped> { ... }
}

    每个状态都是一个单元结构体，编译后不占空间，但借类型系统把非法转移直接拒绝在编译期。

    1. 全局单例 / 统一入口（“命名空间”）

    单元结构体可以实现 trait，也能当命名空间用，避免用 mod 多建文件。

    struct Log;
impl Log {
    fn info(msg: &str)  { ... }
    fn error(msg: &str) { ... }
}

    调用时 Log::info("hello")，语义清晰，且 Log 本身不占用任何内存。

    1. 错误类型聚合（Error Kind）

    当错误不需要额外上下文时，用单元结构体当“错误种类”最轻量。

    使用 #[derive(Debug)] 对结构体进行标记，这样才能使用 println!("{:?}", s); 的方式对其进行打印输出，如果不加，则抛出错误:

    • error[E0277]: xxx doesn't implement std::fmt::Display
    • error[E0277]: xxx doesn't implement Debug
    #[derive(Debug)]
struct Timeout;
impl std::fmt::Display for Timeout { ... }
impl std::error::Error for Timeout {}
    1. 与 PhantomData 组合，做“有签名的零大小”

    在泛型里经常需要“提到”某个类型，但又不存值，此时把单元结构体塞进 PhantomData 可以完美表达：

    struct Ptr<T> {
    _marker: PhantomData<fn() -> T>,   // 零大小，但标记生命周期/协变
}
    1. FFI 对齐/占位

    在 C 接口里偶尔会遇到“空结构体”做不透明指针（void*），Rust 侧用单元结构体表示：

    #[repr(C)]
pub struct cairo_surface_t;   // 外部 C 库的真实定义

    2.结构体的实例化与使用¶

    2.1. 创建结构体实例¶

    实例化结构体时，需为所有字段赋值（顺序可与定义不同，但普通结构体需显式指定字段名）。

    （1）普通结构体实例化¶
    struct User {
    username: String,
    age: u32,
    is_vip: bool,
}

fn main() {
    // 创建实例：指定字段名（顺序无关）
    let user1 = User {
        username: String::from("alice"),
        age: 30,
        is_vip: true,
    };

    // 简化实例化：若变量名与字段名相同，可省略字段名
    let username = String::from("bob");
    let age = 25;
    let user2 = User {
        username, // 等价于 username: username
        age,      // 等价于 age: age
        is_vip: false,
    };
}
    （2）元组结构体实例化¶

    类似元组，按字段顺序赋值（无字段名）：

    struct Point(i32, i32);
struct Color(u8, u8, u8);

fn main() {
    let p = Point(10, 20); // x=10, y=20
    let red = Color(255, 0, 0); // 红=255, 绿=0, 蓝=0
}
    （3）单元结构体实例化¶

    直接使用结构体名称：

    struct DefaultConfig;

fn main() {
    let config = DefaultConfig; // 实例化单元结构体
}
    2.2. 访问和修改结构体字段¶
    • 普通结构体通过 实例.字段名 访问 / 修改字段；
    • 元组结构体通过 实例.索引 访问 / 修改字段（类似元组）；
    • 结构体默认不可变，需用 mut 修饰实例才能修改字段（结构体的可变性是整体的，Rust不支持单独标记某个字段为可变）。
    （1）普通结构体字段操作¶
    struct User {
    username: String,
    age: u32,
    is_vip: bool,
}

fn main() {
    // 不可变实例：字段无法修改
    let user1 = User {
        username: String::from("alice"),
        age: 30,
        is_vip: true,
    };
    println!("用户名：{}", user1.username); // 访问字段：alice
    // user1.age = 31; // 编译错误：不可变实例不能修改字段

    // 可变实例：字段可修改
    let mut user2 = User {
        username: String::from("bob"),
        age: 25,
        is_vip: false,
    };
    user2.age = 26; // 修改字段
    user2.is_vip = true;
    println!("修改后年龄：{}，是否VIP：{}", user2.age, user2.is_vip); // 26, true
}
    （2）元组结构体字段操作¶
    struct Point(i32, i32);

fn main() {
    let mut p = Point(10, 20);
    println!("x: {}", p.0); // 访问第一个字段（x坐标）：10
    p.1 = 30; // 修改第二个字段（y坐标）
    println!("y: {}", p.1); // 30
}
    2.3. 基于现有实例创建新实例（结构体更新语法）¶

    当需要基于一个结构体实例创建新实例，且大部分字段相同（仅少数不同）时，可使用 .. 语法复用原有实例的字段，简化代码。

    struct User {
    username: String,
    age: u32,
    is_vip: bool,
}

fn main() {
    let user1 = User {
        username: String::from("alice"),
        age: 30,
        is_vip: true,
    };

    // 基于 user1 创建 user2，仅修改 username，其他字段复用 user1
    let user2 = User {
        username: String::from("alice_clone"),
        ..user1 // 复用 user1 的 age 和 is_vip 字段
    };

    println!("user2 年龄：{}，是否VIP：{}", user2.age, user2.is_vip); // 30, true
}

    注意：..user1 会转移 user1 中拥有所有权的字段（如 String）的所有权，因此 user1 在 ..user1 之后无法再使用（但基本类型如 u32、bool 是 Copy 类型，会被复制而非转移）。

    3.结构体的方法（Methods）¶

    结构体的 “方法” 是与结构体关联的函数，定义在 impl（实现）块中，用于封装与结构体相关的行为（操作结构体数据）。方法分为实例方法（依赖结构体实例）和关联函数（不依赖实例，类似静态方法）。

    3.1. 实例方法（Instance Methods）¶

    需要通过结构体实例调用，第一个参数通常是 &self（不可变引用）或 &mut self（可变引用），用于访问 / 修改实例的字段。

    1.定义与使用¶
    struct Rectangle {
    width: u32,
    height: u32,
}

// 为 Rectangle 实现方法（impl 块）
impl Rectangle {
    // 实例方法：计算面积（不可变引用 self，仅读取字段）
    fn area(&self) -> u32 {
        self.width * self.height
    }

    // 实例方法：判断是否能包含另一个矩形（可变引用 self 可选，此处仅读取）
    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width >= other.width && self.height >= other.height
    }

    // 实例方法：修改尺寸（可变引用 self，修改字段）
    fn expand(&mut self, factor: u32) {
        self.width *= factor;
        self.height *= factor;
    }
}

fn main() {
    let mut rect = Rectangle { width: 10, height: 20 };

    // 调用实例方法（自动解引用，无需显式写 &rect.area()）
    println!("面积：{}", rect.area()); // 10*20=200

    let small_rect = Rectangle { width: 5, height: 15 };
    println!("能否包含小矩形：{}", rect.can_hold(&small_rect)); // true

    // 调用修改字段的方法（需要可变实例）
    rect.expand(2);
    println!("扩大后尺寸：{}x{}", rect.width, rect.height); // 20x40
}

    关键参数：

    • &self：不可变引用，用于读取实例数据（不获取所有权，调用后实例仍可使用）；
    • &mut self：可变引用，用于修改实例数据（需实例声明为 mut）；
    • self：获取实例所有权（调用后实例不可再使用，较少用）。
    3.2. 关联函数（Associated Functions）¶

    定义在 impl 块中，但不包含 self 参数，无需实例即可调用（类似其他语言的 “静态方法”）。常用作结构体的 “构造函数”（如 new 方法）。

    1.定义与使用¶
    struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    // 关联函数：创建正方形（构造函数）
    fn square(size: u32) -> Self { // Self 等价于 Rectangle
        Self {
            width: size,
            height: size,
        }
    }
}

fn main() {
    // 调用关联函数：使用 :: 语法（无需实例）
    let square = Rectangle::square(10); // 边长为10的正方形
    println!("正方形尺寸：{}x{}", square.width, square.height); // 10x10
}

    常用场景：提供结构体的初始化逻辑（如 new 方法）、工具函数等。

    4.结构体与 Trait：实现接口与多态¶

    结构体可通过实现 trait（类似其他语言的 “接口”）来定义共享行为，实现代码复用和多态。Rust 提供了一些常用的标准 trait（如 Debug、Display），也支持自定义 trait。

    4.1. 派生 Trait（自动实现）¶

    通过 #[derive(TraitName)] 注解，可让编译器为结构体自动实现某些标准 trait（如 Debug 用于打印调试信息）。

    // 派生 Debug trait（用于 {:?} 打印）
#[derive(Debug)]
struct Point {
    x: i32,
    y: i32,
}

fn main() {
    let p = Point { x: 10, y: 20 };
    println!("{:?}", p); // 输出：Point { x: 10, y: 20 }（因派生了 Debug）
    println!("{:#?}", p); // 带格式的 Debug 输出（换行缩进）
    // 输出：
    // Point {
    //     x: 10,
    //     y: 20,
    // }
}

    可派生的常用标准 trait 包括：Debug、Clone（复制实例）、Copy（按位复制）、PartialEq/Eq（相等性比较）等。

    4.2. 手动实现 Trait¶

    若需要自定义行为（如 Display 用于用户友好的输出），需手动实现 trait。

    use std::fmt;

struct Point {
    x: i32,
    y: i32,
}

// 手动实现 Display trait（用于 {} 打印）
impl fmt::Display for Point {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        // 自定义输出格式：(x, y)
        write!(f, "({}, {})", self.x, self.y)
    }
}

fn main() {
    let p = Point { x: 10, y: 20 };
    println!("{}", p); // 输出：(10, 20)（因实现了 Display）
}

    5.结构体的所有权与生命周期¶

    结构体可以包含各种类型的字段，包括拥有所有权的类型（如 String）和引用类型（如 &str）。若包含引用，需显式指定生命周期（确保引用的有效性）。

    5.1. 包含拥有所有权的字段¶

    最常见的场景，结构体拥有字段数据的所有权，无需考虑生命周期（离开作用域时自动释放）。

    struct User {
    username: String, // String 拥有所有权
    age: u32,         // 基本类型，Copy 语义
}

fn main() {
    let user = User {
        username: String::from("alice"),
        age: 30,
    };
    // user 离开作用域时，username 的内存会被自动释放
}
    5.2. 包含引用字段（需生命周期标注）¶

    若结构体包含引用（如 &str），必须通过生命周期标注确保引用的存活时间不短于结构体实例。

    // 生命周期标注：'a 表示 x 和 y 的引用必须至少存活到 Point 实例的生命周期
struct Point<'a> {
    x: &'a i32,
    y: &'a i32,
}

fn main() {
    let x = 10;
    let y = 20;
    let p = Point { x: &x, y: &y }; // p 的生命周期不能超过 x 和 y
    println!("x: {}, y: {}", p.x, p.y); // 10, 20
}

    6.结构体 vs 元组：何时选择结构体？¶

    结构体和元组都是复合类型，但适用场景不同：

    类型 核心区别 适用场景
    结构体 字段有名称，类型更明确 数据需要长期复用、字段含义重要（如用户、订单）
    元组 字段无名称，依赖顺序 临时封装少量数据、函数多返回值（字段含义明显）

    选择准则：

    • 若数据需要被多次复用，且字段含义对可读性至关重要（如 User { id, name, email }），用结构体；
    • 若数据是临时的、字段含义可通过位置推断（如 (x, y) 坐标、(商, 余数)），用元组。

    2.4.枚举(Enums)¶

    1.定义与使用¶

    枚举通过enum关键字定义，可以包含多个变体（variants）。每个变体可以有不同的类型和关联数据。

    2.示例¶

    2.1.基本示例¶
    // 简单枚举
enum Color {
    Red,
    Green,
    Blue,
}
    2.2.带数据的枚举¶

    枚举的变体可以关联数据，这些数据可以是任何类型，甚至是另一个枚举。

    enum IpAddr {
    V4(String),
    V6(String),
}

// 使用
let home = IpAddr::V4(String::from("127.0.0.1"));
let loopback = IpAddr::V6(String::from("::1"));
    2.3.更复杂的枚举¶

    每个变体可以有不同的数据类型和数量。

    enum Message {
    Quit,                       // 没有关联数据
    Move { x: i32, y: i32 },    // 包含一个匿名结构体
    Write(String),              // 包含一个String
    ChangeColor(i32, i32, i32), // 包含三个i32
}

    3.枚举的使用¶

    // 创建枚举实例
let red = Color::Red;
let home = IpAddr::V4(String::from("127.0.0.1"));
let msg = Message::Move { x: 10, y: 20 };

let colors = [Color::Red, Color::Green, Color::Blue];
for color in colors {
    match color {
        Color::Red => println!("Red color"),
        Color::Green => println!("Green color"),
        Color::Blue => println!("Blue color"),
    }
}

// 使用 IpAddr 的所有变体并读取字段
let ip_v4 = IpAddr::V4(String::from("127.0.0.1"));
let ip_v6 = IpAddr::V6(String::from("::1"));

// 读取 IpAddr 字段来避免警告
match &ip_v4 {
    IpAddr::V4(addr) => println!("IPv4 address: {}", addr),
    IpAddr::V6(addr) => println!("IPv6 address: {}", addr),
}

match &ip_v6 {
    IpAddr::V4(addr) => println!("IPv4 address: {}", addr),
    IpAddr::V6(addr) => println!("IPv6 address: {}", addr),
}

// 模式匹配
match msg {
    Message::Quit => println!("Quit"),
    Message::Move { x, y } => println!("Move to ({}, {})", x, y),
    Message::Write(text) => println!("Write: {}", text),
    Message::ChangeColor(r, g, b) => println!("Change color to ({}, {}, {})", r, g, b),
}

// if let 简化匹配
if let Message::Write(text) = msg {
    println!("Writing: {}", text);
}

    5.1.Option<T>¶

    T 是泛型参数

    enum Option<T> {
    Some(T),
    None,
}

// 使用示例
let some_number = Some(5);
let absent_number: Option<i32> = None;

// 安全地处理可能缺失的值
match some_number {
    Some(value) => println!("Got: {}", value),

    5.2.Result<T, E>¶
    • T：代表 Success Type（成功类型）
    • 当操作成功时返回的数据类型
    • 可以是任何类型：基本类型、结构体、枚举、元组等
    • 位于 Ok(T) 变体中
    • E：代表 Error Type（错误类型）
    • 当操作失败时返回的错误信息的类型
    • 通常是实现了 Error trait 的类型
    • 位于 Err(E) 变体中

    Ok(T),
    Err(E),

// 使用示例
fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("Division by zero"))

        Ok(a / b)
    }
}

match divide(10.0, 2.0) {
    Ok(result) => println!("Result: {}", result),
    Err(error) => println!("Error: {}", error),
}

    2.5.数组 (Arrays)¶

    1.基本概念¶

    数组是固定长度的、相同类型元素的集合，在栈上分配内存。与此对应，动态数组 Vector 是存储在堆上，因此长度可以动态改变。

    2.语法和创建方式¶

    // 方式1: 显式指定类型和长度
let arr1: [i32; 5] = [1, 2, 3, 4, 5];

// 方式2: 推断类型
let arr2 = [1, 2, 3, 4, 5];

// 方式3: 创建相同值的数组
let arr3 = [0; 5]; // [0, 0, 0, 0, 0]

// 方式4: 使用常量定义长度
const LENGTH: usize = 3;
let arr4 = [1, 2, 3];

    3.数组操作¶

    let mut numbers = [1, 2, 3, 4, 5];

// 访问元素
let first = numbers[0];
let second = numbers[1];

// 修改元素
numbers[0] = 10;

// 获取长度
let len = numbers.len();

// 数组切片
let slice: &[i32] = &numbers[1..4]; // [2, 3, 4]

// 遍历数组
for element in &numbers {
    println!("{}", element);
}

// 带索引遍历
for (index, value) in numbers.iter().enumerate() {
    println!("Index: {}, Value: {}", index, value);
}

    4.多维数组¶

    // 二维数组
let matrix: [[i32; 3]; 2] = [
    [1, 2, 3],
    [4, 5, 6],
];

// 访问二维数组元素
let element = matrix[0][1]; // 2

// 遍历二维数组
for row in &matrix {
    for element in row {
        print!("{} ", element);
    }
    println!();
}

    5.数组与向量(Vector)的区别¶

    // 数组 - 固定大小，栈上分配
let array: [i32; 3] = [1, 2, 3];

// 向量 - 动态大小，堆上分配
let vector: Vec<i32> = vec![1, 2, 3];

    6.数组应用：固定缓冲区¶

    struct CircularBuffer<T, const N: usize> {
    buffer: [Option<T>; N],
    head: usize,
    tail: usize,
}

impl<T, const N: usize> CircularBuffer<T, N> {
    fn new() -> Self {
        Self {
            buffer: [const { None }; N], // 初始化N个None
            head: 0,
            tail: 0,
        }
    }

    fn push(&mut self, item: T) -> Result<(), &'static str> {
        if (self.tail + 1) % N == self.head {
            Err("Buffer full")
        } else {
            self.buffer[self.tail] = Some(item);
            self.tail = (self.tail + 1) % N;
            Ok(())
        }
    }

    fn pop(&mut self) -> Option<T> {
        if self.head == self.tail {
            None
        } else {
            let item = self.buffer[self.head].take();
            self.head = (self.head + 1) % N;
            item
        }
    }
}

    7.枚举 vs 数组 对比表¶

    特性 枚举 (Enum) 数组 (Array)
    用途 表示多种可能的状态或类型 存储多个相同类型的值
    内存分配 大小是最大变体的大小 固定大小，在栈上分配
    可变性 实例本身可变，但变体类型不可变 元素可变，长度不可变
    元素类型 每个变体可以有不同的类型 所有元素必须是相同类型
    长度 变体数量固定 创建时指定，固定不变
    模式匹配 支持完整的模式匹配 支持通过索引访问
    安全性 编译时检查所有变体都已处理 编译时和运行时边界检查
    典型用例 状态机、错误处理、命令模式 固定大小的数据集合、缓冲区

    2.6.集合类型¶

    在 Rust 中，集合类型（Collections） 是标准库提供的动态数据结构，用于存储和管理多个值。与基础复合类型（如数组、元组）相比，集合类型的核心优势是长度可动态调整（运行时增删元素），且支持更丰富的操作（如查询、排序、过滤等）。它们是处理复杂数据集合的核心工具，广泛应用于实际开发中。

    1.集合类型的共性与分类¶

    Rust 标准库的集合类型主要定义在 std::collections 模块中，提供的通用动态数据结构。

    核心特征是：

    • 可存储多个元素（区别于单个值的基本类型）；
    • 元素类型通用（通过泛型支持任意类型 T，如 Vec<T>、HashMap<K, V>）；
    • 长度动态可变（运行时可增删元素，区别于固定长度的数组 [T; N] 或元组）；
    • 数据存储在堆上（集合变量本身仅持有堆内存的引用或指针）。

    按 功能 可分为 三大类：

    • 序列集合：存储有序的元素序列（如动态数组、双端队列）；
    • 映射集合：存储键值对（如哈希表、B 树映射）；
    • 集合（Set）：存储唯一元素（如哈希集合、B 树集合）。

    所有集合类型均为泛型（支持任意类型的元素），且不存储在栈上（数据存储在堆上，集合变量本身仅持有堆内存的引用或指针）。

fn set_operations() {
    println!("\n=== 集合操作 ===");

    let set_a = BTreeSet::from([1, 2, 3, 4, 5]);
    let set_b = BTreeSet::from([4, 5, 6, 7, 8]);

    println!("集合 A: {:?}", set_a);
    println!("集合 B: {:?}", set_b);

    // 并集 - 所有出现在 A 或 B 中的元素
    let union: BTreeSet<_> = set_a.union(&set_b).cloned().collect();
    println!("并集 A ∪ B: {:?}", union); // {1, 2, 3, 4, 5, 6, 7, 8}

    // 交集 - 同时出现在 A 和 B 中的元素
    let intersection: BTreeSet<_> = set_a.intersection(&set_b).cloned().collect();
    println!("交集 A ∩ B: {:?}", intersection); // {4, 5}

    // 差集 - 在 A 中但不在 B 中的元素
    let difference: BTreeSet<_> = set_a.difference(&set_b).cloned().collect();
    println!("差集 A - B: {:?}", difference); // {1, 2, 3}

    // 对称差集 - 在 A 或 B 中但不同时在两者的元素
    let symmetric_difference: BTreeSet<_> = set_a.symmetric_difference(&set_b).cloned().collect();
    println!("对称差集 A Δ B: {:?}", symmetric_difference); // {1, 2, 3, 6, 7, 8}
}

    3.8. 二叉堆（BinaryHeap<T>）¶

    定义：BinaryHeap<T> 是基于二叉堆实现的优先队列（Priority Queue），核心功能是快速访问和提取 “最大元素”（默认是最大堆），插入和提取操作效率高效。

    核心特性：¶

    • 底层实现：以 Vec<T> 为基础存储的完全二叉树（逻辑结构是树，物理存储是数组），通过索引计算父节点与子节点位置（左子节点索引 2*i+1，右子节点 2*i+2，父节点 (i-1)/2）；

    • 泛型约束：元素类型 T 必须实现 Ord trait（用于维护堆的有序性，默认按 “最大元素在堆顶” 排序）；

    • 无序性：除堆顶元素（最大元素）外，其他元素的顺序不保证有序，遍历结果也不按插入顺序或全排序输出。

    常用操作：¶

    clear() () O(n) 清空所有元素

    查询 len() usize O(1) 元素数量
    is_empty() bool O(1) 是否为空
    capacity() usize O(1) 当前容量
    迭代 iter() Iter<T> O(1) 元素迭代器（无序）
    into_iter() IntoIter<T> O(1) 消费迭代器

    println!("提取元素：{}", val); // 依次输出 5、3、2、1（每次提取当前最大元素）
}

    BinaryHeap 默认是最大堆，若需最小堆（堆顶为最小元素），可通过 Reverse<T> 包装元素（Reverse 会反转 Ord 的排序逻辑）：

    use std::collections::BinaryHeap;
use std::cmp::Reverse;

let mut min_heap = BinaryHeap::new();
min_heap.push(Reverse(3));
min_heap.push(Reverse(1));
min_heap.push(Reverse(5));

// 堆顶为最小元素（Reverse(1)）
println!("最小元素：{:?}", min_heap.peek().map(|r| r.0)); // Some(1)

// 提取顺序：1、3、5
while let Some(Reverse(val)) = min_heap.pop() {
    println!("提取元素：{}", val);
}
    性能与适用场景：¶
    • 优势：
    • 插入元素（push）：时间复杂度 O(log n)（堆调整操作）；
    • 提取最大元素（pop）：时间复杂度 O(log n)；
    • 查看最大元素（peek）：时间复杂度 O(1)。
    • 劣势：
    • 不支持随机访问（无法通过索引访问任意元素）；
    • 遍历结果无序（仅保证堆顶是最大元素）。
    • 适用场景：
    • 需要频繁获取 “优先级最高” 元素的场景（如任务调度：优先级高的任务先执行）；
    • 求 Top K 问题（如从大量数据中快速获取前 10 大元素）；
    • 事件驱动模拟（如按时间戳排序的事件，优先处理最早 / 最晚事件）。

    2.7.联合体（union）¶

    联合体（union）是 Rust 中一种最底层、最不安全的复用内存机制： 所有成员共用同一段字节序列，读写时由程序员自己保证类型正确性。

    除了用 union 代替 struct外，联合体声明使用和结构体声明相同的句法。

    #[repr(C)]
union MyUnion {
    f1: u32,
    f2: f32,
}

    1.语法与最简示例¶

    #[repr(C)]
union MyUnion {
    f1: u32,
    f2: f32,
}

fn main() {
    let u = MyUnion { f1: 0x3F800000 };   // 按 u32 写入
    unsafe {
        println!("as u32: 0x{:x}", u.f1);
        println!("as f32: {}", u.f2);     // 0x3F800000 的位模式 → 1.0
    }
}
    • 访问任何字段都必须 unsafe 块。
    • 所有字段地址相同，size_of::<MyUnion>() == max(size_of(f1), size_of(f2))。

    2.使用场景¶

    1. 与 C ABI 交互 大量 Windows/Linux 系统调用把同一块内存解释为不同结构体。
    #[repr(C)]
union sigval {
    sival_int: c_int,
    sival_ptr: *mut c_void,
}
    1. 实现自解释二进制协议 网络包头部含类型字段，后面载荷按类型解析：
    #[repr(C)]
struct Packet {
    tag: u8,
    payload: Payload,   // union
}
    1. 手写无赘余的可变结构 当 tag 已在外部别处保存，内部不想再浪费 1 字节 tag。

    3.规则与限制¶

    • 字段必须全部实现 Copy（或你手动 impl Drop 并自己保证不双重释放）。
    • 不能直接使用 match；需要外部 tag 判断。
    • 未初始化字段整体视为未初始化内存，读之前必须保证对应字段已写。
    • #[repr(C)] 最常用，确保与 C 布局一致；也可用 #[repr(transparent)] 当只有 1 个非零尺寸字段时。

    ✔️ enum：「枚举型」—— 多个不同的「变体 (Variant)」互斥、同一时间仅一个生效，变体间是「或」的关系（是 A或是 B）；

    3.1.3.导入导出语句¶

    Rust 通过 mod、use、pub 等关键字管理代码的模块化、可见性和路径。其核心逻辑可以用下图概括：

    flowchart TD
    A[Crate（项目包）] --> B[根模块 lib.rs/main.rs]
    B --> C[使用 mod 声明子模块]
    C --> D{模块定义方式}
    D --> E[内联代码块]
    D --> F[指向同名文件]

    C --> G[使用 pub 控制可见性<br>（对外导出）]
    G --> H[pub:公开<br/>pub crate:包内可见<br/>pub super: 父模块可见]

    B --> I[使用 use 引入路径<br>（对内导入）]
    I --> J[缩短调用路径<br>选择性引入<br>as 起别名]
    1.导出语句¶

    mod 即是用作声明模块。

    pub 是 public 的缩写，是 Rust 中唯一的导出关键字，语法上是「可见性修饰符」，可以修饰所有代码项（模块、函数、结构体、枚举、常量、字段、枚举变体等）。

    ✔️ 粒度 1：基础导出 pub → 【上级可见，最常用，90% 场景】¶
    • 语法
    pub 代码项 // 无任何后缀
    • 可见范围

    • 被修饰的代码项，对「当前模块的所有上级模块（父、祖父... 直到根模块）」可见，同时对当前模块的所有子模块可见。

    • 核心规则

    • 要访问模块内的pub项，必须先让「模块本身是可访问的」（模块默认私有，若模块是私有的，内部再pub也无法被外部访问）。

    • 完整示例

    // 定义私有模块（默认）
mod user {
    // pub导出函数 → 外部可访问
    pub fn get_name() -> &'static str { "张三" }
    // 私有函数 → 外部不可访问
    fn get_age() -> u32 { 20 }
}

fn main() {
    println!("{}", user::get_name()); // ✅ 模块内的pub项，可访问
    // println!("{}", user::get_age()); // ❌ 私有项，编译报错：function `get_age` is private
}
    ✔️ 粒度 2：包内导出 pub(crate) → 【项目内可见，生产级首选，重中之重】¶
    • 语法
    pub(crate) 代码项

    • 可见范围

    • 被修饰的代码项，对「整个项目（crate）的所有模块」都可以自由访问，但如果你的项目被当作依赖库引入到其他项目时，这些项对「外部项目」是不可见的。

      crate 是 Rust 的编译单元，一个 cargo new 创建的项目就是一个 crate，一个 lib.rs 定义的库也是一个 crate。

    • 核心价值 & 使用场景: Rust 开发中最实用的导出粒度

    • 价值：实现「内部封装」—— 项目内部的通用工具、公共逻辑可以自由调用，对外隐藏实现细节，避免暴露不必要的接口。

    • 场景：项目内的工具函数、内部常量、业务中间层逻辑，优先使用 pub(crate) 而非基础的 pub。

    • 完整示例

    // 包内可见的工具函数：整个项目的所有模块都能访问，外部依赖不可见
pub(crate) fn format_str(s: &str) -> String {
    format!("[{}]", s)
}

mod a {
    pub fn test() {
        println!("{}", super::format_str("测试")); // ✅ 跨模块访问，包内可见
    }
}

mod b {
    pub fn test2() {
        crate::a::test(); // ✅ 根路径访问，包内可见
    }
}
    ✔️ 粒度 3：父级导出 pub(super) → 【仅父模块可见，极致封装】¶
    • 语法
    pub(super) 代码项

    • 可见范围

    • 被修饰的代码项，仅对「定义它的模块的直接父模块」可见，祖父模块、子模块、外部模块都完全不可访问。

    • 使用场景

    • 父子模块的「专属协作」：子模块封装了一些细节逻辑，只希望给父模块调用，不给其他模块访问，做到极致的代码封装，大型项目模块化治理常用。

    • 完整示例

    mod parent { // 父模块
    mod child { // 子模块
        // 仅父模块可见的函数
        pub(super) fn child_func() -> &'static str { "子模块的专属函数" }
    }

    pub fn parent_func() {
        println!("{}", child::child_func()); // ✅ 父模块可访问，合法
    }
}

fn main() {
    parent::parent_func(); // ✅ 正常调用父模块的公有函数
    // parent::child::child_func(); // ❌ 编译报错：仅父模块可见，外部无法访问
}
    ✔️ 粒度 4：路径导出 pub(in 路径) → 【精准可见，高级封装】¶
    • 语法
    pub(in 完整模块路径) 代码项
// 示例：pub(in crate::a::b) fn func() {}

    • 可见范围

    • 被修饰的代码项，仅对「指定的模块路径」内的所有模块可见，其他任何模块都无法访问。指定的路径必须是「当前模块的祖先路径」（父、祖父、根）。

    • 使用场景

    • 大型项目的精细化模块化治理：精准控制接口的访问范围，杜绝接口被滥用，是 Rust 模块化的高级特性，无性能损耗，库开发 / 大型工程必备。

    • 完整示例

    mod a {
    mod b {
        mod c {
            // 仅在 crate::a 这个路径下的所有模块可见
            pub(in crate::a) fn precise_func() -> &'static str { "精细化可见性函数" }
        }
    }

    pub fn a_func() {
        println!("{}", b::c::precise_func()); // ✅ 指定路径内，合法访问
    }
}

fn main() {
    a::a_func(); // ✅ 正常调用
    // a::b::c::precise_func(); // ❌ 超出指定路径，编译报错
}
    2.导入语句¶

    use 是 Rust 的导入关键字，它的核心行为是：在当前模块的作用域中，为「目标代码项的完整模块路径」创建一个「本地别名」。

    关键补充：use 只是「引用路径」，不会改变代码项的可见性、不会复制代码、不会增加二进制体积，编译后会被编译器优化掉，是「零成本抽象」。

    use 将其他模块中的路径引入当前作用域，简化调用，形成“导入”。

    • 导入项：use std::collections::HashMap;
    • 导入模块：use std::io; 然后通过 io::Read 使用。
    • 相对路径：use self::my_module::some_func; (self 当前模块， super 父模块)。
    • 重命名：use std::io as MyIO;
    • 合并导入：use std::{cmp::Ordering, io};
    • 通配符导入（谨慎使用）：use std::collections::*; 通常不推荐，易导致命名冲突。

    路径类型：

    • ✔️ ① 绝对路径（推荐，最严谨，生产级首选）：从「根模块（crate）」开始的完整路径，以 crate:: 作为起始标识；
    // 绝对路径示例
crate::utils::format_str(); // 根模块 → utils模块 → format_str函数
crate::model::user::User::new(); // 根模块 → model → user → User结构体
    • ✔️ ② 相对路径（适合模块内部嵌套）：从「当前模块」开始的路径，以 self（当前模块）、super（父模块）作为起始标识；
    mod parent {
    pub fn func() {}
    mod child {
        use super::func; // 相对路径：父模块的func函数
        use self::sub_child::func2; // 相对路径：当前模块的子模块
        mod sub_child { pub fn func2() {} }
    }
}
    ✔️ 语法 1：基础单项导入 → use 完整路径; 【最常用，70% 场景】¶
    // 导入标准库的项
use std::collections::HashMap;
use std::fs::read_to_string;

// 导入自定义模块的项（绝对路径，推荐）
mod utils { pub fn format_str(s: &str) -> String { s.to_uppercase() } }
use crate::utils::format_str;

fn main() {
    let mut map = HashMap::new();
    map.insert(1, "a");
    let _ = read_to_string("test.txt");
    let s = format_str("rust");
}
    ✔️ 语法 2：批量导入 → use 路径::{项1,项2,项3}; 【高频，20% 场景】¶
    // 批量导入标准库的多个集合类型
use std::collections::{HashMap, VecDeque, BTreeMap};

// 批量导入自定义模块的多个函数
mod utils {
    pub fn a() {}
    pub fn b() {}
    pub const C: i32 = 100;
}
use crate::utils::{a, b, C};
    ✔️ 语法 3：重命名导入 → use 路径::项 as 别名; 【重中之重，必学，解决命名冲突】¶
    // 场景1：解决命名冲突（最常用）
mod a { pub fn test() { println!("a::test"); } }
mod b { pub fn test() { println!("b::test"); } }
use crate::a::test as test_a;
use crate::b::test as test_b;

// 场景2：简化冗长名称
use std::collections::HashMap as Map;
use std::fmt::Result as FmtResult;

fn main() {
    test_a(); // ✅ 无冲突调用
    test_b(); // ✅ 无冲突调用
    let _: Map<i32, String> = Map::new(); // ✅ 简化名称
}
    ✔️ 语法 4：嵌套导入 → use 路径::{子路径1, 子路径2::{项}}; 【进阶高频，优雅简洁】¶
    // 嵌套导入标准库的项：导入std::io下的stdin，和std::io::Write下的write
use std::io::{stdin, Write};

// 嵌套导入自定义模块的多层嵌套项
mod a {
    pub mod b { pub mod c { pub fn func() {} } }
}
use crate::a::b::{c, c::func};

fn main() {
    let mut buf = String::new();
    stdin().read_line(&mut buf).unwrap(); // ✅ 嵌套导入的项
    func(); // ✅ 嵌套导入的函数
}
    ✔️ 语法 5：通配符导入 → use 路径::*; 【按需使用，谨慎】¶
    // 导入标准库集合模块的所有项
use std::collections::*;

// 导入自定义工具模块的所有项
mod utils { pub fn a() {} pub fn b() {} }
use crate::utils::*;

fn main() {
    let _: HashMap<i32, i32> = HashMap::new(); // ✅ 通配符导入的项
    a(); // ✅ 通配符导入的函数
}
    ✔️ 语法 6：父模块导入 → use 路径::{self, 项}; 【进阶，导入模块本身 + 模块内项】¶
    mod utils {
    pub fn a() {}
    pub mod b { pub fn c() {} }
}
// 导入utils模块本身，以及utils::a、utils::b
use crate::utils::{self, a, b};

fn main() {
    utils::a(); // ✅ 使用模块本身调用
    a(); // ✅ 直接使用导入的项
    b::c(); // ✅ 使用导入的子模块
}
    ✔️ 语法 7：根 / 父模块导入 → use crate::* / use super::* 【模块化必备】¶

// src/main.rs
use crate::utils::format_str; // 根模块导入，跨文件访问
fn main() {
    println!("{}", format_str("rust"));
}
    ✔️ 语法 8：从当前 crate 中的模块导入 （但不是从根开始）¶
    use super::some_module;  // 从父模块导入
use self::inner_mod::*;  // 从当前模块导入

    3.2.表达式（Expression）：有返回值的代码片段¶

    表达式是 Rust 中 “有返回值的代码片段”，函数体由一系列语句组成，也可选择以表达式结尾，几乎所有 Rust 代码都是表达式（或由表达式组成）。

    这是 Rust 与其他语言（如 C、Java）的重要区别。

    表达式：是指会计算出一个值，并可以返回这个值的代码片段，所有能产生「结果」的代码都是表达式。

    语法特征：表达式结尾「不加」分号 ;，如果给表达式加上分号，它会「降级为语句」，失去返回值，只保留执行逻辑。

    特性 语句 (Statement) 表达式 (Expression)
    返回值 固定返回 ()，无实际值 返回具体的、有类型的值（如 i32、String）
    分号规则 必须加分号 ;，否则编译报错 不加;，加了就会变成语句
    核心作用 执行「操作」，如声明、赋值、循环 执行「计算」，得到一个结果
    能否赋值 不能赋值给变量（无返回值） 可以赋值给变量 let a = 表达式;

    3.2.1. 核心定义与特点¶

    • 有返回值：表达式执行后会产生一个值（类型由表达式逻辑决定）；
    • 可作为值使用：表达式的结果可以赋值给变量、作为函数参数或返回值；
    • 无分号结尾：若在表达式后加分号，它会变成语句（Statement），返回值被忽略（返回 ()，单元类型）。

    3.2.2. 常见表达式类型¶

    （1）基础表达式¶
    let a = 1 + 2; // "1 + 2" 是表达式，返回 3（i32 类型）
let b = (5 * 3).to_string(); // "5 * 3" 是表达式（返回 15），后续调用方法也是表达式
    （2）条件/匹配表达式¶

    Rust 中的条件/匹配结构（if-else、 match 等）都是表达式，有返回值：

    // if-else 表达式：返回值类型需一致
let max = if a > b { a } else { b }; // 若 a=3, b=2，返回 3

// match 表达式：所有分支返回值类型需一致
let opt = Some(5);
let val = match opt {
    Some(x) => x * 2, // 返回 10
    None => 0,
};
    （3）流程控制表达式¶

    loop/while/for 循环、break/continue 等，必加分号（部分语法自带无需加），执行循环 / 跳转逻辑，无返回值。

    // loop 表达式：通过 break 返回值
let mut count = 0;
let result = loop {
    count += 1;
    if count == 3 {
        break count * 2; // 跳出循环并返回 6
    }
};
// while / for
let mut i = 0;
while i < 5 {
    println!("{}", i);
    i += 1;
}

for x in 0..3 {
    if x == 1 { continue; }
    println!("{}", x);
}
    （4）函数调用表达式¶

    函数调用本身是表达式，返回值为函数的返回值：

    let sum = add(2, 3); // "add(2, 3)" 是表达式，返回 5
    （5）闭包表达式¶

    匿名函数（闭包）是表达式，可捕获环境变量：

    let x = 10;
let add_x = |y| x + y; // 闭包表达式，返回一个可调用的函数
let result = add_x(5); // 调用闭包，返回 15
    （6）范围表达式¶

    1..10 / 1..=10，生成范围，是表达式

    3.2.3. 表达式 vs 语句¶

    Rust 中语句是 “无返回值的代码片段”，以分号结尾，主要用于执行副作用（如变量赋值、打印）：

    let a = 5; // 变量声明语句（有分号，无返回值，返回 ()）
println!("hi"); // 函数调用语句（有分号，无返回值，返回 ()）

// 表达式 vs 语句的关键区别：
let b = (1 + 2); // "1 + 2" 是表达式，加了分号后整体是语句，但赋值右侧仍取表达式结果 3
let c = { 1 + 2 }; // 代码块是表达式，返回 3（无分号）
let d = { 1 + 2; }; // 代码块内表达式加了分号，变成语句，代码块返回 ()
    类型 特点 示例 返回值
    表达式 无分号，有返回值 1 + 2、if x > 0 { x } else { 0 } 计算结果
    语句 有分号，无返回值 let x = 5;、println!("hi"); ()（单元类型）

    3.3.函数（Function）：代码复用与逻辑封装¶

    函数是一段命名的、可重复调用的代码块，用于封装特定逻辑，接收输入参数并返回输出结果。

    3.3.1. 核心定义与语法¶

    函数通过 fn 关键字定义，基本语法为：

    fn 函数名(参数名: 参数类型, ...) -> 返回值类型 {
    // 函数体（由语句和表达式组成）
}

    关键要素：

    • 函数名：遵循下划线命名法（snake case，直译为蛇形命名法）规范风格（如 add_two，而非 addTwo）；
    • 参数：
    • 参数是特殊变量，是函数签名的一部分。当函数拥有参数（形参）时，可以为这些参数提供具体的值（实参）。
    • 必须显式标注类型（Rust 不推断参数类型），多个参数用逗号分隔；
    • 返回值类型：
    • 用 -> 类型 声明
    • 无返回值时可省略（隐含返回 ()，即单元类型）， 还可以通过; 结尾的语句返回一个 ()：
    • 函数体：由语句和表达式组成，最后一个表达式的结果作为返回值（可省略 return 关键字）。

    3.3.2. 示例与特性¶

    （1）基础函数¶
    // 接收两个 i32 参数，返回 i32
fn add(a: i32, b: i32) -> i32 {
    a + b // 最后一个表达式的结果作为返回值（无分号）
}

// 无参数、无返回值（隐含返回 ()）
fn print_hello() {
    println!("Hello"); // 语句（有分号，无返回值）
}
    （2）返回值与 return 关键字¶

    函数返回值默认是最后一个表达式的结果，return 可用于提前返回：

    fn absolute_value(x: i32) -> i32 {
    if x < 0 {
        return -x; // 提前返回
    }
    x // 最后一个表达式作为返回值
}
    （3）参数特性¶

    • 必须标注类型：fn foo(x) { ... } 会编译错误，必须写 fn foo(x: i32) { ... }；

    • 默认参数：Rust 1.75+ 支持默认参数（需显式指定默认值）：

    fn greet(name: &str, prefix: &str = "Hello") {
    println!("{}: {}", prefix, name);
}
greet("Rust"); // 输出：Hello: Rust（使用默认 prefix）

    3.3.3. 函数的本质¶

    函数是 “命名的代码块”，其核心作用是代码复用和逻辑封装。它将一组相关操作打包，通过参数接收外部输入，通过返回值输出结果，使代码结构更清晰。

    3.4.特殊语句¶

    1.where 子句¶

    具体内容到泛型和约束中进行描述

    简介：

    • where 用于为泛型、关联类型、生命周期添加约束（trait bound / lifetime bound）。
    • 作用位置灵活：函数、方法、impl 块、trait 定义、类型别名、甚至关联类型（如 GATs）上都可以使用。
    • 最大优势是可读性：将复杂、多行的约束从尖括号中解耦出来，结构更清晰。

    语法位置与对比：

    • 内联写法： fn foo<T: Display + Debug>(x: T) { ... }
    • where 写法（等价，适合多约束）： fn foo<T>(x: T) where T: Display + Debug, { ... }

    何时优先使用 where？

    • 约束多、长、且分布在多个类型或关联类型上时。
    • 方法只在特定约束成立时才可用（把约束放到方法的 where 上，避免污染整个 impl）。
    • 需要写生命周期关系、HRTB、?Sized 等时，语义在 where 更清晰。

    实战范式小抄:

    • 对函数泛型加约束： fn f<T, U>(...) where T: A + B, U: C<Assoc: D>, ...
    • 对方法加约束（按需暴露功能）：impl<T> Ty<T> { fn m(&self) where T: Bound { ... } }
    • 对 impl 块整体加约束（条件实现）： impl<T> Trait for Ty<T> where T: Bound { ... }
    • 关联类型：where I: Iterator, I::Item: Display
    • 完整路径： where <T as Trait>::Assoc: Bound
    • 生命周期：where 'a: 'b, T: 'a
    • HRTB： where F: for<'a> Fn(&'a X) -> &'a Y
    • 动态大小： where T: ?Sized
    • 常见并发/内存边界： where T: Send + Sync + 'static

    四、流程控制¶

    在 Rust 中，流程控制（Control Flow） 用于管理代码的执行顺序，通过条件判断、循环和跳转语句，实现复杂逻辑的分支、重复和中断。

    4.1.条件判断：根据条件执行不同分支¶

    条件判断通过对 “布尔值” 的判断，决定执行哪段代码，核心结构包括 if-else 和简化版的 if let。

    1. if-else：多分支条件判断¶

    基本语法：

    if 条件表达式 {
    // 条件为 true 时执行
} else if 另一个条件表达式 {
    // 前一个条件为 false，当前条件为 true 时执行
} else {
    // 所有条件都为 false 时执行
}

    核心特点：

    • 条件必须是 bool 类型：Rust 不允许非布尔值自动转换为布尔值（如 if 0 { ... } 在 C 中合法，在 Rust 中报错）；
    • 无需括号包裹条件：语法上省略 ()，更简洁；
    • 分支块是表达式：if-else 整体是表达式，所有分支的返回值类型必须一致（可用于赋值）。

    示例：

    let num = 5;

// 基础用法
if num > 0 {
    println!("正数");
} else if num < 0 {
    println!("负数");
} else {
    println!("零");
}

// 作为表达式赋值（所有分支返回同类型）
let sign = if num > 0 {
    "positive" // 字符串字面量 &str
} else if num < 0 {
    "negative"
} else {
    "zero"
};
println!("{}", sign); // positive

    2. if let：简化的单分支模式匹配¶

    if let 是 match 表达式的简化版，用于只关心一种匹配情况的场景，避免冗余的 match 块。

    基本语法：

    if let 模式 = 表达式 {
    // 模式匹配成功时执行
} else {
    // 匹配失败时执行（可选）
}

    核心特点：

    • 基于模式匹配：比 if 更灵活，可直接解构枚举、元组、结构体等复合类型；
    • 简化代码：当只需要处理一种匹配结果时，比 match 更简洁。

    示例：

    let opt = Some(5);

// 用 if let 处理 Option<T> 的 Some 情况
if let Some(x) = opt {
    println!("值为：{}", x); // 输出：值为：5
} else {
    println!("无值");
}

// 对比：用 match 处理（更冗长）
match opt {
    Some(x) => println!("值为：{}", x),
    None => println!("无值"),
}

// 解构元组
let t = (1, "hello");
if let (n, s) = t {
    println!("n={}, s={}", n, s); // 输出：n=1, s=hello
}

    4.2.循环：重复执行代码块¶

    Rust 提供三种循环结构：loop（无限循环）、while（条件循环）、for（迭代循环），覆盖不同的重复执行场景。

    1. loop：无限循环（需手动终止）¶

    loop 是最基础的循环，会无限重复执行代码块，必须通过 break 语句手动终止。

    基本语法：

    loop {
    // 循环体
    if 终止条件 {
        break; // 终止循环
    }
}

    核心特点：

    • 可返回值：break 后可跟一个表达式，该值会作为 loop 表达式的返回值（Rust 独有特性）；
    • 适合不确定循环次数的场景（如等待用户输入、重试操作）。

    示例：

    // 基础用法：计数到 3 终止
let mut count = 0;
loop {
    count += 1;
    if count == 3 {
        println!("计数到 3，终止");
        break;
    }
}

// 带返回值的 loop
let mut sum = 0;
let result = loop {
    sum += 2;
    if sum >= 10 {
        break sum; // 返回 sum 的值（10）
    }
};
println!("result = {}", result); // 输出：result = 10

    2. while：条件循环（满足条件时执行）¶

    while 循环在每次迭代前判断条件，条件为 true 时执行循环体，否则终止。

    基本语法：

    while 条件表达式 {
    // 循环体（条件为 true 时执行）
}

    核心特点：

    • 适合已知终止条件的场景（如 “当计数器小于 10 时循环”）；
    • 条件表达式必须是 bool 类型，与 if 一致。

    示例：

    // 倒计时：从 3 到 1
let mut countdown = 3;
while countdown > 0 {
    println!("{}", countdown);
    countdown -= 1;
}
println!("发射！");
// 输出：
// 3
// 2
// 1
// 发射！

    3. for：迭代循环（遍历集合元素）¶

    for 循环是 Rust 中最常用的循环，通过迭代器（Iterator）遍历集合中的元素，无需手动管理索引，安全且简洁。

    基本语法：

    for 元素变量 in 迭代器 {
    // 循环体（依次处理每个元素）
}

    核心特点：

    • 依赖迭代器：支持所有实现 Iterator trait 的类型（如数组、Vec、范围 a..b、HashMap 等）；
    • 安全无越界：无需手动控制索引（如 for i in 0..arr.len() 可替代 C 风格的 for (i=0; i<len; i++)，但更推荐直接迭代元素）；
    • 范围迭代：a..b 表示左闭右开区间（a <= x < b），a..=b 表示左闭右闭区间（a <= x <= b）。

    示例：

    // 遍历范围（1 到 4，不包含 5）
for i in 1..5 {
    println!("{}", i); // 输出：1 2 3 4
}

// 遍历数组元素（直接访问值，无需索引）
let arr = [10, 20, 30];
for num in arr {
    println!("{}", num); // 输出：10 20 30
}

// 遍历 Vec（可变引用修改元素）
let mut vec = vec![1, 2, 3];
for num in &mut vec {
    *num *= 2; // 每个元素乘以 2
}
println!("{:?}", vec); // 输出：[2, 4, 6]

// 遍历 HashMap 键值对
use std::collections::HashMap;
let mut map = HashMap::new();
map.insert("a", 1);
map.insert("b", 2);
for (key, value) in map {
    println!("{}: {}", key, value); // 输出：a:1  b:2（顺序不确定）
}

    4.3.跳转语句：控制循环或函数的执行流程¶

    跳转语句用于中断正常的执行顺序，包括 break（终止循环）、continue（跳过当前迭代）、return（终止函数）。

    4.3.1. break：终止循环¶

    break 用于立即终止当前循环，跳出循环体。在嵌套循环中，可通过标签（label） 指定终止哪个循环。

    基本用法：

    // 基础用法：终止单循环
for i in 1..=10 {
    if i == 5 {
        break; // 当 i=5 时终止循环
    }
    println!("{}", i); // 输出：1 2 3 4
}

// 带标签的 break：终止嵌套循环中的外层循环
'outer: loop {
    println!("外层循环");
    for i in 1..=3 {
        println!("内层循环：{}", i);
        if i == 2 {
            break 'outer; // 终止标签为 'outer 的循环
        }
    }
}
// 输出：
// 外层循环
// 内层循环：1
// 内层循环：2

    标签语法：

    • 标签以单引号 ' 开头，后跟标识符（如 'outer、'loop1）；
    • 标签需写在循环前，通过 break '标签名 指定终止目标。

    4.3.2. continue：跳过当前迭代¶

    continue 用于跳过当前循环的剩余部分，直接进入下一次迭代。同样支持标签，用于嵌套循环中跳过指定循环的当前迭代。

    示例：

    // 基础用法：跳过偶数
for i in 1..=5 {
    if i % 2 == 0 {
        continue; // 跳过偶数
    }
    println!("{}", i); // 输出：1 3 5
}

// 带标签的 continue：跳过外层循环的当前迭代
'outer: for i in 1..=3 {
    for j in 1..=3 {
        if j == 2 {
            continue 'outer; // 跳过外层循环的当前迭代（i=1,2,3 时，j=2 都会触发）
        }
        println!("i={}, j={}", i, j);
    }
}
// 输出：
// i=1, j=1
// i=2, j=1
// i=3, j=1

    4.3.3. return：终止函数并返回值¶

    return 用于立即终止当前函数，并返回指定值（若函数有返回类型）。

    示例：

    fn is_positive(num: i32) -> bool {
    if num > 0 {
        return true; // 提前返回 true
    }
    false // 最后一个表达式返回 false
}

println!("{}", is_positive(5)); // 输出：true
println!("{}", is_positive(-3)); // 输出：false

    4.4.流程控制与表达式的结合¶

    Rust 是 “表达式导向” 的语言，所有流程控制结构本身都是表达式（有返回值），这一特性让代码更简洁灵活。

    4.4.1. if-else 作为表达式赋值¶

    如前文示例，if-else 的返回值可直接赋值给变量：

    let x = 3;
let y = if x > 5 { 10 } else { 20 }; // y = 20

    4.4.2. loop 作为表达式返回值¶

    loop 通过 break 返回值，可用于复杂逻辑的结果计算：

    let mut input = String::new();
let number = loop {
    println!("请输入一个数字：");
    input.clear();
    std::io::stdin().read_line(&mut input).unwrap();
    // 尝试解析输入为数字
    match input.trim().parse::<i32>() {
        Ok(n) => break n, // 解析成功，返回数字
        Err(_) => println!("无效输入，请重试"),
    }
};
println!("你输入的数字是：{}", number);

    4.4.3. for 循环的返回值¶

    for 循环本身返回 ()（单元类型），但循环体中可通过其他方式收集结果：

    let mut sum = 0;
for i in 1..=5 {
    sum += i;
}
// sum = 15（for 循环无返回值，通过变量收集结果）

    4.5.模式匹配¶

    在 Rust 中，模式匹配（Pattern Matching） 是一种强大的语法机制，用于检查值是否符合特定的结构或形式，并从中提取数据。它不仅能实现分支判断（类似其他语言的 switch），还能解构复杂数据类型（如枚举、结构体、元组等），同时通过编译器检查确保逻辑的完整性（如穷尽所有可能的情况）。

    4.5.1.模式的本质：匹配 “结构” 的规则¶

    模式是一种 “模板”，用于描述值的结构。当一个值与模式的结构匹配时，就可以执行对应的逻辑，或从值中提取部分数据。

    常见的模式类型包括：

    • 字面量（如 5、"hello"）；
    • 变量（如 x）；
    • 通配符（如 _ 忽略值，.. 忽略剩余部分）；
    • 复合结构（如枚举变体 Some(x)、结构体 Point { x, y }、元组 (a, b)）；
    • 范围（如 1..=5）；
    • 守卫（if 条件，用于进一步过滤）。

    4.5.2.核心场景：match 表达式¶

    match 是模式匹配最核心的用法，用于对一个值进行多分支匹配，语法为：

    match 目标值 {
    模式1 => 表达式1,
    模式2 => 表达式2,
    // ... 更多模式
}

    关键特性：

    1. 穷尽性（Exhaustiveness）：match 必须覆盖目标值的所有可能情况，否则编译报错（Rust 安全性的体现）。
    2. 不可达性（Irrefutability）：一旦某个模式匹配成功，后续分支不会执行（类似 switch 的 break，但 Rust 无需显式书写）。
    示例 1：匹配基础类型¶
    let num = 3;

match num {
    1 => println!("一"),
    2 => println!("二"),
    3 => println!("三"), // 匹配成功，执行此分支
    _ => println!("其他数字"), // 通配符 _ 匹配所有未覆盖的情况（保证穷尽性）
}
    • _ 是通配符，匹配任何值但忽略它，用于处理 “其他所有情况”，避免遗漏。
    示例 2：匹配枚举（核心场景）¶

    枚举的变体是模式匹配的典型应用，因为枚举的可能值是有限的，match 能确保覆盖所有变体。

    以标准库的 Option<T> 为例（表示 “可能有值或无值”）：

    let opt: Option<i32> = Some(5);

match opt {
    Some(x) => println!("有值：{}", x), // 匹配 Some 变体，提取内部值 x
    None => println!("无值"), // 匹配 None 变体（覆盖所有情况，保证穷尽性）
}
// 输出：有值：5

    自定义枚举的匹配：

    enum Shape {
    Circle(f64), // 半径
    Rectangle(f64, f64), // 长和宽
    Square(f64), // 边长
}

let shape = Shape::Rectangle(3.0, 4.0);

match shape {
    Shape::Circle(r) => println!("圆的面积：{}", 3.14 * r * r),
    Shape::Rectangle(w, h) => println!("矩形面积：{}", w * h), // 匹配并提取长和宽
    Shape::Square(s) => println!("正方形面积：{}", s * s),
}
// 输出：矩形面积：12
    示例 3：解构复合类型（结构体、元组）¶

    模式匹配可直接解构结构体、元组等复合类型，提取内部字段。

    // 结构体
struct Point { x: i32, y: i32 }
let p = Point { x: 10, y: 20 };

match p {
    Point { x, y } => println!("x={}, y={}", x, y), // 提取 x 和 y
    // 也可只关注部分字段，用 .. 忽略其他
    // Point { x, .. } => println!("x={}", x),
}
// 输出：x=10, y=20

// 元组
let t = (1, "hello", 3.14);
match t {
    (a, b, c) => println!("a={}, b={}, c={}", a, b, c), // 提取元组元素
}
// 输出：a=1, b=hello, c=3.14
    示例 4：范围模式与守卫（Guard）¶
    • 范围模式：用 .. 匹配连续范围的值（如数字、字符）。
    • 守卫（Guard）：在模式后加 if 条件，进一步过滤匹配（条件为 true 才执行分支）。
    let ch = 'c';

match ch {
    'a'..='z' => println!("小写字母"), // 范围模式：匹配所有小写字母
    'A'..='Z' => println!("大写字母"),
    _ => println!("其他字符"),
}
// 输出：小写字母

// 带守卫的模式
let num = 6;
match num {
    n if n % 2 == 0 => println!("偶数：{}", n), // 守卫：n 是偶数才匹配
    n => println!("奇数：{}", n),
}
// 输出：偶数：6

    4.5.3.简化模式匹配：if let 与 while let¶

    当只需要匹配一种情况，且不想写完整的 match 时，可用 if let 或 while let 简化代码。

    1. if let：单一模式的条件匹配¶

    if let 模式 = 目标值 { ... } else { ... } 用于只关心一种匹配结果的场景，替代冗长的 match。

    let opt: Option<i32> = Some(3);

// 用 if let 替代 match（只关心 Some 情况）
if let Some(x) = opt {
    println!("有值：{}", x); // 匹配成功，执行
} else {
    println!("无值"); // 匹配失败，执行
}
// 输出：有值：3

// 对比：用 match 实现（更冗长）
match opt {
    Some(x) => println!("有值：{}", x),
    None => println!("无值"),
}
    2. while let：循环中的模式匹配¶

    while let 模式 = 目标值 { ... } 用于循环中持续匹配某个模式，直到匹配失败。

    let mut stack = vec![1, 2, 3];

// 循环弹出栈顶元素（直到栈空）
while let Some(top) = stack.pop() {
    println!("弹出：{}", top);
}
// 输出：
// 弹出：3
// 弹出：2
// 弹出：1

    4.5.4.其他模式匹配场景¶

    模式匹配不仅用于 match/if let，还可在变量声明、函数参数、for 循环等场景中使用，实现 “解构赋值”。

    1. 变量声明中的模式¶

    用模式直接解构值，赋值给变量：

    // 解构元组
let (x, y) = (10, 20);
println!("x={}, y={}", x, y); // x=10, y=20

// 解构结构体（忽略部分字段）
struct Person { name: String, age: u32 }
let p = Person { name: "Alice".to_string(), age: 30 };
let Person { name, .. } = p; // 只提取 name，忽略 age
println!("name: {}", name); // name: Alice
    2. 函数参数中的模式¶

    函数参数可直接用模式解构传入的复合类型：

    // 解构元组参数
fn print_tuple((a, b): (i32, &str)) {
    println!("a={}, b={}", a, b);
}
print_tuple((1, "hello")); // 输出：a=1, b=hello

// 解构结构体参数
struct Point { x: i32, y: i32 }
fn print_point(Point { x, y }: Point) {
    println!("x={}, y={}", x, y);
}
print_point(Point { x: 5, y: 10 }); // 输出：x=5, y=10
    3. for 循环中的模式¶

    遍历集合时，用模式解构迭代器的元素：

    let pairs = vec![(1, "a"), (2, "b"), (3, "c")];

// 解构元组元素
for (num, s) in pairs {
    println!("{}: {}", num, s);
}
// 输出：
// 1: a
// 2: b
// 3: c

    4.5.5.模式的可反驳性与不可反驳性¶

    模式分为两类，决定了它们能在哪些场景使用：

    • 不可反驳的（Irrefutable）：一定能匹配成功的模式（如变量 x、结构体全字段 Point { x, y }）。
    • 可用于 let 声明、函数参数（这些场景要求模式必须匹配，否则编译错误）。
    • 可反驳的（Refutable）：可能匹配失败的模式（如 Some(x)、1..=5）。
    • 只能用于 match 分支、if let、while let（这些场景允许匹配失败）。

    示例：

    // 不可反驳的模式（一定匹配）
let x = 5; // x 是变量模式，不可反驳
let (a, b) = (1, 2); // 元组模式，不可反驳

// 可反驳的模式（可能不匹配）
if let Some(y) = Some(3) { ... } // Some(y) 可能不匹配（如目标是 None）

    4.5.6.核心优势总结¶

    Rust 的模式匹配之所以强大，在于：

    1. 结构解构能力：直接从枚举、结构体、元组等复合类型中提取数据，无需手动调用访问器（如 .field）。
    2. 穷尽性检查：match 必须覆盖所有可能情况，编译器帮你避免遗漏（如忘记处理 Option::None）。
    3. 灵活性：支持范围、守卫、通配符等，简化复杂条件判断。
    4. 代码简洁：用 if let/while let 替代冗长的分支逻辑，可读性更高。

    五、泛型和特征¶

    在 Rust 中，泛型（Generics） 和特征（Trait） 是实现代码复用、抽象通用行为的核心机制。泛型解决 “同一份逻辑适配多种类型” 的问题，特征则定义 “类型必须实现的行为接口”，二者结合可构建灵活且类型安全的抽象。

    专注于参数多态和接口抽象。

    5.1.泛型（Generics）：参数化类型的通用代码¶

    泛型允许我们在定义函数、结构体、枚举、方法等时，不指定具体类型，而是用类型占位符（如 T、U） 代替，从而让同一份代码适配多种类型，避免重复编写相似逻辑。

    1. 泛型的基本用法¶

    （1）泛型函数¶

    定义函数时，在参数 / 返回值类型中使用泛型占位符，使函数可接受多种类型的参数。

    // 泛型函数：返回两个值中较大的一个（暂不考虑类型是否可比较）
fn larger<T>(a: T, b: T) -> T {
    // 注意：这里实际会报错，因为 T 未约束为“可比较”类型，后续会用特征解决
    if a > b { a } else { b }
}

// 调用时，编译器自动推断 T 的具体类型
let num = larger(3, 5); // T = i32，返回 5
let str = larger("apple", "banana"); // T = &str，返回 "banana"
    （2）泛型结构体¶

    结构体的字段类型可声明为泛型，使结构体能存储任意类型的数据。

    // 泛型结构体：存储一对同类型的值
struct Pair<T> {
    first: T,
    second: T,
}

// 为泛型结构体实现方法（方法中可使用泛型 T）
impl<T> Pair<T> {
    fn new(first: T, second: T) -> Self {
        Pair { first, second }
    }
}

// 使用：存储 i32 类型
let int_pair = Pair::new(10, 20);
// 存储 String 类型
let str_pair = Pair::new(String::from("hello"), String::from("world"));
    （3）泛型枚举¶

    枚举的变体也可携带泛型类型，最典型的例子是标准库的 Option<T> 和 Result<T, E>。

    enum Option<T> {
    Some(T),
    None,
}
// 自定义泛型枚举：表示“可能是单个值，也可能是一对值”
enum SingleOrPair<T> {
    Single(T),
    Pair(T, T),
}

// 使用：存储 i32
let s = SingleOrPair::Single(5);
let p = SingleOrPair::Pair(3, 7);
// 存储 &str
let s_str = SingleOrPair::Single("hi");

    2. 泛型约束：限制泛型的范围¶

    泛型的 “通用性” 并非无限制的 —— 如果不对泛型参数 T 做约束，编译器无法保证 T 支持特定操作（如 a > b 中的比较）。此时需要通过特征（Trait） 对泛型进行约束，确保 T 实现了必要的行为。

    例如，修复上面 larger 函数的错误：

    // 引入 Ord 特征（标准库中定义，表示“可比较大小”）
use std::cmp::Ord;

// 泛型约束：T 必须实现 Ord 特征（才能使用 > 比较）
fn larger<T: Ord>(a: T, b: T) -> T {
    if a > b { a } else { b } // 现在合法，因为 T 保证可比较
}

    这里的 T: Ord 是特征约束，表示 “泛型参数 T 必须实现 Ord 特征”。

    3. 泛型的优势：零成本抽象¶

    Rust 编译器通过单态化（Monomorphization） 处理泛型：编译时，会将泛型代码替换为每种具体类型的专属实现（如为 i32、&str 分别生成 larger_i32、larger_str 函数）。

    这意味着：

    • 泛型代码不牺牲性能（无运行时开销）；
    • 保持类型安全（编译时检查每种具体类型的合法性）。

    5.2.特征（Trait）：定义类型的行为接口¶

    特征类似于其他语言中的 “接口（Interface）”，它定义了一组方法签名（可选的默认实现），类型通过 “实现特征” 来提供这些方法的具体逻辑。特征的核心作用是：

    • 抽象不同类型的共同行为（如 “可打印”“可比较”）；
    • 作为泛型约束，限制泛型参数必须具备的能力。

    1. 特征的定义与实现¶

    （1）定义特征¶

    用 trait 关键字定义特征，包含方法签名（可带默认实现）。

    // 定义“可总结”特征：类型实现此特征后可生成摘要
trait Summarizable {
    // 方法签名（无默认实现，必须由类型实现）
    fn summary(&self) -> String;

    // 带默认实现的方法（类型可选择重写或直接使用）
    fn author_summary(&self) -> String {
        format!("(作者：{})", self.summary_author())
    }

    // 辅助方法（仅在特征内部使用，或由类型实现）
    fn summary_author(&self) -> String;
}
    （2）为类型实现特征¶

    用 impl 特征 for 类型 语法为具体类型实现特征，必须提供所有无默认实现的方法。

    // 为 String 类型实现 Summarizable
impl Summarizable for String {
    fn summary(&self) -> String {
        self.clone() // 直接返回字符串本身作为摘要
    }

    fn summary_author(&self) -> String {
        String::from("匿名") // String 的作者默认为“匿名”
    }
}

// 自定义结构体：推文
struct Tweet {
    username: String,
    content: String,
}

// 为 Tweet 实现 Summarizable
impl Summarizable for Tweet {
    fn summary(&self) -> String {
        format!("{}: {}", self.username, self.content)
    }

    fn summary_author(&self) -> String {
        self.username.clone() // Tweet 的作者是用户名
    }

    // 重写默认的 author_summary 方法
    fn author_summary(&self) -> String {
        format!("推文作者：{}", self.username)
    }
}
    （3）使用特征方法¶

    只要类型实现了特征，就可以调用特征中定义的方法（包括默认方法）。

    let s = String::from("这是一段文本");
println!("文本摘要：{}", s.summary()); // 文本摘要：这是一段文本
println!("文本作者：{}", s.author_summary()); // 文本作者：(作者：匿名)

let tweet = Tweet {
    username: String::from("rust_user"),
    content: String::from("Rust 泛型真好用！"),
};
println!("推文摘要：{}", tweet.summary()); // 推文摘要：rust_user: Rust 泛型真好用！
println!("推文作者：{}", tweet.author_summary()); // 推文作者：推文作者：rust_user

    2. 特征作为泛型约束¶

    如前文所述，特征最常用的场景是约束泛型参数，确保泛型类型具备特定行为。

    // 泛型函数：接收任何实现了 Summarizable 特征的类型
fn notify<T: Summarizable>(item: T) {
    println!("新内容：{}", item.summary());
}

// 调用：String 和 Tweet 都实现了 Summarizable，均可传入
notify(String::from("新闻：Rust 1.70 发布"));
notify(Tweet {
    username: String::from("dev"),
    content: String::from("特征约束太方便了！"),
});

    3. 多重约束与 where 子句¶

    一个泛型参数可受多个特征约束，用 + 连接；若约束复杂，可用 where 子句简化语法。

    // 多重约束：T 必须同时实现 Summarizable 和 Display（Display 用于打印）
use std::fmt::Display;
fn print_summary<T: Summarizable + Display>(item: T) {
    println!("摘要：{}，内容：{}", item.summary(), item);
}

// 用 where 子句简化多重约束（更易读）
fn complex_function<T, U>(a: T, b: U)
where
    T: Summarizable + Clone,
    U: Display + Ord,
{
    // 使用 T: Clone + Summarizable
    let a2 = a.clone();
    println!("[complex_function] a 的摘要：{}", a2.summary());
    println!("[complex_function] a 的作者：{}", a2.author_summary());

    // 使用 U: Ord + Display
    // 这里只收到一个 b，但我们仍可以通过排序来“触发” Ord 的约束（哪怕是 1 个元素）
    let mut v = vec![b];
    v.sort(); // 需要 U: Ord
    println!("[complex_function] 最小元素：{}", v[0]); // 需要 U: Display
}

    4. 特征对象：动态多态¶

    特征还支持动态分发：通过 “特征对象（&dyn Trait 或 Box<dyn Trait>）” 在运行时使用不同类型，只要它们实现了同一特征。

    // 接收特征对象：可传入任何实现了 Summarizable 的类型的引用
fn print_any_summary(item: &dyn Summarizable) {
    println!("动态摘要：{}", item.summary());
}

let s = String::from("动态文本");
let tweet = Tweet { username: "a".into(), content: "动态推文".into() };

// 同一函数可接收不同类型（String 和 Tweet）
print_any_summary(&s); // 动态摘要：动态文本
print_any_summary(&tweet); // 动态摘要：a: 动态推文

    特征对象与泛型的区别：

    • 泛型通过单态化实现静态分发（编译时确定具体类型，性能优）；
    • 特征对象实现动态分发（运行时确定类型，灵活性高，有轻微性能开销）。

    5. 特征继承¶

    在 Rust 里，“ trait 继承” 的准确说法是 “ trait 的 super-trait 约束”（supertrait bounds）。 它表示：

    如果某类型实现了 trait Child，那么它必须先实现 trait Parent。 换句话说：Child 依赖 Parent 提供的功能； 对调用者而言，凡是 Child，就一定同时是 Parent。

    5.1.语法与语义¶
    trait Parent {
    fn parent(&self) -> &'static str;
}

trait Child: Parent {          // ← Parent 是 super-trait
    fn child(&self) -> &'static str;
}

// 多重 super-trait
trait A {
    fn a(&self);
}
trait B {
    fn b(&self);
}
trait C: A + B {          // 同时继承 A 和 B
    fn c(&self);
}
// 实现时必须 impl A + impl B + impl C 三者齐全。

// 基础特征：动物（父特征）
trait Animal {
    // 获取动物名称
    fn name(&self) -> &str;
}

// 子特征：宠物，继承自 Animal（要求实现 Pet 的类型必须先实现 Animal）
trait Pet: Animal {
    // 宠物的“玩耍”方法，内部可调用父特征 Animal 的 name 方法
    fn play(&self) {
        // 因为 Pet: Animal，所以 self 必然实现了 Animal，可调用 name()
        println!("{} 在玩耍！", self.name());
    }
}

// 具体类型：狗
struct Dog {
    name: String,
}

// 第一步：实现父特征 Animal
impl Animal for Dog {
    fn name(&self) -> &str {
        &self.name
    }
}

// 第二步：实现子特征 Pet（因 Dog 已实现 Animal，满足 Pet 的约束）
impl Pet for Dog {} // 可直接使用 Pet 的默认 play 方法

// 测试
fn main() {
    let dog = Dog { name: String::from("旺财") };
    dog.play(); // 输出：旺财 在玩耍！（调用 Pet 的 play 方法，内部使用了 Animal 的 name 方法）
}
    • Child: Parent 读作 “Child 继承 Parent”。
    • 任何 impl Child 的代码必须同时 impl Parent，否则编译失败。
    • trait Pet: Animal 表示 “Pet 依赖 Animal”，任何实现 Pet 的类型必须先实现 Animal；
    • Dog 先实现 Animal（提供 name 方法），再实现 Pet（此时满足约束）；
    • Pet 的 play 方法中可以调用 self.name()，因为编译器确保 self 必然实现了 Animal。
    5.2.示例¶
    1.示例 1：基础的特征继承¶
    trait Speak {
    fn speak(&self) -> &'static str;
}

trait Sing: Speak {            // Sing 继承 Speak
    fn sing(&self) -> &'static str;
}

// 具体类型
struct Person;

impl Speak for Person {
    fn speak(&self) -> &'static str {
        "hello"
    }
}

impl Sing for Person {
    fn sing(&self) -> &'static str {
        "la la la"
    }
}

// 泛型函数：要求 T 同时实现 Sing + Speak（由 Sing 隐含）
fn perform<T: Sing>(t: &T) {
    println!("speak: {}", t.speak()); // ← 因为 Sing: Speak，所以一定能调用
    println!("sing : {}", t.sing());
}

fn main() {
    let p = Person;
    perform(&p);
}

// 输出:
// speak: hello
// sing : la la la
    2.示例 2：子特征重写父特征相关逻辑¶

    子特征可以在父特征的基础上扩展功能，甚至重写默认方法，但仍需依赖父特征的核心能力。

    // 父特征：可移动的物体
trait Movable {
    // 移动的基础方法
    fn move_to(&self, x: i32, y: i32);
}

// 子特征：可追踪的移动体（继承自 Movable）
trait Trackable: Movable {
    // 扩展：记录移动轨迹（依赖 Movable 的 move_to 方法）
    fn move_and_track(&self, x: i32, y: i32) {
        // 先调用父特征的移动方法
        self.move_to(x, y);
        // 再添加追踪逻辑
        println!("记录轨迹：({}, {})", x, y);
    }
}

// 具体类型：机器人
struct Robot {
    id: u32,
}

// 实现父特征 Movable
impl Movable for Robot {
    fn move_to(&self, x: i32, y: i32) {
        println!("机器人 {} 移动到 ({}, {})", self.id, x, y);
    }
}

// 实现子特征 Trackable（因 Robot 已实现 Movable，满足约束）
impl Trackable for Robot {}

// 测试
fn main() {
    let robot = Robot { id: 100 };
    robot.move_and_track(5, 10);
    // 输出：
    // 机器人 100 移动到 (5, 10)
    // 记录轨迹：(5, 10)
}

    解析：

    • Trackable: Movable 确保实现 Trackable 的类型一定能移动（有 move_to 方法）；
    • move_and_track 方法结合了父特征的 move_to 和子特征的追踪逻辑，实现功能扩展。
    3.示例 3：多层继承（特征链）¶

    特征可以多层继承，形成依赖链（如 C: B 且 B: A，则实现 C 的类型必须实现 B 和 A）。

    // 第一层：基础特征（可命名）
trait Named {
    fn name(&self) -> &str;
}

// 第二层：继承自 Named（可描述）
trait Describable: Named {
    // 描述自己（依赖 Named 的 name 方法）
    fn describe(&self) -> String {
        format!("名称：{}", self.name())
    }
}

// 第三层：继承自 Describable（可评价）
trait Evaluatable: Describable {
    // 评价自己（依赖 Describable 的 describe 方法）
    fn evaluate(&self) -> String {
        format!("评价：{} 很棒！", self.describe())
    }
}

// 具体类型：书籍
struct Book {
    title: String,
}

// 实现第一层特征 Named
impl Named for Book {
    fn name(&self) -> &str {
        &self.title
    }
}

// 实现第二层特征 Describable（依赖 Named）
impl Describable for Book {}

// 实现第三层特征 Evaluatable（依赖 Describable，间接依赖 Named）
impl Evaluatable for Book {}

// 测试
fn main() {
    let book = Book { title: String::from("Rust 编程之道") };
    println!("{}", book.evaluate());
    // 输出：评价：名称：Rust 编程之道 很棒！
    // 调用链：evaluate → describe → name
}

    解析：

    • 依赖链：Evaluatable → Describable → Named，实现 Evaluatable 的 Book 必须依次实现 Describable 和 Named；
    • 每层特征的方法都能依赖上层特征的能力，形成功能叠加。

    5.4.泛型与特征的关系：相辅相成¶

    泛型和特征是 Rust 抽象机制的核心，二者的配合体现为：

    1. 泛型提供 “通用骨架”：定义不依赖具体类型的逻辑（如 “求最大值”“打印摘要”）；
    2. 特征提供 “约束边界”：通过 T: Trait 确保泛型参数具备必要的行为（如 “可比较”“可生成摘要”）；
    3. 共同实现代码复用：避免为每种类型重复编写逻辑，同时保证类型安全和性能。

    总结:

    • 泛型：通过类型参数化实现代码通用化，支持多种类型复用同一逻辑，编译时单态化确保零成本；
    • 特征：定义类型的行为接口，支持为不同类型实现共同方法，可作为泛型约束或通过特征对象实现动态多态；
    • 结合使用：泛型解决 “通用”，特征解决 “约束”，二者共同构建 Rust 灵活、安全、高效的抽象体系。

    六、所有权和生命周期¶

    6.1.所有权与借用¶

    1.栈(Stack)与堆(Heap)¶

    栈（Stack）与堆（Heap）是程序运行时的两种核心内存区域，区别不在于“语言”，而在于分配方式、生命周期、性能模型

    1.1.栈(Stack) — 自动弹夹¶

    一种 后进先出 的数据结构， 增加数据叫做进栈 (Push)，移出数据则叫做出栈 (Pop)。

    • 分配方式：编译器在进入函数时一次性移动“栈指针”，离开函数时一次性回退。
    • 生命周期：随作用域自动结束，无需程序员手动释放。
    • 性能：单指令级别（sub rsp, N / add rsp, N），缓存友好（连续地址）。
    • 大小：编译期已知（Sized），通常 KB ~ MB 级别；溢出即 stack overflow。
    • 典型数据：局部变量、函数参数、返回地址、小型数组/元组。

    Rust 代码对照：

    let x = 42i32;          // 栈上 4 字节
let p = (1, 2.0);       // 栈上 16 字节
let a = [0u8; 1024];    // 栈上 1 KB（只要不超过线程栈上限）
    1.2.堆(Heap) — 全局仓库¶

    对于 大小未知 或者 可能变化 的数据，我们需要将它存储在堆上。

    当向堆上放入数据时，需要请求一定大小的内存空间。操作系统在堆的某处找到一块足够大的空位，把它标记为已使用，并返回一个表示该位置内存地址 (也称指针 Pointer)，该过程被称为在堆上分配内存，有时简称为 “分配”(allocating)。

    接着，该指针会被推入栈中，因为指针的大小是已知且固定的，在后续使用过程中，你将通过栈中的指针，来获取数据在堆上的实际内存位置，进而访问该数据。

    • 分配方式：运行时向操作系统或内存分配器主动申请（malloc/brk/mmap/GlobalAlloc）。
    • 生命周期：程序员控制（Rust 靠所有权自动 drop；C 靠 free；Java/Go 靠 GC）。
    • 性能：算法 + 系统调用成本，缓存局部性差（随机地址），需额外元数据（块头、对齐）。
    • 大小：运行期决定，理论上仅受系统内存限制。
    • 典型数据：动态数组、字符串、对象、大结构体、递归数据结构（Box<ListNode>）。

    Rust 代码对照：

    let s = String::from("hello");                 // 内部 vec 在堆上分配 5 字节
let v: Vec<i32> = Vec::with_capacity(1000);    // 堆上 4*1000 字节
let b = Box::new(BigStruct([0u8; 1_000_000])); // 大对象放堆，避免栈溢出

    2.所有权¶

    2.1.核心规则¶

    Rust 的所有权机制基于三个基本规则，编译器会在编译时强制检查这些规则，违反规则会直接报错：

    1. 每个值在 Rust 中都有一个 “所有者”（Owner）：一个值只能被一个变量拥有。
    2. 同一时间只能有一个所有者：若将值的所有权转移给另一个变量，原所有者会失去所有权。
    3. 当所有者离开作用域（Scope）时，该值会被自动销毁：Rust 会在此时调用值的 drop 方法（类似析构函数），释放其占用的内存。
    2.2.作用域¶

    在 Rust 中，作用域（Scope） 是变量和值的 “有效范围”—— 即变量从声明到失效的代码区间。作用域是所有权机制的核心载体：当变量（所有者）离开作用域时，它所拥有的值会被自动销毁（释放内存）。

    可以简单理解为 “所有者活着的区间”，从变量被绑定的那一刻起，到离开最近的那对花括号 } 为止。

    在 Rust 中，作用域通常由花括号 {} 界定，常见的作用域场景包括：

    • 函数体（fn main() { ... }）；
    • 代码块（{ ... }）；
    • 控制流语句（if、for、while、match 等的代码块）。

    这个区间内自动保证：

    1. 资源有效；
    2. 离开时被立即 drop；
    3. 不会出现悬垂或二次释放 （所有权转移）。
    // 嵌套作用域：内层先死，外层后死
fn main() {
    let x = 5; // x 开始有效

    {
        let y = 10; // y 开始有效
        println!("x = {}, y = {}", x, y);
    } // y 离开作用域，被丢弃

    // println!("{}", y); // 错误！y 不再存在
    println!("x = {}", x); // x 仍然有效
} // x 离开作用域，被丢弃
    2.3.所有权的 “移动” 与 “复制”¶

    所有权的核心行为体现在值的传递（赋值、函数参数、返回值等场景）中，分为 “移动（Move）” 和 “复制（Copy）” 两种情况。

    2.3.1. 移动（Move）：所有权的转移¶

    当值是非基本类型（如 String、Vec 等，数据存储在堆上，容量可变）时，赋值或传递会导致所有权转移（原变量失去所有权，无法再使用）。

    示例：String 的 所有权移动

    fn main() {
    let s1 = String::from("hello"); // s1 是 "hello" 的所有者
    let s2 = s1; // 所有权从 s1 移动到 s2，s1 失效

    // 错误：s1 已失去所有权，无法再使用
    // println!("s1: {}", s1);
    // 编译报错：use of moved value: `s1`

    println!("s2: {}", s2); // 正确：s2 是当前所有者
}

    为什么会移动？

    • String 由三部分组成（栈上的指针、长度、容量 + 堆上的实际字符数据）：
    • 栈上的元数据（指针、长度、容量）可以快速复制；
    • 堆上的字符数据（"hello"）若复制会消耗大量内存（尤其数据量大时）。
    • Rust 为了避免 “双重释放”（double free）（当 s1 和 s2 离开作用域时，都试图释放堆上的同一份数据），设计为 “所有权转移”：s1 失效，只有 s2 能管理堆数据，最终由 s2 负责释放。
    2.3.2. 复制（Copy）：所有权不转移¶

    当值是基本类型（如 i32、bool、f64 等，数据存储在栈上，容量固定）时，赋值或传递会导致数据复制（原变量仍拥有所有权，可继续使用）。

    这类类型实现了 Copy trait，Rust 会自动复制其数据，无需转移所有权。

    示例：i32 的复制

    fn main() {
    let x = 5; // x 是 5 的所有者
    let y = x; // 复制 x 的数据给 y，x 仍有效（因 i32 实现了 Copy）

    println!("x: {}, y: {}", x, y); // 正确：x 和 y 都可用，输出 x: 5, y: 5
}

    哪些类型实现了 Copy？

    • 所有基本数据类型：i8-i128、u8-u128、f32、f64、bool、char；
    • 仅包含 Copy 类型的元组：如 (i32, bool) 是 Copy，但 (i32, String) 不是（因 String 不实现 Copy）；
    • 自定义类型默认不实现 Copy，需手动通过 #[derive(Copy, Clone)] 派生（且所有字段必须是 Copy 类型）。
    2.3.3. 克隆（Clone）：手动复制堆数据¶

    若希望对非 Copy 类型（如 String）进行 “深复制”（同时复制栈上元数据和堆上数据），需显式调用 clone 方法（对应 Clone trait），此时原变量仍拥有所有权。

    示例：String 的克隆

    fn main() {
    let s1 = String::from("hello");
    let s2 = s1.clone(); // 手动克隆：复制堆上数据，s1 仍有效

    println!("s1: {}, s2: {}", s1, s2); // 正确：s1 和 s2 都可用
}

    Copy vs Clone：

    • Copy 是隐式的、轻量的（栈上数据复制），不可手动重载；
    • Clone 是显式的（需调用 clone），可自定义实现（如深复制堆数据）。
    2.4.函数调用中的所有权¶

    函数参数传递和返回值也会触发所有权的转移（类似赋值）， 值被当作实参的那一刻，它的所有权就永久性地移交给了函数形参（除非形参是引用）。 此后调用者不能再使用该值，函数返回时可以选择：

    1. 把所有权再还给调用者（-> T）；
    2. 转移给别的变量/调用者（-> Some(T)、panic! 等）；
    3. 直接 drop（返回 () 或其他不携带 T 的类型）。

    示例：

    // 函数参数接收 String，会获取所有权
fn take_ownership(s: String) {
    println!("接收的字符串：{}", s);
} // s 离开作用域，堆数据被释放

// 函数返回 String，转移所有权给调用者
fn give_ownership() -> String {
    let s = String::from("返回的字符串");
    s // 返回 s，所有权转移给调用者
}

fn main() {
    let s1 = String::from("hello");
    take_ownership(s1); // s1 的所有权转移给函数参数，s1 失效

    // 错误：s1 已失去所有权
    // println!("s1: {}", s1);

    let s2 = give_ownership(); // s2 获得函数返回值的所有权
    println!("s2: {}", s2); // 正确
}

    3.引用与借用：临时访问值（不转移所有权）¶

    若仅需临时访问值（而非获取所有权），可使用引用（Reference）：通过 & 符号创建对值的引用，引用本身没有所有权，仅临时 “借用” 值的访问权。

    这种方式避免了频繁的所有权转移，是 Rust 中最常用的访问方式。

    3.1. 不可变引用（&T）¶

    创建不可变引用（&s），可以读取值但不能修改，且同一时间可存在多个不可变引用（只读不写，无数据竞争）。

    fn main() {
    let s = String::from("hello");
    let r1 = &s; // 不可变引用：借用 s 的访问权
    let r2 = &s; // 允许：多个不可变引用共存

    println!("r1: {}, r2: {}", r1, r2); // 正确：读取引用
} // r1、r2 先离开作用域（引用失效），然后 s 离开作用域（释放数据）
    3.2. 可变引用（&mut T）¶

    创建可变引用（&mut s），可以修改值，但同一时间只能存在一个可变引用，且不能与不可变引用同时存在（防止数据竞争：多个指针同时读写同一数据）。

    fn main() {
    let mut s = String::from("hello");
    let r1 = &mut s; // 可变引用：借用修改权
    // 错误：同一时间只能有一个可变引用
    // let r2 = &mut s;

    r1.push_str(", world"); // 正确：通过可变引用修改值
    println!("s: {}", s); // 输出：hello, world
}
    3.3. 悬垂引用（Dangling Reference）¶

    Rust 禁止 “悬垂引用”—— 引用指向的内存已被释放的情况。编译器会通过生命周期检查确保引用的生命周期不超过所有者的生命周期。

    // 错误示例：返回悬垂引用（编译报错）
fn dangle() -> &String {
    let s = String::from("dangle");
    &s // 尝试返回 s 的引用，但 s 会在函数结束时被释放
} // s 离开作用域，内存释放，返回的引用成为悬垂引用

// 正确做法：返回值本身（转移所有权）
fn no_dangle() -> String {
    let s = String::from("no dangle");
    s // 返回 s，所有权转移给调用者
}
    3.4. 特殊的引用：切片（Slice）¶

    切片（Slice） 是一种特殊的引用，指向集合（如字符串、数组）中连续的一部分数据，没有所有权，语法为 &[T]（数组切片）或 &str（字符串切片）。

    切片常用于安全地访问集合的部分数据，且能确保不越界。

    fn main() {
    let s = String::from("hello world");
    let hello = &s[0..5];  // 字符串切片：指向 "hello"（0 到 5 索引，不含 5）
    let world = &s[6..11]; // 字符串切片：指向 "world"

    println!("{} {}", hello, world); // 输出：hello world

    // 数组切片
    let arr = [1, 2, 3, 4, 5];
    let slice = &arr[1..3];  // 数组切片：指向 [2, 3]
    println!("{:?}", slice); // 输出：[2, 3]
}
    3.5. 解引用¶

    引用（&T / &mut T）和解引用（*t / *t = val）是 Rust 借用系统的底层操作接口：

    • 引用 = 不拥有的指针，编译期保证非空且合法；
    • 解引用 = 穿透引用层，直接摸到指向的数据；
    • 绝大多数场景 Rust 自动隐式解引用，但遇到裸指针、自定义智能指针或需要写内存时，必须显式写 *。

    显式解引用 —— 必须写 *

    • 裸指针
    let x = 5;
let raw = &x as *const i32;
unsafe {
    assert_eq!(*raw, 5);   // 显式 *
}
    • 可变引用写内存
    let mut y = 10;
let mr = &mut y;
*mr += 1;          // 不写 * 会编译错误
assert_eq!(y, 11);
    • 自定义智能指针
    use std::rc::Rc;
let x = Rc::new(42);
assert_eq!(*x, 42);   // 穿透 Rc<T> 摸到 T
    • 模式匹配中的解引用
    let ref_to_tuple = &(1, String::from("hi"));
let (a, ref b) = *ref_to_tuple; // 先显式解引用，再 ref 绑定第二项

    4.所有权的核心价值¶

    所有权机制通过编译时检查，解决了传统内存管理的痛点：

    • 无需 GC：内存释放时机明确（所有者离开作用域），无运行时开销；
    • 避免双重释放：同一时间只有一个所有者，确保内存只被释放一次；
    • 防止悬垂指针：引用的生命周期严格绑定到所有者，编译时杜绝悬垂；
    • 无数据竞争：可变引用的排他性确保多线程中不会同时读写同一数据。

    6.2.生命周期¶

    在 Rust 中，生命周期（Lifetime） 是一套编译时检查机制，用于跟踪引用的有效时间范围，确保引用不会比它所指向的数据 “存活得更久”（避免悬垂引用）。生命周期的核心目标是：在编译阶段保证所有引用的有效性，无需运行时开销。

    1.为什么需要生命周期？¶

    Rust 的所有权机制确保了 “每个值有唯一所有者，所有者离开作用域后值被销毁”。但当使用引用（&T 或 &mut T）时，引用本身不拥有数据，只是临时访问数据。如果引用指向的数据被销毁，引用就会变成悬垂引用（Dangling Reference），访问悬垂引用会导致内存安全问题（如读取已释放的内存）。

    生命周期的作用就是：通过标注和推断，让编译器知道引用之间的存活关系，确保引用始终指向有效的数据。

    示例：悬垂引用问题:

    // 错误示例：返回悬垂引用（编译报错）
fn dangle() -> &str {
    let s = String::from("hello"); // s 在函数内创建，作用域为函数体
    &s // 尝试返回 s 的引用，但 s 会在函数结束时被销毁
} // s 离开作用域，内存释放 → 返回的引用成为悬垂引用

fn main() {
    let r = dangle(); // r 是悬垂引用，访问 r 会导致未定义行为
    // println!("{}", r); // 编译报错：dangling reference
}

    编译器会拒绝这段代码，因为它能检测到：返回的引用 &s 的 “存活时间”（超出函数作用域）比它指向的数据 s 的存活时间（仅在函数作用域内）更长。

    2.生命周期的基本概念¶

    • 生命周期：引用保持有效的时间范围（从创建到失效的时间段）。在大多数情况下，生命周期是隐式和推断的，但在某些情况下需要显式注解。
    • 生命周期标注：用 'a、'b 等语法（单引号 + 标识符）标记引用的生命周期，用于告诉编译器引用之间的存活关系（标注本身不改变引用的实际生命周期，仅辅助编译器检查）。

    生命周期标注的语法:

    • 标注格式：'标识符（如 'a、'life，通常用小写字母，'a 是最常用的）。
    • 引用的生命周期标注：&'a T（不可变引用）或 &'a mut T（可变引用），表示该引用的生命周期为 'a。

    3.函数中的生命周期标注¶

    当函数参数和返回值都是引用时，编译器可能无法自动推断返回值引用的来源（是来自哪个参数，还是内部创建的），此时需要显式标注生命周期，告诉编译器 “返回值引用的生命周期与哪个参数的生命周期一致”。

    示例：返回两个字符串中较长的一个:

    // 生命周期标注：返回的引用的生命周期与 x 和 y 中较短的那个一致
fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

// 其他：多个生命周期
fn longest_with_an_announcement<'a, 'b>(
    x: &'a str,
    y: &'a str,
    ann: &'b str,  // 注解有不同的生命周期
) -> &'a str
where
    'b: 'a,  // 'b 生命周期至少和 'a 一样长
{
    println!("公告: {}", ann);
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

fn main() {
    let s1 = String::from("long string is long"); // s1 的作用域：main 函数
    let s2 = "short"; // s2 是字符串字面量，生命周期为 'static（整个程序运行期）

    let result = longest(s1.as_str(), s2); // result 的生命周期与 s1 一致（因 s1 比 s2 短）
    println!("最长的字符串：{}", result); // 正确：result 指向 s1 或 s2，且在其生命周期内
}

    解析：

    • 标注 'a 表示：x、y 和返回值的引用都必须存活至少 'a 时间（'a 是 x 和 y 生命周期的交集，即 “较短的那个”）。
    • 编译器通过标注知道：返回的引用要么是 x 要么是 y，且其生命周期不会超过 x 和 y 中存活时间较短的那个，因此不会出现悬垂引用。

    为什么需要标注？

    如果去掉标注，编译器无法确定返回值引用的来源，会直接报错：

    // 错误：缺少生命周期标注，编译器无法推断返回值引用的生命周期
fn longest(x: &str, y: &str) -> &str {
    if x.len() > y.len() { x } else { y }
}
// 编译报错：missing lifetime specifier

    编译器的错误提示本质是：“我不知道返回的引用是来自 x 还是 y，无法确保它的有效性，请告诉我它们的生命周期关系”。

    4.结构体中的生命周期标注¶

    当结构体包含引用类型的字段时，必须为结构体标注生命周期，以确保结构体实例的生命周期不超过其包含的引用的生命周期。

    示例：包含引用的结构体:

    // 结构体 ImportantExcerpt 包含一个字符串引用，标注其生命周期为 'a
struct ImportantExcerpt<'a> {
    part: &'a str, // part 的生命周期为 'a
}

fn main() {
    let novel = String::from("Call me Ishmael. Some years ago..."); // novel 的生命周期：main 函数
    let first_sentence = novel.split('.').next().expect("Could not find a '.'"); // first_sentence 是引用，生命周期与 novel 一致

    // 创建结构体实例：其生命周期不能超过 first_sentence（即 novel）的生命周期
    let i = ImportantExcerpt {
        part: first_sentence,
    };

    println!("摘要：{}", i.part); // 正确：i 的生命周期在 novel 之内
}

    解析：

    • 结构体 ImportantExcerpt<'a> 表示：该结构体的任何实例的生命周期都不能超过其字段 part 的生命周期 'a。
    • 在 main 中，i 的生命周期被约束为与 first_sentence（即 novel）一致，因此 i 不会存活到 novel 被销毁之后，避免了悬垂引用。

    5.生命周期省略规则¶

    手动标注生命周期可能繁琐，因此 Rust 编译器内置了生命周期省略规则（Lifetime Elision Rules），在某些常见场景下会自动推断生命周期，无需手动标注。

    这些规则仅适用于函数参数（输入生命周期） 和返回值（输出生命周期），结构体、枚举等仍需显式标注。

    规则 1：每个引用参数都获得一个独立的生命周期标注¶

    例如，函数 fn foo(x: &str, y: &str) 会被自动推断为 fn foo<'a, 'b>(x: &'a str, y: &'b str)（'a 和 'b 是不同的生命周期）。

    规则 2：若只有一个输入生命周期，则该生命周期被赋予所有输出生命周期¶

    例如，函数 fn bar(x: &str) -> &str 会被自动推断为 fn bar<'a>(x: &'a str) -> &'a str（返回值引用的生命周期与 x 一致）。

    规则 3：若方法有多个输入生命周期，且其中包含 &self 或 &mut self（self 的引用），则 self 的生命周期被赋予所有输出生命周期¶

    这是针对方法的规则，例如：

    impl<'a> ImportantExcerpt<'a> {
    // 方法参数包含 &self（生命周期为 'a），返回值自动推断为 'a
    fn level(&self) -> i32 {
        3
    }

    // 自动推断为：fn announce(&'a self, announcement: &'b str) -> &'a str
    fn announce(&self, announcement: &str) -> &str {
        println!("Attention please: {}", announcement);
        self.part
    }
}

    6.静态生命周期 'static¶

    'static 是一个特殊的生命周期，表示引用可以存活到整个程序运行期间。最常见的 'static 引用是字符串字面量，因为它们被直接嵌入到程序的二进制文件中，在程序启动时加载，直到程序结束才释放。

    示例：'static 生命周期

    // 字符串字面量的生命周期是 'static
let s: &'static str = "I have a static lifetime";

// 函数返回 'static 引用（需确保数据确实能存活整个程序期）
fn static_str() -> &'static str {
    "this is static" // 字符串字面量，生命周期 'static
}

    注意：'static 不意味着 “引用会存活很久”，而是 “引用指向的数据会存活到程序结束”。不要随意将引用标注为 'static，除非确实需要（如字符串字面量），否则可能导致编译错误（编译器会检测到数据无法存活那么久）。

    7.生命周期与作用域的关系¶

    • 作用域：变量在代码中的空间范围（由 {} 界定，如函数体、代码块），是 “可见性” 的范围。
    • 生命周期：引用在程序执行中的时间范围（从创建到失效），是 “有效性” 的范围。

    通常，引用的生命周期被其指向的数据的作用域所约束（引用不能超过数据的作用域）。例如：

    let s = String::from("hello"); // s 的作用域：当前代码块
let r = &s; // r 的生命周期与 s 的作用域一致（不能超过 s 被销毁的时间）

    8.总结¶

    生命周期是 Rust 用于保证引用有效性的编译时机制，核心要点：

    1. 确保内存安全：生命周期跟踪引用的有效时间，防止悬垂引用；
    2. 生命周期标注（'a）用于告诉编译器引用之间的存活关系，不改变实际生命周期；
    3. 函数和结构体中包含引用时，可能需要显式标注生命周期（遵循省略规则可简化标注）；
    4. 'static 表示引用指向的数据存活于整个程序运行期（如字符串字面量）。

    七、返回值与错误处理¶

    在 Rust 中，错误处理的核心原则是显式区分可恢复错误和不可恢复错误，并通过编译时检查确保错误不会被意外忽略。这种设计避免了隐藏的错误（如未捕获的异常），让程序行为更可预测。

    1.错误的两种类型¶

    Rust 将错误分为两类，对应不同的处理策略：

    类型 特点 处理方式 典型场景
    可恢复错误 错误可被处理，程序可继续运行 使用 Result<T, E> 类型 文件未找到、网络连接失败、无效输入
    不可恢复错误 错误严重到程序无法继续运行（逻辑错误） 使用 panic! 宏终止程序 数组越界、空指针解引用、断言失败

    2.不可恢复错误：panic! 宏¶

    当程序遇到无法修复的逻辑错误（如违反 invariants）时，使用 panic! 宏触发不可恢复错误。它会：

    1. 打印错误信息（包括栈跟踪，便于调试）；
    2. 展开（unwind）调用栈（默认行为），释放所有栈上资源；
    3. 终止程序。

    2.1. 手动触发 panic!¶

    fn main() {
    // 手动触发 panic，打印错误信息并终止程序
    panic!("发生严重错误，程序无法继续运行");
    // 以下代码不会执行
    println!("这里不会被打印");
}

    运行后输出（包含错误信息和栈跟踪）：

    thread 'main' panicked at '发生严重错误，程序无法继续运行', src/main.rs:2:5
note: run with `RUST_BACKTRACE=1` environment variable to display a backtrace

    2.2. 自动触发的 panic!¶

    Rust 标准库中的许多操作会在错误时自动触发 panic!，例如：

    • 数组越界访问（vec[10] 当 vec 长度小于 10 时）；
    • assert! 断言失败（用于验证程序 invariants）；
    • unwrap() 或 expect() 在 Result 为 Err 时（后续讲解）。
    fn main() {
    let v = vec![1, 2, 3];
    v[10]; // 数组越界，自动触发 panic
}

    2.3. panic! 的终止策略¶

    panic! 有两种终止程序的策略（可在 Cargo.toml 中配置）：

    • 展开（unwind，默认）：遍历调用栈，依次调用每个变量的 drop 方法释放资源，再终止。优点是资源安全，缺点是增加二进制体积。
    • 终止（abort）：直接终止程序，不清理资源（由操作系统回收）。优点是二进制体积小，适合对大小敏感的场景。

    配置方式（在 Cargo.toml 中）：

    [profile.release]
panic = "abort"  # 发布模式下使用 abort 策略

    适用场景:

    panic! 仅用于真正意外的错误，即 “理论上不应该发生，发生了说明程序有 bug” 的情况（如逻辑错误、违反前置条件）。对于 “预期可能发生的错误”（如用户输入错误），应使用可恢复错误处理。

    3.可恢复错误：Result<T, E> 类型¶

    对于预期可能发生的错误（如 “文件不存在”“网络超时”），Rust 使用 Result<T, E> 枚举类型表示操作结果，让调用者显式处理成功或失败的情况。

    3.1. Result 枚举定义¶

    Result 由标准库定义，本质是一个枚举：

    enum Result<T, E> {
    Ok(T),   // 操作成功，包含结果值 T
    Err(E),  // 操作失败，包含错误信息 E
}
    • T：成功时返回的值的类型；
    • E：失败时返回的错误类型。

    3.2. 处理 Result：match 表达式¶

    最基本的处理方式是用 match 匹配 Result 的两个变体，分别处理成功和失败：

    use std::fs::File;

fn main() {
    // 尝试打开文件：File::open 返回 Result<File, std::io::Error>
    let file_result = File::open("example.txt");

    // 用 match 处理结果
    let file = match file_result {
        Ok(f) => f, // 成功：获取文件句柄
        Err(e) => {
            // 失败：打印错误并提前返回
            eprintln!("打开文件失败：{}", e);
            return; // 终止函数，避免继续执行
        }
    };

    // 成功打开文件后继续操作
    println!("文件打开成功：{:?}", file);
}

    3.3. 简化错误处理：? 运算符¶

    如果希望在错误发生时将错误 “传播” 给调用者（而非在当前函数处理），可以使用 ? 运算符。它的作用是：

    • 若 Result 是 Ok(t)，则返回 t（继续执行）；
    • 若 Result 是 Err(e)，则立即返回 Err(e)（将错误传递给上层）。

    示例：用 ? 传播错误

    use std::fs::File;
use std::io;
use std::io::Read;

// 函数返回 Result<String, io::Error>，表示可能返回字符串或 IO 错误
fn read_file_content() -> Result<String, io::Error> {
    // 打开文件：用 ? 传播错误（若失败，直接返回 Err）
    let mut file = File::open("example.txt")?;

    let mut content = String::new();
    // 读取文件内容：用 ? 传播错误
    file.read_to_string(&mut content)?;

    // 成功：返回 Ok(内容)
    Ok(content)
}

fn main() {
    // 调用函数，用 match 处理最终结果
    match read_file_content() {
        Ok(content) => println!("文件内容：{}", content),
        Err(e) => eprintln!("读取失败：{}", e),
    }
}

    注意：? 运算符只能用于返回 Result 或 Option 的函数（因为它需要将错误 “返回” 给调用者）。若在 main 中使用 ?，需让 main 返回 Result（main 支持返回 Result<(), E>）：

    // main 函数返回 Result<(), io::Error>，允许使用 ? 传播错误
fn main() -> Result<(), io::Error> {
    let content = read_file_content()?; // 直接用 ? 传播，失败时 main 会返回 Err
    println!("文件内容：{}", content);
    Ok(()) // 成功时返回 Ok(())
}

    3.4. 快速处理¶

    3.4.1.unwrap() 与 expect()¶

    对于原型开发或测试代码，可使用 unwrap() 或 expect() 简化错误处理（本质是 “如果错误就 panic”）：

    • unwrap()：若 Result 是 Ok(t) 则返回 t；若为 Err(e) 则触发 panic!（错误信息为 e 的默认描述）。
    • expect(msg)：类似 unwrap()，但可自定义 panic 时的错误信息（更易调试）。

    示例：unwrap 与 expect

    use std::fs::File;

fn main() {
    // unwrap：错误时 panic，使用默认信息
    let file1 = File::open("example.txt").unwrap();

    // expect：错误时 panic，使用自定义信息
    let file2 = File::open("example.txt")
        .expect("打开 example.txt 失败，请检查文件是否存在");
}

    注意：这两个方法会在错误时触发 panic，因此不适合生产代码（应显式处理错误），仅用于 “理论上不会失败” 的场景（如测试资源文件）。

    3.4.2.unwrap_or() 和 unwrap_or_else()¶

    unwrap_or 的作用是：如果 Result 是 Ok(t)，则返回 t；如果是 Err(e)，则返回一个预设的默认值。

    • 签名：fn unwrap_or(self, default: T) -> T
    • 特点：默认值 default 无论 Result 是 Ok 还是 Err 都会被提前计算（即使不需要，也会执行计算逻辑）。

    unwrap_or_else 与 unwrap_or 功能类似，但默认值通过闭包计算，且仅在 Result 为 Err 时才执行闭包。

    • 签名：fn unwrap_or_else<F: FnOnce(E) -> T>(self, f: F) -> T
    • 特点：闭包的参数是 Err 中的错误值 e，可根据错误动态生成默认值；且闭包仅在 Err 时执行（无不必要的计算）。
    fn main() {
    let config_file = std::fs::read_to_string("config.toml")
        .unwrap_or(String::from("default_config")); // 提供默认值

    let config_file = std::fs::read_to_string("config.toml")
        .unwrap_or_else(|error| { // 使用闭包计算默认值
            println!("使用默认配置: {}", error);
            String::from("default_config")
        });
}
    3.4.3.map() 和 and_then()¶

    map 用于对 Ok 中的值进行转换，返回一个新的 Result。如果原结果是 Ok(t)，则应用闭包 f(t) 得到 Ok(f(t))；如果是 Err(e)，则直接返回 Err(e)（错误类型不变）。

    • 签名：fn map<U, F: FnOnce(T) -> U>(self, f: F) -> Result<U, E>
    • 场景：对成功的结果进行简单转换（如类型转换、数据处理），不处理错误。

    and_then 也用于转换 Ok 中的值，但闭包的返回值必须是另一个 Result（而非普通值）。如果原结果是 Ok(t)，则返回闭包的结果 f(t)；如果是 Err(e)，则直接返回 Err(e)。

    • 签名：fn and_then<U, F: FnOnce(T) -> Result<U, E>>(self, f: F) -> Result<U, E>
    • 特点：避免嵌套 Result（如 Result<Result<U, E>, E>），适合 “多步操作，每步都可能出错” 的场景（如先读取文件，再解析内容）。
    fn parse_number(s: &str) -> Result<i32, std::num::ParseIntError> {
    s.parse::<i32>()
}

fn main() {
    let result = "123".parse::<i32>();

    // map: 对成功的值进行转换
    let doubled = result.map(|n| n * 2);

    // and_then: 链式操作，可能返回新的 Result
    let parsed_twice = parse_number("123").and_then(|n| parse_number(&(n * 2).to_string()));
}

    3.5.方法对比与使用场景总结¶

    方法 作用 输入闭包返回值 适用场景
    unwrap_or Err 时返回默认值 直接传值 T 默认值计算简单、无副作用时
    unwrap_or_else Err 时通过闭包计算默认值 闭包返回 T（参数是 E） 默认值计算复杂、有副作用，或需根据错误动态生成
    map 转换 Ok 中的值，返回新 Result 闭包返回 U 对成功结果进行简单转换（无新错误产生）
    and_then 链式处理 Result，返回新 Result 闭包返回 Result<U, E> 多步操作，每步可能出错（避免嵌套 Result）

    典型组合示例:

    结合 map 和 unwrap_or_else 处理 “读取文件并解析为整数” 的逻辑：

    use std::fs;
use std::path::Path;

// 读取文件内容（可能返回 IO 错误）
fn read_file(path: &Path) -> Result<String, std::io::Error> {
    fs::read_to_string(path)
}

// 解析字符串为整数（可能返回解析错误）
fn parse_int(s: &str) -> Result<i32, String> {
    s.parse().map_err(|e| format!("解析错误：{}", e))
}

fn main() {
    let path = Path::new("number.txt");

    // 步骤1：读取文件 → 步骤2：解析为整数 → 步骤3：Err 时返回默认值 -1
    let num = read_file(path)
        .map_err(|e| format!("读取错误：{}", e)) // 将 IO 错误转为 String 错误
        .and_then(|content| parse_int(&content)) // 链式解析（返回 Result<i32, String>）
        .unwrap_or_else(|e| { // Err 时打印错误并返回默认值
            eprintln!("处理失败：{}", e);
            -1
        });

    println!("最终结果：{}", num);
}

    4.自定义错误类型¶

    实际开发中，函数可能遇到多种错误（如 “IO 错误”“解析错误”），此时需要自定义错误类型统一管理。通常用枚举定义自定义错误，包含所有可能的错误变体，并实现 std::error::Error trait（便于兼容标准库错误处理）。

    示例：自定义错误类型:

    use std::error::Error;
use std::fmt;
use std::fs::File;
use std::io::Read;
use std::num::ParseIntError;

// 自定义错误枚举：包含可能的错误类型
#[derive(Debug)]
enum MyError {
    IoError(std::io::Error),       // 包装 IO 错误
    ParseError(ParseIntError),     // 包装解析错误
    CustomError(String),           // 自定义错误信息
}

// 为 MyError 实现 Display trait（错误信息格式化）
impl fmt::Display for MyError {
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        match self {
            MyError::IoError(e) => write!(f, "IO 错误：{}", e),
            MyError::ParseError(e) => write!(f, "解析错误：{}", e),
            MyError::CustomError(msg) => write!(f, "自定义错误：{}", msg),
        }
    }
}

// 为 MyError 实现 Error trait（标准错误接口，需先实现 Display 和 Debug）
impl Error for MyError {}

// 从其他错误类型转换为 MyError（使用 From trait 实现自动转换）
impl From<std::io::Error> for MyError {
    fn from(e: std::io::Error) -> Self {
        MyError::IoError(e)
    }
}

impl From<ParseIntError> for MyError {
    fn from(e: ParseIntError) -> Self {
        MyError::ParseError(e)
    }
}

// 可能返回多种错误的函数，统一返回 MyError
fn process_data() -> Result<(), MyError> {
    // 读取文件：IO 错误会通过 From 自动转换为 MyError::IoError
    let mut file = File::open("data.txt")?;

    let mut content = String::new();
    file.read_to_string(&mut content)?;

    // 解析数字：ParseIntError 会自动转换为 MyError::ParseError
    let num: i32 = content.trim().parse()?;

    // 自定义错误检查
    if num < 0 {
        return Err(MyError::CustomError("数字不能为负数".to_string()));
    }

    Ok(())
}

fn main() -> Result<(), Box<dyn Error>> {
    process_data()?; // 用 ? 传播 MyError（因 MyError 实现了 Error）
    println!("处理成功");
    Ok(())
}

    核心要点:

    • 自定义错误通常用枚举，包含所有可能的错误来源（可包装其他错误类型）；
    • 实现 Display（错误信息）和 Error trait（兼容标准库）；
    • 实现 From<OtherError> 可让 ? 自动将其他错误转换为自定义错误（简化代码）。

    5.错误处理的最佳实践¶

    1. 区分可恢复与不可恢复错误：
    2. 预期可能发生的错误（如文件不存在）用 Result；
    3. 逻辑错误（如数组越界）用 panic!。
    4. 不要忽略错误：Rust 编译器会警告未使用的 Result（如 File::open("a.txt"); 未处理），必须显式处理（match、?、unwrap 等）。
    5. 自定义错误类型统一管理：当函数可能返回多种错误时，用自定义枚举包装，避免返回 Box<dyn Error>（虽然灵活，但丢失类型信息）。
    6. 用 expect 替代 unwrap：expect 提供自定义错误信息，便于调试（明确说明 “为什么这里不该出错”）。
    7. 传播错误而非直接 panic：除非当前函数是 “顶层”（如 main），否则优先用 ? 将错误传播给调用者，让上层决定如何处理（更灵活）。

    八、注释与文档¶

    Rust 提供了强大的注释和文档系统，不仅支持常规代码注释，还能生成漂亮的 HTML 文档。

    Rust 有三种主要注释类型：

    • 普通注释：// 和 /* ... */
    • 文档注释：/// 和 //!
    • 属性注释：#[doc]

    其中文档注释还能通过工具 rustdoc 生成结构化的 HTML 文档，支持 Markdown 语法，甚至可以包含可执行的测试代码。

    1.普通注释：代码内的说明性文字¶

    普通注释用于在代码中添加解释、临时注释掉代码片段等，编译器会完全忽略它们，不影响代码运行。Rust 支持两种普通注释形式：

    1.1. 单行注释（//）¶

    用 // 开头，注释从 // 到行尾的内容，是最常用的注释方式。

    fn main() {
    // 这是单行注释：打印 Hello World
    println!("Hello, World!"); // 行尾的单行注释：输出字符串
}

    1.2. 多行注释（/* ... */）¶

    用 /* 开头、*/ 结尾，可跨多行注释内容，适合注释较长的逻辑或临时注释掉代码块。

    fn calculate(a: i32, b: i32) -> i32 {
    /*
    这是多行注释：
    该函数计算两个整数的和
    参数 a 和 b 为输入，返回它们的和
    */
    a + b
}

fn main() {
    /*
    临时注释掉这段代码：
    let x = 5;
    let y = 10;
    println!("{}", calculate(x, y));
    */
}

    注意：多行注释不能嵌套（/* ... /* 嵌套 */ ... */ 会报错），如需嵌套可改用多个单行注释。

    2.文档注释：生成 API 文档的注释¶

    文档注释（Doc Comments）是一种特殊的注释，用于描述代码的功能、用法、参数、返回值等，支持通过 rustdoc 工具生成 HTML 文档。Rust 提供两种文档注释，分别用于注释 “当前项” 和 “父项”。

    2.1. 外部文档注释（///）¶

    /// 用于注释紧随其后的项（如函数、结构体、枚举、模块等），生成的文档会关联到该项目。

    /// 计算两个整数的和
///
/// # 示例
/// ```rust
/// let result = add(2, 3);
/// assert_eq!(result, 5);
/// ```
fn add(a: i32, b: i32) -> i32 {
    a + b
}

    函数/结构体文档:

    /// WebSocket 连接的唯一标识符
///
/// 使用 UUID 作为底层实现，确保全局唯一性。
///
/// # 示例
///
/// ```rust
/// use crate::ws::connection::WsConnectionId;
///
/// let id = WsConnectionId::new();
/// println!("Generated ID: {}", id.0);
/// ```
#[derive(Debug, Clone, Copy, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct WsConnectionId(pub Uuid);

impl WsConnectionId {
    /// 创建新的连接 ID
    ///
    /// # 返回值
    ///
    /// 返回一个新生成的唯一连接 ID
    pub fn new() -> Self {
        Self(Uuid::new_v4())
    }
}

    2.2. 内部文档注释（//!）¶

    //! 用于注释包含它的父项（如模块、crate、结构体内部等），生成的文档会关联到父项（而非紧随其后的项）。

    //! 数学工具模块
//!
//! 提供基础的数学运算功能，如加法、乘法等。
//! 包含 `add` 和 `multiply` 等函数。

/// 计算两个整数的乘积
fn multiply(a: i32, b: i32) -> i32 {
    a * b
}

    模块级文档示例:

    //! # WebSocket 连接管理模块
//!
//! 该模块提供了 WebSocket 连接的生命周期管理功能，
//! 包括连接的建立、维护和断开处理。
//!
//! ## 主要功能
//!
//! - 连接池管理
//! - 消息路由
//! - 连接状态监控
//!
//! ## 示例
//!
//! ```rust
//! use crate::ws::connection::WsConnectionId;
//!
//! let conn_id = WsConnectionId::new();
//! println!("Connection ID: {:?}", conn_id);
//! ```

pub mod connection;
pub mod manager;

    3.文档注释的 Markdown 支持¶

    rustdoc 原生支持 Markdown 语法，可在文档注释中使用标题、列表、代码块、表格等格式，生成的 HTML 文档会自动渲染这些样式。

    3.1. 常用 Markdown 元素示例¶

    （1）标题与章节（#）¶

    用 # 定义文档章节（如功能说明、参数、示例），提升文档结构清晰度：

    /// 解析字符串为整数
///
/// # 功能说明
/// 将输入的字符串解析为 `i32` 类型，若解析失败返回错误。
///
/// # 参数
/// - `s`：待解析的字符串（如 `"123"`）
///
/// # 返回值
/// - 成功：`Ok(i32)`（解析后的整数）
/// - 失败：`Err(&str)`（错误信息）
///
/// # 示例
/// ```rust
/// assert_eq!(parse_int("42"), Ok(42));
/// assert!(parse_int("abc").is_err());
/// ```
fn parse_int(s: &str) -> Result<i32, &str> {
    s.parse().map_err(|_| "解析失败：不是有效的整数")
}
    （2）代码块（```）¶

    用 `` 包裹代码块，指定语言（如rust`）可启用语法高亮，且代码块会被视为文档测试（见下文）：

    /// 打印向量内容
///
/// 示例代码：
/// ```rust
/// let v = vec![1, 2, 3];
/// print_vec(&v); // 输出：[1, 2, 3]
/// ```
fn print_vec(v: &[i32]) {
    println!("{:?}", v);
}
    （3）表格（| ... |）¶

    用表格展示参数、选项等结构化信息：

    /// 计算图形面积
///
/// # 支持的图形
/// | 图形   | 输入参数          | 面积公式       |
/// |--------|-------------------|----------------|
/// | 正方形 | 边长（`f64`）     | 边长 × 边长    |
/// | 矩形   | 长、宽（`f64`）   | 长 × 宽        |
fn area(shape: &str, params: &[f64]) -> f64 {
    match shape {
        "square" => params[0] * params[0],
        "rectangle" => params[0] * params[1],
        _ => 0.0,
    }
}

    4.文档测试：确保示例可运行¶

    Rust 的文档注释中，标记为 rust 的代码块会被视为文档测试（Doc Tests）。运行 cargo test 时，这些代码会被编译并执行，确保文档中的示例代码是可运行且正确的。

    示例：文档测试的用法:

    /// 检查一个数是否为偶数
///
/// # 测试示例
/// ```rust
/// use my_module::is_even; // 注意：文档测试需显式导入模块
/// assert!(is_even(2));
/// assert!(!is_even(3));
/// ```
pub fn is_even(n: i32) -> bool {
    n % 2 == 0
}

    运行 cargo test 时，文档中的代码块会被自动执行：

    cargo test

    如果示例代码错误（如 assert!(is_even(3))），测试会失败，提示文档与实际功能不符。

    文档测试的注意事项:

    • 文档测试默认在一个独立的模块中运行，需显式用 use 导入被测试的项（如 use my_module::is_even）；
    • 可在代码块前加 ignore 标记跳过测试（如 ```rust,ignore）；
    • 可加 should_panic 标记预期会 panic 的测试（如 ```rust,should_panic）。

    5.文档生成与查看¶

    使用 cargo doc 命令可基于文档注释生成 HTML 文档，--open 选项会自动在浏览器中打开文档：

    # 生成文档（包含依赖项）
cargo doc

# 生成文档（仅当前项目）
cargo doc --no-deps

# 包含私有项的文档
cargo doc --document-private-items

# 生成并打开当前 crate 的文档, 在浏览器中打开文档
cargo doc --open

# 发布模式生成文档（优化后的文档）
cargo doc --release

    生成的文档会包含：

    • 所有带文档注释的项（函数、结构体、模块等）；
    • Markdown 渲染的格式（标题、表格、代码块等）；
    • 交叉引用（点击类型或函数名可跳转至对应文档）。

    6.文档相关属性¶

    Rust 提供了一些属性（Attributes）用于控制文档生成：

    6.1. #[doc(hidden)]：隐藏项不生成文档¶

    标记为 #[doc(hidden)] 的项不会出现在生成的文档中，适合内部使用但不希望暴露的 API：

    /// 公开的加法函数（会生成文档）
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

#[doc(hidden)] // 隐藏，不生成文档
pub fn internal_helper() {
    // 内部辅助函数，无需暴露给用户
}

    6.2. #[doc = "..."]：动态生成文档¶

    可通过 #[doc = "..."] 动态指定文档内容（支持字符串拼接），适合复用文档或生成动态说明：

    const DOC: &str = "这是一个加法函数";

#[doc = DOC] // 复用 DOC 常量作为文档
#[doc = "\n\n# 示例\n```rust\nassert_eq!(add(1, 2), 3);\n```"]
pub fn add(a: i32, b: i32) -> i32 {
    a + b
}

    7.最佳实践¶

    1. 为公共 API 添加文档：所有暴露给用户的函数、结构体、模块等都应添加文档注释，说明功能、参数、返回值和示例；
    2. 使用标准章节：统一使用 # 功能说明、# 参数、# 返回值、# 示例 等章节，让文档风格一致；
    3. 编写文档测试：确保示例代码可运行，避免文档与实际功能脱节；
    4. 简洁明了：文档应突出重点，避免冗余，优先说明 “如何用” 而非 “如何实现”；
    5. 关联相关项：使用 [类型名] 或 [函数名] 生成交叉引用（如 [Vec] 会链接到 Vec 的文档）。
