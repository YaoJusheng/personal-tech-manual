    一、Rust简介¶

    Rust 是由 Mozilla 研究院开发的开源系统级编程语言（2010 年首次发布）、2015 年正式发布的一门 系统级编程语言，设计目标是在 不牺牲性能 的前提下，解决 C/C++ “空指针、悬垂指针、数据竞争” 等内存安全问题，同时提供 现代语言特性（模式匹配、泛型、trait、异步等）。

    “像 C++ 一样快，像 Haskell 一样安全，像 Go 一样易并发。”

    其核心设计理念：

    • 内存安全：编译时防止空指针、数据竞争等内存错误
    • 零成本抽象：高级功能（如泛型）不牺牲运行时性能
    • 并发安全：内置机制防止数据竞争
    • 实用主义：融合函数式与面向对象编程范式

    语言速览:

    // 1. Hello World
fn main() {
    println!("Hello, Rust!");
}

// 2. 所有权示例：编译期保证无悬垂指针
fn take_ownership(s: String) {          // s 进入作用域
    println!("owned: {}", s);
}                                       // s 被 drop，内存释放

fn main() {
    let s = String::from("hello");
    take_ownership(s);                  // 所有权转移
    // println!("{}", s);               // 编译错误：value borrowed here after move
}

// 3. 零成本迭代器 + 并发
use rayon::prelude::*;
fn sum_squares(v: &[i32]) -> i32 {
    v.par_iter()        // 并行迭代器，编译成多线程
     .map(|x| x * x)
     .sum()
}

    二、Rust原理¶

    Rust 的核心原理建立在三大支柱之上：内存安全、零成本抽象和无畏并发。这些原理通过独特的语言机制实现，使其在系统编程领域独树一帜。

    1.所有权系统（Ownership System）—— 内存安全的基石¶

    1.1.核心规则：¶

    1. 每个值有唯一所有者
    2. 值离开作用域自动释放
    3. 所有权可通过转移（move）传递
    fn main() {
    let s1 = String::from("hello"); // s1 拥有字符串
    let s2 = s1;                    // 所有权转移到 s2
    // println!("{}", s1);           // 错误！s1 不再拥有数据
}

    1.2.内存管理机制：¶

    机制 传统语言 Rust
    堆分配 手动管理(易出错) 自动管理(编译时保证)
    释放时机 开发者控制 离开作用域自动 drop
    悬垂指针 常见问题 编译时完全杜绝

    2.借用与生命周期（Borrowing & Lifetimes）¶

    2.1.引用规则：¶

    • 不可变引用：允许多个同时存在 (&T)
    • 可变引用：同一作用域仅允许一个 (&mut T)
    • 引用不能超过值生命周期
    fn main() {
    let mut data = vec![1, 2, 3];

    let ref1 = &data;      // 不可变引用 ✓
    let ref2 = &data;      // 不可变引用 ✓
    // let mut_ref = &mut data; // 错误！不能同时存在可变和不可变引用

    println!("{} {}", ref1[0], ref2[1]);
} // 引用在此结束

    2.2.生命周期标注：¶

    fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() { x } else { y }
}
    • 'a 表示参数和返回值共享相同生命周期
    • 编译器验证引用有效性，防止悬垂引用

    3.零成本抽象原理¶

    3.1. 泛型单态化（Monomorphization）¶

    fn add<T: std::ops::Add<Output=T>>(a: T, b: T) -> T {
    a + b
}

// 编译后生成：
fn add_i32(a: i32, b: i32) -> i32 { a + b }
fn add_f64(a: f64, b: f64) -> f64 { a + b }
    • 编译时生成具体类型实现
    • 无运行时类型检查开销

    3.2. Trait 动态分发¶

    trait Draw { fn draw(&self); }

// 静态分发（编译时确定）
fn render<T: Draw>(item: T) { item.draw() }

// 动态分发（运行时确定）
fn render_dyn(item: &dyn Draw) { item.draw() }
    分发方式 性能 代码体积 使用场景
    静态分发 ★★★ 增大 性能关键路径
    动态分发 ★★☆ 减小 插件系统/异构集合

    4.无畏并发模型¶

    4.1.并发安全保证：¶

    use std::thread;

fn main() {
    let mut data = vec![1, 2, 3];
    // thread::spawn(|| {
    //     data.push(4); // 错误！可能引发数据竞争
    // });

    let handle = thread::spawn(move || { // 所有权转移
        let mut new_data = data;
        new_data.push(4);
        new_data
    });

    let result = handle.join().unwrap();
    println!("{:?}", result); // [1, 2, 3, 4]
}

    4.2.类型系统保障：¶

    1. Send Trait：允许跨线程转移所有权
    2. Sync Trait：允许多线程安全共享 (&T)
    3. Mutex/RwLock：提供线程间同步访问

    2.1.3.包和模块¶
    1.概念理解¶
    • 项目(Packages)：一个 Cargo 提供的 feature，可以用来构建、测试和分享包
    • 由 Cargo.toml 定义的代码单元，包含元数据（名称、版本、依赖）和至少一个 Crate。

    • 一个包最多包含一个库 Crate，但可包含多个二进制 Crate（每个 src/bin/*.rs 文件生成一个可执行文件）。

    • 包(`Crate)：一个由多个模块组成的树形结构，可以作为三方库进行分发，也可以生成可执行文件进行运行, 它是编译的基本单元，分为两类：

    • 库 Crate：根文件为 src/lib.rs，编译为 .rlib 文件供其他代码调用。
    • 二进制 Crate：根文件为 src/main.rs 或 src/bin/*.rs，编译为可执行文件。
    • 模块(`Module)：可以一个文件多个模块，也可以一个文件一个模块，模块可以被认为是真实项目中的代码组织单元
    • 作用：组织代码的命名空间，控制可见性（默认私有，pub 公开）。
    • 声明方式：
      • 内联模块：mod utils { ... }（适合小型工具）
      • 文件模块：mod network;（需同名文件 network.rs 或目录 network/mod.rs）。
    • 路径引用：
      • 绝对路径：crate::module::item
      • 相对路径：super::parent_item（父模块）或 self::child_item（当前模块）。
    2.描述汇总¶
    概念 说明 对应文件/目录
    Package Cargo 管理的项目 ( feature)，可含多个 crate 顶层目录
    Crate 编译单元（lib 或 bin），一个 crate = 一个根文件 src/lib.rs 或 src/main.rs
    Module 在 crate 内部做逻辑分层（`namespace + 可见性） | mod xxx; 声明
    3.优点¶
    • 库 lib.rs 放业务核心，可对外发布；
    • main.rs 只负责 CLI/HTTP 启动，极薄；
    • 额外工具放在 src/bin/*.rs，保持单一职责 。
    4.最佳实践速查表¶
    场景 做法
    可见性 默认 private；API 记得加 pub。
    路径引用 内部用 crate::xxx；外部包用 use serde::*。
    模块拆分 每模块一个目录（含 mod.rs）或同名文件，用 mod gateway; 自动发现。
    重导出 pub use xxx::yyy 把深层路径提到根，使用者更短。
    工作空间 多 crate 大项目用 workspace 统一版本与依赖。

    这也是 Cargo 推荐的目录结构，解释如下：

    • Cargo.toml 和 Cargo.lock 保存在 package 根目录下
    • 源代码放在 src 目录下
    • 默认的 lib 包根是 src/lib.rs, 库 lib.rs 放业务核心，可对外发布；

    • 集成测试代码放在 tests 目录下

    • 按功能拆分模块（如 gateway），每个模块保持 200–800 行代码。
    • 使用 pub(crate) 限制可见性到当前 Crate。

    4.Workspace（工作区）模式¶
    4.1.工作区组成¶
    • 根目录：包含工作区配置的顶级目录
    • Cargo.toml：工作区配置文件（无 [package] 部分）
    • 成员包：工作区中包含的一个或多个 Rust 包
    • 共享资源：
    • 单个 Cargo.lock 文件：统一依赖版本
    • 单个 target 目录：共享构建输出
    • 共享依赖缓存：避免重复下载和编译
    4.2.工作区 vs 单包¶
    特性 单包项目 Workspace 项目
    代码组织 单一代码库 多个独立包
    依赖管理 独立依赖 统一依赖版本
    构建时间 独立构建 共享依赖构建结果
    适用场景 小型应用/库 大型系统/微服务架构
    发布 整体发布 可独立发布子包

    Rust Workspace 是管理大型项目的利器，通过：

    1. 统一依赖管理 - 单个 Cargo.lock 确保版本一致性
    2. 共享构建结果 - 减少重复编译时间
    3. 灵活的项目结构 - 支持多种项目布局
    4. 高效的工作流 - 统一命令管理所有包

    8. Option & Result（空值与错误处理）¶

    8.1.Option<T>¶

    Option<T> 用于表示一个值可能存在，也可能不存在的情况。它有两个变体：

    • Some(T)：包含一个值。
    • None：没有值。
    1.语法定义¶
    enum Option<T> {
    Some(T),
    None,
}
    2.常见用法¶
    1. 基本示例：
    let some_number: Option<i32> = Some(10);
let none_number: Option<i32> = None;

match some_number {
    Some(n) => println!("Got a number: {}", n),
    None => println!("No number"),
}

match none_number {
    Some(n) => println!("Got a number: {}", n),
    None => println!("No number"),
}
    1. 与 unwrap 一起使用： unwrap 可以用于在 Option 为 None 时直接 panic。
    let some_number: Option<i32> = Some(10);
let value = some_number.unwrap(); // value = 10

let none_number: Option<i32> = None;
let value = none_number.unwrap(); // 运行时会 panic，报错："called `Option::unwrap()` on a `None` value"
    1. 使用 map 转换值： 你可以使用 map 方法对 Some 中的值进行操作，None 会直接返回 None。
    let some_number: Option<i32> = Some(5);
let result = some_number.map(|x| x * 2); // Some(10)

let none_number: Option<i32> = None;
let result = none_number.map(|x| x * 2); // None
    1. 与 is_some / is_none 配合判断： 判断一个 Option 是否为 Some 或 None。
    let some_number: Option<i32> = Some(10);
if some_number.is_some() {
    println!("There is a value");
} else {
    println!("There is no value");
}

let none_number: Option<i32> = None;
if none_number.is_none() {
    println!("No value here");
}
    3.应用场景¶
    • 用于函数返回值表示可能为空的情况（例如查找操作、计算过程等）。
    • 作为安全的替代空指针（null）的方式，避免了空指针解引用错误。

    8.2.Result<T, E>¶

    Result 用于表示函数可能返回成功结果，也可能返回错误。它有两个变体：

    • Ok(T)：操作成功，包含一个值。
    • Err(E)：操作失败，包含错误信息。
    1.语法定义¶
    enum Result<T, E> {
    Ok(T),
    Err(E),
}
    2.常见用法¶
    1. 基本示例：
    fn divide(a: f64, b: f64) -> Result<f64, String> {
    if b == 0.0 {
        Err(String::from("Cannot divide by zero"))
    } else {
        Ok(a / b)
    }
}

let result = divide(10.0, 2.0);
match result {
    Ok(val) => println!("Result is: {}", val),
    Err(e) => println!("Error: {}", e),
}

let result = divide(10.0, 0.0);
match result {
    Ok(val) => println!("Result is: {}", val),
    Err(e) => println!("Error: {}", e), // Error: Cannot divide by zero
}
    1. 使用 unwrap / expect： unwrap 和 expect 方法可以简化错误处理，但它们会在遇到 Err 时 panic。expect 方法可以自定义错误信息。
    let result = divide(10.0, 2.0);
let value = result.unwrap(); // 返回结果：5.0

let result = divide(10.0, 0.0);
let value = result.unwrap(); // 会 panic: "called `Result::unwrap()` on an `Err` value: Cannot divide by zero"

    使用 expect 方法：

    let result = divide(10.0, 0.0);
let value = result.expect("Division failed"); // 会 panic: "Division failed"
    1. 错误链 (map_err)： 你可以使用 map_err 来转换 Err 中的错误类型，而不改变 Ok 中的值。
    let result: Result<i32, String> = Err("Something went wrong".to_string());
let new_result = result.map_err(|e| format!("Error occurred: {e}"));
    1. 组合多个 Result 结果 (? 运算符)： 使用 ? 运算符可以自动处理 Result 类型，若遇到 Err，会立即返回该错误。
    fn read_file(file_path: &str) -> Result<String, String> {
    let content = std::fs::read_to_string(file_path)?;
    Ok(content)
}

    ? 运算符可以帮助避免重复的错误检查代码，并使函数短小简洁。

    3.应用场景¶
    • 用于所有可能失败的操作（如文件读取、网络请求、解析数据等），通过返回 Err 来明确标示错误类型。

    • 错误类型（E）通常是 String、Error trait 或自定义类型，具体取决于应用需求。

    • 不可恢复错误：panic! 宏（类似 Python 的 raise）

    panic!("Critical error!");
    • 可恢复错误: Result<T, E> 类型 + match/?操作符
    use std::fs::File;
fn read_file() -> Result<String, std::io::Error> {
    let f = File::open("file.txt")?;
    // 读取文件...
    Ok("content".to_string())
}

    8.3.Option 和 Result 的对比¶

    特性 Option<T> Result<T, E>
    用途 表示一个值可能存在，也可能不存在 表示操作的成功或失败，带有错误信息
    变体 Some(T), None Ok(T), Err(E)
    返回值含义 要么有值，要么没有值 要么成功返回值，要么失败并返回错误
    错误信息 无（仅有 None 表示没有值） 包含详细的错误信息（Err(E)）
    适用场景 可用于表示函数返回的值可能为空 用于表示可能发生错误的操作

    11.1. 泛型函数¶

    泛型函数可以接受不同类型的参数，并返回不同类型的结果。

    for item in list {
        if item > largest {
            largest = item;
        }
    }

    largest
}

fn main() {
    print_item(42);         // 打印一个整数
    print_item("Hello");    // 打印一个字符串
}
    • T 是泛型类型参数，可以是任意类型。
    • Rust 编译器会根据函数调用时传入的类型生成相应的具体类型。

    11.2. 泛型结构体¶

    泛型结构体允许你定义一个具有多个不同类型参数的结构体。

    struct Point<T> {
    x: T,
    y: T,
}

let integer = Point { x: 5, y: 10 };
let float = Point { x: 1.0, y: 4.0 };

    11.3. 泛型约束¶

    有时，我们需要限制泛型类型的行为，例如要求泛型类型实现某些特定的 trait。这就需要用到泛型约束。

    use std::fmt::Debug;

fn print_debug<T: Debug>(item: T) {
    println!("{:?}", item);  // 限制 T 必须实现 Debug trait
}

fn main() {
    print_debug(42);          // 整数实现了 Debug trait
    print_debug("Hello");     // 字符串实现了 Debug trait
}
    • T: Debug 表示类型 T 必须实现 Debug trait。

    11.4. 默认泛型类型¶

    如果你希望为泛型提供默认类型，可以使用 default 约束。

    struct Wrapper<T = i32> {
    value: T,
}

fn main() {
    let default_wrapper = Wrapper { value: 10 }; // T 默认是 i32
    let string_wrapper = Wrapper::<String> { value: String::from("Hello") }; // 显式指定 T 为 String
}
    • T = i32 表示如果没有指定泛型类型，默认类型是 i32。

    11.5. 泛型与生命周期¶

    泛型可以与生命周期一起使用，确保引用的有效性和安全性。

    fn longest<'a, T>(x: &'a T, y: &'a T) -> &'a T {
    if x > y { x } else { y }
}
    • 'a 是生命周期参数，确保函数返回的引用在两个参数的生命周期范围内有效。

    12.所有权系统（Rust 核心特性: 避免 GC，保证安全）¶

    Rust 的所有权系统 (Ownership System) 确保了内存安全，无需垃圾回收。这是 Rust 最具特色的功能之一，它通过三个主要的原则来管理内存：

    1. 每个值都有一个所有者（Owner）。
    2. 一个值同时只能有一个所有者。
    3. 当所有者超出作用域时，值会被丢弃，内存自动释放。

    12.1. 所有权规则¶

    1. Rust 中的每个值都有一个所有者
    2. 一次只能有一个所有者
    3. 当所有者离开作用域，值将被丢弃

    12.2. 移动（Move）语义¶

    let s1 = String::from("hello");
let s2 = s1; // s1 的所有权转移到 s2
// println!("{}", s1); // 错误！s1 不再有效
    • 当值被移动时，原变量不再有效，Rust 会自动释放内存。
    • 拷贝类型（如整数、浮点数、布尔值）在赋值时是复制的，而非移动。

    12.3. 克隆（Clone）¶

    let s1 = String::from("hello");
let s2 = s1.clone(); // 显式复制数据
println!("s1 = {}, s2 = {}", s1, s2); // 两者都有效

    12.4. 引用与借用¶

    let s = String::from("hello");

// 不可变引用
let len = calculate_length(&s);

// 可变引用
let mut s = String::from("hello");
change(&mut s);

fn calculate_length(s: &String) -> usize {
    s.len()
}

fn change(some_string: &mut String) {
    some_string.push_str(", world");
}
    • &T：不可变引用
    • &mut T：可变引用（一个作用域只能有一个）
    • 多个不可变借用是允许的，因为它们不会改变原数据。
    • 只有一个可变借用是允许的，Rust 保证在同一时间只有一个可变引用，以防止数据竞争。
    • 借用必须遵守生命周期：借用的生命周期必须小于或等于原始数据的生命周期。

    12.5.生命周期（Lifetimes）¶

    生命周期是 Rust 的一个高级概念，用于确保引用在使用时不悬挂。Rust 的生命周期机制防止了悬垂指针和内存泄漏。

    fn longest<'a>(x: &'a str, y: &'a str) -> &'a str {
    if x.len() > y.len() {
        x
    } else {
        y
    }
}

fn main() {
    let s1 = String::from("long string");
    let s2 = "short string";
    let result = longest(&s1, s2);
    println!("The longest string is {}", result);
}
    • 'a 表示生命周期参数，longest 函数的返回值生命周期与输入的两个引用相同。

    12.6. 切片（Slice）¶

    let s = String::from("hello world");

let hello = &s[0..5];
let world = &s[6..11];
let whole = &s[..]; // 整个字符串切片

// 数组切片
let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3]; // [2, 3]

    12.7.总结¶

    • 泛型：Rust 的泛型允许我们写出高度灵活和可复用的代码，且不会牺牲性能。泛型在编译时通过单态化转换为具体类型，从而保持性能。
    • 所有权系统：Rust 的所有权系统通过所有权、借用和移动来管理内存，避免了内存泄漏和数据竞争。编译器在编译时会强制检查所有权规则，确保内存安全。
    • 生命周期：通过生命周期，Rust 确保引用在其有效范围内使用，防止悬垂引用和内存问题。
