    1. 为什么搭建本地化 PyPI？¶

    • 内部包管理：方便企业、团队开发内部工具、包、模块并进行版本管理。
    • 离线环境：在无互联网或受限网络的环境中使用。
    • 安全性：内部包不想公开发布到公共 PyPI 上，可使用本地化 PyPI 进行存储和分发。
    • 依赖控制：有更好的依赖管理和版本控制。

    2. 本地化 PyPI + UI 解决方案¶

    简单部署实现: pypiserver

    以下是几种搭建本地化 PyPI 服务器和 UI 界面的推荐解决方案：

    1.Pypiserver¶

    • 安装
    # Env local
pip install pypiserver
mkdir ~/packages
pypi-server run -p 8080 ~/packages &

# Docker
docker run -p 80:8080 -v ~/.htpasswd:/data/.htpasswd pypiserver/pypiserver:latest run -P .htpasswd packages
    • 创建用户
    pip install passlib htpasswd
htpasswd -c packages/.pypipasswd testuser

# 指定密码文件
pypi-server -P packages/.pypipasswd
    • 配置~/.pypirc
    [distutils]
index-servers =
    pypi
    private
    simple

[pypi]
username = __token__
password = pypi-token

[private]
repository: http://localhost:1234
username: yaojusheng
password: aDBae*Lm4fg+xQM

[simple]
repository: http://localhost:1212
username: testuser
password: test123
    • 发布
    twine upload -r simple dist/*
    • 下载
    pip install --index-url http://127.0.0.1:1212/simple/ noeliclight trusted-host 127.0.0.1

    2. Devpi¶

    Devpi 是一个比较流行的私有 PyPI 服务器和缓存代理。它提供 Web UI 和 REST API 来管理包，并支持用户认证和包发布。它同时支持代理官方 PyPI 仓库（类似缓存功能），从而可以加快包的下载速度。

    • 功能特点：

    • 可以作为缓存代理，从官方 PyPI 缓存包，从而提高包的安装速度。

    • 提供 Web 界面，方便浏览和管理包。
    • 支持多用户环境和权限管理。
    • 支持 pip、twine 等工具进行上传、下载、发布。

    • 支持多级包仓库，可以方便不同团队管理不同的包。

    • 搭建步骤：

    • 安装 devpi-server 和 devpi-web：

      pip install devpi-server devpi-web

    • 初始化和启动服务：

      devpi-init
devpi-server run --host 0.0.0.0 --port 3141

    • 安装 Web 界面插件：

      pip install devpi-web

# 生成启动配置
devpi-gen-config
supervisord -c gen-config/supervisord.conf
supervisorctl -c gen-config/supervisord.conf status

# 配置用户
 pip install --no-cache-dir devpi-client
 devpi use http://localhost:3141
 devpi user -c testuser password=test123

    • 访问 Web UI： 默认 Web UI 可以通过 http://localhost:3141 访问，提供用户友好的包管理界面。

    • 配置客户端： 配置 pip 使用本地 devpi 服务器。

      pip config set global.index-url http://localhost:3141/root/pypi/+simple/

    • 上传包： 使用 twine 将包上传至本地 PyPI 服务器。

      twine upload --repository-url http://localhost:3141/root/pypi/ dist/*

    • 优缺点：

    • 优点：支持离线模式；支持多仓库；提供 Web UI；易于配置。

    • 缺点：复杂的权限控制需要额外配置；对于大规模用户可能需要额外的扩展。

    3. Warehouse (官方 PyPI 实现)¶

    Warehouse 是官方 PyPI 的实现。它是用来处理官方 PyPI 的代码库，但也可以在本地环境中搭建作为私有 PyPI。

    • 功能特点：

    • 是官方 PyPI 的实现，功能齐全，完全可以作为替代方案。

    • 提供完整的用户认证和管理机制。

    • 支持 Web UI，功能和官方 PyPI 相同。

    • 搭建步骤：

    • Clone 官方仓库：

      git clone https://github.com/pypa/warehouse.git
cd warehouse

    • 使用 docker-compose 进行构建：

      docker-compose up

    • 访问 Web UI： 默认情况下，可以通过 http://localhost:8000 访问 Web UI。

    • 优缺点：

    • 优点：官方实现，功能齐全，完全可以替代公共 PyPI。

    • 缺点：搭建稍微复杂，资源消耗相对较高。

    • 使用

    • 配置
    # .pypirc
[distutils]
index-servers =
    pypi
    private

[pypi]
username = __token__
password = pypi-token

[private]
repository: http://localhost:1234/legacy
username: yaojusheng
password: aDBae*Lm4fg+xQM
    • 执行
    twine upload -r private dist/* --verbose

    4. Artifactory (私有仓库解决方案)¶

    JFrog Artifactory 是一款强大的通用包管理器，它可以管理各种类型的包，包括 Docker、Maven、NPM、Python 等。Artifactory 也支持作为私有 PyPI 仓库的功能，并带有强大的权限控制和企业级的包管理功能。

    • 功能特点：

    • 支持多种包类型，包括 Python 包。

    • 企业级功能，提供强大的权限管理、镜像、自动化 CI/CD 集成等。
    • 提供功能强大的 Web UI 管理包。

    • 支持与其他工具集成，比如 Jenkins、GitLab CI 等。

    • 搭建步骤：

    • 下载并安装 Artifactory： 可以从 JFrog 官方站点下载 Artifactory 的安装包，并通过命令行或 Docker 启动服务。

    • 创建 Python 仓库： 通过 Artifactory 的 Web UI，可以轻松创建一个 Python 类型的仓库，配置相应的代理源和权限。

    • 配置客户端： 配置 pip 和 twine 使用 Artifactory。

      pip config set global.index-url http://<artifactory-url>/artifactory/api/pypi/<repo-name>/simple

    • Docker部署

      docker pull releases-docker.jfrog.io/jfrog/artifactory-pro:7.71.11

docker run --name artifactory -v /root/jfrog/artifactory/var/:/var/opt/jfrog/artifactory -d -p 8081:8081 -p 8082:8082 releases-docker.jfrog.io/jfrog/artifactory-pro:7.71.11

    • 优缺点：

    • 优点：支持多种包格式和强大的权限控制，适合企业级项目。
    • 缺点：复杂度较高，可能需要付费使用。

    5. Bandersnatch (缓存镜像)¶

    Bandersnatch 是一个用于同步和缓存 PyPI 包的工具。它不会提供 Web UI，但可以实现将 PyPI 仓库缓存到本地，实现镜像功能。适合对 Web UI 要求不高，主要使用本地包缓存的场景。

    • 功能特点：

    • 轻量级，仅提供同步和缓存功能。

    • 支持镜像所有 PyPI 包或部分包。

    • 无 Web UI，适合离线环境。

    • 搭建步骤：

    • 安装 bandersnatch：

      pip install bandersnatch

    • 配置 bandersnatch：

      bandersnatch mirror -c /path/to/your/config.conf

    • 配置客户端： 配置 pip 使用本地的缓存镜像。

      [mirror]
# 镜像存放的路径
directory = /data/pypi
# 你可以选择同步整个 PyPI (所有包) 或指定包
# 如果只同步某些包，取消 restrict-mirrors 的注释，并在 allowlist 中配置
json = true
keep_index_versions = 2
master = https://pypi.org
workers = 3

[plugins]
# 选择包同步的策略插件，比如过滤掉特定包
enabled =
    blacklist_project
    allowlist_project

[blacklist]
# 列出你不希望同步的包
# project =
#     project1
#     project2

[allowlist]
# 如果 restrict-mirrors 为 True，则仅同步此处列出的包
# project =
#     numpy
#     requests

    • 初始化和运行同步: bandersnatch mirror

    • 定期同步:

      为了保持仓库的最新状态，建议使用 cron 定期运行 bandersnatch：

      crontab -e

      在 crontab 中添加一行，例如每 2 小时同步一次：

      0 */2 * * * /path/to/venv/bin/bandersnatch mirror -c /path/to/config.conf

    • 优缺点：

    • 优点：轻量、简单、快速，适合只需离线缓存的场景。
    • 缺点：不提供 UI 界面，功能相对简单。

    6.Nexus¶

    6.1.Nexus 简介¶

    Nexus Repository Manager 是由 Sonatype 提供的一款强大的仓库管理工具，用于管理各种类型的软件包存储库（repositories）。它支持多种格式的包管理（如 Maven、npm、Docker、PyPI 等），并广泛用于 DevOps 流水线中，帮助开发团队高效管理依赖和构件。

    目前有两个主要版本：

    1. Nexus OSS（开源版）：免费的开源版本，支持基本功能。
    2. Nexus Pro（商业版）：需要付费，包含高级功能（如安全扫描、策略管理等）。

    6.2.Nexus 的核心功能¶

    1. 统一的依赖管理：
    2. 支持多种包格式（Maven、Apt、npm、PyPI、Docker、RubyGems、NuGet 等）。
    3. 提供一个中心化的平台，方便团队共享和管理依赖。
    4. 缓存代理仓库：
    5. Nexus 可以代理远程仓库（如 Maven Central、npm Registry 等），加速依赖下载并减少外部访问。
    6. 私有仓库：
    7. 用于存储团队的内部构件和依赖，支持版本管理。
    8. 构件托管：
    9. 支持上传和托管构件（如 Java JAR 包、Python 包、Docker 镜像等）。
    10. 安全和权限管理：
    11. 提供细粒度的用户权限控制，确保构件的安全访问。
    12. 支持 CI/CD 集成：
    13. 可以与 Jenkins、GitLab CI/CD 等工具集成，作为构建流水线中的依赖管理组件。

    6.3.Nexus 的仓库类型¶

    Nexus 支持三种仓库类型：

    1. Hosted（托管仓库）：
    2. 用于存储团队自有的构件或内部依赖。
    3. 例如，上传私有的 Maven JAR 包或 Docker 镜像。
    4. Proxy（代理仓库）：
    5. 用于代理远程仓库，将依赖缓存到本地。
    6. 例如，代理 Maven Central、npm Registry 等。
    7. Group（组仓库）：
    8. 将多个仓库组合起来，通过单个 URL 访问。
    9. 例如，将多个 Maven 仓库（本地仓库 + 代理仓库）合并为一个组仓库。

    6.4.Nexus 安装¶

    1. 系统要求¶
    • 操作系统：
    • Nexus 支持 Linux、Windows 和 macOS。
    • Java 版本：
    • Nexus 3.x 需要 Java 8 或更高版本。
    • 硬件：
    • 推荐至少 4GB 内存和 10GB 磁盘空间。
    2. 安装步骤¶
    (1) Docker 安装¶

    Nexus 最简单的安装方式是通过 Docker。也可参考: Nexus example、nexus3镜像

    • 拉取官方镜像：
    docker pull sonatype/nexus3
    • 启动容器：
    docker run -d -p 8081:8081 --name nexus -v nexus-data:/nexus-data sonatype/nexus3
    • 容器启动后，访问 http://<服务器IP>:8081。
    • 默认管理员用户名为 admin，初始密码存储在 /nexus-data/admin.password 文件中。
    (2) 二进制安装¶

    1. 下载 Nexus：

    2. 到 官方下载页面 下载二进制文件。

    3. 解压安装:

    tar -xvf nexus-<version>.tar.gz
cd nexus-<version>
    1. 启动 Nexus：
    ./bin/nexus start
    • 默认监听端口为 8081。
    • 访问 http://<服务器IP>:8081，使用默认管理员账户登录。

    3.1.场景方案分析¶

    3.1.1.方案分析¶

    • Pypiserver：
    • 适合小型团队或个人开发者，追求简单部署和轻量使用。
    • Devpi：
    • 适合中小型团队，需求包括私有包托管、代理缓存和基本权限管理。
    • Warehouse：
    • 适合中大型团队，需要功能与官方 PyPI 一致，同时有能力维护复杂部署。
    • Artifactory：
    • 适合大型企业，追求多种包格式支持、企业级功能和安全性。
    • Bandersnatch：
    • 适合需要 PyPI 镜像的团队，特别是离线环境或对官方 PyPI 访问受限的场景。
    • Nexus：
    • 适合中大型团队或企业，需求包括多语言支持、企业级权限管理和可靠的缓存功能。

    3.1.2.总结¶

    • 小团队：使用推荐 Pypiserver（简单轻量）, 或者 Devpi，它功能全面且相对简单，支持缓存和多用户。
    • 中型团队：推荐 Devpi（功能全面）或 Nexus OSS（支持多语言）。
    • 企业级或复杂环境：可以选择 Artifactory或 Nexus Pro（企业级功能强大），它支持各种包管理类型和复杂的权限管理。
    • 离线环境或仅需缓存：使用 Bandersnatch，非常轻量级，适合仅需要 PyPI 缓存的场景。对于资源丰富，想要全量同步、建立镜像，为公司内部、甚至外部提供高速访问服务的人来说，bandersnatch是唯一的选择。
    • 对官方功能一致性要求高：推荐 Warehouse（官方实现）。
