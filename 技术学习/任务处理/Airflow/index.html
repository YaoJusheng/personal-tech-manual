    1. Airflow 简介¶

    Apache Airflow 是一个开源的工作流管理平台，用于开发、调度和监控批处理工作流。它允许您将工作流定义为代码，使其更易于维护、版本控制、测试和协作。

    1.1 核心概念¶

    • DAG (Directed Acyclic Graph): 有向无环图，定义任务之间的依赖关系
    • Task: 工作流中的单个工作单元
    • Operator: 定义如何执行任务的模板
    • Executor: 决定如何执行任务的组件
    • Scheduler: 调度器，负责触发工作流执行
    • Webserver: Web UI界面

    1.2 主要特性¶

    • 动态工作流: 工作流用Python代码定义，允许动态生成管道
    • 可扩展性: 丰富的操作符和执行器生态系统
    • 优雅的用户界面: 清晰的工作流概览，监控进度和调试问题
    • 强大的调度: 支持复杂的调度逻辑和依赖管理

    2. 安装与配置¶

    2.1 安装方式¶

    使用 pip 安装¶

    # 基础安装
pip install apache-airflow

# 带额外依赖的安装
pip install apache-airflow[postgres,redis,celery]

# 指定版本安装
pip install apache-airflow==2.7.3 \
  --constraint "https://raw.githubusercontent.com/apache/airflow/constraints-2.7.3/constraints-3.8.txt"

    3. DAG 开发¶

    3.1 基础DAG结构¶

    from datetime import datetime, timedelta
from airflow import DAG
from airflow.operators.python import PythonOperator
from airflow.operators.bash import BashOperator

# 默认参数
default_args = {
    'owner': 'data-team',
    'depends_on_past': False,
    'start_date': datetime(2024, 1, 1),
    'email_on_failure': False,
    'email_on_retry': False,
    'retries': 1,
    'retry_delay': timedelta(minutes=5),
}

# 定义DAG
dag = DAG(
    'example_dag',
    default_args=default_args,
    description='示例DAG',
    schedule_interval=timedelta(days=1),
    catchup=False,
    tags=['example'],
)

def extract_data():
    """数据提取函数"""
    print("正在提取数据...")
    return "extracted_data"

def transform_data(**context):
    """数据转换函数"""
    data = context['task_instance'].xcom_pull(task_ids='extract')
    print(f"正在转换数据: {data}")
    return "transformed_data"

def load_data(**context):
    """数据加载函数"""
    data = context['task_instance'].xcom_pull(task_ids='transform')
    print(f"正在加载数据: {data}")

# 定义任务
extract_task = PythonOperator(
    task_id='extract',
    python_callable=extract_data,
    dag=dag,
)

transform_task = PythonOperator(
    task_id='transform',
    python_callable=transform_data,
    dag=dag,
)

load_task = PythonOperator(
    task_id='load',
    python_callable=load_data,
    dag=dag,
)

# 设置任务依赖
extract_task >> transform_task >> load_task

    3.3 高级特性¶

    动态任务生成¶

    def create_dynamic_tasks():
    tasks = []
    for i in range(5):
        task = PythonOperator(
            task_id=f'dynamic_task_{i}',
            python_callable=lambda x=i: print(f"执行任务 {x}"),
            dag=dag,
        )
        tasks.append(task)
    return tasks

dynamic_tasks = create_dynamic_tasks()

# 设置依赖关系
for i in range(len(dynamic_tasks) - 1):
    dynamic_tasks[i] >> dynamic_tasks[i + 1]

    分支逻辑¶

    from airflow.operators.python import BranchPythonOperator

def choose_branch(**context):
    """根据条件选择分支"""
    import random
    if random.choice([True, False]):
        return 'task_a'
    else:
        return 'task_b'

branch_task = BranchPythonOperator(
    task_id='branching',
    python_callable=choose_branch,
    dag=dag,
)

task_a = PythonOperator(
    task_id='task_a',
    python_callable=lambda: print("执行任务A"),
    dag=dag,
)

task_b = PythonOperator(
    task_id='task_b',
    python_callable=lambda: print("执行任务B"),
    dag=dag,
)

branch_task >> [task_a, task_b]

    4. 执行器类型¶

    4.1 SequentialExecutor¶

    • 特点: 单线程执行，适用于开发和测试
    • 配置: executor = SequentialExecutor
    • 使用场景: 本地开发、调试

    4.2 LocalExecutor¶

    • 特点: 多进程执行，适用于单机部署
    • 配置: executor = LocalExecutor
    • 使用场景: 中小型工作负载

    4.3 CeleryExecutor¶

    • 特点: 分布式执行，支持水平扩展
    • 配置: 需要Redis或RabbitMQ作为消息代理
    • 使用场景: 生产环境，大规模工作负载
    # Celery Worker 启动
celery -A airflow.executors.celery_executor worker

    4.4 KubernetesExecutor¶

    • 特点: 在Kubernetes集群中执行任务
    • 配置: 每个任务在独立的Pod中运行
    • 使用场景: 云原生环境，资源隔离需求

    6. 最佳实践¶

    6.1 DAG设计原则¶

    1. 幂等性: 确保任务可以安全重复执行
    2. 原子性: 每个任务应该是独立的工作单元
    3. 可观测性: 添加充分的日志和监控
    4. 错误处理: 实现适当的重试和失败处理机制

    6.2 性能优化¶

    # 优化DAG解析性能
dag = DAG(
    'optimized_dag',
    max_active_runs=1,  # 限制并发运行数
    max_active_tasks=10,  # 限制并发任务数
    catchup=False,  # 禁用回填
    dagrun_timeout=timedelta(hours=2),  # 设置超时
)

# 使用连接池
from airflow.hooks.base import BaseHook

def get_connection():
    return BaseHook.get_connection('my_db_conn')

    6.4 测试策略¶

    import pytest
from airflow.models import DagBag

def test_dag_loaded():
    """测试DAG是否正确加载"""
    dag_bag = DagBag()
    dag = dag_bag.get_dag(dag_id='example_dag')
    assert dag is not None
    assert len(dag.tasks) == 3

def test_task_dependencies():
    """测试任务依赖关系"""
    dag_bag = DagBag()
    dag = dag_bag.get_dag(dag_id='example_dag')

    extract_task = dag.get_task('extract')
    transform_task = dag.get_task('transform')

    assert transform_task in extract_task.downstream_list
