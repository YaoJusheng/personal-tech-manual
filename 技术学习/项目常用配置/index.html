    一、常用配置¶

    1.Pyenv & GVM & Rustup & fnm¶

    pyenv: 一个Python版本管理工具，它允许您在同一系统上安装和切换不同版本的Python。

    gvm: go版本管理工具

    rustup: rust 工具琏管理

    fnm/nvm/n: Node.js 版本管理器

    Usage:

    export THOME="/usr/local"
# pyenv
git clone --depth 1 https://github.com/pyenv/pyenv.git ~/.pyenv
export PYENV_ROOT="$THOME/pyenv"
export PATH="$HOME/.local/bin:$PYENV_ROOT/bin:/usr/local/bin:$PATH"
if command -v pyenv 1>/dev/null 2>&1; then
  eval "$(pyenv init -)"
fi
export PYTHON_BUILD_MIRROR_URL_SKIP_CHECKSUM=1
export PYTHON_BUILD_MIRROR_URL="https://registry.npmmirror.com/-/binary/python"

# gvm
sudo git clone https://github.com/moovweb/gvm.git /usr/local/gvm
export GOPROXY=https://goproxy.cn,direct
export GVM_ROOT="$THOME/gvm"
export PATH="$GVM_ROOT/bin:/usr/local/go/bin:$PATH"
[[ -s "$GVM_ROOT/scripts/gvm" ]] && source "$GVM_ROOT/scripts/gvm"

# rustup
sudo mkdir -p /usr/local/cargo /usr/local/rustup
sudo chown -R $(whoami):$(whoami) /usr/local/cargo /usr/local/rustup
export RUSTUP_HOME=/usr/local/rustup
export CARGO_HOME=/usr/local/cargo
export RUSTUP_DIST_SERVER=https://mirrors.ustc.edu.cn/rust-static
export RUSTUP_UPDATE_ROOT=https://mirrors.ustc.edu.cn/rust-static/rustup
curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
. "$THOME/cargo/env"

# nvm
sudo git clone https://github.com/nvm-sh/nvm.git /usr/local/nvm
export NVM_DIR=/usr/local/nvm
# Load NVM
[ -s "$NVM_DIR/nvm.sh" ] && \. "$NVM_DIR/nvm.sh"
[ -s "$NVM_DIR/bash_completion" ] && \. "$NVM_DIR/bash_completion"

source ~/.bashrc

# tool
alias cf='fzf --height=40% --layout=reverse --info=inline --border --margin=1 --padding=1'
alias fd='fdfind'

template1：一个模板数据库，但与 template0 不同，它是可写的。template1 通常包含一些用户自定义的设置或扩展模块。

    6.3.3.输出指标¶
    • Latency（延迟）：
    • Latency 表示每个请求的平均延迟时间，通常以毫秒为单位。它是从发送请求到接收到响应的时间。
    • Requests/sec（每秒请求数）：
    • Requests/sec 表示每秒完成的请求数量。这个指标显示了服务器每秒能够处理的请求数量。
    • Transfer/sec（传输速率）：
    • Transfer/sec 表示每秒传输的数据量，通常以 MB/s 或者 KB/s 为单位。它表示服务器的带宽使用情况。
    • Socket errors（Socket错误）：
    • Socket errors 列出了在测试期间发生的各种 socket 错误的数量，比如连接错误、读取错误、写入错误、超时等。
    • Non-2xx or 3xx responses（非 2xx 或 3xx 响应）：
    • 显示了除了 2xx 和 3xx 状态码之外的响应数量，这些状态码通常代表着请求的失败或错误。
    • Errors：
    • Errors 列出了在测试期间发生的各种错误数量，可能包括超时、连接错误、HTTP 错误等。
    • Duration（持续时间）：
    • Duration 表示整个测试的持续时间，通常以秒为单位。
    6.3.4.特点与比较¶
    • 特点：
    • 高度可定制，支持多线程，适用于HTTP基准测试。
    • 提供较为详细的性能统计和结果。
    • 适用场景：
    • 需要更高级定制和更详细结果分析的场景。
    • 限制：
    • 需要一定的命令行技能和理解性能统计指标的知识。

    ab 和 siege 更适合简单的测试场景，对于快速进行基准测试或较简单的压力测试是不错的选择。 wrk 更适合对于 HTTP 请求和并发性能有更高要求的场景，需要更高级的定制和更详细的结果分析。

    7.4.Charles¶

    Charles是一款http抓包工具，它是通过代理来实现的抓包。也就是我们在访问网页时需要配置代理，将代理指向Charles监听的端口，之后我们的http请求都会发向Charles的端口，之后Charles会帮我们转发并记录协议内容。

    wget -q -O - https://www.charlesproxy.com/packages/apt/PublicKey | sudo apt-key add
sudo sh -c 'echo deb https://www.charlesproxy.com/packages/apt/ charles-proxy main > /etc/apt/sources.list.d/charles.list'
# 安装
sudo apt-get update
sudo apt-get install charles-proxy
# 卸载
sudo apt-get remove charles-proxy

    7.5.Mitmproxy¶

    mitmproxy是python写的一款http抓包工具，虽然只支持http抓包，但是它的特性非常强大，它不仅可以抓包，还可以对请求进行拦截、重现等操作。和Charles一样，它的原理也是基于代理，使用的时候需要设置代理指向它。

    mitmproxy是命令行工具，但是也自带了mitmweb工具，可以让用户在网页上操作。另外，mitmproxy还支持用户自行编写插件，可以编写脚本对请求进行处理，然后把修改后的请求发出去。

    pip3 install mitmproxy
