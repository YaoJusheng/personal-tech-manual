    一、图片处理¶

    1.1.图片处理的一般方案¶

    1. 选择合适的图片格式：根据需求选择合适的图片格式，如JPEG、PNG、GIF、TIFF等。
    2. 调整图片大小：根据使用场景调整图片的分辨率，以优化文件大小和显示效果。
    3. 颜色空间转换：在不同颜色空间之间进行转换，如BGR到RGB。
    4. 色彩校正：使用直方图匹配等技术进行色彩校正，以确保颜色的一致性。
    5. 压缩：根据需求选择合适的压缩方式，如无损压缩或有损压缩。
    6. 传输优化：在传输过程中，确保图片数据的完整性和兼容性。

    1.2.常见图片格式对比¶

    格式 特点 优点 缺点 适用场景
    JPEG 有损压缩 压缩率高，文件体积小，适合照片和复杂图像 有损压缩可能导致图像质量损失，不支持透明背景 照片、网页图片、社交媒体
    PNG 无损压缩 无损压缩，支持透明背景，适合图标和图形 文件大小较大 图标、图形、需要透明背景的图像
    GIF 支持动画 支持简单动画，文件小，适合图标和简单图像 色彩受限，不适合复杂图像 简单动画、图标、表情包
    TIFF 无损压缩 无损压缩，支持多通道图像，适合印刷和专业图像处理 文件大小较大，不适合Web应用 印刷、专业摄影、图像编辑
    BMP 未压缩 无损压缩，适用于Windows系统和位图编辑 文件非常大，不适合网络传输 Windows系统和位图编辑
    WebP 支持有损和无损压缩 高压缩率，支持透明背景和动画，适合Web应用 兼容性较差，部分浏览器不支持 Web应用、需要高压缩率的图像
    SVG 矢量图格式 矢量图，放大或缩小不会失真，适合图标和Logo 不适合复杂照片或图像 图标、Logo、图表、网页设计中的矢量图形

    总结：

    • JPEG：适合照片或复杂色彩渐变的图片，文件小但质量有损。
    • PNG：适合需要透明背景的高质量图片，文件大但无损压缩。
    • GIF：适合简单动画和图标，颜色有限，支持动画。
    • TIFF：适合印刷、专业摄影、图像编辑，文件大但质量高。
    • BMP：适合图像编辑和处理过程中保存的临时图像，文件大。
    • WebP：适合现代网页图像，兼顾文件大小和质量。
    • SVG：适合图标、Logo等矢量图，放大不失真，适合网页设计。

    1.3.PIL¶

    PIL（Python Imaging Library）是一个功能强大的图片处理库，提供了广泛的图片处理功能。以下是 PIL 常用的图片处理方法：

    1.3.1.基本操作¶

    • Image.open()： 打开图片
    • Image.save()：保存图片
    • Image.show()： 显示图片
    • 获取图片信息：使用 Image.size、Image.mode 等属性获取图片的尺寸和模式。

    1.3.2.裁剪和调整¶

    • 裁剪图片：使用 Image.crop(left, upper, right, lower) 方法裁剪图片。img_pil.crop((100, 100, 400, 400))
    • 调整大小：使用 Image.resize() 方法调整图片大小。
    • 旋转图片：使用 Image.rotate() 方法旋转图片。
    • 翻转图片：使用 Image.transpose() 方法翻转图片。

    1.3.3. 颜色空间转换¶

    • 转换颜色模式：使用 Image.convert() 方法转换图片的颜色模式（如 RGB 到灰度图）。e.g.: img_pil.convert("RGB").save("converted.jpg", "JPEG")

    1.3.4. 图片的滤波¶

    • 应用滤波器：使用 ImageFilter 模块中的各种滤波器（如高斯模糊、中值滤波等）。

    1.3.5. 图片的增强¶

    • 调整亮度和对比度：使用 ImageEnhance 模块调整图片的亮度和对比度。
    • 应用颜色映射：使用 ImageOps 模块应用颜色映射。

    1.3.6. 图片的合成¶

    • 合成图片：使用 Image.blend() 方法合成两张图片。

    1.3.7.其他¶

    • 添加水印： ImageDraw.Draw(img_pil).text((x, y), "Watermark")

    1.4.OpenCV¶

    OpenCV 是一个功能强大的计算机视觉库，提供了广泛的图片处理功能。以下是 OpenCV 常用的图片处理方法：

    1.4.1. 图片的基本操作¶

    • 读取图片：使用 cv2.imread() 方法读取图片文件。
    • 显示图片：使用 cv2.imshow() 方法显示图片。
    • 保存图片：使用 cv2.imwrite() 方法保存图片。
    • 获取图片信息：使用 shape 属性获取图片的尺寸和通道信息。

    1.4.2. 颜色空间转换¶

    • 转换颜色空间：使用 cv2.cvtColor() 方法转换颜色空间（如 BGR 到 RGB、HSV 等）。

    1.4.3. 图片的滤波¶

    • 高斯模糊：使用 cv2.GaussianBlur() 方法应用高斯模糊。
    • 中值滤波：使用 cv2.medianBlur() 方法应用中值滤波。
    • 双边滤波：使用 cv2.bilateralFilter() 方法应用双边滤波。

    1.4.4. 边缘检测¶

    • Canny 边缘检测：使用 cv2.Canny() 方法进行边缘检测。
    • Sobel 边缘检测：使用 cv2.Sobel() 方法进行边缘检测。

    1.4.5. 特征检测和描述¶

    • SIFT 特征检测：使用 cv2.SIFT_create() 方法创建 SIFT 特征检测器。
    • SURF 特征检测：使用 cv2.SURF_create() 方法创建 SURF 特征检测器。
    • ORB 特征检测：使用 cv2.ORB_create() 方法创建 ORB 特征检测器。

    1.4.6. 图片的变换¶

    • 仿射变换：使用 cv2.warpAffine() 方法进行仿射变换。
    • 透视变换：使用 cv2.warpPerspective() 方法进行透视变换。

    1.4.7. 图片的增强¶

    • 直方图均衡化：使用 cv2.equalizeHist() 方法进行直方图均衡化。
    • 自适应直方图均衡化：使用 cv2.createCLAHE() 方法进行自适应直方图均衡化。

    1.4.8. 图片的分割¶

    • 阈值分割：使用 cv2.threshold() 方法进行阈值分割。
    • Otsu 阈值分割：使用 cv2.threshold() 方法结合 Otsu 算法进行自动阈值分割。
    • GrabCut 分割：使用 cv2.grabCut() 方法进行图像分割。

    1.4.9. 图片的匹配¶

    • 模板匹配：使用 cv2.matchTemplate() 方法进行模板匹配。
    • 特征匹配：使用 cv2.BFMatcher 或 cv2.FlannBasedMatcher 进行特征匹配。

    1.6.总结¶

    PIL 和 OpenCV 都是功能强大的图片处理库，但它们的侧重点不同：

    • PIL：适合基本的图片处理任务，如裁剪、调整大小、颜色转换等。
    • OpenCV：适合更复杂的计算机视觉任务，如特征检测、边缘检测、图片分割等。
    • scikit-image：适合科研和工程应用，基于 NumPy。
    • ImageMagick（Wand）：适合需要高级图片处理功能的场景。
    • pngquant：专注于 PNG 图片的高效压缩。
    • Aspose.Imaging：提供全面的图片处理功能，适合企业级应用。

    4. 图像缩放：运用 resize 方法对图像大小进行调整。
    5. JPEG 编码：借助 io.BytesIO 存储编码后的 JPEG 数据，再通过 save 方法将图像保存为 JPEG 格式。
    6. 获取二进制数据：从 io.BytesIO 里读取二进制数据。
    2.方法二：OpenCV¶
    ret, frame = cap.read()
if ret:
    frame = cv2.cvtColor(frame, cv2.COLOR_BGR2RGB)
    ret, buffer = cv2.imencode(".jpeg", frame)
    img_bytes = buffer.tobytes()
    await websocket.send_bytes(img_bytes)
    1. 使用 OpenCV 读取帧：同样使用 cv2.VideoCapture 读取视频帧。
    2. 颜色空间转换：把帧的颜色空间从 BGR 转换为 RGB。
    3. JPEG 编码：采用 cv2.imencode 直接对帧进行 JPEG 编码，将结果存储于 buffer 中。
    4. 获取二进制数据：调用 tobytes 方法把编码后的 NumPy 数组转换为二进制数据。
    3.比较¶
    3.1.性能差异¶

    • 内存占用

    • 方法 1：在转换为 PIL.Image 对象时，会额外占用一定内存。并且，io.BytesIO 也会占用内存来存储编码后的图像数据。

    • 方法 2：直接在 OpenCV 的 NumPy 数组上进行编码，减少了中间对象的创建，内存占用相对较低。

    • 处理速度

    • 方法 1：由于涉及到 PIL.Image 对象的转换和操作，以及 io.BytesIO 的使用，处理速度可能较慢。

    • 方法 2：直接使用 OpenCV 的编码函数，处理速度通常较快，因为 OpenCV 是用 C++ 实现的，底层优化较好。

    3.2.灵活性差异¶

    • 图像操作

    • 方法 1：使用 PIL 库可以进行更多的图像操作，如裁剪、旋转、滤镜等。在需要对图像进行复杂处理时，PIL 提供了丰富的 API。

    • 方法 2：OpenCV 主要侧重于计算机视觉任务，对于简单的图像缩放、颜色空间转换等操作比较方便，但对于一些高级的图像编辑功能，不如 PIL 灵活。

    • 编码参数

    • 方法 1：PIL 的 save 方法可以通过参数设置 JPEG 编码的质量，例如 img.save(buffer, format="JPEG", quality=80)。

    • 方法 2：cv2.imencode 可以通过第二个参数设置编码参数，例如 ret, buffer = cv2.imencode(".jpg", frame, [cv2.IMWRITE_JPEG_QUALITY, 80])。

    3.3.兼容性差异¶

    • 其他库的兼容性

    • 方法 1：PIL 库在 Python 社区中广泛使用，与其他图像处理库（如 scikit-image）的兼容性较好。

    • 方法 2：OpenCV 与其他计算机视觉库（如 NumPy、SciPy）的集成度较高，适合进行大规模的图像数据处理。

    3.4.适用场景¶

    方法 1

    • 适用于需要进行复杂图像操作和编辑的场景，例如图像处理软件、图像增强等。
    • 当需要与其他使用 PIL 库的代码集成时，方法 1 更为合适。

    方法 2

    • 适用于对性能要求较高的实时视频处理场景，例如视频监控、视频直播等。
    • 当主要进行计算机视觉任务，如目标检测、人脸识别时，方法 2 更为合适。

    1.5.2.示例：图像与base64互转¶

    import base64
from io import BytesIO

# PIL图像 → Base64字符串
def pil_to_base64(img_pil, format="JPEG"):
    buffered = BytesIO()
    img_pil.save(buffered, format=format)
    return base64.b64encode(buffered.getvalue()).decode("utf-8")

# Base64字符串 → PIL图像
def base64_to_pil(base64_str):
    img_data = base64.b64decode(base64_str.split(",")[-1])  # 处理可能的数据头
    return Image.open(BytesIO(img_data))

# OpenCV图像 → Base64字符串
def cv_to_base64(img_cv, format=".jpg"):
    _, buffer = cv2.imencode(format, img_cv)
    return base64.b64encode(buffer).decode("utf-8")

# Base64字符串 → OpenCV图像
def base64_to_cv(base64_str):
    img_data = base64.b64decode(base64_str.split(",")[-1])
    np_arr = np.frombuffer(img_data, np.uint8)
    return cv2.imdecode(np_arr, cv2.IMREAD_COLOR)

    2.1.方案与对比¶

    2.1.1.一般方案¶

    1. 选择合适的库：根据具体需求选择合适的视频处理库，如 MoviePy、OpenCV、FFmpeg 等。
    2. 读取视频：使用所选库读取视频文件。

    4. 保存或输出视频：将处理后的视频保存为新的文件或输出到其他媒介。

    import av

input_container = av.open(input_file)
input_stream = input_container.streams.video[0]

output_container = av.open(output_file, mode='w')
output_stream = output_container.add_stream('libx264', rate=target_fps)
output_stream.width = input_stream.width
output_stream.height = input_stream.height
output_stream.pix_fmt = input_stream.pix_fmt

for packet in input_container.demux(input_stream):
    for frame in packet.decode():
        new_packet = output_stream.encode(frame)
        if new_packet:
            output_container.mux(new_packet)

remaining_packets = output_stream.encode(None)
for packet in remaining_packets:
    output_container.mux(packet)

input_container.close()
output_container.close()

    2.7.常见视频格式对比¶

    1. MP4¶

    • 特点：
    • 兼容性：广泛支持，几乎所有的设备和播放器都能播放 MP4 格式的视频。
    • 压缩效率：采用先进的编码标准（如 H.264、H.265），具有较高的压缩效率，能在较小的文件大小下保持较好的视频质量。
    • 应用场景：网络视频、在线教育、移动视频等。

    2. AVI¶

    • 特点：
    • 兼容性：早期的视频格式，兼容性较好，但随着技术发展，一些新的设备和播放器可能对其支持不如 MP4。
    • 压缩效率：相对较低，文件大小通常较大。
    • 应用场景：传统视频编辑、早期视频存储等。

    3. MOV¶

    • 特点：
    • 兼容性：主要由苹果公司开发，在苹果设备和软件中具有良好的兼容性。
    • 压缩效率：支持多种编码标准，压缩效率和视频质量取决于具体的编码设置。
    • 应用场景：苹果生态系统内的视频制作、分享等。

    4. MKV¶

    • 特点：
    • 兼容性：是一种开放的容器格式，支持多种音视频编码，兼容性较好。
    • 压缩效率：可以封装多个音轨、字幕等，适合包含多种元素的视频文件。
    • 应用场景：高清视频、蓝光原盘提取等。

    5. FLV¶

    • 特点：
    • 兼容性：主要用于网络流媒体，在网页浏览器中具有良好的兼容性。
    • 压缩效率：文件大小较小，适合网络传输。
    • 应用场景：在线直播、视频网站等。
