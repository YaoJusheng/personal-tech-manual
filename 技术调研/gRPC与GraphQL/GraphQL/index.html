    1. GraphQL 概述¶

    1.1 什么是 GraphQL¶

    GraphQL 是一种用于 API 的查询语言和运行时环境，由 Facebook 于 2012 年开发并于 2015 年开源。它提供了一种更高效、强大和灵活的替代 REST API 的方案。

    1.2 核心优势¶

    • 精确数据获取：客户端可以准确指定需要的数据，避免过度获取或获取不足
    • 强类型系统：完整的类型系统确保 API 的可靠性和可预测性
    • 单一端点：所有数据通过单一 URL 端点访问，简化 API 管理
    • 实时订阅：内置支持实时数据更新
    • 内省能力：API 可以描述自身，支持强大的开发工具
    • 版本无关：通过字段弃用而非版本控制来演进 API

    1.3 与 REST 的对比¶

    特性 GraphQL REST
    数据获取 精确获取所需数据 可能过度获取或不足
    端点数量 单一端点 多个端点
    缓存 复杂但可控 HTTP 缓存简单
    学习曲线 较陡峭 相对简单
    工具生态 快速发展 成熟丰富

    3. 核心概念¶

    3.1 GraphQL 操作类型¶

    GraphQL 支持三种主要操作类型：

    Query（查询）¶

    用于读取数据，类似于 REST 中的 GET 请求。

    query GetUser($id: ID!) {
  user(id: $id) {
    id
    name
    email
    posts {
      title
      content
    }
  }
}

    Mutation（变更）¶

    用于修改数据，类似于 REST 中的 POST、PUT、DELETE 请求。

    mutation CreateUser($input: CreateUserInput!) {
  createUser(input: $input) {
    id
    name
    email
  }
}

    Subscription（订阅）¶

    用于实时数据更新，基于 WebSocket 或 Server-Sent Events。

    subscription OnCommentAdded($postId: ID!) {
  commentAdded(postId: $postId) {
    id
    content
    author {
      name
    }
  }
}

    3.2 类型系统¶

    GraphQL 使用强类型系统来定义 API 的结构：

    标量类型¶

    • String - UTF-8 字符串
    • Int - 32位有符号整数
    • Float - 双精度浮点数
    • Boolean - 布尔值
    • ID - 唯一标识符

    对象类型¶

    type User {
  id: ID!
  name: String!
  email: String!
  age: Int
  posts: [Post!]!
}

type Post {
  id: ID!
  title: String!
  content: String!
  author: User!
  createdAt: DateTime!
}

    输入类型¶

    input CreateUserInput {
  name: String!
  email: String!
  age: Int
}

    3.3 Schema 定义语言（SDL）¶

    GraphQL Schema 定义了 API 的完整结构：

    # 根查询类型
type Query {
  users: [User!]!
  user(id: ID!): User
  posts: [Post!]!
  post(id: ID!): Post
}

# 根变更类型
type Mutation {
  createUser(input: CreateUserInput!): User!
  updateUser(id: ID!, input: UpdateUserInput!): User!
  deleteUser(id: ID!): Boolean!
}

# 根订阅类型
type Subscription {
  userCreated: User!
  postUpdated: Post!
}

    4. GraphQL 语法规则¶

    GraphQL是一种用于API的查询语言和类型系统，其语法规则如下：

    • 查询语句由字段和参数组成。
    • 查询语句由关键字query或mutation或subscription开头，后面跟着可选的操作名称和可选的参数列表。
    • 每个查询包含一个或多个字段。每个字段都有一个名称和一个返回类型。
    • 每个字段可以有一个或多个参数，每个参数都有一个名称和一个类型。
    • 参数可以是必选的，也可以是可选的。必选参数在字段名称后面带有一个冒号和参数类型，可选参数在参数名称后面带有一个问号。
    • 每个字段都可以有一个或多个子字段，用来指定返回的子级数据。
    • 通过在字段和子字段之间添加圆括号，可以将字段分组为嵌套结构。
    • 查询可以有一个或多个操作变量，用来在查询之外传递参数。
    • 变量在查询中使用$符号和名称定义，并在查询中引用。
    • 通过使用查询片段，可以在不重复编写查询的情况下复用字段和子字段。
    • 查询片段可以使用...符号引用，或者嵌套在其他查询片段或查询操作中。
    • 查询可以有指令，用于在查询执行过程中控制查询的行为。
    • 每个指令都有一个名称和可选的参数列表。
    • 查询可以包含注释，注释以#开头，直到行末为止。
    • 查询语句由一个或多个查询组成，每个查询以;结尾。

    4.1 GraphQL-Core 示例¶

    下面是一个示例，演示如何在Python中使用graphql-core库来构建GraphQL查询语句和解析查询结果。

    安装graphql-core库¶

    pip install graphql-core

    查询语句示例¶

    from graphql import build_ast_schema, parse, print_ast
from graphql.language.ast import (
    FieldNode,
    NameNode,
    SelectionSetNode,
    DocumentNode,
    OperationDefinitionNode,
)
from graphql.type.definition import GraphQLObjectType, GraphQLField, GraphQLString, GraphQLFloat, GraphQLList

# 定义电影类型
movie_type = GraphQLObjectType(
    name="Movie",
    fields=lambda: {
        "name": GraphQLField(GraphQLString),
        "rating": GraphQLField(GraphQLFloat)
    }
)

# 定义查询类型
query_type = GraphQLObjectType(
    name="Query",
    fields=lambda: {
        "movies": GraphQLField(
            GraphQLList(movie_type),
            resolver=lambda obj, info: [
                {"name": "Movie 1", "rating": 8.5},
                {"name": "Movie 2", "rating": 7.2},
                {"name": "Movie 3", "rating": 6.9}
            ]
        )
    }
)

# 构建Schema
schema = build_ast_schema(DocumentNode(definitions=[OperationDefinitionNode(
    operation="query",
    selection_set=SelectionSetNode(
        selections=[
            FieldNode(
                name=NameNode(value="movies"),
                selection_set=SelectionSetNode(
                    selections=[
                        FieldNode(name=NameNode(value="name")),
                        FieldNode(name=NameNode(value="rating")),
                    ]
                )
            )
        ]
    )
)]))

# 解析查询
result = schema.execute("query { movies { name rating } }")

# 打印结果
print(result.data)

    上述代码中，我们使用graphql-core库构建了一个GraphQL Schema，并定义了一个Movie类型和一个Query类型。 然后，我们使用GraphQL语法规则构建了一个查询语句，并使用schema.execute方法解析了查询，并返回了结果。 最后，我们打印了结果，结果应该是一个包含电影名称和评分的列表。

    4.2 Python 实现详细对比¶

    在Python中，有许多流行的GraphQL库，下面介绍几个常用的：

    库名 特点 适用场景 学习曲线
    Graphene 成熟稳定，Django集成好 企业级应用 中等
    Strawberry 现代化，类型注解支持 新项目推荐 较低
    Ariadne Schema-first 方法 大型团队协作 中等
    Tartiflette 高性能，异步支持 高并发场景 较高

    详细库介绍¶

    • Graphene: Graphene是一个跨语言的GraphQL库，它支持多种语言，包括Python、JavaScript、Ruby、Go等。它的优点是易于使用和学习，提供了简单和清晰的API和文档。Graphene支持异步查询和自定义扩展，并提供了一些扩展插件和工具，如graphene-django、graphene-sqlalchemy等。缺点是文档相对较少。

    • Graphene-Python: Graphene-Python是Graphene的Python实现，它提供了一个易于使用和学习的API，并且能够轻松地与Django、SQLAlchemy和其他Python库进行集成。Graphene-Python支持异步查询和自定义扩展，并提供了一些扩展插件和工具，如graphene-file-upload、graphene-relay等。缺点是文档和社区支持相对较少。

    • Sanic-GraphQL: Sanic-GraphQL是一个基于Sanic框架的GraphQL库，它的主要优点是速度快和易于使用。Sanic-GraphQL支持异步查询和自定义扩展，并提供了一些扩展插件和工具，如sanic-graphql-playground、sanic-jwt等。缺点是文档和社区支持相对较少。

    • Ariadne: Ariadne是一个轻量级的GraphQL库，其主要优点是易于使用和学习，提供了简单和清晰的API和文档，并支持异步查询和HTTP服务。Ariadne还支持使用Python类型自动生成GraphQL模式，并提供了一些扩展插件和工具，如ariadne-django、ariadne-sqlalchemy等。

    • Strawberry: Strawberry是一个基于Python 3.7+类型注释和PEP 563的GraphQL库。它的优点是易于使用和学习，提供了简单和清晰的API和文档，并支持异步查询和HTTP服务。Strawberry还支持使用Python类型自动生成GraphQL模式，可以更轻松地进行类型检查和错误处理，并提供了一些扩展插件和工具，如:strawberry-asyncio、strawberry-django等。缺点是仍处于相对较早的开发阶段，文档和社区支持相对较少。

    • gql: gql是一个轻量级的GraphQL库，它支持Python类型注释自动生成GraphQL模式，并提供了一些常用的GraphQL操作，如查询、变异、订阅等。优点是易于使用和学习，提供了简单和清晰的API和文档，并支持异步查询和HTTP服务。gql还提供了一些扩展插件和工具，如gql-next、gqlalchemy、gql-subscriptions等。缺点是文档相对较少，扩展插件和工具支持相对较少。

    • Tartiflette: Tartiflette是一个高性能的GraphQL库，它的主要优点是速度快和可扩展性好，可以处理大量的请求和数据。Tartiflette支持异步查询和自定义扩展，并提供了一些扩展插件和工具，如 tartiflette-aiohttp、tartiflette-asgi等。缺点是学习曲线相对较陡峭，文档和社区支持相对较少。

    • GraphLayer: GraphLayer是一个GraphQL库和开发框架，它提供了可扩展的GraphQL服务和工具集。GraphLayer支持Python类型注释自动生成GraphQL模式，并提供了一些常用的GraphQL操作，如查询、变异、订阅等。GraphLayer还支持异步查询和自定义扩展，并提供了一些扩展插件和工具，如graphlayer-django、graphlayer-sqlalchemy等。缺点是文档和社区支持相对较少。

    总结：Python中的GraphQL库都有各自的优缺点，选择适合自己的库取决于具体的需求和项目规模。需要注意的是，在使用GraphQL库时，应该确保安全性和性能，避免GraphQL查询被滥用或用于恶意目的，同时需要进行适当的缓存和优化来提高查询性能。

  Subscription: {
    messageAdded: {
      subscribe: () => pubsub.asyncIterator(['MESSAGE_ADDED'])
    }
  },
  Mutation: {
    addMessage: (_, { message }) => {
      pubsub.publish('MESSAGE_ADDED', { messageAdded: message });
      return message;
    }
  }
};
