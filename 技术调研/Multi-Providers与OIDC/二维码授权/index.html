    概述¶

    二维码授权是一种利用二维码实现 OAuth 2.0 认证的用户体验优化方案。用户可以通过扫描二维码在另一设备上完成身份验证，特别适用于：

    • 桌面应用：桌面客户端通过手机扫码授权
    • 无设备认证：浏览器中通过手机授权（无浏览器登录场景）
    • 设备配对：IoT 设备与账户的绑定
    • 多设备同步：跨设备认证流程

    与传统授权的区别¶

    传统方式 二维码方式
    在当前设备输入用户名密码 扫描二维码，使用已登录设备授权
    需要在受限设备输入凭据 凭据仅出现在可信设备上
    授权链接复制粘贴 自动识别，无需手动输入

    实现方案¶

    方案一：Device Authorization Grant + 二维码¶

    这是最标准的方案，RFC 8628 明确支持二维码展示 verification_uri_complete。

    流程：

    1. 设备请求设备授权端点
    2. 服务器返回 device_code、user_code 和 verification_uri_complete
    3. 设备生成二维码，内容为 verification_uri_complete
    4. 用户使用已登录的手机扫码并授权
    5. 设备轮询 token 端点获取 token

    二维码内容示例：

    https://auth.example.com/device?user_code=WDJB-MJHT

    优点：

    • 符合 RFC 标准
    • 支持手动输入 user_code
    • 有超时机制，安全性高

    缺点：

    • 需要设备轮询，增加复杂度

    方案二：OAuth Authorization Code + 二维码¶

    桌面应用生成包含认证请求参数的二维码，用户使用手机扫码后通过浏览器完成认证。

    流程：

    1. 桌面应用生成 state 和 code_verifier（如使用 PKCE）
    2. 构建授权 URL 并生成二维码
    3. 用户扫码后在手机浏览器打开
    4. 手机上完成认证，服务器重定向到桌面应用的回调
    5. 桌面应用通过数据通道（如 WebSocket、Server-Sent Events）获取回调

    Mermaid 流程图：

    sequenceDiagram
    participant Desktop as 桌面应用
    participant Phone as 手机浏览器
    participant Auth as 授权服务器
    participant WS as WebSocket服务

    Desktop->>Desktop: 1. 生成 state, code_verifier
    Desktop->>Desktop: 2. 计算 code_challenge

    Desktop->>Desktop: 3. 生成二维码<br/>auth URL + code_challenge

    Phone->>Phone: 4. 扫码访问授权URL
    Phone->>Auth: 5. 请求授权页面
    Auth->>Phone: 显示登录/授权页面
    Phone->>Auth: 6. 用户完成认证授权
    Auth->>Phone: 7. 重定向到回调URI<br/>code=xxx&state=xyz

    Note over Phone: 回调通过 Custom URI Scheme<br/>唤起桌面应用

    Phone->>Desktop: 8. 唤起桌面应用回调
    Desktop->>Auth: 9. Token 请求<br/>(code + code_verifier)
    Auth->>Desktop: 10. 返回 Token

    &redirect_uri=com.example.app://callback

    &state=xyz789

    方案三：Token Binding + 二维码¶

    使用硬件令牌或安全密钥绑定实现无密码认证。

    流程：

    1. 用户在设备 A 打开登录页面
    2. 设备 A 显示二维码（包含临时会话 ID）
    3. 用户使用已绑定安全密钥的设备 B 扫描二维码
    4. 用户在设备 B 确认身份（生物识别/PIN）
    5. 设备 B 将认证信息推送回服务器
    6. 设备 A 获取认证结果

    Mermaid 流程图：

    sequenceDiagram
    participant DeviceA as 设备A<br/>(登录设备)
    participant Auth as 授权服务器
    participant DeviceB as 设备B<br/>(安全密钥/手机)

    DeviceA->>Auth: 1. 请求登录会话
    Auth->>DeviceA: 2. 返回会话ID<br/>生成二维码
    DeviceA->>DeviceA: 3. 显示二维码

    DeviceB->>DeviceB: 4. 扫码
    DeviceB->>Auth: 5. 获取会话信息
    Auth->>DeviceB: 显示确认页面
    DeviceB->>DeviceB: 6. 用户确认<br/>(生物识别/PIN)
    DeviceB->>Auth: 7. 推送认证确认
    Auth->>Auth: 8. 验证并记录认证结果

    loop 轮询/长连接
        DeviceA->>Auth: 9. 查询认证状态
        Auth->>DeviceA: 认证完成
    end

    DeviceA->>Auth: 10. 获取最终 Token
    Auth->>DeviceA: 11. 返回 Token

    二维码生成与解析¶

    二维码规范要求¶

    • 版本：建议 QR Code Version 3 以上（29x29 模块）
    • 纠错级别：M 级别（15% 恢复能力）
    • 编码模式：Byte 模式（UTF-8）
    • 尺寸：在设备屏幕上清晰可扫描，建议至少 200x200 像素

    生成二维码¶

    JavaScript 示例：

    import QRCode from 'qrcode';

async function generateQRCode(data: string): Promise<string> {
  return QRCode.toDataURL(data, {
    errorCorrectionLevel: 'M',
    version: 5,
    margin: 2,
    width: 300,
    color: {
      dark: '#000000',
      light: '#FFFFFF',
    },
  });
}

// 使用
const authUrl = 'https://auth.example.com/device?user_code=WDJB-MJHT';
const qrCodeDataUrl = await generateQRCode(authUrl);

    Go 示例：

    package auth

import (
    "github.com/skip2/go-qrcode"
)

func GenerateQRCodePNG(content string, size int) ([]byte, error) {
    return qrcode.Encode(content, qrcode.Medium, size)
}

func GenerateQRCodeBase64(content string) (string, error) {
    png, err := qrcode.Encode(content, qrcode.Medium, 300)
    if err != nil {
        return "", err
    }
    return "data:image/png;base64," + base64.StdEncoding.EncodeToString(png), nil
}

    二维码安全性¶

    潜在风险：

    • 二维码替换攻击：恶意二维码替换合法二维码
    • 中间人攻击：不安全网络环境下二维码内容被窃取
    • 会话劫持：通过二维码获取的会话被攻击者利用

    防护措施：

    1. HTTPS 强制：所有认证相关流量必须使用 HTTPS
    2. 短期有效性：二维码设置较短的有效期（如 5-10 分钟）
    3. 一次性使用：二维码只能使用一次
    4. 设备绑定：二维码与特定设备/会话绑定
    5. 用户确认：敏感操作需要用户二次确认
    6. 监控异常：检测异常的扫码和授权模式

    实现示例¶

    桌面应用扫码登录完整示例¶

    Go 后端实现：

    package auth

import (
    "context"
    "crypto/rand"
    "encoding/base64"
    "encoding/json"
    "fmt"
    "net/http"
    "time"

    "github.com/go-oauth2/oauth2/v4"
    "github.com/google/uuid"
)

// QRCodeSession 管理二维码会话
type QRCodeSession struct {
    SessionID   string    `json:"session_id"`
    AuthURL     string    `json:"auth_url"`
    CallbackURL string    `json:"callback_url"`
    CreatedAt   time.Time `json:"created_at"`
    ExpiresAt   time.Time `json:"expires_at"`
    Status      string    `json:"status"` // pending, authorized, expired
    TokenInfo   *oauth2.TokenInfo `json:"token_info,omitempty"`
}

type QRCodeService struct {
    sessions     map[string]*QRCodeSession
    tokenStore    oauth2.TokenStore
    authCodeStore oauth2.AuthorizeCodeStore
}

func NewQRCodeService() *QRCodeService {
    return &QRCodeService{
        sessions: make(map[string]*QRCodeSession),
    }
}

// 创建二维码会话
func (s *QRCodeService) CreateSession(clientID, redirectURI string) (*QRCodeSession, error) {
    sessionID := generateSessionID()

    session := &QRCodeSession{
        SessionID:   sessionID,
        AuthURL:     fmt.Sprintf("https://auth.example.com/qr/login/%s", sessionID),
        CallbackURL: fmt.Sprintf("com.example.app://qr-callback/%s", sessionID),
        CreatedAt:   time.Now(),
        ExpiresAt:   time.Now().Add(10 * time.Minute),
        Status:      "pending",
    }

    s.sessions[sessionID] = session
    return session, nil
}

// 获取会话状态（用于轮询）
func (s *QRCodeService) GetSessionStatus(sessionID string) (*QRCodeSession, error) {
    session, ok := s.sessions[sessionID]
    if !ok {
        return nil, fmt.Errorf("session not found")
    }

    if time.Now().After(session.ExpiresAt) {
        session.Status = "expired"
    }

    return session, nil
}

// 完成授权
func (s *QRCodeService) CompleteAuthorization(sessionID string, tokenInfo *oauth2.TokenInfo) error {
    session, ok := s.sessions[sessionID]
    if !ok {
        return fmt.Errorf("session not found")
    }

    session.TokenInfo = tokenInfo
    session.Status = "authorized"
    return nil
}

// 处理移动端回调
func (s *QRCodeService) HandleMobileCallback(w http.ResponseWriter, r *http.Request) {
    sessionID := r.URL.Query().Get("session_id")
    if sessionID == "" {
        http.Error(w, "missing session_id", http.StatusBadRequest)
        return
    }

    // 获取当前用户信息并完成授权
    tokenInfo := &oauth2.TokenInfo{
        // ... 从授权服务器获取的 token
    }

    if err := s.CompleteAuthorization(sessionID, tokenInfo); err != nil {
        http.Error(w, err.Error(), http.StatusInternalServerError)
        return
    }

    // 返回成功页面
    w.Header().Set("Content-Type", "text/html")
    w.Write([]byte("<html><body><h1>Authorization Successful</h1><p>You can close this page.</p></body></html>"))
}

func generateSessionID() string {
    b := make([]byte, 16)
    rand.Read(b)
    return base64.URLEncoding.EncodeToString(b)
}

    桌面客户端实现：

    package desktop

import (
    "context"
    "fmt"
    "time"

    "yourapp/auth"
)

func (c *DesktopClient) QRLogin() (*oauth2.TokenInfo, error) {
    // 1. 创建二维码会话
    session, err := c.qrService.CreateSession(c.clientID, c.redirectURI)
    if err != nil {
        return nil, fmt.Errorf("create session: %w", err)
    }

    // 2. 生成并显示二维码
    qrDataURL, err := auth.GenerateQRCodeBase64(session.AuthURL)
    if err != nil {
        return nil, fmt.Errorf("generate QR code: %w", err)
    }
    c.showQRCode(qrDataURL)

    // 3. 轮询等待授权完成
    ctx, cancel := context.WithTimeout(context.Background(), 10*time.Minute)
    defer cancel()

    for {
        select {
        case <-ctx.Done():
            return nil, fmt.Errorf("login timeout")
        case <-time.After(2 * time.Second):
            status, err := c.qrService.GetSessionStatus(session.SessionID)
            if err != nil {
                continue
            }

            switch status.Status {
            case "authorized":
                return status.TokenInfo, nil
            case "expired":
                return nil, fmt.Errorf("session expired")
            }
        }
    }
}

    最佳实践¶

    1. 用户体验¶

    • 清晰的视觉反馈：显示加载状态、倒计时、授权结果
    • 多语言支持：根据用户偏好显示界面语言
    • 错误处理：网络波动时提供重试选项
    • 超时提示：清晰告知二维码有效期

    2. 安全性¶

    • 二维码有效期：限制在 5-10 分钟内
    • 一次性使用：扫码后使二维码失效
    • 设备绑定：检测扫码设备与授权设备的关系
    • 异常监控：记录和监控异常授权行为
    • PKCE 增强：结合 PKCE 防止 CSRF

    3. 兼容性¶

    • 多种扫码工具：支持微信、支付宝、各类浏览器扫码
    • 降级方案：二维码失效时提供手动输入码
    • 响应式设计：确保在不同尺寸设备上可正常显示
