    1. 基础规则¶

    1.1 缩进、空格、换行¶

    • 【强制】2 空格缩进，不使用 Tab。
    • 【强制】每个声明独立一行，末尾分号必写。
    • 【强制】选择器与 { 之间保留空格。
    • 【强制】属性名与 : 之间不留空格，: 后一个空格。
    .card {
  margin: 0;
  padding: 12px;
}

    1.2 行宽与断行¶

    • 【推荐】单行不超过 120 字符。
    • 【推荐】长值在逗号或空格后断行。
    • 【推荐】多选择器一行一个。
    .dialog,
.drawer,
.popover {
  box-shadow:
    0 4px 12px rgb(0 0 0 / 12%),
    0 2px 4px rgb(0 0 0 / 8%);
}

    1.3 声明规范¶

    • 【强制】属性值中的字符串统一双引号。
    • 【推荐】颜色格式在同一项目统一（hex 或 rgb）。
    • 【推荐】0 值省略单位（0px → 0）。

    1.4 统一单位策略¶

    • 【推荐】字体与间距优先使用 rem。
    • 【推荐】边框可使用 px。
    • 【强制】同一属性族不要混用多种单位体系（如一组 spacing 既有 px 又有 rem）。

    2. 命名与组织¶

    2.1 类名命名¶

    • 【强制】类名使用 kebab-case。
    • 【强制】禁止拼音和无语义缩写。
    • 【推荐】采用 BEM 或等价命名体系。
    /* 推荐 */
.user-card {}
.user-card__avatar {}
.user-card--active {}

/* 不推荐 */
.UserCard {}
.yonghu {}
.a1 {}

    2.2 文件组织¶

    • 【强制】按业务域拆分样式，禁止单文件无限增长。
    • 【推荐】全局层仅放 reset、token、工具类。
    • 【推荐】组件样式与组件就近放置。

    2.3 工具类约束¶

    • 【推荐】工具类统一前缀（如 u-）。
    • 【强制】工具类只处理单一职责（如 u-text-center）。
    • 【强制】禁止工具类直接绑定业务语义。

    3. 选择器与优先级¶

    3.1 复杂度控制¶

    • 【强制】嵌套层级不超过 3 层（极限 4 层）。
    • 【强制】避免后代链式深层选择器。
    • 【推荐】避免标签 + 类选择器耦合（div.card）。
    /* 推荐 */
.comment-list__item {}

/* 不推荐 */
.page .content .main .list .item {}

    3.2 关系选择器¶

    • 【强制】>、+、~ 两侧留空格。
    • 【强制】属性选择器值使用双引号。
    • 【推荐】优先使用类选择器，降低重排风险与维护成本。

    3.3 优先级治理¶

    • 【强制】禁止通过叠加大量选择器提高优先级。
    • 【推荐】通过组件作用域、层级设计解决覆盖问题。
    • 【推荐】使用 :where() 降低默认优先级，便于被业务覆写。

    3.4 禁止项¶

    • 【强制】禁止 ID 选择器驱动样式（除遗留迁移场景）。
    • 【强制】禁止超过 4 层嵌套。
    • 【强制】禁止使用 !important 覆盖架构问题。

    4. 属性顺序与可读性¶

    建议按功能分组：

    1. 布局定位（display、position、inset）
    2. 盒模型（width/height、margin/padding、border）
    3. 排版（font、line-height、text-*）
    4. 视觉（background、color、shadow）
    5. 动画（transition、animation）
    .sidebar {
  position: sticky;
  top: 16px;

  width: 280px;
  padding: 12px;
  border: 1px solid var(--color-border);

  font-size: 14px;
  line-height: 1.5;

  background: var(--color-bg-card);
  color: var(--color-text-primary);

  transition: box-shadow 0.2s ease;
}

    5. 现代 CSS 实践¶

    5.1 变量与设计令牌¶

    • 【强制】颜色、字号、间距、圆角统一走 Token（CSS 变量）。
    • 【推荐】按语义命名变量，而不是视觉名。
    :root {
  --space-2: 8px;
  --radius-2: 8px;
  --color-primary: #2563eb;
  --color-text-primary: #111827;
}

    5.2 响应式设计¶

    • 【推荐】优先移动端优先（mobile first）。
    • 【推荐】优先 min-width 媒体查询。
    • 【推荐】断点统一在 token 中管理。

    推荐断点（示例）¶

    • --bp-sm: 576px
    • --bp-md: 768px
    • --bp-lg: 992px
    • --bp-xl: 1200px
    @media (min-width: 768px) {
  .layout {
    display: grid;
    grid-template-columns: 240px 1fr;
  }
}

    5.3 布局体系¶

    • 【推荐】一维布局优先 flex，二维布局优先 grid。
    • 【推荐】尽量使用 gap 替代子元素 margin 拼接。
    • 【推荐】避免使用浮动做主布局。

    5.4 逻辑属性（i18n）¶

    • 【推荐】使用逻辑属性支持多书写方向：
    • margin-inline、padding-block、inset-inline-start

    5.5 @layer 分层（推荐）¶

    • 【推荐】将样式按层组织：reset、base、components、utilities。
    • 【推荐】避免跨层反复覆盖。
    @layer reset, base, components, utilities;

@layer components {
  .btn {
    border-radius: var(--radius-2);
  }
}

    6. 动画与可访问性¶

    6.1 动效规则¶

    • 【推荐】过渡时长统一在 150ms~300ms。
    • 【推荐】优先动画 transform 和 opacity。
    • 【强制】避免触发高代价布局属性动画（如频繁改 top/left/width）。

    6.2 减少动态偏好¶

    • 【推荐】支持 prefers-reduced-motion。
    @media (prefers-reduced-motion: reduce) {
  * {
    animation: none !important;
    transition: none !important;
  }
}

    7. 性能规则¶

    • 【强制】避免无边界 * 选择器与深层后代选择器。
    • 【推荐】避免无意义阴影和滤镜叠加。
    • 【推荐】审慎使用 will-change，并在动画结束后移除。
    • 【推荐】大体积样式按页面或模块拆分。

    7.1 浏览器渲染性能细则¶

    • 【推荐】避免高频使用 filter: blur(...)。
    • 【推荐】避免大范围 box-shadow 动态变化。
    • 【推荐】谨慎使用 position: fixed 与复杂重绘场景叠加。

    8. !important 与 z-index¶

    8.1 !important¶

    • 【强制】默认禁止 !important。
    • 【例外】第三方样式覆盖、紧急线上修复可临时使用并标注原因。

    8.2 z-index 分层¶

    • 【推荐】统一维护分层表，禁止魔法值乱写。
    • 【推荐】建议区间（示例）：
    • 普通内容：1-10
    • 吸顶元素：100-199
    • 浮层：1000-1999
    • 模态：2000-2999
    • 全局提示：3000+

    8.3 覆盖冲突处理顺序¶

    当样式冲突时，处理顺序必须为：

    1. 调整结构层级
    2. 调整样式分层
    3. 调整选择器策略
    4. 最后才考虑临时 !important

    9. Sass / SCSS 规范¶

    9.1 变量与 Map¶

    • 【强制】变量使用 kebab-case。
    • 【推荐】变量按语义分组管理（color/spacing/typography）。
    • 【推荐】复杂主题参数优先 map。
    $color-primary: #2563eb;
$spacing: (
  xs: 4px,
  sm: 8px,
  md: 12px,
  lg: 16px
);

    9.2 嵌套与 &¶

    • 【强制】嵌套层级不超过 3 层。
    • 【推荐】& 只用于状态、修饰符、伪类。
    .button {
  &--primary {
    background: $color-primary;
  }

  &:hover {
    filter: brightness(1.05);
  }
}

    9.3 抽象复用¶

    • 【推荐】重复声明提取为 @mixin。
    • 【推荐】纯计算提取为 @function。
    • 【强制】禁止在 mixin 中隐式篡改全局状态。

    9.4 禁止项¶

    • 【强制】禁止在循环中生成海量无业务意义类。
    • 【强制】禁止深度嵌套拼接导致难读选择器。
    • 【推荐】避免为每个细粒度值都生成 utility（造成 CSS 体积膨胀）。

    10. stylelint 配置模板（示例）¶

    {
  "extends": [
    "stylelint-config-standard",
    "stylelint-config-standard-scss"
  ],
  "rules": {
    "selector-max-id": 0,
    "max-nesting-depth": 3,
    "declaration-no-important": true,
    "color-hex-length": "short"
  }
}

    11. 样式审查清单¶

    • 是否遵守命名和层级规则？
    • 是否存在选择器复杂度过高问题？
    • 是否引入了无必要 !important？
    • 是否所有颜色和间距都走 token？
    • 是否影响性能（阴影、滤镜、动画）？
    • 是否存在层级覆盖设计缺陷？
    • 是否符合断点与响应式策略？

    12. 反例速查¶

    • 深层链式选择器。
    • 组件内硬编码主题颜色。
    • 通过 !important 打补丁。
    • 过度嵌套 SCSS。
    • z-index 魔法数横飞。
    • 工具类与业务类混写。
    • 同模块混用多种 spacing 单位。
