    协议使用与分析¶

    一、web开发¶

    1.http/sse/ws¶

    1.1. 基础特性梳理¶

    技术类型 连接模型 通信方向 核心协议开销 实时性（端到端延时） 高并发支撑能力 开发 / 运维成本 兼容性
    普通 HTTP（轮询 / 长轮询） 短连接（轮询）/ 伪长连接（长轮询） 单向（前端→后端） 高：每次请求 TCP 握手 + HTTP 头（几十～几百字节） 低（轮询≥500ms，长轮询≥100ms） 差（短连接数爆炸，端口 / CPU 耗尽） 极低 全兼容
    SSE（Server-Sent Events） 长连接（HTTP/1.1+）+ 普通 HTTP 上报 半双向（后端→前端推送，前端→后端 HTTP 上报） 中：SSE 长连接仅 1 次握手，上报用 HTTP/2 可复用连接 中高（推送≤10ms，上报≤50ms） 好（长连接数 = 客户端数，HTTP/2 可多路复用） 中 好（除 IE，占比 < 5%）
    WebSocket 全双工长连接（TCP 升级） 全双向 低：1 次握手，帧协议仅几字节开销 高（双向≤10ms） 优（需优化，支撑 10 万 + 连接） 高 较好（需适配网关 / 代理）

    1.2. 场景适配性深度分析¶

    先明确场景的核心技术诉求，这是选型的根本依据，假设为上传大文件的分片上传场景：

    核心诉求 具体指标要求
    实时性（低延时） 前端上报进度 → 后端计算 → 前端接收进度，整体延时≤100ms
    高并发支撑 服务端能稳定支撑≥1000 个客户端同时上传（分片并发上传 + 进度交互）
    性能优化 连接 / 消息开销最小化，服务端 CPU / 内存 / 网络占用可控
    开发 / 运维成本 协议复杂度低、兼容性好、网关 / 代理适配成本低
    功能适配 支持 “前端主动上报进度”+“后端主动推送计算结果” 双向交互
    （1）普通 HTTP（轮询 / 长轮询）—— 直接排除¶
    • 核心问题：
    • 轮询：前端定时（如 1s）发请求，高并发下（1000 客户端）每秒产生 1000 次请求，服务端 TCP 握手 / HTTP 头解析消耗大量 CPU / 内存，端口耗尽风险高；
    • 长轮询：后端挂起连接直到有进度更新，但本质仍是 “请求 - 响应” 模型，重连仍有开销，且后端无法主动推进度（需等前端发起请求）；
    • 实时性差：轮询间隔决定延时，无法满足 “低延时” 要求。
    • 仅适用场景：极低并发（<100 客户端）、老旧系统（IE）兜底，无实时性要求。
    （2）SSE —— 通用最优解（90% 场景）¶
    • 核心优势：
    • 性能平衡：长连接复用降低开销，HTTP/2 下前端上报可复用 SSE 的 TCP 连接，整体开销仅为 WebSocket 的 1.2 倍，但开发成本低 50%；
    • 开发简单：前端用EventSource原生支持，内置重连机制，无需手动处理心跳；后端仅需返回text/event-stream格式响应；
    • 运维成本低：基于 HTTP 协议，网关（Nginx）仅需配置长连接超时，无需特殊适配；
    • 核心短板：
    • 半双向：前端上报仍需 HTTP 请求，但分片上传场景中 “上报粒度为分片级”（而非字节级），消息量少，该短板可忽略；
    • 仅支持文本传输：需序列化 JSON，但分片进度数据量小（<1KB），无性能影响。
    （3）WebSocket —— 极致性能解（10% 场景）¶
    • 核心优势：
    • 全双工低开销：一次握手后双向传输，帧协议仅几字节，延时比 SSE 低 30%~50%；
    • 极致实时性：毫秒级双向交互，适合超大规模并发（10 万 + 客户端）、低延时要求（如云存储平台）；
    • 核心短板：
    • 开发复杂度高：需手动实现重连、心跳、消息重发、异常处理（如连接断开检测）；
    • 运维成本高：网关（Nginx）需配置Upgrade/Connection头，老旧代理可能不支持；高并发下需调整系统参数（文件描述符、epoll）；
    • 兼容性风险：部分企业内网代理可能拦截 WebSocket 连接。

    🚀 推荐方案：基于WebSocket的实现架构¶

    1. 整体架构设计¶
    客户端                                           后端服务                                      对象存储
   |                                                |                                            |
   |--- 1. 创建上传任务 (HTTP POST) --------------->|                                            |
   |<-- 2. 返回预签名URLs和任务ID ------------------|                                            |
   |                                                |                                            |
   |--- 3. 建立WebSocket连接 (任务ID认证) --------->|                                            |
   |                                                |                                            |
   |--- 4. 并发直传分片到对象存储 -------------------|------------------------------------------>|
   |     (每个分片进度变化时)                         |                                            |
   |--- 5. 通过WebSocket上报分片进度 -------------->|                                            |
   |                                                |-- 6. 更新任务状态，计算整体进度 ----------|
   |<-- 7. 实时推送整体进度 ------------------------|                                            |
   |                                                |                                            |
   |--- 8. 所有分片完成，通知合并 ------------------>|                                            |
   |                                                |--- 9. 调用对象存储合并API --------------->|
   |<-- 10. 返回最终结果 ---------------------------|                                            |
    2. 核心数据结构¶
    // 前端上报的单分片进度
type ChunkProgress struct {
    TaskID     string  `json:"taskId"`     // 任务唯一ID
    ChunkIndex int     `json:"chunkIndex"` // 分片索引
    Status     string  `json:"status"`     // uploading/completed/failed
    Loaded     int64   `json:"loaded"`     // 已上传字节
    Total      int64   `json:"total"`      // 分片总字节
    Timestamp  int64   `json:"timestamp"`  // 上报时间戳
}

// 后端返回的整体进度
type OverallProgress struct {
    TaskID         string   `json:"taskId"`
    TotalChunks    int      `json:"totalChunks"`    // 总分片数
    Completed      int      `json:"completed"`      // 已完成数
    Failed         int      `json:"failed"`         // 失败数
    Progress       float64  `json:"progress"`       // 整体进度百分比
    Status         string   `json:"status"`         // uploading/completed/merging
    Speed          float64  `json:"speed"`          // 上传速度(KB/s)
    EstimatedTime  int64    `json:"estimatedTime"`  // 预计剩余时间(秒)
}

    3.websocket处理器

    // internal/handler/websockethandler.go
package handler

import (
    "encoding/json"
    "log"
    "net/http"
    "sync"
    "time"

    "github.com/gorilla/websocket"
    "github.com/zeromicro/go-zero/core/logx"
)

var upgrader = websocket.Upgrader{
    ReadBufferSize:  1024,
    WriteBufferSize: 1024,
    CheckOrigin: func(r *http.Request) bool {
        // 生产环境应严格验证来源
        return true
    },
}

// UploadTaskManager 管理所有上传任务
type UploadTaskManager struct {
    tasks sync.Map // taskID -> *UploadTask
}

type UploadTask struct {
    TaskID      string
    TotalChunks int
    Chunks      []*ChunkStatus
    StartTime   time.Time
    ClientConn  *websocket.Conn
    mu          sync.RWMutex
}

type ChunkStatus struct {
    Index    int
    Status   string // pending, uploading, completed, failed
    Loaded   int64
    Total    int64
    LastUpdate time.Time
}

func (h *UploadTaskManager) HandleWebSocket(w http.ResponseWriter, r *http.Request) {
    // 1. 升级为WebSocket连接
    conn, err := upgrader.Upgrade(w, r, nil)
    if err != nil {
        logx.Errorf("WebSocket升级失败: %v", err)
        return
    }
    defer conn.Close()

    // 2. 等待客户端发送认证信息（包含taskId）
    _, authMsg, err := conn.ReadMessage()
    if err != nil {
        logx.Errorf("读取认证信息失败: %v", err)
        return
    }

    var auth struct {
        TaskID string `json:"taskId"`
        Token  string `json:"token"`
    }
    if err := json.Unmarshal(authMsg, &auth); err != nil {
        conn.WriteJSON(map[string]string{"error": "认证信息格式错误"})
        return
    }

    // 3. 验证任务ID和Token（简化示例）
    task, ok := h.tasks.Load(auth.TaskID)
    if !ok {
        conn.WriteJSON(map[string]string{"error": "任务不存在"})
        return
    }

    uploadTask := task.(*UploadTask)
    uploadTask.ClientConn = conn

    // 4. 持续处理客户端消息
    for {
        _, message, err := conn.ReadMessage()
        if err != nil {
            logx.Infof("连接断开: %v", err)
            break
        }

        // 处理进度上报
        var chunkProgress ChunkProgress
        if err := json.Unmarshal(message, &chunkProgress); err != nil {
            logx.Errorf("解析进度消息失败: %v", err)
            continue
        }

        // 更新任务状态
        overallProgress := h.updateTaskProgress(uploadTask, chunkProgress)

        // 推送整体进度给客户端
        if err := conn.WriteJSON(overallProgress); err != nil {
            logx.Errorf("推送进度失败: %v", err)
            break
        }
    }
}

func (h *UploadTaskManager) updateTaskProgress(task *UploadTask, chunk ChunkProgress) OverallProgress {
    task.mu.Lock()
    defer task.mu.Unlock()

    // 更新分片状态
    if chunk.ChunkIndex < len(task.Chunks) {
        task.Chunks[chunk.ChunkIndex].Status = chunk.Status
        task.Chunks[chunk.ChunkIndex].Loaded = chunk.Loaded
        task.Chunks[chunk.ChunkIndex].Total = chunk.Total
        task.Chunks[chunk.ChunkIndex].LastUpdate = time.Now()
    }

    // 计算整体进度
    completed := 0
    failed := 0
    totalLoaded := int64(0)
    totalSize := int64(0)

    for _, chunk := range task.Chunks {
        totalSize += chunk.Total
        if chunk.Status == "completed" {
            completed++
            totalLoaded += chunk.Total
        } else if chunk.Status == "failed" {
            failed++
        } else {
            totalLoaded += chunk.Loaded
        }
    }

    progress := 0.0
    if totalSize > 0 {
        progress = float64(totalLoaded) / float64(totalSize) * 100
    }

    // 计算上传速度
    elapsed := time.Since(task.StartTime).Seconds()
    speed := 0.0
    if elapsed > 0 {
        speed = float64(totalLoaded) / 1024 / elapsed // KB/s
    }

    return OverallProgress{
        TaskID:        task.TaskID,
        TotalChunks:   task.TotalChunks,
        Completed:     completed,
        Failed:        failed,
        Progress:      progress,
        Status:        "uploading",
        Speed:         speed,
        EstimatedTime: int64((totalSize - totalLoaded) / 1024 / int64(speed+0.1)),
    }
}
