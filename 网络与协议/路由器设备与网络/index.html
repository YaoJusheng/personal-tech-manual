    1.简介¶

    1.1.路由¶

    • 路由的定义 ：路由是指导数据包在网络中转发的路径信息。它就像是公路交通中的导航系统，告诉数据包从源节点到目的节点应该经过哪些中间节点。在网络中，路由器是负责执行路由功能的主要设备。路由器工作在网络层，它通过分析数据包的头部信息（如目的 IP 地址）来决定数据包的转发方向。

    • 路由表的作用 ：路由器中有一个关键的数据结构 —— 路由表。路由表包含了网络拓扑信息，如目的网络地址、下一跳路由器的 IP 地址、出口接口等。当路由器收到一个数据包时，它会查找路由表，根据目的 IP 地址所在的网络段来确定应该将数据包转发到哪个下一跳路由器或者直接连接的网络接口。例如，假设路由表中有这样一条记录：目的网络为 192.168.1.0/24，下一跳路由器为 10.0.0.1，出口接口为 eth0。当路由器收到一个目的 IP 地址为 192.168.1.10 的数据包时，就会将数据包从 eth0 接口转发给下一跳路由器 10.0.0.1。

    • 静态路由和动态路由 ：

    • 静态路由 ：是由网络管理员手动配置的路由信息。它的优点是配置简单，适合于小型、拓扑结构简单的网络。例如，在一个企业内部的小型网络中，只有几个子网，网络管理员可以预先配置好各个子网之间的路由关系。但静态路由的缺点也很明显，当网络拓扑发生变化（如链路中断、新增设备等）时，需要手动更新路由表，这在大型复杂网络中是非常繁琐的工作。

    • 动态路由 ：是通过动态路由协议（如 RIP、OSPF、BGP 等）自动学习和维护路由信息。动态路由协议可以在网络拓扑发生变化时自动调整路由表。以 OSPF（开放最短路径优先）协议为例，它是一种基于链路状态的动态路由协议。每个路由器会广播自己的链路状态信息（如连接的网络、链路的带宽、链路是否正常等），其他路由器收到这些信息后，根据 Dijkstra 算法计算出到各个网络的最短路径，从而实现动态更新路由表。这使得网络能够快速适应拓扑变化，保证数据包的高效传输。

    • 区别：

      • 配置方式

      • 静态路由 ：配置相对简单，管理员只需在路由器上指定目的网络地址、子网掩码、下一跳路由器的 IP 地址或出口接口等信息即可。

      • 动态路由 ：配置相对复杂，需要在路由器上启用相应的动态路由协议，并进行一定的参数配置，如指定网络范围、认证信息等。

      • 维护难度
      • 静态路由 ：在小型、拓扑结构简单的网络中，维护相对容易。但如果网络拓扑发生变化，如链路中断、新增设备等，需要手动更新路由表，维护成本较高。
      • 动态路由 ：能够自动适应网络拓扑的变化，当网络发生改变时，路由器之间会自动交换路由信息并更新路由表，大大降低了维护难度。
      • 适应能力

      • 静态路由 ：适应能力较差，不能自动感知网络的变化。如果网络拓扑发生变化而管理员没有及时更新静态路由，就可能导致数据包无法正确转发。

      • 动态路由 ：具有较强的适应能力，能够快速响应网络拓扑的变化，保证数据包的正确转发。

      • 安全性

      • 静态路由 ：相对安全，因为所有的路由信息都是由管理员手动配置的，不存在路由信息泄露或被篡改的风险。
      • 动态路由 ：存在一定的安全隐患，如路由协议的认证机制不完善可能导致路由信息泄露或被篡改，从而影响网络的安全性。
      • 性能开销
      • 静态路由 ：由于不需要进行路由信息的自动学习和更新，路由器的性能开销较小。

      • 动态路由 ：路由器需要运行动态路由协议，不断接收、处理和发送路由信息，这会占用一定的 CPU 和内存资源，增加路由器的性能开销。

      • 适用场景

      • 静态路由 ：适用于小型、拓扑结构简单且相对稳定的网络，如企业内部的小型局域网。
      • 动态路由 ：适用于大型、复杂的网络，尤其是那些网络拓扑经常发生变化的网络，如互联网服务提供商的网络、大型企业的广域网等。

    1.2.网络¶

    • 网络的定义 ：网络是由若干节点和连接这些节点的链路构成的，它是一种通信系统，用于在不同的设备之间传输数据。这些设备可以是计算机、服务器、打印机、路由器等各种硬件设备。网络可以按照不同的方式进行分类，如按地理范围分为局域网（LAN）、城域网（MAN）和广域网（WAN）。局域网通常覆盖一个有限的区域，如一个办公室或学校，它具有高数据传输速率和低误码率的特点。广域网则覆盖范围很广，可以是一个国家甚至全球，像互联网就是全球最大、最典型的广域网。
    • 网络协议的重要性 ：网络协议是为进行网络中的数据交换而建立的规则、标准或约定。它规定了数据传输的格式、顺序、速度和错误纠正方法等。例如，TCP/IP 协议是互联网的基础协议。TCP（传输控制协议）主要负责将数据分成数据包，保证数据的可靠传输，包括数据包的顺序、重复数据包的处理等。IP（网际协议）则负责将数据包从源地址传输到目的地址，它根据 IP 地址来确定数据包的转发路径。没有这些协议，网络中的设备就像没有交通规则的车辆，无法有序地通信。

    1.3.网络与路由在实际应用中的关系¶

    • 企业网络中的应用 ：在企业网络中，通常会构建多个子网用于隔离不同的部门（如财务部、市场部、研发部等）。通过路由器和合适的路由协议（如静态路由或 OSPF 等），可以实现这些子网之间的通信。同时，企业网络还需要与互联网连接，这时路由器会配置到互联网服务提供商（ISP）的路由信息，使得企业内部设备能够访问外部网络资源，如访问网站、收发电子邮件等。
    • 互联网服务提供商的应用 ：ISP 通过复杂的路由网络将不同的用户网络和内容提供商网络连接起来。他们会使用高级的动态路由协议，如 BGP（边界网关协议），来在不同的自治系统（AS）之间交换路由信息。BGP 通过传递网络可达性信息和路由属性，帮助 ISP 选择最佳的路径来转发数据，从而优化网络流量，提高网络的稳定性和可靠性。这对于全球互联网的正常运行至关重要，因为互联网是由无数个 ISP 网络相互连接组成的庞大网络。

    2.配置使用¶

    2.1.查看网络¶

    1.查看IP/网卡/连通性¶

    • [Windows 系统 ]：
    • ipconfig 命令 ：在命令提示符中输入 ipconfig，可查看网络接口的 IP 地址、子网掩码、默认网关等基本信息。如 ipconfig /all 会显示更详细的网络配置，包括 DNS 服务器地址、网卡的物理地址等。
    • ping 命令 ：用于检测网络连通性。如 ping www.baidu.com，可测试本地主机与目标主机之间的网络连接是否正常，通过发送 ICMP 回显请求消息，查看是否有回应以及回应的时间等。
    • tracert 命令 ：用于跟踪数据包的传输路径。如 tracert www.baidu.com，能显示数据包从本地主机到目标主机所经过的路由器等网络设备的 IP 地址及每一跳的传输时间，有助于了解网络的拓扑结构和路由情况。
    • [Linux 系统] ：
    • ifconfig 命令 ：在终端中输入 ifconfig，可查看网络接口的详细信息，包括 IP 地址、子网掩码、广播地址、网卡的运行状态等。若要查看指定网卡的信息，可在命令后加上网卡名称，如 ifconfig eth0。
    • ping 命令 ：与 Windows 系统类似，用于检测网络连通性，语法和功能基本相同。
    • traceroute 命令 ：功能与 Windows 系统的 tracert 命令相似，用于跟踪数据包的路由路径，语法为 traceroute www.example.com。

    2.查看网关¶

    1. 使用 ip route 命令：
    ip route show

    或者只查看默认网关：

    ip route | grep default
    1. 使用 route 命令（如果已安装）：
    route -n

    3.查看 DNS 信息¶

    1. 查看 /etc/resolv.conf 文件：
    cat /etc/resolv.conf

    这个文件包含了 DNS 服务器的地址，通常以 nameserver 开头。

    1. 使用 nmcli 命令（如果使用 NetworkManager）：
    nmcli dev show | grep DNS
    1. 使用 systemd-resolve 或 resolvectl 命令（如果使用 systemd-resolved）：
    systemd-resolve --status

    或者在较新版本中：

    resolvectl status

    2.2.查看路由¶

    • Windows 系统 ：在命令提示符中输入 route print 命令，可查看本地计算机的路由表信息，包括网络目标、子网掩码、网关、接口等列，显示了数据包发往不同网络时所经过的路由。

    • Linux 系统 ：可以使用 ip route 或 netstat -r 命令来查看路由表。ip route 命令的输出会以一种更直观的方式显示路由信息，如目标网络、子网掩码、下一跳地址、出接口等。netstat -r 命令则会显示内核路由表的详细内容，包括目的网络、网关、子网掩码、接口等信息。

    • 查看： 执行 ip route show， 输出：

    default via 192.168.2.1 dev wlo1 proto dhcp metric 20600
169.254.0.0/16 dev wlo1 scope link metric 1000
172.17.0.0/16 dev docker0 proto kernel scope link src 172.17.0.1 linkdown
192.168.2.0/24 dev wlo1 proto kernel scope link src 192.168.2.72 metric 600
198.18.0.0/30 dev Meta proto kernel scope link src 198.18.0.1

    解释输出：

    • default：表示这是默认路由。
    • via：指定下一跳的网关地址。
    • dev：指定数据包发送时使用的网络接口（例如 wlo1 是无线接口）。

    1. route命令语法¶

    route [-n] [-e] [add|del] [ -net | -host ] 目标网络 [mask 子网掩码] [gw 网关] [dev 接口]
    • 常用选项：
    • -n：以数字形式显示 IP 地址（不解析主机名，输出更清晰）。
    • -e：以 详细格式（BSD 风格） 显示路由表。
    • add/del：添加或删除路由条目。
    • -net：指定目标为网络地址（默认）。
    • -host：指定目标为单播主机地址。

    2. 查看路由表输出详解¶

    执行 route -n 输出示例：

    $ route
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
default         routerlogin.cn  0.0.0.0         UG    20600  0        0 wlo1
link-local      0.0.0.0         255.255.0.0     U     1000   0        0 wlo1
172.17.0.0      0.0.0.0         255.255.0.0     U     0      0        0 docker0
192.168.2.0     0.0.0.0         255.255.255.0   U     600    0        0 wlo1
198.18.0.0      0.0.0.0         255.255.255.252 U     0      0        0 Meta

$ route -n
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.2.1     0.0.0.0         UG    20600  0        0 wlo1
169.254.0.0     0.0.0.0         255.255.0.0     U     1000   0        0 wlo1
172.17.0.0      0.0.0.0         255.255.0.0     U     0      0        0 docker0
192.168.2.0     0.0.0.0         255.255.255.0   U     600    0        0 wlo1
198.18.0.0      0.0.0.0         255.255.255.252 U     0      0        0 Meta

    各字段含义：

    字段 说明
    Destination 目标网络或主机地址（0.0.0.0 表示默认路由）。
    Gateway 下一跳网关地址（0.0.0.0 表示直接连接，无需经过网关）。
    Genmask 子网掩码（255.255.255.0 表示 /24 网络）。
    Flags 路由标志（常用标志： U= 启用，G= 使用网关，H= 目标为单播主机）。
    Metric 路由优先级（跃点数），数值越小优先级越高（与 ifmetric 关联）。
    Ref 引用计数（当前未使用，恒为 0）。
    Use 该路由的数据包统计数（累计通过该路由的流量）。
    Iface 网络接口名称（如 eth0、wlan0）。

    3. 常用操作示例¶

    （1）添加静态路由¶
    • 添加到目标网络的路由：
    sudo route add -net 192.168.2.0/24 gw 192.168.1.254 dev eth0

    • 目标网络：192.168.2.0/24，通过网关 192.168.1.254，经接口 eth0 转发。

    • 添加默认路由（网关）：

    sudo route add default gw 192.168.1.1 dev eth0
    （2）删除路由¶
    sudo route del -net 192.168.2.0/24 gw 192.168.1.254 dev eth0
    （3）设置路由 Metric¶
    sudo route add -net 10.0.0.0/8 gw 10.8.0.1 dev tun0 metric 50
    • 通过 metric 参数指定路由优先级（数值越小越优先）。

    4. 永久生效配置（非临时）¶

    Linux 中 route 命令的修改为临时配置，重启后失效。永久配置需修改网络配置文件：

    • Ubuntu/Debian（netplan）： 在 /etc/netplan/*.yaml 中添加 routes 字段：
    network:
  version: 2
  ethernets:
    eth0:
      addresses: [192.168.1.10/24]
      routes:
        - to: 192.168.2.0/24
          via: 192.168.1.254
          metric: 100  # 可选优先级
      nameservers:
        addresses: [8.8.8.8]

    应用配置：sudo netplan apply。

    • CentOS/RHEL（ifcfg-ethX）： 在 /etc/sysconfig/network-scripts/ifcfg-eth0 中添加：
    IPROUTE2_RT_0="dst 192.168.2.0/24 via 192.168.1.254 dev eth0 table main metric 100"

    重启网络服务：sudo systemctl restart network。

    2.3.设置路由网络优先级¶

    当设备同时连接有线、Wi-Fi、4G/5G时，需手动设置优先级。

    1. Windows系统¶

    • 方法1：修改跃点数

    • 打开“网络连接” (ncpa.cpl) → 右击网卡 → “属性” → “IPv4协议” → “高级”。

    • 取消“自动跃点” → 输入跃点数（值越小优先级越高）。

      • 有线网：建议设为10
      • 无线网：建议设为20
      • 4G/5G：设为308。

    • 方法2：PowerShell命令

    # 查看网卡索引号
Get-NetIPInterface
# 设置跃点数（例：将索引10的网卡设为优先级最高）
Set-NetIPInterface -InterfaceIndex 10 -InterfaceMetric 15:cite[8]

# 示例：添加到 192.168.2.0/24 的路由，跃点数 10，通过接口 10（需以管理员身份运行）
route add 192.168.2.0 mask 255.255.255.0 192.168.1.2 metric 10 if 10 # 添加路由，metric值越小优先级越高

route change 目标网络 mask 子网掩码 网关地址 metric 新跃点数
# 示例：将到 192.168.2.0/24 的路由跃点数改为 5
route change 192.168.2.0 mask 255.255.255.0 192.168.1.254 metric 5  # 修改现有路由的metric
# 添加 -p 参数, 永久生效（重启后保留）
route -p add 192.168.2.0 mask 255.255.255.0 192.168.1.254 metric 10

# 验证
route print

    2. Linux系统¶

    • 修改网关metric值：

    • 删除默认网关：route del default gw 192.168.1.1。

    • 重新设置优先级：

      • 传统方式
      # 添加带 Metric 的路由
sudo route add -net 目标网络 netmask 子网掩码 gw 网关 dev 接口名 metric 数值
# e.g.
route add default gw 192.168.1.1 metric 10    # 高优先级
route add default gw 192.168.2.1 metric 20    # 低优先级:cite[2]
# 示例：添加到 192.168.3.0/24 的路由，Metric 为 20
sudo route add -net 192.168.3.0 netmask 255.255.255.0 gw 192.168.1.254 dev eth0 metric 20
      • ip route设置
      ip route del <目标网络>/<子网掩码>

# 添加带 Metric 的路由
sudo ip route add 目标网络/掩码 via 网关 dev 接口名 metric 数值
# 示例：添加到 10.0.0.0/8 的路由，Metric 为 50，经 eth0 接口
sudo ip route add 10.0.0.0/8 via 10.8.0.1 dev eth0 metric 50

# 修改现有路由的 Metric
sudo ip route change 目标网络/掩码 via 网关 dev 接口名 metric 新数值

# e.g.
sudo ip route add 192.168.2.0/24 via 192.168.1.2 metric 1
sudo ip route add default via 192.168.1.1 dev eth0 metric 50
sudo ip route change 192.168.2.0 mask 255.255.255.0 192.168.1.2 metric 5  # 修改现有路由的metric

    • 重新加载

      systemctl restart network | networking

    • 使用 Netplan

    可以在 /etc/netplan/ 目录下的 YAML 配置文件中添加路由配置。例如，编辑 /etc/netplan/50-cloud-init.yaml 文件：

    network:
  version: 2
  ethernets:
    eth0:
      dhcp4: no
      addresses:
        - 192.168.1.100/24
      gateway4: 192.168.1.1
      routes:
        - to: 192.168.100.0/24
          via: 192.168.1.1
          metric: 100

    然后应用配置：

    sudo netplan apply

    3. MacOS系统¶

    • 调整 Wi-Fi 网络优先级：

    • 系统偏好设置 → 网络 → 选择 Wi-Fi → 高级 → Wi-Fi。

    • 在 首选网络 列表中拖动网络名称调整顺序，越靠前优先级越高1415。

    • 手动设置路由 metric（需管理员权限）：

    sudo route add -net 192.168.2.0/24 192.168.1.2 -metric 10

    • 永久生效： macOS 可通过 启动脚本 或 NetworkManager 配置 实现永久生效（需手动创建脚本）：

    • 创建脚本/Library/LaunchDaemons/com.user.route.plist（需管理员权限）：

    <?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>com.user.route</string>
  <key>ProgramArguments</key>
  <array>
    <string>/sbin/route</string>
    <string>add</string>
    <string>-net</string>
    <string>192.168.2.0/24</string>
    <string>192.168.1.254</string>
    <string>-metric</string>
    <string>10</string>
  </array>
  <key>RunAtLoad</key>
  <true/>
</dict>
</plist>
    1. 加载脚本：
    sudo launchctl load /Library/LaunchDaemons/com.user.route.plist
    1. 查看路由表验证
    netstat -rn  # 显示路由表（-n 不解析域名，-r 显示路由）

    4.优先级配置原则¶

    • 有线 > Wi-Fi > 蜂窝网络（默认优先级）。
    • 同类型网络按链路速度自动分配跃点（速度越快优先级越高）。

    2.4.配置静态 IP¶

    1. Windows系统¶

    • 步骤：

    • 打开“网络和共享中心” → “更改适配器设置”。

    • 右击当前网络（如以太网）→ “属性” → 双击“Internet协议版本 (TCP/IPv4)”。
    • 选择“使用下面的IP地址”，输入：
      • IP地址：需在路由器网段内（如路由器IP为192.168.1.1，则设备IP设为192.168.1.100）。
      • 子网掩码：通常为255.255.255.0。
      • 默认网关：路由器IP（如192.168.1.1）。
      • DNS服务器：可填路由器IP或公共DNS（如8.8.8.8）。

    • 关闭备用配置：防止自动切换回动态IP。

    • 命令行配置：

    netsh interface ipv4 set address name="以太网" static 192.168.1.100 255.255.255.0 192.168.1.1
netsh interface ipv4 set dns name="以太网" static 8.8.8.8 primary

    2. Linux系统¶

    • 配置方法：

    • 编辑网络配置文件（如/etc/network/interfaces （Ubuntu/Debian）或/etc/sysconfig/network-scripts/ifcfg-ens33、/etc/sysconfig/network-scripts/ifcfg-eth0 （CentOS/RHEL ））。

    • 修改以下参数：

      # centos
BOOTPROTO=static
IPADDR=192.168.1.100
NETMASK=255.255.255.0
GATEWAY=192.168.1.1
DNS1=8.8.8.8

# ubuntu
dhcp=static
address=192.168.1.100
netmask=255.255.255.0
gateway=192.168.1.1
dns1=8.8.8.8

    • 重启网络服务：systemctl restart network。

    • Netplan 配置（Ubuntu 20.04+）： 编辑 /etc/netplan/00-installer-config.yaml，示例：

    network:
  version: 2
  renderer: networkd
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.1.100/24]
      routes:
        - to: default
          via: 192.168.1.1
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114]

    应用配置：

    sudo netplan apply

    3.通用方法：通过 nmcli 命令设置（推荐）¶

    nmcli 是跨发行版的 NetworkManager 命令行工具，适合快速设置静态 IP（无需编辑文件）。

    nmcli 步骤：

    • 步骤 1：查看当前连接
    nmcli connection show  # 列出所有网络连接（找到你的网口连接名，如 "Wired connection 1"）
    • 步骤 2：修改连接为静态 IP
    # 命令格式：
# nmcli connection modify "连接名" ipv4.method manual ipv4.addresses "IP/子网" ipv4.gateway "网关" ipv4.dns "DNS"

# 示例：
sudo nmcli connection modify "Wired connection 1" \
  ipv4.method manual \
  ipv4.addresses "192.168.1.100/24" \
  ipv4.gateway "192.168.1.1" \
  ipv4.dns "8.8.8.8,114.114.114.114"
    • 步骤 3：重启连接生效
    sudo nmcli connection down "Wired connection 1"  # 断开连接
sudo nmcli connection up "Wired connection 1"    # 重新连接

    7.静态IP适用场景¶

    • 服务器、监控摄像头、远程设备控制（需固定地址）。
    • 避免IP变动导致的服务中断（如直播、VPN）。

    ⚠️ 注意事项：

    • 确保IP地址唯一，避免冲突（通过路由器DHCP客户端列表检查）。
    • 子网掩码和网关必须与路由器一致。
