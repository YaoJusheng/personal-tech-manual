    1.3.快捷键¶

    在 Ubuntu（GNOME）里，Ctrl + Alt + ← / → 既被系统占用（切换工作区）

    方案 1：直接“卸掉”系统快捷键（最干净）:

    打开终端，把 GNOME 的左右工作区切换设成空：

    gsettings set org.gnome.desktop.wm.keybindings switch-to-workspace-left  "['']"
gsettings set org.gnome.desktop.wm.keybindings switch-to-workspace-right "['']"

    生效立刻，无需重启。 以后 IDE 的 Ctrl + Alt + ← / → 就能正常前后跳转代码了。 （如果你还想保留上下切换，可保留 switch-to-workspace-up/down 不动）

    方案 2：给系统换一套不冲突的快捷键:

    例如改成 Super + Alt + ← / →（Win 键当作 Super）：

    gsettings set org.gnome.desktop.wm.keybindings switch-to-workspace-left  "['<Super><Alt>Left']"
gsettings set org.gnome.desktop.wm.keybindings switch-to-workspace-right "['<Super><Alt>Right']"

    恢复默认（想反悔时）:

    gsettings reset org.gnome.desktop.wm.keybindings switch-to-workspace-left
gsettings reset org.gnome.desktop.wm.keybindings switch-to-workspace-right

    图形化修改（可选）:

    1. 安装 dconf-editor
    sudo apt install dconf-editor
    1. 运行 dconf-editor → 依次展开/org/gnome/desktop/wm/keybindings/ 找到 switch-to-workspace-left/right 手动修改即可。

    方法二：纯 Shell 命令（无需安装）¶

    如果你不想安装工具，可以用 find + wc：

    统计文件个数（按扩展名）：

    # 统计所有 .py 文件个数
find . -name "*.py" -type f | wc -l

# 统计多种代码文件总数
find . \( -name "*.py" -o -name "*.js" -o -name "*.rs" -o -name "*.c" -o -name "*.cpp" -o -name "*.go" -o -name "*.java" -o -name "*.zig" \) -type f | wc -l

    统计代码行数（含空行和注释）：

    # 统计所有 .py 文件的总行数
find . -name "*.py" -type f -exec cat {} + | wc -l

# 统计多种文件的总行数
find . \( -name "*.py" -o -name "*.js" -o -name "*.rs" \) -type f -exec cat {} + | wc -l

    更完善的脚本（保存为 codecount.sh）：

    #!/bin/bash
DIR="${1:-.}"

echo "=== 代码文件统计 ==="

# 定义要统计的扩展名
exts=("py" "js" "ts" "jsx" "tsx" "rs" "c" "cpp" "h" "hpp" "go" "java" "zig" "rb" "php" "sh" "md")

total_files=0
total_lines=0

for ext in "${exts[@]}"; do
    count=$(find "$DIR" -name "*.$ext" -type f 2>/dev/null | wc -l)
    if [ "$count" -gt 0 ]; then
        lines=$(find "$DIR" -name "*.$ext" -type f -exec cat {} + 2>/dev/null | wc -l)
        printf "%-6s: %4d 个文件, %6d 行\n" ".$ext" "$count" "$lines"
        total_files=$((total_files + count))
        total_lines=$((total_lines + lines))
    fi
done

echo "------------------------"
echo "总计  : $total_files 个文件, $total_lines 行"

    使用方法：

    chmod +x codecount.sh
./codecount.sh          # 统计当前目录
./codecount.sh /path/to/project  # 统计指定目录

    8.综合示例（排错套路）¶

    1. lsmod | grep nvidia 有输出 → 模块已加载；无输出 → 看 2。
    2. cat /proc/driver/nvidia/version 存在 → 记录版本号 A；不存在 → 模块没加载，跳到 4。
    3. nvidia-smi 能启动 → 用户态版本 B；若 B ≠ A → API mismatch，需重装/重启。
    4. zgrep nvidia /var/log/dpkg.log* 看最近一条是 install 还是 remove，确认 APT 层面到底该有还是不该有。
    5. dkms status | grep nvidia 检查 DKMS 是否已为当前内核编译对应版本；若状态为 added 而非 installed，执行 sudo dkms autoinstall 即可。

    1.2.路由¶
    1.简介¶

    • 可以使用 ip route 或 netstat -r 命令来查看路由表。ip route 命令的输出会以一种更直观的方式显示路由信息，如目标网络、子网掩码、下一跳地址、出接口等。netstat -r 命令则会显示内核路由表的详细内容，包括目的网络、网关、子网掩码、接口等信息。

    • 查看： 执行 ip route show， 输出：

    default via 192.168.2.1 dev wlo1 proto dhcp metric 20600
169.254.0.0/16 dev wlo1 scope link metric 1000
172.17.0.0/16 dev docker0 proto kernel scope link src 172.17.0.1 linkdown
192.168.2.0/24 dev wlo1 proto kernel scope link src 192.168.2.72 metric 600
198.18.0.0/30 dev Meta proto kernel scope link src 198.18.0.1

    解释输出：

    • default：表示这是默认路由。
    • via：指定下一跳的网关地址。
    • dev：指定数据包发送时使用的网络接口（例如 wlo1 是无线接口）。
    2.route命令¶

    语法：

    route [-n] [-e] [add|del] [ -net | -host ] 目标网络 [mask 子网掩码] [gw 网关] [dev 接口]
    • 常用选项：
    • -n：以数字形式显示 IP 地址（不解析主机名，输出更清晰）。
    • -e：以 详细格式（BSD 风格） 显示路由表。
    • add/del：添加或删除路由条目。
    • -net：指定目标为网络地址（默认）。

    • -host：指定目标为单播主机地址。

    • 执行 route -n 输出示例：

    $ route
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
default         routerlogin.cn  0.0.0.0         UG    20600  0        0 wlo1
link-local      0.0.0.0         255.255.0.0     U     1000   0        0 wlo1
172.17.0.0      0.0.0.0         255.255.0.0     U     0      0        0 docker0
192.168.2.0     0.0.0.0         255.255.255.0   U     600    0        0 wlo1
198.18.0.0      0.0.0.0         255.255.255.252 U     0      0        0 Meta

$ route -n
Kernel IP routing table
Destination     Gateway         Genmask         Flags Metric Ref    Use Iface
0.0.0.0         192.168.2.1     0.0.0.0         UG    20600  0        0 wlo1
169.254.0.0     0.0.0.0         255.255.0.0     U     1000   0        0 wlo1
172.17.0.0      0.0.0.0         255.255.0.0     U     0      0        0 docker0
192.168.2.0     0.0.0.0         255.255.255.0   U     600    0        0 wlo1
198.18.0.0      0.0.0.0         255.255.255.252 U     0      0        0 Meta
    • 各字段含义：
    字段 说明
    Destination 目标网络或主机地址（0.0.0.0 表示默认路由）。
    Gateway 下一跳网关地址（0.0.0.0 表示直接连接，无需经过网关）。
    Genmask 子网掩码（255.255.255.0 表示 /24 网络）。
    Flags 路由标志（常用标志： U= 启用，G= 使用网关，H= 目标为单播主机）。
    Metric 路由优先级（跃点数），数值越小优先级越高（与 ifmetric 关联）。
    Ref 引用计数（当前未使用，恒为 0）。
    Use 该路由的数据包统计数（累计通过该路由的流量）。
    Iface 网络接口名称（如 eth0、wlan0）。

    3.优先级设置¶

    （1）添加静态路由¶
    • 添加到目标网络的路由：
    sudo route add -net 192.168.2.0/24 gw 192.168.1.254 dev eth0

    目标网络：192.168.2.0/24，通过网关 192.168.1.254，经接口 eth0 转发。

    • 添加默认路由（网关）：
    sudo route add default gw 192.168.1.1 dev eth0
    （2）删除路由¶
    sudo route del -net 192.168.2.0/24 gw 192.168.1.254 dev eth0
    （3）设置路由 Metric¶
    sudo route add -net 10.0.0.0/8 gw 10.8.0.1 dev tun0 metric 50
    • 通过 metric 参数指定路由优先级（数值越小越优先）。
    （4）IP Route¶
    $ ip route del <目标网络>/<子网掩码>

# 添加带 Metric 的路由
$ sudo ip route add 目标网络/掩码 via 网关 dev 接口名 metric 数值
# 示例：添加到 10.0.0.0/8 的路由，Metric 为 50，经 eth0 接口
$ sudo ip route add 10.0.0.0/8 via 10.8.0.1 dev eth0 metric 50

# 修改现有路由的 Metric
$ sudo ip route change 目标网络/掩码 via 网关 dev 接口名 metric 新数值

# e.g.
$ sudo ip route add 192.168.2.0/24 via 192.168.1.2 metric 1
$ sudo ip route add default via 192.168.1.1 dev eth0 metric 50
$ sudo ip route change 192.168.2.0 mask 255.255.255.0 192.168.1.2 metric 5  # 修改现有路由的metric
    （5）永久生效配置（非临时）¶

    Linux 中 route 命令的修改为临时配置，重启后失效。永久配置需修改网络配置文件：

    • Ubuntu/Debian（netplan）： 在 /etc/netplan/*.yaml 中添加 routes 字段：
    network:
  version: 2
  ethernets:
    eth0:
      addresses: [192.168.1.10/24]
      routes:
        - to: 192.168.2.0/24
          via: 192.168.1.254
          metric: 100  # 可选优先级
      nameservers:
        addresses: [8.8.8.8]

    应用配置：sudo netplan apply。

    • CentOS/RHEL（ifcfg-ethX）： 在 /etc/sysconfig/network-scripts/ifcfg-eth0 中添加：
    IPROUTE2_RT_0="dst 192.168.2.0/24 via 192.168.1.254 dev eth0 table main metric 100"

    重启网络服务：sudo systemctl restart network。

    -S, --shutdown 关机（系统完全关闭）时的灯光状态 true（开启）/false（关闭）

detect_gpu_vendor() {
    local gpu_info
    gpu_info=$(lspci | grep -i vga)

    if echo "$gpu_info" | grep -qi nvidia; then
        echo "nvidia"
    elif echo "$gpu_info" | grep -qi amd; then
        echo "amd"
    else
        error_exit "未检测到NVIDIA或AMD GPU, 请检查硬件"
    fi
}

        return 1
    fi

    if ! grep -q "blacklist nouveau" /etc/modprobe.d/nouveau-blacklist.conf; then
        log "禁用nouveau驱动..."
        echo -e "blacklist nouveau\noptions nouveau modeset=0" | tee /etc/modprobe.d/nouveau-blacklist.conf >/dev/null
        update-initramfs -u >/dev/null
    fi

    # 安装推荐驱动
    log "自动检测并安装推荐的NVIDIA驱动..."
    apt update >/dev/null
    ubuntu-drivers autoinstall >/dev/null
    # or
    #driver=$(ubuntu-drivers devices | grep recommended | cut -d':' -f2 | cut -d ' ' -f2)
    #install_apt_packages "$driver"

    log "NVIDIA驱动安装完成, 需重启系统生效"
    read -p "是否现在重启？[y/n] " -n 1 -r
    echo
    if [[ $REPLY =~ ^[Yy]$ ]]; then
        reboot
    else
        log "请手动重启系统以完成驱动加载"
    fi
}


# 定义CUDA版本兼容性映射（持续更新）
declare -A cuda_map=(
    # 格式：驱动版本最低要求 -> CUDA版本
    ["575"]="12.8"
    ["570"]="12.8"
    ["565"]="12.7"
    ["560"]="12.5"
    ["555"]="12.4"
    ["550"]="12.4"
    ["545"]="12.3"
    ["540"]="12.3"
    ["535"]="12.2"
    ["530"]="12.1"
    ["525"]="12.0"
    ["510"]="11.6"
    ["450"]="11.0"
)

get_cuda_version() {
    local driver_version=$1
    major_version=$(echo "$driver_version" | awk -F'.' '{print $1}')
    # minor_version=$(echo "$driver_version" | awk -F'.' '{print $2}')
    # 驱动版本 -> 推荐CUDA版本映射（根据NVIDIA官方兼容性表）
    # 查找兼容的最高CUDA版本
    local selected_cuda=""
    for drv in $(echo "${!cuda_map[@]}" | tr ' ' '\n' | sort -nr); do
        if [ "$major_version" -ge "$drv" ]; then
            selected_cuda=${cuda_map[$drv]}
            break
        fi
    done
    if [ -z "$selected_cuda" ]; then
        error_exit "未找到与驱动 $driver_version 兼容的CUDA版本, 请参考NVIDIA CUDA兼容性表"
    fi
    echo "$selected_cuda"
}


verify_cuda() {
    log "验证CUDA安装..."
    if ! command -v nvcc &>/dev/null; then
        error_exit "nvcc未找到, CUDA安装失败"
    fi
    local cuda_version
    cuda_version=$(nvcc --version | grep "release" | awk '{print $6}' | cut -d',' -f1)
    log "CUDA版本验证通过: $cuda_version"
}

install_cuda() {
    local cuda_version=$1
    # 获取系统信息
    # ubuntu_version=$(lsb_release -cs)  # 如jammy（22.04）、focal（20.04）
    distro=$(lsb_release -si 2>/dev/null || echo "unknown")
    version=$(lsb_release -sr 2>/dev/null)

    log "开始安装 CUDA $cuda_version..."

    # 添加NVIDIA CUDA仓库
    log "添加 CUDA 仓库..."
    case $distro in
      "Ubuntu")
            repo_version="ubuntu$(echo "$version" | tr -d '.')"
            keyring_pkg="cuda-keyring_1.1-1_all.deb"
            wget -q "https://developer.download.nvidia.com/compute/cuda/repos/$repo_version/x86_64/$keyring_pkg"
            sudo dpkg -i $keyring_pkg > /dev/null
            ;;
        "CentOS")
            repo_version="rhel$(cat /etc/redhat-release | grep -oP '[0-9]+' | head -1)"
            sudo rpm --erase --nodeps nvidia-repo-* > /dev/null
            sudo rpm -i "https://developer.download.nvidia.com/compute/cuda/repos/$repo_version/x86_64/cuda-repo-$repo_version-12-2-local-12.2.2_535.104.05-1.x86_64.rpm"
            ;;
        *)
            error_exit "不支持的发行版：$distro"
            ;;
    esac

    # 安装CUDA工具包（仅核心组件）
    log "安装 CUDA $cuda_version 核心组件..."
    case $distro in
        "Ubuntu")
            sudo apt-get update &> /dev/null
            sudo apt-get install -y wget software-properties-common cuda-toolkit-"${cuda_version//./-}" > /dev/null
            # or
            # download_file https://developer.download.nvidia.com/compute/cuda/12.8.0/local_installers/cuda_12.8.0_570.86.10_linux.run cuda_12.8.0_570.86.10_linux.run
            # sudo sh cuda_12.8.0_570.86.10_linux.run
            # 清理临时文件
            rm -f cuda-keyring_1.1-1_all.deb
            # 添加NVIDIA Container Toolkit
            gpu_path=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg
            if [ -f "$gpu_path" ]; then
                log "NVIDIA Container Toolkit 已存在, 跳过添加"
            else
                log "添加NVIDIA Container Toolkit..."
                curl -fsSL https://nvidia.github.io/libnvidia-container/gpgkey | sudo gpg --dearmor -o /usr/share/keyrings/nvidia-container-toolkit-keyring.gpg  || true
            fi
            curl -s -L https://nvidia.github.io/libnvidia-container/stable/deb/nvidia-container-toolkit.list | \
              sed 's#deb https://#deb [signed-by=/usr/share/keyrings/nvidia-container-toolkit-keyring.gpg] https://#g' | \
              sudo tee /etc/apt/sources.list.d/nvidia-container-toolkit.list
            install_apt_packages nvidia-container-toolkit
            ;;
        "CentOS")
            sudo yum clean all > /dev/null
            sudo yum install -y cuda-"$(echo "$cuda_version" | tr -d .)" > /dev/null
            # 添加NVIDIA Container Toolkit
            distribution=$(. /etc/os-release;echo "$ID$VERSION_ID")
            curl -s -L https://nvidia.github.io/nvidia-container-runtime/$distribution/nvidia-container-runtime.repo | \
              sudo tee /etc/yum.repos.d/nvidia-container-runtime.repo
            sudo yum install -y nvidia-container-toolkit
            ;;
    esac

    # 验证安装
    verify_cuda

    # 配置CUDA环境变量
    log "配置 CUDA 环境变量..."
    if [ -d "/usr/local/cuda-$cuda_version" ]; then
        echo "CUDA $cuda_version 安装成功！"
        echo "请将以下内容添加到 ~/.bashrc:"
        echo "export PATH=/usr/local/cuda-$cuda_version/bin:\$PATH"
        echo "export LD_LIBRARY_PATH=/usr/local/cuda-$cuda_version/lib64:\$LD_LIBRARY_PATH"
        # 将环境变量添加到用户文件
        if echo "$LD_LIBRARY_PATH" | grep -wq "cuda-$cuda_version"; then
            log "CUDA环境变量已存在, 跳过添加"
        else
            log "自动添加CUDA环境变量到用户文件..."
            {
                echo "
# Cuda
export PATH=\"\$PATH:/usr/local/cuda-$cuda_version/bin\"
export LD_LIBRARY_PATH=\"\$LD_LIBRARY_PATH:/usr/local/cuda-$cuda_version/lib64\"
export LIBRARY_PATH=\"\$LIBRARY_PATH:/usr/local/cuda-$cuda_version/lib64\"


"
            } >> "$user_file"
        fi
        # or
        # echo 'export PATH="/usr/local/cuda/bin:\$PATH"' >> /etc/profile.d/cuda.sh
        # echo 'export LD_LIBRARY_PATH="/usr/local/cuda/lib64:\$LD_LIBRARY_PATH"' >> /etc/profile.d/cuda.sh
        # source /etc/profile.d/cuda.sh
    else
        echo "安装失败，请检查日志。"
        exit 1
    fi

    log "CUDA $cuda_version 安装完成"
}

# 一键安装CUDA
oneclick_cuda_installer() {
    log "[驱动CUDA安装] 一键安装进行中..."

    # 1. 检测GPU类型（仅支持NVIDIA）
    GPU_VENDOR=$(detect_gpu_vendor)
    log "检测到 GPU 厂商: $GPU_VENDOR"
    if [ "$GPU_VENDOR" != "nvidia" ]; then
        log "仅支持NVIDIA GPU的CUDA安装, 当前检测到非NVIDIA GPU"
        return 1
    fi
    # 2. 检查NVIDIA驱动是否已安装并获取版本
    DRIVER_VERSION=$(recheck_nvidia_driver)

    # 3. 根据驱动版本匹配推荐的CUDA版本
    CUDA_VERSION=$(get_cuda_version "$DRIVER_VERSION")
    log "推荐CUDA版本: $CUDA_VERSION (与当前驱动版本 [$DRIVER_VERSION] 兼容)"

    # 4. 安装CUDA
    install_cuda "$CUDA_VERSION"

    log "[CUDA安装流程] 一键安装完成"
}

install_graphic() {
    log "===== 开始GPU驱动检测与安装 ====="
    # 检测GPU类型
    GPU_VENDOR=$(detect_gpu_vendor)
    log "检测到GPU厂商: $GPU_VENDOR"

    if [ "$GPU_VENDOR" == "nvidia" ]; then
        if check_nvidia_driver; then
            exit 0
        else
            install_nvidia_driver
        fi
    elif [ "$GPU_VENDOR" == "amd" ]; then
        if check_amd_driver; then
            exit 0
        else
            install_amd_driver
        fi
    else
        error_exit "未知GPU厂商"
    fi
    log "===== 驱动安装完成 ====="
}

install_printer() {
    install_apt_packages cups \
      cups-browsed \
      bluez-cups \
      cups-pdf \
      system-config-printer \
      libnss-mdns \
      foomatic-db \
      hplip
    sudo systemctl enable cups
    sudo systemctl enable cups-browsed
    sudo systemctl start cups
    sudo systemctl start cups-browsed
    # 1. 备份原始配置文件（关键步骤，防止配置错误）
    sudo cp /etc/cups/cupsd.conf /etc/cups/cupsd.conf.bak
    # 2. 启用 Browsing（允许网络发现打印机）
    sudo sed -i 's/^Browsing .*/Browsing Yes/' /etc/cups/cupsd.conf
    # 3. 修改监听地址为 localhost:631（CUPS 默认端口）
    sudo sed -i 's/^Listen localhost$/Listen localhost:631/' /etc/cups/cupsd.conf
    # 4. 在 <Location /> 块中添加 Allow @LOCAL（允许本地用户访问）
    sudo sed -i '/<Location \/>/a \    Allow @LOCAL' /etc/cups/cupsd.conf
    # 5. 注释管理页面的 Order allow,deny（兼容新访问控制规则）
    sudo sed -i '/<Location \/admin>/,/<\/Location>/ s/^Order allow,deny/#Order allow,deny/' /etc/cups/cupsd.conf
    # 6. 重启 CUPS 服务使配置生效
    sudo systemctl restart cups
}
