    激活：Microsoft Activation Scripts (MAS) | MAS (massgrave.dev)

    powershell: irm https://get.activated.win | iex, 然后选择1

    0.0.Winget¶

    winget 是Windows 10 和 Windows 11 中内置，用户无需额外安装。微软推出的 Windows 包管理器，旨在简化软件的发现、安装、升级、删除和配置过程。（广泛但支持应用有限）

    # 安装
winget install <software/package/tool>

    0.1.安装 Scoop¶

    Scoop 是一个简洁的命令行包管理器，专注于开发人员工具的安装和管理。（不需要管理员权限，应用较丰富，特别是开发者工具）

    Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
Invoke-RestMethod -Uri https://get.scoop.sh | Invoke-Expression  # irm get.scoop.sh | iex

# Add the extras bucket(extras 是 Scoop 官方维护的一个社区仓库，里面包含了许多官方仓库中没有的额外软件包。添加这个仓库后，用户可以访问和安装这些额外的软件。)
scoop bucket add extras

    0.2.安装 Chocolatey¶

    Chocolatey 是一个强大的包管理器，旨在简化 Windows 平台上的软件安装和管理。 （需要管理员权限，应用非常丰富）

    # install with cmd.exe
@"%SystemRoot%\System32\WindowsPowerShell\v1.0\powershell.exe" -NoProfile -InputFormat None -ExecutionPolicy Bypass -Command "[System.Net.ServicePointManager]::SecurityProtocol = 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))" && SET "PATH=%PATH%;%ALLUSERSPROFILE%\chocolatey\bin"

# install with powershell
Set-ExecutionPolicy Bypass -Scope Process -Force; [System.Net.ServicePointManager]::SecurityProtocol = [System.Net.ServicePointManager]::SecurityProtocol -bor 3072; iex ((New-Object System.Net.WebClient).DownloadString('https://community.chocolatey.org/install.ps1'))

# install with winget
winget install --id chocolatey.chocolatey --source winget

# install with scoop
scoop install chocolatey

# 升级
choco upgrade chocolatey

    如需终端刷新环境变量：

    Import-Module $env:ChocolateyInstall\helpers\chocolateyProfile.psm1'
refreshenv

# 直接初始化刷新整个终端
. $PROFILE

    0.3.安裝 gsudo¶

    gsudo 是一个提升权限的工具，类似于 Unix 系统中的 sudo，用于在 Windows 环境中以管理员权限运行命令。

    winget install gsudo
# or
scoop install gsudo

oh-my-posh font install --user

    3.2.配置 oh-my-posh¶

    # 查看配置文件
echo $PROFILE
# C:\Users\Jushe\Documents\WindowsPowerShell\Microsoft.PowerShell_profile.ps1

# 添加
oh-my-posh init pwsh --config $env:POSH_THEMES_PATH\kushal.omp.json | Invoke-Expression

# 查看所有主题
Get-ChildItem -Path "~\AppData\Local\Programs\oh-my-posh\themes\*" -Include '*.omp.json' | Sort-Object Name | ForEach-Object -Process {
    $esc = [char]27
    Write-Host ""
    Write-Host "$esc[1m$($_.BaseName)$esc[0m"
    Write-Host ""
    oh-my-posh --config $($_.FullName) --pwd $PWD
    Write-Host ""
}

# 美化终端图标
Install-Module -Name Terminal-Icons -Repository PSGallery

# 重启终端
# 执行 Set-ExecutionPolicy RemoteSigned 修改策略
# 执行 Get-ExecutionPolicy 确认策略是否已修改

    3.3.历史命令-自动补全¶

    # 为用户安装（CurrentUser/AllUsers）
Install-Module -Name PowerShellGet -Force
Install-Module PSReadLine -Scope CurrentUser -Force
Install-Module posh-git -Scope CurrentUser

    然后修改前面的配置文件 Microsoft.PowerShell_profile.ps1，在末尾追加以下内容：

    Import-Module -Name Terminal-Icons
Import-Module posh-git
# 提示插件
Import-Module PSReadLine

# 设置预测文本来源为历史记录
Set-PSReadLineOption -PredictionSource History
# 设置 Tab 为菜单补全和 Intellisenses
Set-PSReadLineKeyHandler -Key "Tab" -Function MenuComplete
# 每次回溯输入历史，光标定位于输入内容末尾
Set-PSReadLineOption -HistorySearchCursorMovesToEnd
# 设置向上键为后向搜索历史记录
Set-PSReadLineKeyHandler -Key UpArrow -Function HistorySearchBackward
# 设置向下键为前向搜索历史纪录
Set-PSReadLineKeyHandler -Key DownArrow -Function HistorySearchForward

    5.安装WSL¶

    # 参考：https://learn.microsoft.com/en-us/windows/wsl/install
# https://learn.microsoft.com/zh-cn/windows/wsl/install-manual

wsl --install
# 查看可用的发行版本
wsl --list --online
# 列出已安装的版本
wsl -l -v
# 安装22.04
wsl --install -d Ubuntu-22.04
# 下载并安装 WSL2内核包
# 启动WSL2
wsl --set-default-version 2

# 完全卸载
wsl --shutdown
dism.exe /online /disable-feature /featurename:Microsoft-Windows-Subsystem-Linux /norestart
dism.exe /online /disable-feature /featurename:VirtualMachinePlatform /norestart
# 重新安装
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
# 修复镜像
dism /online /cleanup-image /restorehealth

usermod -d /home/new_name -m new_name
# 修改用户组
groupmod -n new_name new_name
# 修改UID
usermod -u 1000 new_name

    12.1.Mesa3D 文件的放置位置¶

    Mesa3D 的核心文件是 opengl32.dll（OpenGL 兼容层），其放置位置决定了哪些程序会使用它：

    1. 单应用生效（推荐）¶

    将 Mesa3D 文件放在 目标应用程序的执行目录（与 .exe 文件同目录），仅对该应用生效，不影响系统其他程序。 例如：若要让 game.exe 使用 Mesa3D，文件结构如下：

    C:\Games\MyGame\
├─ game.exe          # 目标应用
├─ opengl32.dll      # Mesa3D 提供的 OpenGL 实现
├─ libgallium_wgl.dll
└─ dxil.dll          # D3D12 支持库（部分渲染器需要）

    2. 全局生效（谨慎使用）¶

    若需让系统中所有程序默认使用 Mesa3D（不推荐，可能引发兼容性问题），可将文件放在系统目录：

    • 64 位系统：C:\Windows\System32\（64 位 DLL）
    • 32 位程序：C:\Windows\SysWOW64\（32 位 DLL）

    注意：全局替换可能导致依赖系统原生 opengl32.dll 的程序出错，建议仅在测试环境使用。

    12.2.Windows 虚拟机中配置使用 Mesa3D 的步骤¶

    1. 下载适配的 Mesa3D 版本¶

    • 访问 mesa-dist-win releases，根据虚拟机系统选择：
    • 64 位系统：下载 mesa64-xxxx.zip（如 mesa64-24.1.0.zip）
    • 32 位系统：下载 mesa32-xxxx.zip
    • 解压后，核心文件包括：
    • opengl32.dll：OpenGL 软件渲染核心
    • dxil.dll：支持 D3D12 加速渲染（可选，提升性能）
    • libgallium_wgl.dll：WGL 接口支持（解决 WGL 相关报错的关键）

    2. 针对单应用配置（以虚拟机中运行 test.exe 为例）¶

    假设虚拟机中需运行 C:\Apps\test.exe，步骤如下：

    1. 将解压后的 opengl32.dll 和 dxil.dll 复制到 C:\Apps\ 目录（与 test.exe 同目录）。
    2. （可选）设置环境变量指定渲染器（提升性能）：
    3. 右键「此电脑」→「属性」→「高级系统设置」→「环境变量」→「新建系统变量」：
      • 变量名：GALLIUM_DRIVER
      • 变量值：llvmpipe（默认，兼容性好）或 d3d12（利用 D3D12 加速，需虚拟机支持 D3D12）
    4. 双击 test.exe 运行，程序会自动加载同目录的 opengl32.dll（Mesa3D 版本），替代系统默认的 OpenGL 驱动。

    3. 验证 Mesa3D 是否生效¶

    在虚拟机中安装 GL Extensions Viewer，运行后查看「Renderer」字段：

    • 若显示 llvmpipe (Mesa) 或 D3D12 (Mesa)，说明 Mesa3D 已成功加载。
    • 若仍显示系统原生驱动（如 Intel UHD Graphics），检查文件是否放对目录（需与应用同目录），或是否存在同名 DLL 冲突。

    4. 虚拟机性能优化（可选）¶

    • 启用 3D 加速：在虚拟机软件（如 VMware、VirtualBox）中开启 3D 加速：
    • VMware：虚拟机设置 →「硬件」→「显示器」→ 勾选「加速 3D 图形」。
    • VirtualBox：虚拟机设置 →「显示」→「显卡控制器」选择 VMSVGA，勾选「启用 3D 加速」。
    • 选择合适的渲染器：
    • 纯 CPU 渲染：GALLIUM_DRIVER=llvmpipe（兼容性最佳，性能一般）。
    • 利用虚拟机 3D 加速：GALLIUM_DRIVER=d3d12（需虚拟机支持 D3D12，性能更好）。

    13.0.Shelll/Lua¶

    13.0.1.Shell¶

    • 类型: Shell 是一种命令行解释器，是用户与操作系统之间的接口。常见的 Shell 包括 Bash、Zsh、PowerShell 等。
    • 主要用途:
    • 脚本编写: 编写自动化脚本来完成重复性的系统管理任务。
    • 命令行操作: 提供用户与系统进行交互的界面，如文件管理、进程管理等。
    • 优点:
    • 强大的命令行工具: Shell 提供了丰富的命令行工具和命令组合的强大功能。
    • 跨平台性: 大部分 Shell 都支持多种操作系统，如 Linux、macOS 和 Windows（通过 WSL、Cygwin 或 Git Bash）。

    13.0.2.Lua¶

    • 类型: Lua 是一种轻量级的嵌入式脚本语言。
    • 主要用途:
    • 嵌入式系统: 由于其小巧且高效的特性，Lua 常用于嵌入到其他应用程序中，如游戏（如《魔兽世界》的插件）和应用程序脚本。
    • 配置脚本: Lua 常用于编写配置文件和数据描述脚本。
    • 优点:
    • 小巧、快速: Lua 是一种设计简洁、高效的语言。

    • 易于嵌入: Lua 可以轻松嵌入 C/C++ 程序中。

    • 版本管理

    # luaver
curl https://raw.githubusercontent.com/dhavalkapil/luaver/master/install.sh -o install.sh && . ./install.sh
# usage
luaver install 5.3.1             # Installs Lua version 5.3.1
luaver install 5.3.0             # Installs Lua version 5.3.0
luaver use 5.3.1                 # Switches to Lua version 5.3.1
luaver install-luarocks 2.3.0    # Installs Luarocks version 2.3.0
luaver uninstall 5.3.0           # Uninstalls Lua version 5.3.0
    • 包管理
    scoop install Luarocks

    13.0.3.比较和选择¶

    • 嵌入性和轻量性:
    • 如果需要嵌入到应用程序中，尤其是对性能和内存占用有严格要求的场景（如嵌入式设备、游戏引擎），Lua 是更好的选择。
    • 系统管理和自动化:
    • 对于系统管理任务、脚本自动化和命令行操作，Shell 是更适合的选择，尤其是 Unix 环境下的 Bash、Zsh 等。
    • 学习曲线和生态系统:
    • Shell 更加广泛用于系统管理，生态系统丰富，学习资料众多，适合系统管理员和 DevOps 工程师。
    • Lua 由于其简洁性和嵌入能力，也有很多应用场景，适合开发嵌入式系统和需要轻量级脚本语言的场景。

    13.2.1.pyenv版本管理¶

    pyenv是一个Python版本管理工具，它允许您在同一系统上安装和切换不同版本的Python。

    每个Python版本都有其自己的用途和特点，您可以根据项目需求选择适当的版本。使用pyenv可以方便地在不同Python版本之间切换，以满足您的开发需求。

    Usage:

    # 安装
scoop install pyenv

# --- 使用 ---
# 1.安装python
pyenv install 3.8.12
# 2.创建env
pyenv virtualenv 3.8.12 common-default-3.8.12
# 3.使用
pyenv activate common-default-3.8.12
pyenv shell common-default-3.8.12
# 4.查看当前
pyenv version
# 5.退出
pyenv deactivate
# 6.列出所有virtualenv
pyenv virtualenvs
# 7.删除
pyenv virtualenv-delete
# 查看所有版本
pyenv versions [--bare] [--skip-aliases ]
# all available versions
pyenv install --list
# 更多
pyenv help
