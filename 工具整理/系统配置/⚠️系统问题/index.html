    一、Mac¶

    1.1.mysqlclient¶

    # 1. mysqlclient 报错 _mysql undefined (old)
export LD_LIBRARY_PATH="/usr/local/mysql/lib"
sudo ln -s /usr/local/mysql/lib/libmysqlclient.21.dylib /usr/lib/libmysqlclient.21.dylib

# 2. /usr 目录下提示无操作权限
# 1) 重启，过程中按住 command+R，进入保护模式
# 2）打开terminal终端，输入 csrutil disable
# 3）再次重启，即可对 /usr/bin 目录下文件进行修改
sudo chown -R $(whoami) /usr/local/
# 上面可能还会报错: Read-only file systen
sudo ln -s /usr/local/mysql/lib/libmysqlclient.21.dylib /usr/local/lib/libmysqlclient.21.dylib
# 如果要恢复保护机制，重新进入保护模式，输入 csrutil enable

# 3. 依赖安装
# Assume you are activating Python 3 venv
$ brew install mysql-client pkg-config
$ export LDFLAGS="-L$(brew --prefix)/opt/mysql-client/lib"
$ export CPPFLAGS="-I$(brew --prefix)/opt/mysql-client/include"
$ export PKG_CONFIG_PATH="$(brew --prefix)/opt/mysql-client/lib/pkgconfig"
$ pip install mysqlclient

# 其他: 手动指定路径
$ export MYSQLCLIENT_CFLAGS=$(mysql_config --cflags)
$ export MYSQLCLIENT_LDFLAGS=$(mysql_config --libs)

    二、Ubuntu¶

    1.Xorg日志过大¶

    1.1.原因分析¶

    1. Xorg 持续报错
    2. 显卡驱动不兼容或配置错误，导致 Xorg 不断记录错误信息。
    3. 日志轮转（logrotate）未生效
    4. /etc/logrotate.d/xserver-xorg 配置缺失或错误，未对日志进行压缩和清理。
    5. 应用程序或桌面环境冲突
    6. 某些图形应用或桌面扩展（如 GNOME 插件）与 Xorg 交互异常，触发大量日志输出。
    7. 硬件问题
    8. 显卡硬件故障或连接问题，导致 Xorg 频繁尝试重连。

    1.2.解决方法¶

    1. 立即释放磁盘空间¶

    清空当前日志文件（无需重启服务）：

    sudo truncate -s 0 /var/log/Xorg.1.log
    2. 检查日志内容¶

    查看日志末尾的报错信息，定位问题根源：

    tail -n 100 /var/log/Xorg.1.log

    常见关键词：

    • (EE)：严重错误（如显卡驱动崩溃）。
    • (WW)：警告（如分辨率设置失败）。
    3. 修复日志轮转配置¶

    创建或修改日志轮转策略：

    sudo nano /etc/logrotate.d/xserver-xorg

    添加以下内容：

    /var/log/Xorg.*.log {
    weekly
    missingok
    rotate 4
    compress
    delaycompress
    notifempty
    create 0644 root root
}

    手动触发轮转：

    sudo logrotate -vf /etc/logrotate.d/xserver-xorg
    4. 更新或重装显卡驱动¶
    • NVIDIA 显卡：
    sudo apt purge nvidia-*
sudo ubuntu-drivers autoinstall
sudo reboot
    • Intel/AMD 集成显卡：
    sudo apt install --reinstall xserver-xorg-core
sudo dpkg-reconfigure xserver-xorg
sudo reboot
    5. 检查 Xorg 配置文件¶

    备份并删除自定义配置（如有）：

    sudo mv /etc/X11/xorg.conf /etc/X11/xorg.conf.bak
sudo systemctl restart gdm  # 重启显示管理器（如 GDM、LightDM）
    6. 监控日志增长¶

    实时监控日志文件大小：

    watch -n 60 'du -sh /var/log/Xorg.1.log'
    7. 临时禁用 Xorg 日志（可选）¶

    在 /etc/X11/xorg.conf 中添加：

    Section "ServerFlags"
    Option "Log" "none"
EndSection

    重启 Xorg 服务生效。

    1.3.预防措施¶

    1. 定期检查日志轮转
    2. 确保 /etc/logrotate.d/xserver-xorg 配置正常。
    3. 更新系统和驱动
    4. 使用 sudo apt update && sudo apt upgrade 保持系统最新。
    5. 硬件诊断
    6. 运行 sudo lspci -k | grep -A 2 VGA 检查显卡驱动状态。
    7. 使用 sudo dmesg | grep -i error 查看内核级错误。

    2.GLIBCXX_3.4.32 not found¶

    报错：

    ImportError: /lib/x86_64-linux-gnu/libstdc++.so.6: version `GLIBCXX_3.4.32' not found

    解决方法：

    1. 添加 GCC 官方仓库：
    sudo add-apt-repository ppa:ubuntu-toolchain-r/test
sudo apt update
    1. 检查安装libstdc++6 包
    # 查看 OS 与 libstdc++ 包版本
cat /etc/os-release
apt-cache policy libstdc++6

# 安装libstdc++6
sudo apt-get install -y libstdc++6
    1. 安装新版本的 GCC 和 G++：
    sudo apt install gcc-13 g++-13
# or
sudo apt install -y gcc g++ build-essential
    1. 设置新 GCC 为默认版本：
    sudo update-alternatives --install /usr/bin/gcc gcc /usr/bin/gcc-13 60
sudo update-alternatives --install /usr/bin/g++ g++ /usr/bin/g++-13 60
    1. 验证安装：
    gcc --version
strings /usr/lib/x86_64-linux-gnu/libstdc++.so.6 | grep GLIBCXX_3.4.32
# or
strings /usr/lib/aarch64-linux-gnu/libstdc++.so.6 | grep GLIBCXX_3.4.32

    3.NVIDIA 驱动未签名导致内核污染¶

    3.1.错误信息¶

    nvidia: module verification failed: signature and/or required key missing - tainting kernel

    3.2.原因¶

    若启用了Secure Boot，未签名的 NVIDIA 驱动会被内核拒绝加载，导致驱动无法生效并污染内核（tainting kernel）。

    3.3.解决方案¶

    1. 关闭 Secure Boot（推荐）

    2. 重启进入 BIOS（ROG 主板通常按F2）。

    3. 找到 Security > Secure Boot 选项，设置为 Disabled。

    4. 保存并退出 BIOS，重启系统。

    5. 为 NVIDIA 驱动签名（高级） 若需保留 Secure Boot，需为驱动模块签名：

    # 生成签名密钥
openssl req -new -x509 -newkey rsa:2048 -keyout MOK.priv -outform DER -out MOK.der -nodes -days 3650 -subj "/CN=My Custom Driver Signing Key/"

# 注册密钥到MOK（Machine Owner Key）
sudo mokutil --import MOK.der

# 安装NVIDIA驱动（使用DKMS自动签名）
sudo apt install nvidia-driver-535-dkms

# 重启并完成MOK认证（按提示选择"Enroll MOK"）
sudo reboot

    4.温度传感器读取失败（thermal thermal_zone5）¶

    4.1.错误信息¶

    thermal thermal_zone5: failed to read out thermal zone (-61)

    4.2.原因¶

    错误 -61 表示设备无响应，可能是主板温度传感器驱动未正确加载，或内核不支持该硬件。

    4.3.解决方案¶

    1. 安装传感器工具检查
    sudo apt install lm-sensors
sensors-detect  # 自动检测传感器
sensors  # 查看温度数据（若thermal_zone5仍异常，继续下一步）

    1. 更新内核或禁用传感器（临时）

    2. 升级内核到 6.10+（如前所述），新版内核可能修复传感器驱动。

    3. 若需临时禁用该传感器，编辑 Grub 配置：

      sudo nano /etc/default/grub

      添加参数：

      GRUB_CMDLINE_LINUX_DEFAULT="quiet splash acpi_thermal_relaxed"

      更新 Grub 并重启：

      sudo update-grub && sudo reboot

    5.ASUS 设备探测失败（asus: probe failed）¶

    5.1.错误信息¶

    asus: probe of 0003:0B05:19B6.0003 failed with error -12

    5.2.原因¶

    错误 -12 表示内存不足（ENOMEM），可能是 ASUS 硬件驱动（如 RGB 灯效、键盘驱动）在启动时申请内存失败，常见于内核版本不兼容。

    5.3.解决方案¶

    1. 安装 ASUS Linux 驱动 社区工具 asusctl 可修复 ROG 设备的硬件功能（如 RGB 灯效）：
    # 添加仓库并安装
sudo add-apt-repository ppa:asus-linux/asus
sudo apt update
sudo apt install asusctl rog-control-center

# 启动服务
sudo systemctl enable --now asusd
    1. 调整内核内存管理参数 编辑 /etc/sysctl.conf，增加内存分配阈值：
    sudo nano /etc/sysctl.conf

    末尾添加：

    vm.overcommit_memory=1  # 允许内核过度分配内存
vm.panic_on_oom=0       # 禁用OOM（内存不足）时panic

    生效配置：

    sudo sysctl -p

    6.version `Qt_6.7' not found¶

    6.1.错误信息¶

    xxx: /lib/x86_64-linux-gnu/libQt6Core.so.6: version `Qt_6.7' not found

    6.2.解决方案¶

    1.检查当前 Qt 版本¶

    首先确认系统中已安装的 Qt 版本：

    # 检查 Qt6Core 库版本
strings /lib/x86_64-linux-gnu/libQt6Core.so.6 | grep Qt_6

    2.安装检查qt6¶
    sudo apt install qt6-base-dev

    在 Ubuntu 22.04 中，目前存在一个未解决的错误： QtChooser 不支持 qt6 。（参考：Ubuntu 22.04）

    设置qt6的两种方案

    • 全系统设置qt6
    # 根据 qmake6 的路径生成 qt6.conf
qtchooser -install qt6 $(which qmake6)
# 将 qt6.conf 移动到系统范围目录，若以 root 身份运行 qtchooser -install （例如，使用 sudo ），它会将配置添加到 qt-default 目录，而不是用户的主目录，从而无需移动 conf 文件。
sudo mv ~/.config/qtchooser/qt6.conf /usr/share/qtchooser/qt6.conf
# 将 Qt6 设置为默认选项
sudo mkdir -p /usr/lib/$(uname -p)-linux-gnu/qt-default/qtchooser
sudo ln -n /usr/share/qtchooser/qt6.conf /usr/lib/$(uname -p)-linux-gnu/qt-default/qtchooser/default.conf
    • 为当前用户设置qt6
    # 根据 qmake6 的路径生成 qt6.conf
qtchooser -install qt6 $(which qmake6)
# 选择 Qt6 作为默认值（放在 ~/.bashrc 中以保持持久性）：
export QT_SELECT=qt6

    8.树莓派内核检查错误¶

    8.1.错误信息¶

    Failed to check for processor microcode upgrades.
No services need to be restarted.
No containers need to be restarted.
No user sessions are running outdated binaries.
No VM guests are running outdated hypervisor (qemu) binaries on this host.

    8.2.解决方法¶

    • 检查 needrestart 的版本
    dpkg -l | grep needrestart
    • 检查内核版本
    uname -r
    • 检查 needrestart 的服务状态
    sudo systemctl status needrestart
    • 检查处理器微代码
    cat /proc/cpuinfo | grep microcode
    • 检查 needrestart 的依赖项
    dpkg -l | grep iucode-tool
dpkg -l | grep libpam-systemd
dpkg -l | grep systemd
    方法 1：修改 needrestart 配置文件¶

    你可以通过修改 needrestart 的配置文件来禁用这些检查：

    1. 打开配置文件：
    sudo nano /etc/needrestart/needrestart.conf
    1. 找到以下两行并取消注释，将它们的值改为 0：
    #$nrconf{kernelhints} = 0;
#$nrconf{ucodehints} = 0;

    修改后应为：

    $nrconf{kernelhints} = 0;
$nrconf{ucodehints} = 0;
    1. 保存并关闭文件。
    方法 2：更新系统和固件¶

    确保你的系统和处理器微代码是最新的：

    1. 更新系统：
    sudo apt update && sudo apt upgrade -y
    1. 安装最新的处理器微代码：
    sudo apt install intel-microcode

    9.Disk has no space (xxx index out of range)¶

    • check space
    df -h

# 交互式逐层磁盘空间统计并清理（d：清理目录）
sudo apt install ncdu -y
sudo ncdu /
    • clean cache
    sudo apt clean                    # 缓存
sudo apt autoremove --purge       # 旧内核
sudo apt --purge autoremove
sudo journalctl --vacuum-time=3d  # 保留3天
sudo rm -rf /var/crash/*
    • check Docker
    # 看容器日志大小
sudo sh -c 'du -h /var/lib/docker/containers/*/*-json.log 2>/dev/null | sort -hr | head'
# 立即截断超大日志（容器无需停）
sudo sh -c 'truncate -s 50M /var/lib/docker/containers/*/*-json.log'
# 永久解决：新建 /etc/docker/daemon.json
{
  "log-driver": "json-file",
  "log-opts": { "max-size": "50m", "max-file": "3" }
}
sudo systemctl restart docker
    • system log
    sudo du -h /var/log | sort -hr | head
# 手动轮转并压缩
sudo logrotate -f /etc/logrotate.conf
# 仍过大就删旧归档
sudo find /var/log -name '*.log.*.gz' -mtime +30 -delete
    • top 20
    # 单文件 >500 MB
sudo find / -type f -size +500M -exec ls -lh {} \; 2>/dev/null
# 看前 20 吃货
sudo du -ahx / | sort -rh | head -n 20
