    1.简介¶

    Docker 是一个开源的应用容器引擎，让开发者可以打包他们的应用以及依赖包到一个可移植的容器中，然后发布到任何流行的 Linux 机器上。 Docker 容器不使用硬件虚拟化，它的守护进程是宿主机上的一个进程，换句话说，应用直接运行在宿主机内核上。 Docker 由镜像(Image)、容器(Container)、仓库(Repository)三部分组成。

    Docker 的镜像可以简单的类比为电脑装系统用的系统盘，包括操作系统，以及必要的软件。 Docker的容器可以简单理解为提供了系统硬件环境，它是真正跑项目程序、消耗机器资源、提供服务的东西。 Docker的仓库用于存放镜像。

    3.镜像/容器操作¶

    3.1.镜像基础操作¶

    • [搜索镜像]：docker search image_name
    • [拉取镜像]：docker pull 镜像仓库
    • [查看镜像]：
    • docker images
    • docker image ls
    • [删除镜像]：
    • docker rmi IMAGEID：删除单个镜像
    • docker rmi $(docker images -q)：删除所有的镜像
    • [镜像重命名]：docker tag IMAGEID REPOSITORY:TAG
    • [镜像更改历史]：docker history IMAGEID

    3.2.容器基础操作¶

    • [开启镜像容器]：
    • docker run -itd --name mycontainer -v localpath:containerpath -p 8888:8080 python3:latest bash( /bin/bash)
      • -d：后台运行
      • --name：指定容器名
      • -v：设置本地目录与容器目录共享
      • -p：端口映射
    • [查看运行容器id]：
    • docker ps：查看运行中的容器
    • docker ps –a：查看所有容器
    • docker ps –aq：查看所有容器ID
    • [进入容器]：
    • docker exec -it <id> bash
    • docker attach <id>
    • [停止容器]：
    • docker stop <id or name>：停止单个容器
    • docker stop $(docker ps -aq)：停止所有容器
    • [关闭容器（杀进程）]：docker kill <id or name>
    • [暂停容器或结束暂停]：docker pause/unpause <id or name>
    • [重启容器]：docker restart <id or name>
    • [删除容器]：
    • docker rm CONTAINERID：删除单个容器
    • docker rm $(docker ps -aq)：删除所有容器（未运行的）
    • docker rm (docker ps -a|grep Exited|awk '{print $1}')：删除所有已停止容器
    • docker container prune：删除孤立的容器（1.13版本以后）
    • [容器重命名]：docker rename 原容器名 新容器名
    • [显示容器资源信息]：docker stats <id or name>
    • [查看容器日志]：docker logs [-f] <id or name>
    • [查看容器进程信息]：docker top <id or name>
    • [查看容器属性信息（JSON）]：docker inspect <id or name>
    • [查看容器绑定端口信息]：docker port <id or name>
    • [查看容器文件变化信息]：docker diff <id or name>

    查看所有容器和镜像大小：docker system df -v

    3.3.与宿主机文件交互¶

    1) 向宿主机传文件

    docker cp container_id:<docker容器内的路径> <本地保存文件的路径>

    2) 向容器传文件

    docker cp 本地文件路径 ID全称:容器路径

    3.8.3.docker-compose.yml 属性¶

    3.8.3.1.基础属性¶
    • version: Compose 文件格式版本.
    • services: 多个容器集合.
    • image: 指定服务的镜像名称或镜像ID,如果镜像在本地不存在，Compose将会尝试拉取镜像.
    • container_name: Compose的容器名称.
    • hostname: Compose的容器设置主机名.
    • domainname: 声明要用于服务容器的自定义域名.
    • dns: 定义要在容器网络接口配置上设置的自定义 DNS 服务器.
    • build: 服务除了可以基于指定的镜像，还可以基于一份Dockerfile.
    • context: dockerfile目录.
    • dockerfile: 文件路径.
    • args: 带入build参数.
    • working_dir: 工作目录.
    • volumes: 挂载一个目录或者一个已存在的数据卷容器.
    • ports: ports用于映射端口的标签.
    • environment: 环境变量配置.
    • env_file: 从文件中获取环境变量.
    • command: 使用command可以覆盖容器启动后默认执行的命令.
    • network_mode: 网络模式, host、bridge ...
    • extends: 可以扩展另一个服务，扩展内容可以是来自在当前文件，也可以是来自其他文件，相同服务的情况下，后来者会有选择地覆盖原有配置.
    extends:
  file: common.yml
  service: webapp
    3.8.3.2.网络¶
    1. links 默认情况下，服务之间可使用服务名称相互访问。links允许我们定义一个别名，从而使用该别名访问其他服务。 example:
    version: '2'
services:
  web:
    build: .
    links:
      - "db:database"
  db:
    image: postgres

    1. external_links 在使用Docker过程中，我们会有许多单独使用Docker run启动的容器，为了使Compose能够连接这些不在Docker-compose.yml中定义的容器，我们需要一个特殊的标签，就是external_links, 可以让Compose项目里面的容器连接到那些项目配置外部的容器. example:

    2. 自定义一个网络net-a

    docker network create net-a --driver bridge
    • 创建外部容器
    docker run --name redis-external --net net-a -d redis
    • compose配置文件
    version: "3.4"
services:
  web:
    image: redis-web:1.0
    container_name: redis-web
    restart: always
    environment:
      REDIS_HOST: redis-external
    ports:
      - 8001:8001
    external_links:
      - redis-external

networks:
  default:
    external:
      name: net-a
    1. extra_hosts 添加主机名映射。使用与Docker client --add-host参数相同的值。
    version: "3.4"
services:
  web:
    # ...
    extra_hosts:
      - "dockerhost:172.18.0.1"

    1. 自定义网络 Docker 默认提供了 5 种网络驱动模式:

    2. bridge: 默认的网络驱动模式。如果不指定驱动程序，bridge 便会作为默认的网络驱动模式。当应用程序运行在需要通信的独立容器(standalone containers)中时，通常会选择 bridge 模式。

    3. host：移除容器和 Docker 宿主机之间的网络隔离，并直接使用主机的网络。host 模式仅适用于 Docker 17.06+。
    4. overlay：overlay 网络将多个 Docker 守护进程连接在一起，并使集群服务能够相互通信。您还可以使用 overlay 网络来实现 swarm 集群和独立容器之间的通信，或者不同 Docker 守护进程上的两个独立容器之间的通信。该策略实现了在这些容器之间进行操作系统级别路由的需求。
    5. macvlan：Macvlan 网络允许为容器分配 MAC 地址，使其显示为网络上的物理设备。 Docker 守护进程通过其 MAC 地址将流量路由到容器。对于希望直连到物理网络的传统应用程序而言，使用 macvlan 模式一般是最佳选择，而不应该通过 Docker 宿主机的网络进行路由。
    6. none：对于此容器，禁用所有联网。通常与自定义网络驱动程序一起使用。none 模式不适用于集群服务。

    docker network ls 查看网络

    networks 关键字自定义网络:

    version: '3'
services:
  proxy:
    build: ./proxy
    networks:
      - front
  app:
    build: ./app
    network_mode: bridge  # 网络模式
    networks:
      - front
      - back
  db:
    image: postgres
    networks:
      - back

networks:  # 定义了back和front两个网络
  default:  # 默认网络
    # Use a custom driver
    driver: bridge
    external:  # 已存在的
        name: net-a
  front:
    # Use a custom driver
    driver: custom-driver-1
  back:
    # Use a custom driver which takes special options
    driver: custom-driver-2
    driver_opts:
      foo: "1"
      bar: "2"

    5.问题汇总¶

    5.1.Docker 和虚拟机区别¶

    虚拟机（VM）是共享一个服务器的物理资源的操作系统。 容器是一个不依赖于操作系统，运行应用程序的环境。 Docker 是轻量级的沙盒，在其中运行的只是应用，虚拟机里面还有额外的系统。

    5.2.如何清理后台停止的容器¶

    可以使用 sudo docker rm $sudo( docker ps -a -q) 命令。

    5.3.如何查看镜像支持的环境变量¶

    可以使用 docker run IMAGE env 命令。

    5.4.如何退出一个镜像的 bash，而不终止它¶

    按 Ctrl-p、Ctrl-q。

    5.5.仓库（Repository）、注册服务器（Registry）、注册索引（Index）有何关系¶

    首先，仓库是存放一组关联镜像的集合，比如同一个应用的不同版本的镜像，注 册服务器是存放实际的镜像的地方，注册索引则负责维护用户的账号，权限，搜索， 标签等管理。注册服务器利用注册索引来实现认证等管理。

    5.6.如何将一台宿主机的 Docker 环境迁移到另外一台宿主机¶

    停止 Docker 服务，将整个 Docker 存储文件复制到另外一台宿主机上，然后调整 另外一台宿主机的配置即可。

    5.10.解决MacOS中主机访问不了Docker分配的IP的问题¶

    在宿主机上看不到 docker0，无法访问容器所在的网络，导致宿主机 ping 不通 Docker 给 Container 所分配的 IP 地址。

    • 方法一: 在 Docker for Mac 的虚拟机里跑一个 VPN代理
    • 安装 Tunnelblick 客户端: brew install tunnelblick
    • 克隆 Docker-mac-network 项目: git clone https://github.com/wojas/docker-mac-network.git
    • 修改 Docker-mac-network/helpers/run.sh, 将 ip 和 子网掩码 修改为 Docker 的 ip 和子网页码.
    • 在 项目根目录执行: docker-compose up -d, 找到 docker-for-mac.vpn
    • 修改 VPN 文件(docker-for-mac.vpn), 添加 comp-lzo no

    • 双击启动 docker-for-mac.vpn,开启tunnelblick连接.

    • 方法二: 使用 Docker for Mac 里的一个实验性功能：SOCKS 代理。

    • 安装一下 jq: brew install jq
    • 执行:
    cd ~/Library/Group\ Containers/group.com.docker/
mv settings.json settings.json.backup
cat settings.json.backup | jq '.["socksProxyPort"]=8888' > settings.json
    • 重启 Docker for Mac, 前往 系统偏好设置 -> 网络 -> 高级 -> 代理

    • 设置 SOCKS 代理：localhost:8888 并保存和应用

    • 方法三: 使用 docker-connector

    • 安装 Docker-connector: brew install wenjunxiao/brew/docker-connector
    • 把 Docker 的所有 bridge 网络都添加到路由中:
    docker network ls --filter driver=bridge --format "{{.ID}}" | xargs docker network inspect --format "route {{range .IPAM.Config}}{{.Subnet}}{{end}}" >> /usr/local/etc/docker-connector.conf
# 也可以手动修改 /usr/local/etc/docker-connector.conf 文件中的路由，格式是: route 172.17.0.0/16
sudo docker-connector -config /usr/local/etc/docker-connector.conf
    • 配置完成，直接启动服务:
    sudo brew services restart docker-connector
# 运行容器
docker run -it -d --restart always --net host -v /etc/hosts:/etc/hosts --cap-add NET_ADMIN --name mac-docker-connector -e DISPLAY=192.168.251.2 wenjunxiao/mac-docker-connector

    • *注:

      • 1.共享网络
      # 如果需要将对应的网络共享给其他人访问，
# step-1: 需要修改配置文件 /usr/local/etc/docker-connector.conf, 在 route 后增加 expose
route 172.17.0.0/16 expose
# step-2: 增加监听端口、token 和 ip 分配, example:
expose 0.0.0.0:2512
token user1 192.168.251.10
token user2 192.168.251.1
# step-3: 其他机器要安装 docker-accessor
# MacOS
brew install wenjunxiao/brew/docker-accessor
sudo docker-accessor -remote 192.168.1.100:2512 -token user1

# Linux
curl -L -o- https://github.com/wenjunxiao/mac-docker-connector/releases/download/v2.0/docker-accessor-linux.tar.gz | tar -xzf - -C /usr/local/bin
sudo docker-accessor -remote 192.168.1.100:2512 -token user1

# Windows
# 从 OpenVPN 下载并安装 tap 驱动: [tap-windows](http://build.openvpn.net/downloads/releases/latest/tap-windows-latest-stable.exe)
# 从 mac-docker-connector/releases 下载最新适合当前操作系统的 docker-accessor-win-i686.zip 或 docker-accessor-win-x86_64.zip 解压即可
docker-accessor.exe -remote 192.168.1.100:2512 -token user2  # 管理员运行

# 如果和本地的 docker 网络发生了冲突，可以使用 -exclude 来避免
docker-accessor -remote 192.168.1.100:2512 -token my-token -exclude 172.1.0.0/24,172.2.0.0/24
