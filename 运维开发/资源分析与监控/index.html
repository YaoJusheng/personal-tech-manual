    一、系统级监控工具¶

    1. Linux 平台¶

    （1）实时资源监控¶

    # 交互式全功能监控（推荐）
atop -P MEM  # 按内存排序进程
atop -P CPU  # 按CPU排序进程

    核心指标解读：

    • PRC 列：显示进程状态（S 睡眠 / R 运行 / Z 僵尸）
    • VIRT：虚拟内存占用（含共享库）
    • RES：实际物理内存占用
    • SWAP：交换空间使用量
    • %CPU：进程 CPU 使用率（多核累加）

    （2）线程阻塞分析¶

    # 显示进程所有线程栈
pstack $(pgrep -n your_process)

# 跟踪系统调用（排查I/O阻塞）
strace -p $(pgrep -n your_process) -T -tt -e trace=file,network

# 性能剖析（火焰图生成）
perf record -g -p $(pgrep -n your_process)
perf script | stackcollapse-perf.pl | flamegraph.pl > flamegraph.svg

    （3）内存深度分析¶

    # 内存泄漏检测
valgrind --leak-check=full --show-leak-kinds=all --track-origins=yes ./your_process

# 内存访问越界检测
ASan_OPTIONS=detect_leaks=1 ./your_process

    2. Windows 平台¶

    （1）图形化工具¶

    • Process Monitor：实时监控文件、注册表、网络操作
    • 任务管理器：性能选项卡显示实时 CPU / 内存 / 磁盘 / 网络占用
    • perfmon：监控内存页交换（Page/sec）、进程私有字节（Process\Private Bytes）

    （2）命令行工具¶

    # 显示进程详细信息
tasklist /SVC /FO CSV | findstr "your_process"

# 内存使用分析
wmic process where name="your_process.exe" get WorkingSetSize,VirtualSize

    3. macOS 平台¶

    （1）图形化工具¶

    • Activity Monitor：查看进程树、能耗、网络
    • Stats：菜单栏实时显示 CPU / 内存 / 网络
    • Instruments：线程分析（Time Profiler）、内存泄漏检测

    （2）命令行工具¶

    # 进程资源占用
top -o rsize  # 按内存排序
top -o cpu    # 按CPU排序

# 内存分析
leaks your_process

    4.进程状态核心指标¶

    4.1.资源占用¶

    • CPU 使用率：进程占用的 CPU 时间百分比（用户态 / 内核态）
    • 内存占用：
    • 虚拟内存（VSZ）：进程虚拟地址空间大小
    • 常驻内存（RSS）：实际驻留在物理内存中的大小
    • 共享内存（SHR）：共享库或内存映射文件大小
    • 文件描述符：进程打开的文件 / 套接字数量

    4.2.线程状态¶

    • 运行态：正在 CPU 执行
    • 就绪态：等待 CPU 调度
    • 阻塞态：等待 I/O 或同步资源
    • 僵死态：子进程已终止但父进程未回收

    5.跨平台监控工具¶

    5.1.Linux/macOS：¶

    1.基础工具¶
    • ps：静态进程快照
    ps aux | grep python  # 查看Python进程

    • top/htop：动态资源监控

    • pidstat：CPU 细分统计

    • lsof：查看文件描述符

    lsof -p <pid>  # 查看进程打开的文件
    2.高级工具¶
    • perf：性能分析
    • pstack：线程堆栈分析
    • valgrind：内存泄漏检测
    • strace：系统调用跟踪

    5.2.Windows：¶

    1.内置工具¶

    • 任务管理器：性能标签页

    • Resource Monitor：详细资源监控

    • PowerShell：

    Get-Process -Name python | Format-List *  # 获取进程详细信息
    2.第三方工具¶
    • Process Explorer：增强版任务管理器
    • DebugDiag：内存诊断工具

    6.内存深度分析¶

    6.1.内存类型¶

    • 堆内存：动态分配（malloc/new）
    • 栈内存：函数调用栈
    • 共享内存：进程间通信

    6.2.内存泄漏检测¶

    • Linux：valgrind --tool=memcheck

    • Windows：DebugDiag, Visual Leak Detector

    • 示例输出：

    ==1234== 4 bytes in 1 blocks are definitely lost in loss record 1 of 1
==1234==    at 0x4C2AB80: malloc (vg_replace_malloc.c:309)
==1234==    by 0x804860F: main (test.c:4)

    7.线程阻塞诊断¶

    7.1.死锁检测¶

    • Linux：pstack <pid> | grep -A 10 'pthread_mutex_lock'
    • Windows：Process Explorer -> Find Handle or DLL

    7.2.锁竞争分析¶

    • 使用 perf 记录锁事件：
    perf record -g -e lock:* -p <pid>

    （3）异步线程监控¶

    import threading
import time

def monitor_threads(process):
    while True:
        for thread in process.threads():
            thread_info = psutil.Process(thread.id).as_dict(attrs=['name', 'status'])
            print(f"线程ID: {thread.id}, 状态: {thread_info['status']}")
        time.sleep(5)

# 启动监控线程
threading.Thread(target=monitor_threads, args=(process,), daemon=True).start()

    三、进阶分析方法¶

    1. 线程阻塞定位¶

    （1）死锁检测¶

    # Python 死锁检测
import threading
import sys

def detect_deadlocks():
    for thread in threading.enumerate():
        if thread.ident == threading.get_ident():
            continue
        sys.settrace(lambda frame, event, arg: sys.settrace(None) or None)
        print(f"线程 {thread.name} 状态: {thread.is_alive()}")
        print(f"线程 {thread.name} 堆栈:")
        for line in traceback.format_stack(thread._stack):
            print(line.strip())

# 定期执行检测
threading.Timer(60, detect_deadlocks).start()

    （2）系统调用跟踪¶

    # Linux 跟踪进程系统调用
strace -p $(pgrep -n your_process) -T -tt -e trace=all

    2. 内存泄漏分析¶

    （1）工具链组合¶

    # Valgrind + Massif 内存分配分析
valgrind --tool=massif ./your_process
ms_print massif.out.*

    （2）Python 内存泄漏检测¶

    import objgraph

# 跟踪对象引用
objgraph.show_refs(your_object, filename='refs.png')

    3. 性能瓶颈定位¶

    （1）火焰图生成¶

    # Linux 生成CPU火焰图
perf record -g -p $(pgrep -n your_process)
perf script | stackcollapse-perf.pl | flamegraph.pl > flamegraph.svg

    （2）异步性能分析¶

    # Python 异步任务监控
import asyncio
from pyinstrument import Profiler

profiler = Profiler()

from fastapi import FastAPI
import psutil

app = FastAPI()

@app.get("/metrics")
def get_metrics():
    process = psutil.Process()
    return {
        "cpu_percent": process.cpu_percent(),
        "memory_rss": process.memory_info().rss,
        "num_threads": process.num_threads(),
        "open_files": len(process.open_files()),
        "connections": len(process.connections())
    }

    2. 日志集成¶

    # Python 日志记录资源使用
import logging
import psutil
from logging.handlers import RotatingFileHandler

logger = logging.getLogger(__name__)
logger.setLevel(logging.INFO)

handler = RotatingFileHandler('process.log', maxBytes=1e6, backupCount=3)
logger.addHandler(handler)

def log_system_info():
    process = psutil.Process()
    logger.info(f"CPU: {process.cpu_percent()}%")
    logger.info(f"Memory: {process.memory_info().rss / 1024 ** 2:.2f} MB")
    logger.info(f"Threads: {process.num_threads()}")

# 定时记录
import threading
threading.Timer(60, log_system_info).start()
