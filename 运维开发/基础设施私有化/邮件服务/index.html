    1.邮件服务简介¶

    邮件服务是通过电子邮件系统实现用户之间信息传递的服务。它允许用户发送、接收和管理电子邮件，是现代通信中不可或缺的一部分。邮件服务通常包括以下几个核心组件：

    • SMTP（Simple Mail Transfer Protocol）：用于发送邮件的协议。
    • IMAP（Internet Message Access Protocol）：用于接收和管理邮件的协议。
    • POP3（Post Office Protocol 3）：另一种用于接收邮件的协议，但不如 IMAP 普遍。
    • Webmail：通过网页界面访问邮件的服务。

    2.准备¶

    1.安装 Docker 和 Docker Compose¶

    • 安装 Docker：
    curl -fsSL https://get.docker.com | sh
systemctl start docker && systemctl enable docker
    • 安装 Docker Compose：
    curl -L "https://github.com/docker/compose/releases/latest/download/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose
chmod +x /usr/local/bin/docker-compose
    • 验证安装：
    docker --version && docker-compose --version

    2.域名解析配置¶

    在 DNS 服务商处添加以下记录（以域名 example.com 为例）：

    • A 记录：mail.example.com → 服务器 IP
    • MX 记录：@ → mail.example.com
    • TXT 记录：SPF（v=spf1 mx ~all）和 DKIM（部署后从日志或工具中获取）.

    3.开源工具¶

    3.1.邮件服务工具¶

    工具名称 特点和性能分析 部署和使用复杂度 使用场景
    Postfix+Dovecot 是一个快速、稳定、安全的邮件传输代理（MTA），被广泛用于搭建邮件服务器。支持多种邮件传输协议，如 SMTP、LMTP 等，具备灵活的路由和转发功能。提供丰富的安全特性，如 SMTP - AUTH、TLS/SSL 支持等，可有效防范垃圾邮件和恶意攻击 部署需要一定的技术知识，要对各种配置文件进行设置和调整。使用复杂度中等，需要了解邮件传输的相关知识和 Postfix 的配置选项 适用于各种规模的邮件系统，尤其是对邮件传输性能、安全性和稳定性有较高要求的场景，常作为大型邮件系统的核心传输组件
    EwoMail 基于 Linux，稳定高效，可 365 天不重启不间断运行，保证邮箱接收与发送效率。支持基于 TLS/SSL 的邮件传输加密，邮箱账号防密码破解，提供专业的反垃圾和防病毒功能。所有操作可在网页进行，无需进入服务器和安装客户端，管理方便，还提供邮箱管理后台、邮箱群组、组织架构、邮箱监控等丰富功能 部署相对简单，按照安装向导进行操作即可。使用简单，通过网页管理界面可轻松进行各种设置和管理 适用于企业和组织，对邮件系统的稳定性、安全性和功能完整性有较高要求，且希望管理便捷，能够方便地进行邮箱管理、监控和组织架构设置等操作的场景
    iRedMail 功能全面的开源邮件服务器解决方案，预集成Postfix、Dovecot、Roundcube等主流开源组件，支持多种Linux/BSD发行版，提供向导式安装，降低部署门槛。性能较好，能满足企业级邮件服务需求；开箱即用，配置相对简单；资源占用相对较高，可能在轻量服务器上出现性能瓶颈。 中等，传统架构，容器化支持不足，Web管理界面功能较基础，高级配置需命令行。 适合中小型企业，特别是对邮件服务器的快速部署和易用性有一定要求，且希望利用成熟的一体化解决方案的用户。
    Modoboa 开源邮件托管和管理平台，包含现代化的Web用户界面，集成邮件、日历、域名管理等功能，开箱即用，提供付费技术支持。性能较好，能满足中小企业的邮件服务需求；功能集成度高，部分功能可能冗余；资源占用在轻量场景下可能较高。 中等，需要一定的技术背景进行配置和管理，依赖Python环境，定制化需二次开发。 适合中小型企业，特别是对一体化邮件管理平台有需求，且希望快速部署的用户。

    3.2.容器化工具¶

    工具名称 特点和性能分析 部署和使用复杂度 使用场景
    Docker Mailserver 集成 SMTP、IMAP、LDAP、防垃圾邮件和防病毒等功能。基于 Postfix、Dovecot 等强大开源组件构建，提供全面邮件解决方案。通过配置文件管理，无需数据库，便于版本控制和升级。安全强化，内置多层防护，支持多种认证方式和证书管理策略 部署简单，通过容器化实现一键部署。使用较简单，主要通过配置文件进行管理 适用于小型企业、个人开发者以及中大型组织，可快速搭建内部邮件系统，尤其适合需要高度定制但不想陷入复杂系统管理工作的团队
    Mailu 基于Docker容器技术的邮件服务器，设计精简但功能齐全，原生支持ARM架构及Kubernetes部署，内置隐私安全功能，提供直观的Web管理界面。启动速度较快，但因包含更多组件，启动时间略长于Docker-Mailserver；资源消耗略高，适合对功能有较高要求的场景；处理能力较强，能够应对较高并发的邮件处理需求，适合大型企业使用；扩展性好，支持多节点部署，易于横向扩展 较低，利用Docker的优势简化了部署过程，适合技术敏感型用户。 适合追求“够用就好”的中小企业，以及对隐私保护有较高要求的用户，强调部署灵活性和架构兼容性。
    mailcow 基于 Docker 的开源组件 / 电子邮件套件，依赖许多广为人知且长期使用的组件。核心是图形化 Web 界面 mailcow UI，几乎提供所有设置入口，可轻松创建新域名和电子邮件地址，完成各种复杂任务，如 DKIM 和 ARC 支持 / 生成、黑白名单设置、垃圾邮件评分管理等。数据存储在 Docker 卷中，需要注意备份 部署前需要进行一些准备工作，如安装 Docker 和 Docker Compose，以及满足操作系统、硬件等要求，但整体部署过程有详细文档指导。使用较方便，通过 Web 界面进行操作 适用于各种规模的企业和组织，提供全方位的邮件服务解决方案，满足企业对邮件系统功能和管理的多样化需求
    poste.io 可方便地搭建 SMTP + IMAP + POP3 + 反垃圾邮件 + 防病毒 + Web 管理 + Web 电子邮件。支持 SPF、DKIM、DMARC、SRS 原生实现，带有防病毒引擎 ClamAV、垃圾邮件过滤器 RSPAMD 等。提供不同权限的 Web 管理，支持 HTTPS 上的 Webmail 客户端 Roundcube，以及电子邮件重定向、自动回复和其他过滤功能。整个邮件服务器容器由 Docker 与其他应用程序隔离 部署简单，只需运行 Docker 命令并进行一些参数配置。使用较简单，通过 Web 管理界面进行操作 适用于个人和小型企业，希望以较低成本搭建功能齐全、安全可靠的邮件服务器，且对部署和管理的便捷性有较高要求的场景
    Catapulte 通过 Docker 容器化部署，支持 AMD64、i386 及 ARM64V8 架构，适应性强。结合 MJML 模板渲染和 SMTP 发送机制，支持通过简单 HTTP 请求触发邮件发送，集成 OpenAPI 规范，为系统集成提供标准化接口 部署简单，通过环境变量配置即可。使用方便，对于开发者和非技术人员都比较友好 特别适合需要频繁调整邮件模板、控制数据主权或希望实现特定 SMTP 认证流程的企业，如初创团队降低运营成本，或大企业解决数据安全、内部网络限制等问题
    Postal 功能齐全的开源邮件投递平台，用于收发电子邮件。启动速度较快，资源消耗较低，处理能力强，支持多节点部署，易于横向扩展，能够满足企业不断增长的邮件服务需求。 部署复杂度较高，需要预先安装多个依赖包，并且对操作系统有一定要求，如 Ubuntu 22.04 或 Ubuntu 20.04，且在其他系统上可能存在兼容性问题。配置复杂，使用复杂度中等 企业邮件系统，SaaS产品集成

    3.3.选择建议¶

    一般来说，如果你更倾向于轻量级和高效的解决方案，Docker-Mailserver 非常适合，特别是当您已经在使用 Docker 进行容器化部署时。

    Postal 也是一个强大的选择，尤其适合需要高并发邮件处理能力的用户。具体选择应基于您的技术能力、资源情况和业务需求。

    如果你需要一个功能全面且易于管理的邮件服务，Mailcow 是一个非常好的选择，作为私有化邮件服务的首选方案，mailcow 凭借其 容器化部署、图形化管理、全功能集成 的特性，在中小型企业和团队中表现尤为突出。以下是其核心优势与适用场景：

    1.核心优势¶
    1. 功能全面性
    2. 集成反垃圾邮件（Rspamd）、反病毒（ClamAV）、Webmail（Roundcube）、多域名支持、Let's Encrypt 证书自动签发等功能，满足企业级需求311。
    3. 支持邮件审计、用户配额管理、邮件列表（Mailing List）等高级功能，适合对邮件系统管理有深度需求的场景。
    4. 部署与管理便利性
    5. 基于 Docker Compose 的容器化部署，可在 30 分钟内完成搭建，且支持一键升级412。
    6. 提供直观的 Web 管理界面（https://mail.example.com），几乎所有配置均可通过界面完成，降低运维门槛311。
    7. 安全性与合规性
    8. 内置 TLS 加密传输、SPF/DKIM/DMARC 认证，支持 OAuth 2.0 登录，有效防范钓鱼攻击和数据泄露1112。
    9. 符合 GDPR、HIPAA 等合规要求，适合对数据安全敏感的企业。
    2.适用场景¶
    • 中小型企业（100-1000 用户）：需可视化管理、多域名 / 组织架构管理，且希望降低运维成本。
    • 技术团队 / 开发者：追求快速部署、灵活配置，同时需要邮件服务与其他系统（如 CRM、OA）集成。
    • 教育机构 / 非盈利组织：预算有限但需专业邮件服务，支持邮件归档和审计。

    4.部署和使用¶

    1.Docker-Mailserver¶

    1.1.服务器要求¶

    • 操作系统：Ubuntu 20.04 或 22.04（推荐）
    • 硬件配置：至少 2GB 内存，2 核 CPU，20GB 磁盘空间
    • 域名：确保已注册域名，并解析到服务器 IP

    1.2.部署 Docker-Mailserver¶

    1.创建目录和配置¶

    services:
  mailserver:

    hostname: mail.example.com
    domainname: example.com
    ports:
      - "25:25"    # SMTP

    volumes:

      - ./mail-state/:/var/mail-state/
      - /etc/letsencrypt:/etc/letsencrypt  # 证书目录映射

      - ENABLE_CLAMAV=1  # 启用防病毒
      - ENABLE_FAIL2BAN=1  # 启用防暴力破解
      - ENABLE_RSPAMD=1:cite[3]:cite[7]  # 反垃圾邮件

    核心区别与最佳实践:

    特性 mail-data mail-state
    数据类型 用户邮件内容、邮箱文件夹 (包括用户邮件、邮件箱配置和邮件内容等) 服务状态、临时队列、动态配置、日志 (包括服务运行时的数据和服务配置)
    持久化目的 永久存储用户数据，防止邮件丢失 维持服务运行状态，加速重启恢复
    关键文件 vmail/ 下的 Maildir 结构 postfix/ 队列、dovecot/ 缓存、DKIM 密钥
    备份优先级 最高（用户数据不可丢失） 高（影响服务稳定性，但可重建部分数据）
    权限要求 需匹配容器内用户组（如 vmail:vmail） 需读写权限，允许容器进程修改状态文件

    最佳实践：

    1. 目录权限：通过环境变量 PUID 和 PGID 匹配主机用户组（如 PUID=1000 PGID=1000），避免权限冲突。
    2. 备份策略：定期备份 mail-data（全量备份）和 mail-state（增量备份队列和配置）。
    3. 存储位置：建议将两个目录挂载到高速存储（如 SSD），提升邮件读写和队列处理效率。

    2.2.配置 DNS 记录¶

    • MX 记录：指向 mail.example.com，优先级 10。
    • SPF 记录：添加 TXT 记录，内容为 v=spf1 a mx ~all。
    • DKIM 记录：在 mailcow 界面生成密钥后，添加 TXT 记录（如 default._domainkey.example.com）。
    • DMARC 记录：添加 TXT 记录 _dmarc.example.com，内容为 v=DMARC1; p=quarantine; rua=mailto:dmarc@example.com。
