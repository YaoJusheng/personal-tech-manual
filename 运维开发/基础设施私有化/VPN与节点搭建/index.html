    1.VPS购买准备¶

    准备工作:

    1. 选择云服务商： 推荐阿里云、腾讯云、AWS、Vultr 等，选择适合的配置和操作系统（如 Ubuntu 20.04）。
    2. 准备域名（可选）： 如需使用域名，提前购买并解析到服务器 IP。
    3. SSH 客户端： 如 Xshell、PuTTY 或 Terminal。

    购买并配置云服务器:

    1. 注册并登录云服务商账号。
    2. 选择服务器配置： 根据需求选择 CPU、内存、带宽等。
    3. 选择操作系统： 推荐 Ubuntu 20.04。
    4. 设置安全组： 开放 SSH（22）、Xray-core 和 Sing-box 所需端口。
    5. 购买并获取服务器 IP 地址、用户名和密码。

    1.0.云服务器与 VPS¶

    云服务器与 VPS 的区别及使用场景分析:

    对比维度 云服务器 VPS
    核心技术 基于云计算架构，资源池化管理，支持分布式存储和弹性扩展 基于虚拟化技术，单台物理服务器划分，资源静态分配
    资源弹性 ✅ 按需动态调整 CPU、内存、存储，支持秒级扩容 ❌ 资源固定，需手动更换套餐或迁移数据
    性能稳定性 ✅ 多节点冗余，故障自动迁移，数据多副本存储 ❌ 依赖单台物理服务器，硬件故障可能导致服务中断
    管理权限 ✅ 提供控制台、API 及自动化工具，支持容器化部署 ✅ 拥有 root 权限，可自定义系统配置，但需自行管理底层运维
    技术支持 ✅ 7×24 小时专业团队支持，提供安全防护、监控等增值服务 ❌ 基础支持为主，高级运维需额外付费
    数据备份 ✅ 自动快照备份，支持跨区域恢复 ❌ 需手动备份或依赖第三方工具
    适用场景 高并发网站、企业级应用、AI / 大数据、弹性业务 个人博客、小型网站、测试环境、预算有限的静态项目
    运营商推荐（国内） 腾讯云、阿里云（合规性强，适合商业用途） 搬瓦工（CN2 GIA 线路，免备案，适合个人用途）
    运营商推荐（国外） AWS、Google Cloud（全球化布局，技术生态完善） Vultr（节点多，按小时计费）、DigitalOcean（开发者友好）
    价格预估（2025） - 入门级：36 元 / 年（阿里云轻量应用服务器）5 - 中级：199 元 / 年（通用算力型）5 - 高级：3000 元 +/ 年（高配置企业级） - 入门级：20 美元 / 月（Vultr）7 - 中级：50-100 美元 / 月（DigitalOcean）7 - 高级：150 美元 / 月 +（独立服务器）
    核心优势 弹性扩展、高可用性、自动化运维 成本低、管理灵活、适合中小项目
    核心劣势 长期使用成本可能高于 VPS 性能上限低、资源共享可能导致波动

    1.0.1. 定义与架构¶

    • 云服务器（ECS/CVM 等）：
    • 基于云计算技术构建，整合了大规模的计算、存储和网络资源。
    • 通过分布式存储和虚拟化技术，将资源池化，用户可以根据需求动态申请和释放资源。
    • VPS（Virtual Private Server）：
    • 通过虚拟化技术将一台物理服务器分割成多个独立的虚拟服务器。
    • 每个 VPS 拥有独立的操作系统、IP 地址、内存和存储空间。

    1.0.2. 资源分配与性能¶

    • 云服务器：
    • 资源动态分配，支持弹性伸缩，可根据业务需求实时调整配置。
    • 性能和稳定性更高，通过多节点冗余和故障自恢复能力，确保服务的高可用性。
    • VPS：
    • 资源分配相对固定，性能受限于物理服务器的配置。
    • 在高负载情况下，可能会受到同一物理服务器上其他 VPS 的影响。

    1.0.3. 成本与价格¶

    • 云服务器：
    • 采用按需付费模式，用户只需为实际使用的资源付费，适合资源需求波动较大的场景。
    • 虽然基础配置价格可能与 VPS 相近，但高性能配置时成本较高。
    • VPS：
    • 价格相对较低，适合资源使用量稳定的用户。
    • 成本结构简单，用户可以根据预设资源选择套餐。

    1.0.4. 管理与运维¶

    • 云服务器：
    • 提供更丰富的管理工具和接口，支持自动化部署和监控。
    • 云服务提供商通常提供全面的技术支持和增值服务。
    • VPS：
    • 管理相对简单，用户可以通过控制面板进行基本操作。
    • 高级管理任务（如安全配置和性能优化）需要用户具备一定的技术知识。

    1.0.5. 使用场景¶

    • 云服务器：
    • 适用于需要弹性伸缩和高可用性的业务，如大型网站、电商平台、大数据处理等。
    • 适合对资源使用量不确定或有突发需求的场景。
    • VPS：

    2.0.6.Nginx配置¶

    1.HTTPS 加密¶

    HTTPS 是基于 SSL/TLS 加密的 HTTP 协议，用于保护数据传输的安全性。

    • 基本配置:
    server {
    listen 443 ssl;  # 监听 443 端口并启用 SSL
    server_name your_domain.com;  # 替换为您的域名

    # SSL 证书和私钥路径
    ssl_certificate /path/to/your_domain.crt;  # 替换为您的证书文件路径
    ssl_certificate_key /path/to/your_domain.key;  # 替换为您的私钥文件路径

    # SSL 配置
    ssl_session_cache shared:SSL:1m;  # 配置 SSL 会话缓存，用于存储 SSL 会话参数，以减少 SSL 握手开销。
    ssl_session_timeout 10m;  # SSL 会话超时时间
    ssl_protocols TLSv1.2 TLSv1.3;  # 支持的 SSL 协议版本
    ssl_ciphers ECDHE-RSA-AES128-GCM-SHA256:HIGH:!aNULL:!MD5:!RC4;  # 加密套件,用于定义客户端和服务器之间支持的加密算法。
    # PROFILE=SYSTEM：使用系统默认的加密套件配置文件。
    ssl_prefer_server_ciphers on;  # 优先使用服务器端加密套件

    # 网站根目录和默认首页
    root /usr/share/nginx/html;  # 替换为您的网站根目录
    index index.html index.htm;  # 默认首页文件

    location / {
        try_files $uri $uri/ =404;  # 尝试加载请求的文件或返回 404 错误
    }
    location /.well-known/acme-challenge {
        alias /var/www/html/.well-known/acme-challenge;
        try_files $uri =404;
    }
}

# HTTP 到 HTTPS 的重定向
server {
    listen 80;
    server_name your_domain.com;
    return 301 https://$host$request_uri;
}
    • 验证
    curl -I http://your_domain.com
curl --http1.1 -I http://your_domain.com
    2.HTTP/2 配置¶

    HTTP/2 是 HTTP/1.1 的升级版本，支持多路复用、头部压缩等特性，可以显著提升页面加载速度。

    配置前提：

    • 确保 Nginx 版本为 1.10.0 或更高。
    • 需要有效的 SSL 证书，因为 HTTP/2 通常运行在 HTTPS 上。

    3.HTTP/3¶

    HTTP/3 是基于 QUIC 协议的下一代 HTTP 标准，支持更低的延迟和更高的性能。

    配置前提：

    • 确保 Nginx 版本支持 HTTP/3（通常需要 Nginx 1.21.0 或更高版本）。
    • 需要支持 QUIC 的 SSL 证书。
    server {
    listen 443 quic reuseport;  # 启用 HTTP/3
    listen 443 ssl;  # 同时支持 HTTP/2 和 HTTP/1.1

    server_name your_domain.com;

    ssl_certificate /path/to/your_domain.crt;
    ssl_certificate_key /path/to/your_domain.key;

    ssl_protocols TLSv1.2 TLSv1.3;  # 必须支持 TLSv1.3
    ssl_early_data on;  # 启用 0-RTT QUIC 连接恢复

    add_header alt-svc 'h3=":443"; ma=86400';  # 用于协商 HTTP/3
    add_header QUIC-Status $http3;  # 发送 QUIC 状态头

    root /usr/share/nginx/html;
    index index.html index.htm;

    location / {
        try_files $uri $uri/ =404;
    }
}

    验证:

    curl --http3 https://your_domain.com

    • listen 443 ssl：监听 443 端口并启用 SSL 加密。
    • listen 443 ssl http2：启用 HTTP/2 支持。
    • listen 443 quic：启用 HTTP/3 支持。
    • ssl_certificate 和 ssl_certificate_key：指定 SSL 证书和私钥的路径。
    • ssl_protocols：指定支持的 SSL/TLS 协议版本，推荐使用 TLSv1.2 和 TLSv1.3。
    • ssl_ciphers：指定加密套件，推荐使用高强度的加密算法。
    • ssl_early_data：启用 0-RTT QUIC 连接恢复。
    • add_header alt-svc：用于协商 HTTP/3。
    • add_header QUIC-Status：发送 QUIC 状态头。

    2.1.3.优化¶

    1.开启BBR¶

    实际的 BBR = Bottleneck Bandwidth and Round-trip propagation time，是一种 TCP 的拥塞控制算法。简单粗暴的理解就是数据流量的交通管理 ：当公路不再塞车的时候，每辆车自然就能保持较快的车速了。

    一般而言，有BBR 和 没有BBR 会有可以感知的差别（速度、稳定性、延迟方面都会有一些改善），所以 【非常建议开启 BBR】。

    但开启之后，BBR 在 4.x 和 5.x 之间的差异往往比较微妙、见仁见智，造成体验差别的决定性因素仍然是线路质量。所以 【不必纠结版本、不必盲目追新、跟随你的发行版更新内核即可】

    BBR 的更新和发布，都是跟随 Linux 的内核（Kernel）进行的。换言之，只要你用的是比较新的内核，就自然会使用到新版BBR。

    1. 给 Debian 10 添加官方 backports 源，获取更新的软件库
    sudo nano /etc/apt/sources.list
    1. 然后把下面这一条加在最后，并保存退出。
    deb http://archive.debian.org/debian buster-backports main
    1. 刷新软件库并查询 Debian 官方的最新版内核并安装。请务必安装你的 VPS 对应的版本（本文以比较常见的【amd64】为例）。
    sudo apt update && sudo apt -t buster-backports install linux-image-amd64

    注意

    如果你的 VPS 支持，可以尝试【云服务器专用内核】linux-image-cloud-amd64，优点就是精简、资源占用低，缺点嘛是有同学反馈不支持的系统强行安装会导致无法开机（Kernel 无法识别）。
